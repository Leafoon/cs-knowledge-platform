## 2.1 数据通信基础

### 2.1.1 信息、数据与信号

在数据通信领域，三个核心概念经常被混用但含义截然不同：

**信息（Information）** 是通信的最终目的，是对接收方有意义的内容。比如"今天的气温是25°C"就是一条信息。信息具有主观性——同样的数据对不同接收者可能意味着不同的信息量。

**数据（Data）** 是信息的载体，是信息的符号化表示。数据可以是数字的（discrete）或模拟的（continuous）。上面那条信息可以编码为数据 `"temp=25"` 或二进制 `00011001`（25的二进制表示）。

**信号（Signal）** 是数据的电气或电磁表现形式，是可以在传输介质上传播的物理量。数据 `00011001` 可以用电压信号 +5V 表示 1、0V 表示 0。

三者的关系可以用一个类比来理解：

```
信息 = 你想说的话
数据 = 你写下的文字
信号 = 信纸上墨水的物理形态
```

### 2.1.2 模拟信号与数字信号

根据信号在时间和幅度上的连续性，信号分为四类：

```
┌──────────────┬────────────────────┬────────────────────┐
│              │  时间连续(模拟)     │  时间离散(数字)     │
├──────────────┼────────────────────┼────────────────────┤
│ 幅度连续      │  模拟信号           │  采样信号           │
│ (模拟)       │  (如正弦波)         │  (如PCM采样)        │
├──────────────┼────────────────────┼────────────────────┤
│ 幅度离散      │  脉冲幅度调制       │  数字信号           │
│ (数字)       │  (PAM)             │  (如方波)           │
└──────────────┴────────────────────┴────────────────────┘
```

**模拟信号（Analog Signal）**：在时间和幅度上都是连续的。自然界中的声音、光、温度等都是模拟信号。

```
模拟信号波形（正弦波）:

     振幅 A
      ↑
      │    ╭──╮          ╭──╮          ╭──╮
      │   ╱    ╲        ╱    ╲        ╱    ╲
  0 ──┼──╱──────╲──────╱──────╲──────╱──────╲──→ 时间 t
      │          ╲    ╱        ╲    ╱
      │           ╰──╯          ╰──╯
      ↓
     -A

      |←─ T ──→|
      周期 T = 1/f
```

**数字信号（Digital Signal）**：在时间和幅度上都是离散的。计算机内部处理的信号就是典型的数字信号。

```
数字信号波形（方波）:

  高电平 ───┐     ┌─────┐     ┌─────┐
            │     │     │     │     │
            │     │     │     │     │
  低电平 ───┴─────┘     └─────┘     └────

            bit 1  bit 0  bit 1  bit 0
```

### 2.1.3 周期信号的参数

任何周期信号都可以用三个基本参数来描述：

**振幅（Amplitude）**：信号偏离零值的最大幅度。类比：声波的振幅对应声音的音量大小。单位可以是伏特（V）、安培（A）等。

**频率（Frequency）**：信号每秒重复的次数，单位赫兹（Hz）。频率与周期的关系：

```
f = 1/T

其中：
  f = 频率 (Hz)
  T = 周期 (s)
```

**相位（Phase）**：信号在一个周期内的相对位置，用角度（0°~360°）或弧度（0~2π）表示。相位描述的是波形相对于时间原点的偏移。

一个完整的正弦信号可以用数学公式表示为：

```
s(t) = A × sin(2πft + φ)

其中：
  A = 振幅
  f = 频率 (Hz)
  t = 时间 (s)
  φ = 初始相位 (弧度)
  2πf = 角频率 ω (rad/s)
```

**数值示例**：假设有信号 `s(t) = 5 × sin(2π × 1000t + π/4)`
- 振幅 A = 5V
- 频率 f = 1000 Hz
- 周期 T = 1/1000 = 1 ms
- 初始相位 φ = π/4 = 45°

### 2.1.4 基带传输与宽带传输

**基带传输（Baseband Transmission）**：信号直接在传输介质上传输，不经过调制。信号占据从零频率（DC）到某个最大频率的全部频带。

```
基带信号频谱:
  功率
   ↑
   │ ████
   │ ██████
   │ ████████
   │ ██████████
   └──────────────→ 频率
   0    f_max

   频带从 0 开始
```

基带传输的特点：
- 适用于短距离通信（局域网、USB、HDMI）
- 使用数字信号
- 双向传输需要两条线路（或分时复用）

**宽带传输（Broadband Transmission）**：信号经过调制后传输，占据以某个载波频率为中心的一段频带。

```
宽带信号频谱:
  功率
   ↑
   │       ████
   │      ██████
   │     ████████
   │      ██████
   │       ████
   └──────────────────→ 频率
         f_c
     频带在 f_c 附近
```

宽带传输的特点：
- 适用于长距离通信（有线电视、ADSL、无线通信）
- 使用模拟信号
- 单根电缆可同时传输多路信号（频分复用）

### 2.1.5 时域图与频域图

信号可以从两个维度来分析：

**时域（Time Domain）**：展示信号幅度随时间的变化。示波器看到的就是时域图。

**频域（Frequency Domain）**：展示信号中包含的各频率分量及其幅度。频谱分析仪看到的就是频域图。

根据傅里叶分析，任何周期信号都可以分解为一系列正弦波的叠加：

```
周期方波 = 基波 + 三次谐波 + 五次谐波 + 七次谐波 + ...

方波的傅里叶级数展开:
              4    [sin(2πft) + (1/3)sin(6πft) + (1/5)sin(10πft) + ...]
  s(t) = ───────
              π

时域:                      频域:
  ↑  ┌──┐  ┌──┐           ↑  │   │       │       │
  │  │  │  │  │           │  │   │       │       │
  │  │  │  │  │           │  │   │   │   │   │   │
  └──┘  └──┘  └──→        └──┴───┴───┴───┴───┴───→ f
                           0  f0  3f0  5f0  7f0
  方波时域波形               方波频谱（只有奇次谐波）
```

**带宽（Bandwidth）** 在频域中的含义：信号占据的频率范围。一个理想方波需要无限带宽，但实际信道带宽有限，因此方波经过带限信道后会发生失真（波形变圆滑）。

```
带宽对方波的影响:

原始方波（无限带宽）:        经过有限带宽信道后:
  ┌──┐  ┌──┐                 ╭──╮  ╭──╮
  │  │  │  │                ╱    ╲╱    ╲
  │  │  │  │    ───→       ╱            ╲
  └──┘  └──┘              ╰──╯  ╰──╯
  (理想方波)               (波形变圆滑，有振铃)
```

## 2.2 带宽与数据速率

### 2.2.1 带宽的定义

带宽（Bandwidth）在不同语境下有不同含义：

**模拟带宽**：信号或信道占据的频率范围，单位赫兹（Hz）。

```
模拟带宽 = f_high - f_low

例：电话信道带宽 = 3400 Hz - 300 Hz = 3100 Hz ≈ 4 kHz
```

**数字带宽**：信道的最大数据传输速率，单位比特每秒（bps）。

```
例：以太网带宽 = 100 Mbps（100兆比特每秒）
    Wi-Fi 6 带宽 = 9.6 Gbps
```

两者的关系：模拟带宽越大，理论上的数字带宽也越大。这是奈奎斯特定理和香农定理的核心思想。

### 2.2.2 带宽的单位

**赫兹（Hz）** 用于模拟带宽：

```
1 kHz  = 1,000 Hz
1 MHz  = 1,000,000 Hz
1 GHz  = 1,000,000,000 Hz
```

**比特每秒（bps）** 用于数字带宽：

```
1 Kbps = 10³ bps
1 Mbps = 10⁶ bps
1 Gbps = 10⁹ bps
1 Tbps = 10¹² bps
```

注意：网络领域中，1 Kbps = 1000 bps（十进制），而不是 1024（二进制）。存储领域的 KB 才是 1024 字节。

### 2.2.3 信道容量

**信道容量（Channel Capacity）** 是信道在给定条件下能够达到的最大数据传输速率。它由以下因素决定：

1. **信道带宽（B）**：可用的频率范围，单位 Hz
2. **信号电平数（V）**：可以区分的离散信号电平数
3. **信噪比（SNR）**：信号功率与噪声功率的比值

信道容量的两个关键公式：

```
奈奎斯特（无噪声）: C = 2 × B × log₂(V)
香农（有噪声）:     C = B × log₂(1 + SNR)
```

## 2.3 数字数据的数字编码

数字数据在数字信道上传输时，需要将二进制数据（0和1）转换为数字信号（电压电平的变化）。不同的编码方案在时钟同步、带宽效率、抗干扰能力等方面各有优劣。

### 2.3.1 NRZ编码（Non-Return-to-Zero）

**NRZ-L（Non-Return-to-Zero Level）**：电平直接对应比特值。

```
NRZ-L 编码规则：
  逻辑 1 → 高电平 (+V)
  逻辑 0 → 低电平 (-V)

数据:  1  0  1  1  0  0  1  0
       ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓

NRZ-L: ┌────┐        ┌────┐              ┌────┐
       │    │        │    │              │    │
  +V ──┘    └────────┘    │              │    │
                          │              │    │
  -V ─────────────────────┘──────────────┘    └────
       
       |←1→|←0→|← 1 →|← 0 →|← 0 →|← 1 →|←0→|
```

**NRZI（Non-Return-to-Zero Inverted）**：在比特 1 时信号翻转，比特 0 时信号保持不变。

```
NRZI 编码规则：
  逻辑 1 → 电平翻转（在比特起始处）
  逻辑 0 → 电平保持不变

数据:  1  0  1  1  0  0  1  0
       ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓

NRZI:  ┌────────────────────┐        ┌────
       │                    │        │
  +V ──┘                    └────────┘
       
  翻转  保持 翻转 翻转 保持 保持 翻转 保持
```

NRZ编码的问题：
- **直流分量**：连续的相同比特会导致信号长时间保持在高电平或低电平，产生直流分量，这在某些传输介质（如变压器耦合线路）中是不允许的。
- **时钟同步困难**：长串相同比特（如 `11111111`）会导致信号长时间不变，接收端无法从中提取时钟信息。

### 2.3.2 RZ编码（Return-to-Zero）

RZ编码在每个比特周期中间都回到零电平：

```
RZ 编码规则：
  逻辑 1 → 前半周期高电平，后半周期归零
  逻辑 0 → 前半周期低电平，后半周期归零

数据:  1    0    1    1    0    0    1    0

RZ:    ┌──┐       ┌──┐ ┌──┐             ┌──┐
       │  │       │  │ │  │             │  │
  +V ──┘  │       └──┘ └──┘             └──┘
          │
  0V ─────┴────────────────────────────────────
       
       |← 1 →|← 0 →|← 1 →|← 1 →|← 0 →|← 0 →|← 1 →|← 0 →|
```

RZ编码的优缺点：
- ✅ 每个比特都有跳变，便于时钟恢复
- ❌ 需要三种电平（+V, 0, -V），实现复杂
- ❌ 比NRZ编码需要更大带宽（每个比特有两次变化）

### 2.3.3 Manchester编码

**Manchester编码** 是以太网（IEEE 802.3）使用的编码方案。它将时钟和数据信号合并编码。

```
Manchester 编码规则：
  逻辑 1 → 前半周期低电平，后半周期高电平（↓跳变）
  逻辑 0 → 前半周期高电平，后半周期低电平（↑跳变）

  注意：跳变发生在比特周期的中间

数据:    1      0      1      1      0      0      1      0

Manchester:
         ┌──┐   ┌─────────┐   ┌──┐   ┌──┐         ┌──┐
         │  │   │         │   │  │   │  │         │  │
    +V ──┘  │   │         │   │  │   │  │         │  │
             │   │         │   │  │   │  │         │  │
    0V ──────┘   │         │   │  │   │  │         │  │
                 │         │   │  │   │  │         │  │
    -V ──────────┘         └───┘  └───┘  └─────────┘  └──
         
         |← 1 →|← 0 →|← 1 →|← 1 →|← 0 →|← 0 →|← 1 →|← 0 →|
```

Manchester编码的特点：
- ✅ 每个比特中间都有跳变，时钟恢复容易
- ❌ 编码效率只有50%（信号变化速率是数据速率的2倍）
- ❌ 需要更大带宽

### 2.3.4 差分Manchester编码

差分Manchester编码（IEEE 802.5令牌环网使用）：

```
差分Manchester 编码规则：
  每个比特中间必有跳变（用于时钟恢复）
  逻辑 0 → 比特起始处有额外跳变
  逻辑 1 → 比特起始处无跳变

数据:    1      0      1      1      0      0      1      0

差分Manchester:
    +V ──┐     ┌───┐   ┌─────────┐   ┌───┐   ┌───┐   ┌───┐
         │     │   │   │         │   │   │   │   │   │   │
    0V ──┤     │   │   │         │   │   │   │   │   │   │
         │     │   │   │         │   │   │   │   │   │   │
    -V ──┘─────┘   └───┘         └───┘   └───┘   └───┘   └───
         
         |← 1 →|← 0 →|←  1  →|←  1  →|← 0 →|← 0 →|← 1 →|← 0 →|
```

差分Manchester与Manchester的区别：
- 差分Manchester通过比特起始处有/无跳变来区分0/1，而Manchester通过跳变方向来区分
- 差分Manchester是差分编码，不受接线极性影响（A线和B线接反不影响解码）

### 2.3.5 4B/5B编码

4B/5B编码将每4位数据映射为5位编码，然后使用NRZI传输。这种编码确保了编码后的比特流中不会出现过长的连续0或1。

```
4B/5B 编码表：
┌────────┬────────┬────────────────────────┐
│ 4位数据 │ 5位编码 │        含义            │
├────────┼────────┼────────────────────────┤
│ 0000   │ 11110  │ 数据 0                 │
│ 0001   │ 01001  │ 数据 1                 │
│ 0010   │ 10100  │ 数据 2                 │
│ 0011   │ 10101  │ 数据 3                 │
│ 0100   │ 01010  │ 数据 4                 │
│ 0101   │ 01011  │ 数据 5                 │
│ 0110   │ 01110  │ 数据 6                 │
│ 0111   │ 01111  │ 数据 7                 │
│ 1000   │ 10010  │ 数据 8                 │
│ 1001   │ 10011  │ 数据 9                 │
│ 1010   │ 10110  │ 数据 A                 │
│ 1011   │ 10111  │ 数据 B                 │
│ 1100   │ 11010  │ 数据 C                 │
│ 1101   │ 11011  │ 数据 D                 │
│ 1110   │ 11100  │ 数据 E                 │
│ 1111   │ 11101  │ 数据 F                 │
├────────┼────────┼────────────────────────┤
│ --     │ 11000  │ 静默 (Idle)            │
│ --     │ 11001  │ 帧开始 (Start)         │
│ --     │ 10001  │ 帧结束 (End)           │
└────────┴────────┴────────────────────────┘
```

4B/5B编码的关键特性：
- 每个5位编码中最多有3个连续的0
- 编码效率 = 4/5 = 80%
- 100 Mbps以太网使用 4B/5B + NRZI → 实际信号速率 125 Mbaud

### 2.3.6 编码效率比较

```
┌──────────────────┬──────────────┬───────────────┬──────────────┬───────────────┐
│    编码方案       │  编码效率     │  时钟同步能力  │  直流分量     │  典型应用      │
├──────────────────┼──────────────┼───────────────┼──────────────┼───────────────┤
│ NRZ-L            │ 100%         │ 差            │ 有           │ 磁带记录       │
│ NRZI             │ 100%         │ 差            │ 有           │ USB, 磁盘      │
│ RZ               │ 50%          │ 好            │ 无           │ 较少使用       │
│ Manchester       │ 50%          │ 好            │ 无           │ 10Base-T以太网  │
│ 差分Manchester   │ 50%          │ 好            │ 无           │ 令牌环网       │
│ 4B/5B            │ 80%          │ 好            │ 有           │ 100BASE-TX     │
│ 8B/10B           │ 80%          │ 好            │ 近似无       │ 千兆以太网      │
│ 128B/130B        │ 98.5%        │ 好            │ 有           │ 万兆以太网      │
└──────────────────┴──────────────┴───────────────┴──────────────┴───────────────┘
```

编码效率（Code Efficiency）的计算：

```
编码效率 = 数据比特数 / 编码后比特数 × 100%

Manchester: η = 1 / 2 = 50%
4B/5B:      η = 4 / 5 = 80%
8B/10B:     η = 8 / 10 = 80%
```

### 2.3.7 时钟恢复问题

时钟恢复（Clock Recovery）是数字通信中的关键问题。发送端和接收端的时钟不可能完全同步，微小的频率偏差会导致采样点漂移。

```
时钟偏差导致的问题：

发送端时钟: ──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──
              │ │  │ │  │ │  │ │  │ │
              └─┘  └─┘  └─┘  └─┘  └─┘
              ↑    ↑    ↑    ↑    ↑
              理想采样点

接收端时钟（略微偏快）:
              ↑   ↑   ↑   ↑   ↑   ↑
              采样点逐渐偏移，最终可能采到错误的值
```

解决方案：
1. **自同步编码**：如Manchester编码，每个比特都有跳变，接收端可以从信号中提取时钟
2. **添加起始位**：异步串行通信（UART）中，每个字节前加一个起始位来重新同步
3. **PLL（锁相环）**：硬件电路，从接收到的信号中恢复出精确的时钟

## 2.4 数字数据的模拟调制

当数字数据需要在模拟信道上传输时（如通过电话线拨号上网），需要将数字信号调制（modulate）到模拟载波上。三种基本调制方式分别改变载波的振幅、频率和相位。

### 2.4.1 ASK（振幅键控）

ASK通过改变载波的振幅来表示数字数据：

```
ASK 调制原理：
  载波信号: s(t) = A × cos(2πf_c × t)

  逻辑 1 → 发送载波（A = A1）
  逻辑 0 → 不发送载波（A = 0）

数据:    1      0      1      1      0      0      1

原始载波（连续发送）:
         ∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿

ASK信号:
  +A1 ─  ∿∿∿∿∿           ∿∿∿∿∿∿∿∿           ∿∿∿∿∿
   0  ─        ∿∿∿∿∿∿∿∿∿          ∿∿∿∿∿∿∿∿∿        
  -A1 ─  ∿∿∿∿∿           ∿∿∿∿∿∿∿∿           ∿∿∿∿∿

         |← 1 →|← 0 →|← 1 →|← 1 →|← 0 →|← 0 →|← 1 →|
```

ASK的星座图（Constellation Diagram）：

```
正交分量 Q
    ↑
    │
    │      ●  (A1, 0) 代表 1
────┼──────────→ 同相分量 I
    │
    │      ●  (0, 0) 代表 0
    │

两点之间的距离 = 判决距离
距离越大，抗噪声能力越强
```

ASK的特点：
- 实现简单，但抗噪声能力差（噪声直接影响振幅）
- 频带利用率高
- 光纤通信中广泛使用（有光=1，无光=0）

### 2.4.2 FSK（频移键控）

FSK通过改变载波的频率来表示数字数据：

```
FSK 调制原理：
  逻辑 1 → 使用频率 f1（较高频率）
  逻辑 0 → 使用频率 f2（较低频率）

数据:    1      0      1      1      0      0      1

FSK信号:
  f1:  ∿∿∿∿∿∿∿           ∿∿∿∿∿∿∿∿∿           ∿∿∿∿∿∿∿
       ∿∿∿∿∿∿∿           ∿∿∿∿∿∿∿∿∿           ∿∿∿∿∿∿∿
  
  f2:         ∿∿∿∿∿∿∿∿∿           ∿∿∿∿∿∿∿∿∿
              ∿∿∿∿∿∿∿∿∿           ∿∿∿∿∿∿∿∿∿

       |← 1 →|← 0 →|← 1 →|← 1 →|← 0 →|← 0 →|← 1 →|
       
  f1 密集（高频）   f2 稀疏（低频）
```

FSK的应用：
- 调制解调器（低速，如 Bell 103 标准：f1=1270Hz, f2=1070Hz）
- 来电显示（Caller ID）
- 无线遥控器
- 蓝牙（GFSK调制）

### 2.4.3 PSK（相移键控）

**BPSK（Binary Phase Shift Keying）**：用两种相位表示0和1。

```
BPSK 调制原理：
  逻辑 1 → 相位 0°（与载波同相）
  逻辑 0 → 相位 180°（与载波反相）

数据:    1      0      1      1      0      0      1

BPSK信号:
         ∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿
  
  ───────∿∿∿∿∿∿∿─────────∿∿∿∿∿∿∿────────────────∿∿∿∿∿∿∿
         相位翻转     相位翻转              相位翻转
```

BPSK星座图：

```
正交分量 Q
    ↑
    │
────●──────────→ 同相分量 I
   (-1,0)       (1,0)
   代表 0        代表 1
```

**QPSK（Quadrature Phase Shift Keying）**：用四种相位表示每2个比特。

```
QPSK 调制原理：
  每个符号携带 2 比特信息

  00 → 相位 45°   (I=+1, Q=+1)
  01 → 相位 135°  (I=-1, Q=+1)
  11 → 相位 225°  (I=-1, Q=-1)
  10 → 相位 315°  (I=+1, Q=-1)

QPSK星座图：
  正交分量 Q
      ↑
  ●   │   ●
  (01)│   (00)
  ────┼────→ 同相分量 I
  ●   │   ●
  (11)│   (10)

  每个符号间隔的角度 = 90°
  相邻状态的比特差为1位（格雷码）
```

QPSK的优势：
- 每个符号传输2比特，频谱效率是BPSK的2倍
- 在相同误码率下，QPSK所需的信噪比与BPSK相同（因为判决距离相同）

### 2.4.4 QAM（正交幅度调制）

QAM同时改变载波的振幅和相位，是现代通信中最常用的调制方式。

**16-QAM**：16个星座点，每个符号传输4比特。

```
16-QAM 星座图：
  正交分量 Q
      ↑
  ●   ●   ●   ●  (+3)
  ●   ●   ●   ●  (+1)
  ●   ●   ●   ●  (-1)
  ●   ●   ●   ●  (-3)
  ──────────────→ 同相分量 I
 (-3)(-1)(+1)(+3)

每个 ● 代表一个4比特的组合
例如：(0000) (0001) (0010) ... (1111)
```

**64-QAM**：64个星座点，每个符号传输6比特。

```
64-QAM 星座图（简化表示）：
  Q
  ↑
  ●●●●●●●●  (+7)
  ●●●●●●●●  (+5)
  ●●●●●●●●  (+3)
  ●●●●●●●●  (+1)
  ●●●●●●●●  (-1)
  ●●●●●●●●  (-3)
  ●●●●●●●●  (-5)
  ●●●●●●●●  (-7)
  ────────→ I

  每行8个点，共8行 = 64个点
  每个符号传输 log₂(64) = 6 比特
```

各调制方式的比较：

```
┌────────────┬──────────────┬──────────────┬───────────────┬──────────────┐
│ 调制方式    │ 每符号比特数  │ 星座点数      │ 频谱效率       │ 抗噪声能力    │
├────────────┼──────────────┼──────────────┼───────────────┼──────────────┤
│ BPSK       │ 1            │ 2            │ 1 bps/Hz      │ 最强         │
│ QPSK       │ 2            │ 4            │ 2 bps/Hz      │ 强           │
│ 8-PSK      │ 3            │ 8            │ 3 bps/Hz      │ 中等         │
│ 16-QAM     │ 4            │ 16           │ 4 bps/Hz      │ 较弱         │
│ 64-QAM     │ 6            │ 64           │ 6 bps/Hz      │ 弱           │
│ 256-QAM    │ 8            │ 256          │ 8 bps/Hz      │ 很弱         │
│ 1024-QAM   │ 10           │ 1024         │ 10 bps/Hz     │ 极弱         │
└────────────┴──────────────┴──────────────┴───────────────┴──────────────┘
```

### 2.4.5 比特率与波特率

**比特率（Bit Rate）**：每秒传输的比特数，单位 bps。
**波特率（Baud Rate）**：每秒传输的符号数，单位 baud（波特）。

两者的关系：

```
比特率 = 波特率 × 每符号比特数
R = S × log₂(N)

其中：
  R = 比特率 (bps)
  S = 波特率 (baud)
  N = 调制阶数（星座点数）
```

**示例**：一个信道的带宽为 4 kHz，使用 16-QAM 调制

```
最大波特率 = 2 × B = 2 × 4000 = 8000 baud（奈奎斯特）
每符号比特数 = log₂(16) = 4
最大比特率 = 8000 × 4 = 32,000 bps = 32 kbps
```

### 2.4.6 误码率与信噪比

**误码率 BER（Bit Error Rate）**：接收到的错误比特数与总传输比特数的比值。

```
BER = 错误比特数 / 总传输比特数

典型要求：BER < 10⁻⁶（每百万比特最多1个错误）
```

不同调制方式的BER与SNR的关系：

```
BPSK:   BER = Q(√(2×SNR))
QPSK:   BER = Q(√(2×SNR))    （与BPSK相同！）
M-QAM:  BER ≈ (4/log₂M) × (1 - 1/√M) × Q(√(3×log₂M×SNR/(M-1)))

其中 Q(x) 是Q函数（标准正态分布的右尾概率）
Q(x) = (1/√(2π)) × ∫[x,∞] exp(-t²/2) dt
```

**关键观察**：
- 调制阶数越高，每符号传输的比特越多，但要求更高的SNR
- 从64-QAM升级到256-QAM，比特率增加33%，但所需SNR增加约5-7 dB

## 2.5 模拟数据的数字编码

### 2.5.1 采样定理（奈奎斯特-香农采样定理）

**采样定理**：要从采样信号中完美重建原始模拟信号，采样频率必须至少是信号最高频率的两倍：

```
f_s ≥ 2 × f_max

其中：
  f_s = 采样频率 (Hz)
  f_max = 模拟信号的最高频率分量 (Hz)
  2 × f_max = 奈奎斯特频率
```

**为什么是两倍？** 直觉理解：一个正弦波每个周期至少需要2个采样点才能确定其频率和幅度。

```
足够采样 (f_s > 2f_max):
  原始信号: ∿∿∿∿∿∿∿∿∿
  采样点:   │ │ │ │ │ │ │ │   → 能重建波形

不足采样 (f_s < 2f_max) —— 混叠(Aliasing):
  原始信号: ∿∿∿∿∿∿∿∿∿
  采样点:   │   │   │   │   → 采到的点连成的波形与原始不同！
                                产生"假"的低频信号
```

**混叠（Aliasing）**：当采样率不足时，高频信号被错误地"伪装"成低频信号。解决方案：采样前用低通滤波器（抗混叠滤波器）去除高于 f_s/2 的频率分量。

### 2.5.2 PCM（脉冲编码调制）

PCM（Pulse Code Modulation）是将模拟信号转换为数字信号的标准方法，包含三个步骤：

**第一步：采样（Sampling）**

```
连续模拟信号:
      ╭──╮          ╭──╮
     ╱    ╲        ╱    ╲
────╱──────╲──────╱──────╲────

在固定时间间隔采样:
      │    │        │    │
      ↓    ↓        ↓    ↓
      S1   S2       S3   S4

采样值（幅度）: S1=0.8, S2=0.3, S3=-0.5, S4=-0.9 ...
```

**第二步：量化（Quantization）**

将连续的采样值映射到有限个离散电平：

```
量化级数 = 2^n（n为量化位数）

假设使用 3 位量化（8个量化级）：

  量化范围      量化值    编码
  ─────────────────────────────
  0.75 ~ 1.00    0.875    111
  0.50 ~ 0.75    0.625    110
  0.25 ~ 0.50    0.375    101
  0.00 ~ 0.25    0.125    100
 -0.25 ~ 0.00   -0.125    011
 -0.50 ~-0.25   -0.375    010
 -0.75 ~-0.50   -0.625    001
 -1.00 ~-0.75   -0.875    000

采样值 S1=0.8 → 量化为 0.875 → 编码 111
采样值 S2=0.3 → 量化为 0.375 → 编码 101
采样值 S3=-0.5 → 量化为 -0.500 → 编码 001
采样值 S4=-0.9 → 量化为 -0.875 → 编码 000
```

**第三步：编码（Encoding）**

将量化值转换为二进制码：

```
PCM 输出比特流:
  S1=111  S2=101  S3=001  S4=000  ...
  ↓       ↓       ↓       ↓
  111101001000...

比特率 = 采样率 × 每样本比特数
例：电话语音 = 8000 samples/s × 8 bits/sample = 64 kbps
```

### 2.5.3 量化噪声

量化过程引入的误差称为**量化噪声（Quantization Noise）**：

```
量化噪声 = 原始采样值 - 量化值

例：原始值 0.8，量化值 0.875
    量化误差 = |0.8 - 0.875| = 0.075

量化噪声功率:
  N_q = Δ²/12

  其中 Δ = 量化步长 = (最大值 - 最小值) / 量化级数
```

**信噪比与量化位数的关系**：

```
SNR = 6.02n + 1.76 dB

其中 n = 量化位数

  8位量化: SNR ≈ 49.9 dB  （电话语音质量）
 16位量化: SNR ≈ 98.1 dB  （CD音质）
 24位量化: SNR ≈ 146.2 dB （专业录音）
```

### 2.5.4 采样率与音质

不同应用场景采用不同的采样率和量化位数：

```
┌──────────────────┬──────────────┬──────────────┬──────────────┐
│     应用          │  采样率       │  量化位数     │  比特率       │
├──────────────────┼──────────────┼──────────────┼──────────────┤
│ 电话语音          │  8 kHz       │  8 位        │  64 kbps     │
│ AM 广播           │  11.025 kHz  │  16 位       │  176 kbps    │
│ FM 广播质量       │  22.05 kHz   │  16 位       │  352 kbps    │
│ CD 音质           │  44.1 kHz    │  16 位       │  705.6 kbps  │ (单声道)
│ DVD 音频          │  48 kHz      │  16 位       │  768 kbps    │ (单声道)
│ 专业录音          │  96 kHz      │  24 位       │  2.304 Mbps  │ (单声道)
│ 高解析度音频      │  192 kHz     │  24 位       │  4.608 Mbps  │ (单声道)
└──────────────────┴──────────────┴──────────────┴──────────────┘
```

**为什么CD采样率是44.1kHz？** 人耳听觉范围约为20Hz~20kHz，根据采样定理需要 f_s ≥ 40kHz。44.1kHz的选择有历史原因：早期数字音频使用视频录像带存储，NTSC和PAL制式的视频帧率分别为29.97fps和25fps，每帧可嵌入的音频样本数分别为49和58，计算：29.97×49×3 = 44055.3 ≈ 44100，25×58×3 = 43500 ≈ 44100。

## 2.6 奈奎斯特定理（Nyquist Theorem）

### 2.6.1 无噪声信道的最大数据速率

**奈奎斯特定理**：对于带宽为 B Hz 的无噪声信道，最大数据传输速率为：

```
C = 2 × B × log₂(V)    (bps)

其中：
  C = 信道容量（最大数据速率）(bps)
  B = 信道带宽 (Hz)
  V = 可区分的离散信号电平数
```

### 2.6.2 公式推导

**推导思路**：

1. 一个带宽为 B Hz 的信道，根据奈奎斯特采样定理，每秒最多可以传输 2B 个独立的信号样本。

2. 如果有 V 个离散电平，每个样本可以携带 log₂(V) 比特的信息。

3. 因此最大数据速率 = 2B × log₂(V) bps。

**数学推导**：

```
设带宽为 B Hz 的理想低通滤波器的冲激响应为:
  h(t) = sin(2πBt) / (πt)

这是 sinc 函数，其零点在 t = n/(2B)，n = ±1, ±2, ...

这意味着每 1/(2B) 秒可以传输一个独立的符号，
即每秒最多 2B 个符号（奈奎斯特速率）。

如果每个符号有 V 种可能的电平，
每个符号携带的信息量 = log₂(V) 比特。

因此：C = 2B × log₂(V) bps
```

### 2.6.3 数值例子

**例1**：一个带宽为 3 kHz 的无噪声信道，使用二进制信号（V=2）

```
C = 2 × 3000 × log₂(2)
  = 2 × 3000 × 1
  = 6000 bps = 6 kbps
```

**例2**：同一信道，使用 16 个电平（V=16）

```
C = 2 × 3000 × log₂(16)
  = 2 × 3000 × 4
  = 24000 bps = 24 kbps
```

**例3**：一个带宽为 6 MHz 的无噪声信道（类似电视信道），使用 4 个电平

```
C = 2 × 6,000,000 × log₂(4)
  = 2 × 6,000,000 × 2
  = 24,000,000 bps = 24 Mbps
```

**关键洞察**：
- 增加电平数 V 可以提高数据速率，但在有噪声的信道中，电平数越多，相邻电平之间的间隔越小，越容易被噪声干扰导致误判。
- 实际系统需要在数据速率和可靠性之间权衡。

## 2.7 香农定理（Shannon Theorem）

### 2.7.1 有噪声信道的最大数据速率

**香农定理**：对于带宽为 B Hz、信噪比为 SNR 的有噪声信道，最大数据传输速率（信道容量）为：

```
C = B × log₂(1 + SNR)    (bps)

其中：
  C = 信道容量 (bps)
  B = 信道带宽 (Hz)
  SNR = 信号功率/噪声功率（线性值，不是dB）
```

### 2.7.2 信噪比（SNR）

**信噪比的定义**：

```
SNR（线性值）= 信号功率 P_signal / 噪声功率 P_noise

SNR_dB = 10 × log₁₀(SNR) = 10 × log₁₀(P_signal / P_noise)

两者之间的转换：
  SNR（线性）= 10^(SNR_dB / 10)
  SNR_dB = 10 × log₁₀(SNR)
```

**数值示例**：

```
SNR_dB = 30 dB → SNR = 10^(30/10) = 10^3 = 1000
  （信号功率是噪声功率的1000倍）

SNR_dB = 20 dB → SNR = 10^(20/10) = 10^2 = 100
  （信号功率是噪声功率的100倍）

SNR_dB = 10 dB → SNR = 10^(10/10) = 10^1 = 10
  （信号功率是噪声功率的10倍）

SNR_dB = 0 dB  → SNR = 10^0 = 1
  （信号功率等于噪声功率）
```

### 2.7.3 香农定理的推导与解释

香农定理的核心思想是**信息论**中的一个基本结论：

```
推导思路（启发式）：

1. 假设信道带宽为 B，信噪比为 SNR（线性值）。

2. 接收端收到的信号 = 原始信号 + 噪声

3. 根据奈奎斯特定理，每秒可以传输 2B 个样本。

4. 每个样本的信号功率为 P_signal，
   噪声功率为 P_noise = P_signal / SNR。

5. 每个样本可以区分的有效电平数受噪声限制：
   有效电平数 ≈ √(1 + SNR)

6. 因此每样本的比特数 = log₂(√(1 + SNR)) = 0.5 × log₂(1 + SNR)

7. 数据速率 = 2B × 0.5 × log₂(1 + SNR) = B × log₂(1 + SNR)

严格证明需要用到随机过程和信息熵的理论，超出了本课程的范围。
```

**香农定理的深刻含义**：
- 信道容量取决于带宽和信噪比，与调制方式无关
- 任何调制方式都无法超过这个极限
- 香农容量是理论上界，实际系统通常只能达到其 50%-80%

### 2.7.4 数值例子

**例1：标准电话线路**

```
电话线路参数：
  带宽 B = 3100 Hz（300 Hz ~ 3400 Hz）
  信噪比 SNR_dB = 30 dB → SNR = 1000

香农容量：
  C = B × log₂(1 + SNR)
  C = 3100 × log₂(1 + 1000)
  C = 3100 × log₂(1001)
  C = 3100 × 9.97
  C ≈ 30,907 bps ≈ 31 kbps

这就是为什么56K调制解调器的理论上限约为35 kbps（实际上行约33.6 kbps）。
```

**例2：光纤通信**

```
光纤参数：
  带宽 B = 25 THz（25 × 10¹² Hz）（单模光纤在1550nm窗口）
  信噪比 SNR_dB = 40 dB → SNR = 10,000

香农容量：
  C = 25 × 10¹² × log₂(1 + 10,000)
  C = 25 × 10¹² × 13.29
  C ≈ 332 × 10¹² bps ≈ 332 Tbps

目前实验室记录：单根光纤传输速率已超过 1 Pbps（10¹⁵ bps），
使用空分复用等技术。实际商用系统通常为 10-100 Tbps。
```

**例3：Wi-Fi信道**

```
Wi-Fi 6 参数：
  带宽 B = 160 MHz
  信噪比 SNR_dB = 35 dB → SNR ≈ 3162

香农容量：
  C = 160 × 10⁶ × log₂(1 + 3162)
  C = 160 × 10⁶ × 11.63
  C ≈ 1.86 Gbps

Wi-Fi 6 实际最大速率 ≈ 9.6 Gbps（通过 MIMO 等技术超越单信道极限）
```

### 2.7.5 奈奎斯特和香农定理的结合使用

在实际问题中，两个定理通常需要结合使用：

```
实际信道容量 = min(奈奎斯特极限, 香农极限)

步骤：
1. 用香农定理计算信道容量上限
   C_shannon = B × log₂(1 + SNR)

2. 用奈奎斯特定理计算所需的电平数
   C_nyquist = 2B × log₂(V)
   → V = 2^(C/(2B))

3. 如果所需的 V 在实际中不可实现（太大），则需要降低数据速率
```

**综合例题**：

```
问题：一个带宽为 4 kHz 的信道，SNR_dB = 24 dB，
      使用 QPSK 调制，最大数据速率是多少？

解法：
1. SNR = 10^(24/10) = 10^2.4 ≈ 251

2. 香农极限：
   C_shannon = 4000 × log₂(252) = 4000 × 7.98 ≈ 31.9 kbps

3. QPSK 的 V = 4（4种相位），奈奎斯特极限：
   C_nyquist = 2 × 4000 × log₂(4) = 2 × 4000 × 2 = 16 kbps

4. 实际最大速率 = min(31.9 kbps, 16 kbps) = 16 kbps

5. 验证：16 kbps < 31.9 kbps，说明 QPSK 能在此信道上正常工作。
   如果使用 16-QAM (V=16)：
   C_nyquist = 2 × 4000 × 4 = 32 kbps > 31.9 kbps
   → 超过香农极限，16-QAM 在此信道上会有很多误码！
```

### 2.8.1 调制解调器的演进

调制解调器（Modem）将计算机的数字信号转换为电话线可以传输的模拟信号。其演进体现了物理层技术的进步：

│ V.34         │ 1994         │  28.8 kbps   │  960-QAM      │ 预编码       │
│ V.34+        │ 1996         │  33.6 kbps   │  1664-QAM     │ 多维星座     │

**V.90/V.92下行56 kbps的原理**：

```
电话网络结构:
  用户A ──[Modem]──→ 模拟线路 → [局端Codec] → 数字干线 → [局端Codec] → 模拟线路 ←──[Modem]── 用户B
  
  V.90下行: 局端Codec直接将PCM样本发送到数字干线，
           用户端Modem从PCM信号中恢复数据
           采样率 8000 Hz × 8 bits/sample = 64 kbps
           实际最大 56 kbps（受量化噪声限制，使用 128 个电平: 8000 × log₂(128) = 56 kbps）
  
  V.90上行: 仍然使用传统的模拟调制，最大 33.6 kbps
```

```
DMT 调制示意:
  子信道1: ∿∿∿  (QPSK, 2 bits)
  子信道2: ∿∿∿  (16-QAM, 4 bits)
  子信道3: ∿∿∿  (64-QAM, 6 bits)  ← 信道质量好，用高阶调制
  子信道4: ∿∿∿  (QPSK, 2 bits)    ← 信道质量差，用低阶调制

  子信道256: ∿∿ (BPSK, 1 bit)
  
  总速率 = Σ(每个子信道的速率)
```

### 2.8.3 光纤通信的容量极限

光纤通信是现代互联网的骨干，其容量增长遵循一些有趣的规律：

```
光纤通信容量增长趋势:
  1980年代:  45 Mbps (DS3)
  1990年代:  2.5 Gbps (OC-48/STM-16)
  2000年代:  10 Gbps (OC-192/STM-64)
  2010年代:  100 Gbps (100GbE)
  2020年代:  400 Gbps / 800 Gbps (400GbE / 800GbE)
  未来:      1.6 Tbps / 更高
```

**单根光纤的理论极限**：

```
光纤在1550nm窗口（C波段）的可用带宽:
  波长范围: 1530 nm ~ 1565 nm
  频率范围: 191.7 THz ~ 196.1 THz
  带宽 ≈ 4.4 THz

假设 SNR_dB = 30 dB (SNR = 1000):
  C = 4.4 × 10¹² × log₂(1001)
  C ≈ 4.4 × 10¹² × 9.97
  C ≈ 43.9 Tbps （每根光纤，单波长）

使用 WDM（波分复用）:
  80个波长 × 43.9 Tbps ≈ 3.5 Pbps

使用空分复用（多芯光纤）:
  理论极限可达 数十 Pbps
```

**实际挑战**：
- 非线性效应（四波混频、自相位调制）
- 光纤衰减（约0.2 dB/km）
- 放大器噪声积累
- 色散限制

各模块的功能详解：

**线路接口（Line Interface）**：连接电话线的物理接口，通常使用RJ-11连接器。包含：
- 摘/挂机电路（模拟电话的摘机/挂机操作）
- 振铃检测电路
- 线路阻抗匹配网络

**DAA（Data Access Arrangement）**：隔离Modem与电话网络的关键模块：
- 提供电气隔离（通常使用变压器耦合）
- 防止电话线上的高压（如振铃电压约90V AC）损坏Modem
- 实现2线到4线的转换（电话线是2线双向，Modem内部收发分离需要4线）
- 各国电话网络标准不同，DAA负责适配

**ADC/DAC（模数/数模转换器）**：
- ADC：将接收到的模拟信号数字化（解调方向）
- DAC：将数字信号转换为模拟信号（调制方向）
- 采样率：通常为 8 kHz 或更高
- 量化精度：通常 14-16 位

**DSP处理器（Digital Signal Processor）**：Modem的核心，负责：
- 调制/解调算法
- 均衡和回波消除
- 编解码（V.42, MNP等）
- 数据压缩

**控制器（CPU）**：协调各模块工作，执行AT命令集

**UART/USB接口**：连接计算机
- UART（通用异步收发器）：RS-232串口，传统Modem使用
- USB：现代Modem使用

### A.2 调制过程：数字→模拟

以V.34标准（28.8 kbps）为例，详细描述调制过程：

```
调制过程信号流:

  计算机数字数据
       │
       ↓
  ┌─────────────┐
  │ 1. 数据分组  │  将连续比特流分成符号帧
  │    每帧96比特│  (每符号6比特, 共16个符号)
  └──────┬──────┘
         ↓
  ┌─────────────┐
  │ 2. 扰码     │  防止长串相同比特
  │    (Scrambler)│  使用伪随机序列进行XOR
  └──────┬──────┘
         ↓
  ┌─────────────┐
  │ 3. 纠错编码  │  添加冗余比特用于错误检测和纠正
  │    (FEC)     │  V.34使用格型编码(Trellis Coding)
  └──────┬──────┘
         ↓
  ┌─────────────┐
  │ 4. 星座映射  │  将比特映射到复数星座点
  │    960-QAM   │  V.34使用960点星座
  │    I + jQ    │  每个符号映射为(I,Q)坐标对
  └──────┬──────┘
         ↓
  ┌─────────────┐
  │ 5. 脉冲成形  │  使用升余弦滤波器限制带宽
  │    (Shaping) │  减少码间干扰(ISI)
  └──────┬──────┘
         ↓
  ┌─────────────┐
  │ 6. 载波调制  │  s(t) = I(t)cos(2πf_c×t) - Q(t)sin(2πf_c×t)
  │    (Modulation)  将基带信号搬移到载波频率
  └──────┬──────┘
         ↓
  ┌─────────────┐
  │ 7. D/A转换   │  DAC将数字波形转换为模拟信号
  └──────┬──────┘
         ↓
  ┌─────────────┐
  │ 8. 线路驱动  │  放大信号功率，发送到电话线
  └──────┬──────┘
         ↓
      电话线
```

**步骤详解**：

**步骤4 - 星座映射**：
```
V.34的960-QAM星座（简化示意）:

  Q轴
   ↑
   │  ●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●
   │  ●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●
   │  ●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●
   │  ...
   │  ●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●
   └──────────────────────────────────→ I轴

  960个点，每个符号携带 log₂(960) ≈ 9.91 比特
  波特率 = 3429 baud
  比特率 ≈ 3429 × 9.91 ≈ 33.6 kbps（理论值）
  实际考虑开销后为 28.8 kbps
```

**步骤5 - 脉冲成形**：
```
升余弦滤波器的时域和频域特性:

时域:                         频域:
  ↑  主瓣                      ↑
  │ ╱╲                        │ ┌──────────┐
  │╱  ╲                       │ │          │
──┼────╲──╱╲──╱╲──→t         ─┼─┘          └───→ f
  │     ╲╱  ╲╱               │
  ↓                           ↓
  
  通过限制带宽来减少ISI
  滚降系数 α 决定了频谱的形状
  α = 0: 理想矩形滤波器（不可实现）
  α = 1: 最大滚降（带宽翻倍，但无ISI）
```

**步骤6 - 载波调制**：
```
正交调制（I/Q调制）:

  I路: I(t) × cos(2πf_c×t)  ──→ +
                                    ├──→ s(t) = I(t)cos(ωt) - Q(t)sin(ωt)
  Q路: Q(t) × sin(2πf_c×t)  ──→ -

这允许在同一个载波频率上同时传输两路独立的数据
频谱效率翻倍
```

### A.4 Modem标准演进详解

**V.32（9600 bps, 1984）**：
```
调制方式: 32-QAM (或 trellis-coded 32点星座)
波特率: 2400 baud
每符号比特: log₂(32) = 5 bits
比特率: 2400 × 4 = 9600 bps (有效数据)
        2400 × 5 = 12000 bps (含纠错码)
载波频率: 1800 Hz
全双工: 使用回波消除(Echo Cancellation)
```

**V.32bis（14400 bps, 1991）**：
```
调制方式: 128-QAM
波特率: 2400 baud
每符号比特: log₂(128) = 7 bits
比特率: 2400 × 6 = 14400 bps (有效数据)
        2400 × 7 = 16800 bps (含纠错码)
新增: 速率自适应(14400/12000/9600/7200/4800 bps)
```

**V.34（28800/33600 bps, 1994/1996）**：
```
调制方式: 960-QAM (或 1664-QAM for V.34+)
波特率: 3429 baud
每符号比特: ~10 bits (含编码增益)
比特率: 3429 × 8.4 ≈ 28800 bps (V.34)
        3429 × 9.8 ≈ 33600 bps (V.34+)
关键技术:
  - 预编码(Tomlinson-Harashima): 发送端预失真
  - 多维格型编码: 编码增益约3-4 dB
  - 自适应波特率: 根据信道条件选择最佳波特率
  - 自适应载波频率: 不固定使用1800Hz
```

**V.90（56000 bps 下行, 1998）**：
```
下行(服务器→客户端):
  信道: 服务器端Codec直接发送PCM样本
  采样率: 8000 Hz
  量化电平: 128个（使用7位PCM，第8位用于信令）
  比特率: 8000 × 7 = 56000 bps（理论最大）
  调制方式: 实际是PCM编码，不是传统调制
  限制: 只在下行方向有效，要求ISP端是数字连接

上行(客户端→服务器):
  仍然使用V.34调制
  最大: 33600 bps
```

│ 距离限制        │ 无明显限制           │ 3-5 km（受距离影响大）│
│ 成本           │ 低                   │ 中等                 │
└────────────────┴─────────────────────┴─────────────────────┘
```

### A.6 错误纠正和压缩

**MNP（Microcom Networking Protocol）错误纠正**：

```
MNP Class 1-4: 错误纠正协议
  - Class 1: 异步到同步转换，半双工
  - Class 2: 异步到同步转换，全双工
  - Class 3: 同步传输，帧化
  - Class 4: 自适应帧大小，优化吞吐量

V.42: ITU标准，包含LAPM(Link Access Procedure for Modems)
  - 基于HDLC的帧协议
  - CRC错误检测
  - ARQ重传机制
  - 滑动窗口流控
```

**V.42bis数据压缩**：

```
压缩算法: 基于LZ78的字典压缩

工作原理:
1. 发送端和接收端维护相同的字典
2. 数据中重复出现的字符串被字典索引替代
3. 字典动态更新

压缩比: 理论最高4:1，实际通常2:1到3:1

例: 原始数据 28800 bps → V.42bis压缩后有效数据率可达 ~115200 bps (4:1)

注意: 压缩比取决于数据的可压缩性
  - 文本文件: 通常 3:1 ~ 4:1
  - 已压缩文件(JPG, ZIP): 接近 1:1（无法进一步压缩）
  - 二进制数据: 通常 1.5:1 ~ 2:1
```

### A.8 性能指标

**波特率 vs 比特率**：

```
波特率(Baud): 每秒传输的符号数
比特率(bps): 每秒传输的比特数

关系: 比特率 = 波特率 × log₂(M)
其中 M = 调制阶数（每个符号的可能状态数）

例: V.34 Modem
  波特率: 3429 baud
  调制阶数: 960 (V.34) 或 1664 (V.34+)
  比特率: 3429 × log₂(960) ≈ 3429 × 9.91 ≈ 33,977 bps
  
  考虑纠错编码开销后:
  有效比特率: 3429 × 8.4 ≈ 28,800 bps (V.34)
  有效比特率: 3429 × 9.8 ≈ 33,600 bps (V.34+)
```

**信噪比要求**：

```
不同Modem标准所需的最低SNR:

  V.22bis (2400 bps):    ~15 dB
  V.32    (9600 bps):    ~21 dB
  V.32bis (14400 bps):   ~24 dB
  V.34    (28800 bps):   ~30 dB
  V.34+   (33600 bps):   ~33 dB
  V.90    (56000 bps):   ~37 dB (下行)

SNR每增加3dB，理论上数据速率可以翻倍（根据香农定理）
```

## 组件B：编码器/解码器的设计

### B.1 硬件编码器的实现（以Manchester编码为例）

Manchester编码的硬件实现通常使用异或门和时钟信号：

```
Manchester编码器电路原理:

  数据信号 ───→ ┌───────┐
                │  XOR  │──→ Manchester输出
  时钟信号 ───→ └───────┘

逻辑:
  数据=1, 时钟=0 → 输出=1 (前半周期高)
  数据=1, 时钟=1 → 输出=0 (后半周期低)
  数据=0, 时钟=0 → 输出=0 (前半周期低)
  数据=0, 时钟=1 → 输出=1 (后半周期高)

时序图:
  数据:    ────┐    ┌────┐    ┌────
               │    │    │    │
               └────┘    └────┘
  
  时钟:    ─┐  ┌─┐  ┌─┐  ┌─┐  ┌─
            │  │ │  │ │  │ │  │ │
            └──┘ └──┘ └──┘ └──┘
  
  XOR输出: ──┐  ┌────┐  ┌────┐  ┌──
              │  │    │  │    │  │
              └──┘    └──┘    └──┘
  
  解释:
  数据=1, 时钟↓(0→1): XOR = 1⊕1 = 0 → 输出从高到低 (↓跳变)
  数据=1, 时钟↑(1→0): XOR = 1⊕0 = 1 → 输出保持在0? 不对...
```

实际上Manchester编码的标准实现更为精确：

```
Manchester编码器精确实现:

采用NRZ-L到Manchester的转换:
  Manchester = NRZ-L XOR 时钟(方波)
  
  规则:
  - 逻辑1: 前半周期低(-V), 后半周期高(+V)
  - 逻辑0: 前半周期高(+V), 后半周期低(-V)

Verilog实现:
```verilog
module manchester_encoder (
    input  wire clk,        // 系统时钟 (2倍数据速率)
    input  wire data_in,    // NRZ数据输入
    input  wire rst_n,      // 复位信号
    output wire man_out     // Manchester编码输出
);
    reg data_reg;
    reg clk_div;           // 分频时钟
    reg [1:0] state;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            clk_div <= 1'b0;
            data_reg <= 1'b0;
        end else begin
            clk_div <= ~clk_div;
            if (clk_div)  // 在分频时钟上升沿采样数据
                data_reg <= data_in;
        end
    end

    // Manchester编码: 数据 XOR 分频时钟
    assign man_out = data_reg ^ clk_div;

endmodule
```

### B.2 时钟恢复电路（PLL）

PLL（Phase Locked Loop，锁相环）是从接收到的信号中恢复时钟的关键电路：

```
PLL 结构框图:

                     ┌──────────────────────────────────┐
                     │              PLL                  │
                     │                                   │
  接收信号 ──→ ┌─────────┐ ┌──────────┐ ┌─────────┐ ──→ 恢复时钟
               │ 鉴相器   │→│ 环路滤波器 │→│  VCO    │
               │ (PD)     │ │  (LPF)   │ │(压控振荡)│
               └────┬─────┘ └──────────┘ └────┬────┘
                    │                          │
                    │    ┌──────────┐          │
                    └────│  分频器   │←─────────┘
                         │  (÷N)    │
                         └──────────┘

工作原理:
1. 鉴相器(PD): 比较输入信号和VCO输出的相位差
   输出电压 ∝ Δφ (相位差)

2. 环路滤波器(LPF): 滤除高频噪声，平滑控制电压
   决定PLL的响应速度和噪声带宽

3. VCO(压控振荡器): 根据控制电压调整输出频率
   f_out = f_center + K_vco × V_control

4. 反馈环路: VCO输出通过分频器反馈到鉴相器
   PLL会自动调整VCO使相位差趋近于零
```

**PLL在时钟恢复中的应用**：

```
从Manchester编码信号中恢复时钟:

  Manchester信号: ─┐  ┌────┐  ┌────┐  ┌──
                   │  │    │  │    │  │
                   └──┘    └──┘    └──┘
                   ↑↑      ↑↑      ↑↑
                   ││      ││      ││
                   每个比特中间都有跳变
                   ││
                   ↓↓
  PLL锁定这些跳变，产生恢复时钟:
  恢复时钟:     ─┐  ┌─┐  ┌─┐  ┌─┐  ┌─
                 │  │ │  │ │  │ │  │ │
                 └──┘ └──┘ └──┘ └──┘

  在每个比特中间的跳变处，PLL输出时钟边沿
  接收端在时钟边沿处采样数据，获得正确的比特值
```

**PLL的关键参数**：

```
1. 捕获范围(Capture Range): PLL能锁定的频率范围
   通常为 f_center ± 10% ~ 50%

2. 锁定范围(Lock Range): PLL保持锁定的频率范围
   通常大于捕获范围

3. 锁定时间(Lock Time): PLL从失锁到锁定所需的时间
   取决于环路带宽，典型值 10-1000 个信号周期

4. 相位噪声(Phase Noise): PLL输出的相位抖动
   影响恢复时钟的精度

5. 环路带宽(Loop Bandwidth): PLL的响应速度
   带宽大 → 响应快，但抗噪声能力差
   带宽小 → 响应慢，但抗噪声能力强
```

### B.3 编码的数学表示

各种编码方案可以用数学公式来精确描述：

**NRZ-L编码**：
```
s(t) = A × sign(d[n])    for nT ≤ t < (n+1)T

其中:
  d[n] ∈ {0, 1}  (第n个数据比特)
  sign(0) = -1, sign(1) = +1
  T = 比特周期
  A = 信号幅度
```

**Manchester编码**：
```
s(t) = A × sign(d[n]) × sign(c(t))    for nT ≤ t < (n+1)T

其中:
  c(t) = 方波时钟信号，周期为 T/2
  sign(c(t)) = +1  for nT ≤ t < nT + T/2
                -1  for nT + T/2 ≤ t < (n+1)T

等价形式:
  s(t) = -A    for nT ≤ t < nT + T/2, d[n]=1
  s(t) = +A    for nT + T/2 ≤ t < (n+1)T, d[n]=1
  s(t) = +A    for nT ≤ t < nT + T/2, d[n]=0
  s(t) = -A    for nT + T/2 ≤ t < (n+1)T, d[n]=0
```

**NRZI编码**：
```
s(t) = A × (-1)^(p[n])    for nT ≤ t < (n+1)T

其中:
  p[n] = p[n-1] ⊕ d[n]   (p[-1] = 0)
  ⊕ 表示异或运算

含义: 当 d[n]=1 时，p[n] 翻转（信号极性改变）
     当 d[n]=0 时，p[n] 不变（信号极性保持）
```

**频谱分析**：

```
编码信号的功率谱密度(PSD):

NRZ-L (随机数据):
  S(f) = A²T × sinc²(fT)
  主瓣带宽: 1/T = 比特率
  频谱从 DC (0 Hz) 开始

Manchester:
  S(f) = A²T × sinc²(fT/2) × sin²(πfT/2)
  主瓣带宽: 2/T = 2×比特率
  频谱从 DC 开始，但在 DC 处为零（无直流分量）

4B/5B + NRZI:
  带宽介于 NRZ 和 Manchester 之间
  主瓣带宽: 1.25/T = 1.25×比特率
```

### B.4 各编码方式的编码器实现差异

```
┌────────────────┬────────────────────────────────────────────────────┐
│    编码方式     │              硬件实现特点                           │
├────────────────┼────────────────────────────────────────────────────┤
│ NRZ-L          │ 最简单: 直接将数据比特转换为电平                    │
│                │ 电路: 查表或简单逻辑门                              │
│                │ 不需要额外时钟处理                                  │
├────────────────┼────────────────────────────────────────────────────┤
│ NRZI           │ 需要一个触发器记录当前状态                          │
│                │ 电路: D触发器 + XOR门                               │
│                │ 当数据=1时翻转输出，数据=0时保持                     │
├────────────────┼────────────────────────────────────────────────────┤
│ Manchester     │ 需要2倍数据速率的时钟                               │
│                │ 电路: XOR门 + 时钟分频器                            │
│                │ 实现简单，但时钟频率高                               │
├────────────────┼────────────────────────────────────────────────────┤
│ 差分Manchester │ 需要触发器和状态机                                  │
│                │ 电路: JK触发器 + 组合逻辑                           │
│                │ 比Manchester复杂，需要记忆前一比特的状态             │
├────────────────┼────────────────────────────────────────────────────┤
│ 4B/5B          │ 需要编码表(ROM或组合逻辑)                           │
│                │ 电路: 4→5 查表 + NRZI编码器                         │
│                │ 编码表较大，但可用简单的ROM实现                       │
├────────────────┼────────────────────────────────────────────────────┤
│ 8B/10B         │ 需要更大的编码表 + 运行不均等性(Running Disparity)   │
│                │ 电路: 8→10 查表 + 不均等性控制器                     │
│                │ 比4B/5B复杂得多，但编码效率更高                       │
└────────────────┴────────────────────────────────────────────────────┘
```

## 组件C：信道的物理特性与性能指标

### C.1 信道模型

**有线信道**：

```
有线信道等效电路模型:

  发送端                                    接收端
    │                                        │
    ├──[Rs]──┬──[L]──[R]──[L]──┬──[Rl]──┤
    │        │                  │        │
    │       [C]                [C]       │
    │        │                  │        │
    │        └──────────────────┘        │
    │              (地线)                 │

  参数:
    Rs = 源阻抗
    Rl = 负载阻抗
    R = 导线电阻（随长度增加）
    L = 电感（随长度增加）
    C = 并联电容（随长度增加）

  传输线方程 (电报方程):
    ∂V/∂z = -(R + jωL) × I
    ∂I/∂z = -(G + jωC) × V

  其中 G = 并联电导（绝缘损耗）
```

**无线信道**：

```
无线信道传播模型:

  发射功率 P_t
       │
       ↓
  ┌──────────┐
  │ 自由空间  │  路径损耗: L_fs = (4πd/λ)²
  │ 路径损耗  │  dB形式: L_fs(dB) = 20log₁₀(d) + 20log₁₀(f) + 32.44
  └────┬─────┘  (d单位: km, f单位: MHz)
       ↓
  ┌──────────┐
  │ 阴影衰落  │  对数正态分布，标准差 6-10 dB
  │ (Shadowing)  由建筑物、地形等大尺度障碍物引起
  └────┬─────┘
       ↓
  ┌──────────┐
  │ 多径衰落  │  瑞利分布或莱斯分布
  │ (Multipath)  由反射、衍射、散射引起
  └────┬─────┘
       ↓
  接收功率 P_r = P_t - L_fs - L_shadow - L_multipath (dB)
```

### C.2 噪声类型

通信系统中存在多种噪声源：

```
┌────────────────┬─────────────────────────────────────────────────────┐
│    噪声类型     │              特性与来源                              │
├────────────────┼─────────────────────────────────────────────────────┤
│ 热噪声         │ 来源: 导体中电子的随机热运动                          │
│ (Thermal Noise)│ 特性: 白噪声，功率谱密度均匀                          │
│                │ 功率: N = kTB                                        │
│                │  k = 1.38×10⁻²³ J/K (玻尔兹曼常数)                   │
│                │  T = 绝对温度(K)                                     │
│                │  B = 带宽(Hz)                                        │
│                │ 例: T=290K, B=1MHz → N = 4×10⁻¹⁵ W = -114 dBm      │
│                │ 不可消除，是通信系统的基本噪声极限                      │
├────────────────┼─────────────────────────────────────────────────────┤
│ 串扰           │ 来源: 相邻线路之间的电磁耦合                          │
│ (Crosstalk)    │ 类型:                                                  │
│                │  - 近端串扰(NEXT): 干扰源在同一端                      │
│                │  - 远端串扰(FEXT): 干扰源在另一端                      │
│                │ 特性: 与频率相关，频率越高串扰越严重                    │
│                │ 减小方法: 双绞线、屏蔽、增加线间距                      │
├────────────────┼─────────────────────────────────────────────────────┤
│ 脉冲噪声       │ 来源: 电气设备开关、雷电、引擎点火                     │
│ (Impulse Noise)│ 特性: 持续时间短，幅度大，不可预测                     │
│                │ 对数字通信影响严重（可能瞬间破坏多个比特）              │
│                │ 对模拟通信影响较小（听起来像"咔嗒"声）                 │
│                │ 对抗方法: 纠错编码、交织                               │
├────────────────┼─────────────────────────────────────────────────────┤
│ 互调噪声       │ 来源: 非线性器件产生的频率混合                         │
│ (Intermodulation)│ 当两个频率 f1, f2 通过非线性器件时，                  │
│                │ 产生 f1±f2, 2f1±f2, f1±2f2 等新频率                  │
│                │ 这些新频率可能落入信号频带内，造成干扰                  │
├────────────────┼─────────────────────────────────────────────────────┤
│ 宇宙噪声       │ 来源: 太阳、银河系等天体的辐射                         │
│ (Cosmic Noise) │ 主要影响卫星通信和射电天文                             │
│                │ 频率范围: 主要在 MHz-GHz                               │
└────────────────┴─────────────────────────────────────────────────────┘
```

### C.3 信道衰减与频率的关系

信道衰减（Attenuation）是信号在传输过程中功率的损失：

```
衰减的定义:
  A(dB) = 10 × log₁₀(P_in / P_out)
  
  或
  A(dB/km) = 衰减(dB) / 传输距离(km)
```

**不同传输介质的衰减特性**：

```
双绞线衰减 vs 频率:
  频率(kHz)    衰减(dB/km)    相对衰减
  ────────────────────────────────────
    1           0.5            1×
    10          1.5            3×
    100         5.0            10×
    1000        20.0           40×
    10000       100.0          200×

观察: 衰减随频率增加而急剧增加
原因: 趋肤效应、介质损耗、辐射损耗

趋肤效应:
  高频电流集中在导体表面流动
  有效截面积减小 → 电阻增加 → 衰减增加
  
  趋肤深度: δ = √(ρ/(πfμ))
  例: 铜在1MHz时，δ ≈ 66 μm
```

**光纤衰减 vs 波长**：

```
光纤衰减谱:

  衰减(dB/km)
    ↑
  5 │                    
    │ ╲                  ╱
  3 │  ╲                ╱
    │   ╲     ┌──┐     ╱
  1 │    ╲───┘  └───╲╱─────
    │     1310nm    1550nm
  0 └────────────────────────→ 波长(nm)
    800  1000  1200  1400  1600

两个低损耗窗口:
  1310 nm窗口: 衰减 ≈ 0.35 dB/km
  1550 nm窗口: 衰减 ≈ 0.20 dB/km (最低)

衰减来源:
  - 瑞利散射: ∝ 1/λ⁴ (短波长主导)
  - OH⁻离子吸收峰: 1385 nm (水峰)
  - 红外吸收: 长波长主导
```

### C.4 信道容量的物理限制

信道容量不仅受带宽和SNR限制，还受以下物理因素限制：

```
1. 热噪声极限:
   N = kTB
   最小可检测信号功率: P_min = kTB × SNR_min
   
   例: SNR_min = 10 dB, B = 1 MHz, T = 290 K
   P_min = 1.38×10⁻²³ × 290 × 10⁶ × 10 = 4×10⁻¹⁴ W = -104 dBm

2. 量子极限:
   光通信中的量子噪声: N_q = hνB
   其中 h = 普朗克常数, ν = 光频率
   
   量子极限下的最小接收光功率:
   P_min ≈ hνB × (1/η)
   η = 量子效率

3. 非线性限制:
   光纤中的非线性效应:
   - 自相位调制(SPM): 功率引起折射率变化
   - 交叉相位调制(XPM): 相邻信道干扰
   - 四波混频(FWM): 新频率产生
   - 受激布里渊散射(SBS): 功率阈值限制
   
   这些效应限制了光纤中的最大传输功率和信道数

4. 色散限制:
   - 模式色散: 多模光纤中不同模式的传播速度不同
   - 色度色散: 不同波长的光传播速度不同
   - 偏振模色散: 不同偏振态传播速度不同
   
   色散导致脉冲展宽，限制最大传输距离和速率
   色散限制: B² × L < D_max
   其中 B = 比特率, L = 距离, D_max = 色散容限
```

### C.5 多径效应与码间干扰

**多径效应（Multipath Effect）**：

```
多径传播示意:

  发射机 ─────────────────────────────────→ 直射路径 ──→ 接收机
     │                                        ↑
     └──→ 建筑物反射 ─────────────────────────┘
     │                                        ↑
     └──→ 地面反射 ───────────────────────────┘
     │                                        ↑
     └──→ 山体反射 ──────────→ 建筑物反射 ────┘

  多径导致:
  1. 接收信号是多个延迟版本的叠加
  2. 不同路径的信号可能相长或相消干涉
  3. 造成信号幅度和相位的快速波动（衰落）
```

**码间干扰（ISI, Inter-Symbol Interference）**：

```
码间干扰的产生:

  发送脉冲:    ┌───┐           ┌───┐
               │   │           │   │
               │   │           │   │
          ─────┘   └───────────┘   └─────

  经过信道后:  ╭───╮           ╭───╮
              ╱   ╲           ╱   ╲
             ╱     ╲    ←→   ╱     ╲
          ──╱───────╲───────╱───────╲──
            ↑       ↑       ↑
            前一符号的尾巴  当前符号的头部  后一符号的前奏
            干扰当前符号    干扰后一符号

  ISI的数学描述:
  接收信号: r(t) = Σ a[n] × h(t - nT) + n(t)
  
  其中 h(t) 是信道冲激响应
  当 h(t) 在 t = kT (k≠0) 处不为零时，产生ISI
```

### C.6 均衡器的作用

均衡器（Equalizer）用于补偿信道失真和消除ISI：

```
均衡器类型:

1. 线性均衡器 (Linear Equalizer):
   结构: FIR滤波器
   ┌──────────────────────────────────────────┐
   │  z⁻¹   z⁻¹   z⁻¹   z⁻¹                  │
   │──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──→ 输出  │
   │  │  │  │  │  │  │  │  │  │  │          │
   │ [c0][c1][c2][c3][c4][c5][c6][c7][c8][c9]│
   │  │  │  │  │  │  │  │  │  │  │          │
   │  └──┴──┴──┴──┴──┴──┴──┴──┴──┘          │
   │                +                        │
   │               输出                       │
   └──────────────────────────────────────────┘
   
   输出: y[n] = Σ c[k] × x[n-k]
   
   系数 c[k] 通过训练序列自适应调整
   常用算法: LMS (Least Mean Squares)
   
   缺点: 会放大噪声（在信道深衰落处）

2. 判决反馈均衡器 (DFE, Decision Feedback Equalizer):
   ┌─────────────────────────────────────────────┐
   │                ┌───────────┐                │
   │  输入 ──→ [前馈均衡器] ──→ + ──→ 判决 ──→ 输出│
   │                              ↑               │
   │                ┌───────────┐│               │
   │                │[反馈均衡器]│←───────────────│
   │                └───────────┘                │
   └─────────────────────────────────────────────┘
   
   使用已判决的符号来消除其对后续符号的干扰
   优点: 不会放大噪声
   缺点: 错误传播（一个判决错误可能影响后续多个符号）

3. 最大似然序列估计 (MLSE, Maximum Likelihood Sequence Estimation):
   使用Viterbi算法搜索最可能的发送序列
   理论上最优，但复杂度随信道记忆长度指数增长
```

**LMS自适应算法**：

```python
import numpy as np

def lms_equalizer(input_signal, desired, n_taps=11, mu=0.01):
    """
    LMS自适应均衡器
    
    参数:
      input_signal: 接收到的信号（含ISI）
      desired: 期望的输出（训练序列）
      n_taps: 均衡器抽头数
      mu: 步长（学习率）
    
    返回:
      output: 均衡后的信号
      weights: 最终的均衡器系数
    """
    n = len(input_signal)
    weights = np.zeros(n_taps)
    output = np.zeros(n)
    error = np.zeros(n)
    
    for i in range(n_taps, n):
        # 提取输入向量
        x = input_signal[i:i-n_taps:-1]
        if len(x) < n_taps:
            x = np.pad(x, (0, n_taps - len(x)))
        
        # 计算均衡器输出
        output[i] = np.dot(weights, x)
        
        # 计算误差
        error[i] = desired[i] - output[i]
        
        # LMS更新
        weights = weights + 2 * mu * error[i] * x
    
    return output, weights, error

# 仿真示例
np.random.seed(42)
n_symbols = 1000

# 生成BPSK信号 (+1/-1)
data = 2 * np.random.randint(0, 2, n_symbols) - 1

# 信道模型 (3径)
channel = np.array([0.5, 1.0, 0.3])  # 主径 + 2个多径
received = np.convolve(data, channel)[:n_symbols]

# 添加噪声
snr_db = 20
noise_power = 10 ** (-snr_db / 10)
noise = np.sqrt(noise_power) * np.random.randn(n_symbols)
received += noise

# LMS均衡
train_len = 200  # 训练序列长度
equalized, weights, error = lms_equalizer(
    received, data, n_taps=11, mu=0.01
)

# 计算均衡前后的BER
before_ber = np.mean(np.sign(received[:train_len]) != data[:train_len])
after_ber = np.mean(np.sign(equalized[train_len:]) != data[train_len:])
print(f"均衡前BER: {before_ber:.4f}")
print(f"均衡后BER: {after_ber:.4f}")
```

**均衡器设计的关键权衡**：

```
1. 抽头数 (n_taps):
   - 太少: 无法完全消除ISI
   - 太多: 计算复杂度增加，收敛慢
   - 经验法则: n_taps ≥ 信道记忆长度 × 3

2. 步长 (mu):
   - 太大: 收敛快但不稳定（稳态误差大）
   - 太小: 稳定但收敛慢
   - 经验法则: 0 < mu < 1/(n_taps × signal_power)

3. 训练序列长度:
   - 必须足够长以让均衡器收敛
   - 典型值: 100-1000个符号
   - 收敛后可切换到判决引导模式(Decision-Directed)
```
