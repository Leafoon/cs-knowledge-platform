## 30.1 VTA 概述

### 30.1.1 为什么需要可编程加速器？

深度学习硬件加速器面临一个核心矛盾：

- **专用加速器**（如 Google TPU）：高性能但不可编程，难以支持新算子
- **通用处理器**（如 CPU/GPU）：灵活但效率低，能耗比差

VTA 采用**可编程加速器**的折中方案：

| 方案 | 性能 | 灵活性 | 能效 | 开发成本 |
|------|------|--------|------|---------|
| **CPU** | 低 | 最高 | 低 | 低 |
| **GPU** | 高 | 高 | 中 | 中 |
| **专用 ASIC** | 最高 | 低 | 最高 | 最高 |
| **VTA (FPGA)** | 中高 | 中高 | 高 | 低 |

VTA 的设计目标：
1. **可编程**：支持多种算子（矩阵乘、向量运算、激活函数）
2. **可定制**：通过 FPGA 实现，可根据需求调整硬件参数
3. **TVM 集成**：与 TVM 编译器深度集成，自动化代码生成
4. **开源**：完整的硬件设计和软件栈

### 30.2.4 缓冲区层次

VTA 的片上存储采用三级缓冲区：

| 缓冲区 | 容量 | 访问模式 | 用途 |
|--------|------|---------|------|
| **Weight Buffer** | BLOCK_IN × BLOCK_OUT × WGT_BITS | 只读 | 存储权重块 |
| **Input Buffer** | BATCH × BLOCK_IN × INP_BITS | 只读 | 存储输入激活 |
| **Accumulator Buffer** | BATCH × BLOCK_OUT × ACC_BITS | 读写 | 存储累加结果 |
| **Uop Buffer** | μOP 条目数 | 只读 | 存储微操作指令 |

def annotate_vta_ops(mod):
    """将 Relay 算子注解为 VTA 操作"""

    class VTAAnnotator(relay.ExprMutator):
        def visit_call(self, call):
            if call.op == relay.op.get("nn.conv2d"):
                # 注解为 VTA GEMM 操作
                return vta.gemm(call.args[0], call.args[1], call.attrs)
            elif call.op == relay.op.get("nn.relu"):
                # 注解为 VTA ALU 操作
                return vta.alu("max", call.args[0], relay.const(0))
            elif call.op == relay.op.get("nn.max_pool2d"):
                # 注解为 VTA ALU 操作
                return vta.alu("max", call.args[0], call.attrs)
            return super().visit_call(call)

    return VTAAnnotator().visit(mod)

# 大矩阵乘法：C[M, N] = A[M, K] @ B[K, N]
# 分块为：C[m, n] = Σ_k A[m, k] @ B[k, n]
# 其中 m ∈ [0, M/BATCH), n ∈ [0, N/BLOCK_OUT), k ∈ [0, K/BLOCK_IN)

def vta_tiling_schedule(s, C, M, N, K):
    """VTA GEMM 的分块调度"""
    # 获取循环轴
    m, n = s[C].op.axis
    k = s[C].op.reduce_axis[0]

    # 分块
    mo, mi = s[C].split(m, factor=BATCH)
    no, ni = s[C].split(n, factor=BLOCK_OUT)
    ko, ki = s[C].split(k, factor=BLOCK_IN)

    # 重排循环顺序
    # 外层：遍历所有块
    # 内层：VTA 硬件处理单个块
    s[C].reorder(mo, no, ko, mi, ki, ni)

    # 外层并行
    s[C].parallel(mo)

    return s
```

import tvm
from tvm import te
import vta

def vta_custom_gemm(A, B, M, N, K):
    """自定义 VTA GEMM 算子"""
    env = vta.get_env()

    # 定义计算
    k = te.reduce_axis((0, K), name="k")
    C = te.compute(
        (M, N),
        lambda i, j: te.sum(A[i, k] * B[k, j], axis=k),
        name="C"
    )

    # VTA 调度
    s = te.create_schedule(C.op)

    # 分块适配 VTA 硬件参数
    batch = env.BATCH
    block_in = env.BLOCK_IN
    block_out = env.BLOCK_OUT

    m, n = s[C].op.axis
    k = s[C].op.reduce_axis[0]

    mo, mi = s[C].split(m, factor=batch)
    no, ni = s[C].split(n, factor=block_out)
    ko, ki = s[C].split(k, factor=block_in)

    s[C].reorder(mo, no, ko, mi, ki, ni)

    # 绑定到 VTA
    s[C].pragma(mo, "vta.gemm")

    return s, C

### 30.99.2 业务意义

1. VTA 的业务价值不只是让模型跑得更快，而是让同一个模型可以在不同成本、功耗和延迟约束下交付。
2. 在服务器场景中，核心指标通常是吞吐、P95/P99 延迟、资源利用率和多租户隔离。
3. 在移动端场景中，核心指标通常是首帧时间、持续发热、内存峰值和包体大小。
4. 在嵌入式场景中，核心指标通常是 Flash 占用、静态内存、实时性和掉电恢复能力。
5. 在云端批处理场景中，编译时间可以接受更长，但调优记录和缓存复用变得非常重要。
6. 在在线服务场景中，编译产物需要可回滚、可审计、可灰度，而不能只在开发机上验证。
7. 业务方关心的是 SLA、成本和稳定性，编译器工程师关心的是 IR 正确性、优化空间和后端能力。
8. 优秀的 TVM 项目需要把这两类语言翻译成共同的指标体系。
9. 当优化收益只有少量百分点时，应评估它是否值得引入新的维护复杂度。
10. 当优化收益很大但只在少数输入上成立时，应评估输入分布变化后的风险。

### 30.99.3 TVM 内部机制

1. TVM 的核心机制是分层表示和逐层降低，高层 IR 保留模型语义，低层 IR 暴露循环、内存和并行结构。
2. Relay 更适合表达静态计算图和传统深度学习算子。
3. Relax 更强调动态形状、函数式组合和跨层优化。
4. TIR 是最终性能调优的关键，因为它决定循环嵌套、内存作用域、向量化和线程映射。
5. PassContext 会影响优化级别、禁用或启用的 pass，以及某些后端特定配置。
6. Target 不只是字符串，它包含硬件特征、指令集、运行时约定和 codegen 选择。
7. Executor 决定模型入口如何组织，Runtime 决定编译产物如何加载和调用。
8. 参数绑定会影响常量折叠、内存规划和代码体积。
9. Layout rewrite 会影响算子融合和后端库调用。
10. AutoTVM 与 MetaSchedule 的调优记录会把搜索结果沉淀为可复用资产。

### 30.99.4 适用场景

1. 当模型结构相对稳定、目标硬件明确、性能收益可以通过基准测试确认时，VTA 相关技术最容易发挥价值。
2. 当团队需要支持多种硬件后端时，TVM 的统一 IR 和 Target 抽象可以降低重复适配成本。
3. 当模型中存在框架运行时开销、算子融合机会或布局转换冗余时，编译优化通常能带来明显收益。
4. 当部署环境不能依赖完整 Python 栈时，AOT、CRT 或导出后的 runtime artifact 更有意义。
5. 当硬件厂商提供高性能库但模型图需要复杂切分时，BYOC 和外部 codegen 是常见选择。
6. 当输入形状变化频繁时，应提前设计 shape 策略，而不是在上线前才补动态形状支持。
7. 当模型版本迭代频繁时，应把编译、调优、验证和发布纳入 CI/CD。
8. 当业务对精度非常敏感时，应把优化收益和数值回归一起评估。
9. 当系统存在多模型串联时，应评估端到端 pipeline，而不是只优化单个模型。
10. 当部署设备数量很大时，编译产物的一致性和可追踪性比单次实验性能更重要。

### 30.99.5 限制条件

1. TVM 并不能自动解决所有性能问题，尤其不能替代对模型结构、硬件层次和数据分布的理解。
2. 如果前端导入阶段已经丢失语义，后续 pass 很难恢复原始意图。
3. 如果目标硬件缺少成熟 codegen 或 runtime 支持，理论上的 IR 优化收益可能无法兑现。
4. 如果模型包含大量小算子，调度优化可能被调用开销和内存同步抵消。
5. 如果输入分布与校准集或 benchmark 数据差异较大，性能和精度结论都可能失真。
6. 如果编译参数没有固定，调优结果难以复现。
7. 如果只看平均延迟，可能忽略 P99 抖动和内存峰值。
8. 如果只在开发机验证，可能忽略生产驱动、内核版本和设备温控策略。
9. 如果项目缺少 IR dump 和 artifact 存档，出现线上问题时很难追溯。
10. 如果团队没有维护编译工具链的能力，过度定制会形成长期负担。

### 30.99.6 工程经验

1. 第一条经验是先建立可靠 baseline，再做任何复杂优化。
2. baseline 应包含原框架结果、TVM 未调优结果、TVM 调优结果和目标平台参考结果。
3. 第二条经验是每次只改变一个变量，例如 target、layout、executor、batch 或调优记录。
4. 第三条经验是把编译日志、IR dump、调优数据库和运行时产物一起归档。
5. 第四条经验是为每个模型维护最小可复现输入，便于定位前端导入和数值误差。
6. 第五条经验是把 shape、dtype、layout 写入模型契约，而不是散落在脚本里。
7. 第六条经验是使用真实业务样本做最终验证，因为随机输入无法覆盖预处理和分布偏移。
8. 第七条经验是把冷启动和热启动分开统计。
9. 第八条经验是为不同硬件维护独立调优记录，不要假设一个 schedule 可以跨平台复用。
10. 第九条经验是对编译产物做哈希和版本标记，方便灰度和回滚。

### 30.99.7 常见误区

1. 误区一是认为 TVM build 成功就等于模型可以生产上线。
2. 误区二是认为单次 benchmark 的最小值代表真实线上延迟。
3. 误区三是把调优时间算进推理收益，或者完全忽略调优成本。
4. 误区四是只关注算子融合数量，却不检查内存带宽和 cache 行为。
5. 误区五是看到 INT8 就默认更快，却忽略硬件是否有高效低精度指令。
6. 误区六是看到动态形状支持就默认所有输入尺寸都高效。
7. 误区七是把外部库调用当成万能方案，却忽略数据布局转换成本。
8. 误区八是把示例代码复制到生产环境，却没有补齐错误处理和 artifact 管理。
9. 误区九是只比较平均精度，不检查关键类别或长尾输入。
10. 误区十是把编译器问题、模型问题和硬件问题混在一起排查。

### 30.99.9 与同类系统对比

1. 与 TensorRT 相比，TVM 的硬件覆盖更开放，TensorRT 在 NVIDIA GPU 上的工程成熟度更高。
2. 与 XLA 相比，TVM 更强调可调度性和多后端扩展，XLA 更强调与框架图执行的深度集成。
3. 与 MLIR 相比，TVM 更像面向深度学习部署的完整编译器，MLIR 更像可构建编译器的基础设施。
4. 与 ONNX Runtime 相比，TVM 更关注提前编译和内核生成，ONNX Runtime 更强调运行时图优化和 execution provider 生态。
5. 与 Triton 相比，TVM 覆盖端到端模型编译，Triton 更适合手写或自动生成 GPU kernel。
6. 与 TFLite 相比，TVM 的后端扩展更灵活，TFLite 在移动端生态和模型格式上更标准化。
7. 与厂商 NPU SDK 相比，TVM 更中立，厂商 SDK 往往能访问更底层的私有能力。
8. 选择系统时不应只看峰值性能，还应看调试成本、团队经验、社区活跃度和长期维护风险。
9. 如果项目只部署到单一成熟硬件，专用推理引擎可能更省事。
10. 如果项目需要跨硬件、跨模型长期演进，TVM 的编译器化路线更有战略价值。

// 3rdparty/vta-hw/hardware/chisel/src/vta/GEMM.scala（简化）

class GEMM(implicit p: Parameters) extends Module {
  val io = IO(new Bundle {
    val i_post = Input(Bool())           // 启动信号
    val inp = Input(Vec(BLOCK_IN, SInt(INP_WIDTH.W)))   // 输入向量
    val wgt = Input(Vec(BLOCK_IN, Vec(BLOCK_OUT, SInt(WGT_WIDTH.W)))) // 权重矩阵
    val acc = Output(Vec(BLOCK_OUT, SInt(ACC_WIDTH.W))) // 累加结果
  })

  // 脉动阵列：BLOCK_IN × BLOCK_OUT 个 MAC 单元
  val mac = VecInit.tabulate(BLOCK_IN, BLOCK_OUT) { (i, j) =>
    val prod = io.inp(i) * io.wgt(i)(j)  // INT8 × INT8
    prod
  }

  // 累加器
  val accumulator = RegInit(VecInit(Seq.fill(BLOCK_OUT)(0.S(ACC_WIDTH.W))))

  when(io.i_post) {
    // 重置累加器
    accumulator := VecInit(Seq.fill(BLOCK_OUT)(0.S(ACC_WIDTH.W)))
  }.otherwise {
    // 累加 MAC 结果
    for (j <- 0 until BLOCK_OUT) {
      accumulator(j) := accumulator(j) + mac.map(_(j)).reduce(_ + _)
    }
  }

  io.acc := accumulator
}
```

class LoadStore(implicit p: Parameters) extends Module {
  val io = IO(new Bundle {
    val start = Input(Bool())          // 启动信号
    val dram_addr = Input(UInt(32.W)) // DRAM 地址
    val sram_addr = Input(UInt(16.W)) // SRAM 地址
    val x_size = Input(UInt(16.W))    // 传输宽度
    val y_size = Input(UInt(16.W))    // 传输高度
    val x_stride = Input(UInt(16.W))  // 步长
    val done = Output(Bool())          // 完成信号
  })

  // DMA 状态机
  val sIdle :: sBusy :: sDone :: Nil = Enum(3)
  val state = RegInit(sIdle)

  val x_cnt = RegInit(0.U(16.W))
  val y_cnt = RegInit(0.U(16.W))

  switch(state) {
    is(sIdle) {
      when(io.start) {
        state := sBusy
        x_cnt := 0.U
        y_cnt := 0.U
      }
    }
    is(sBusy) {
      // 2D DMA 传输
      when(x_cnt === io.x_size - 1.U) {
        x_cnt := 0.U
        when(y_cnt === io.y_size - 1.U) {
          state := sDone
        }.otherwise {
          y_cnt := y_cnt + 1.U
        }
      }.otherwise {
        x_cnt := x_cnt + 1.U
      }
    }
    is(sDone) {
      state := sIdle
    }
  }

  io.done := (state === sDone)
}
```

def generate_vta_instructions(conv_params):
    """为卷积生成 VTA 指令序列"""

    instructions = []

    # 1. 加载权重
    for i in range(conv_params["num_weight_blocks"]):
        instructions.append({
            "type": "LOAD",
            "mem_type": "weight",
            "sram_offset": i * BLOCK_IN * BLOCK_OUT,
            "dram_offset": conv_params["weight_dram_offset"] + i * ...,
            "y_size": BLOCK_IN,
            "x_size": BLOCK_OUT,
        })

    # 2. 计算循环
    for oc in range(conv_params["output_channels"] // BLOCK_OUT):
        for ic in range(conv_params["input_channels"] // BLOCK_IN):
            # 加载输入
            instructions.append({
                "type": "LOAD",
                "mem_type": "input",
                "sram_offset": 0,
                "dram_offset": ...,
                "y_size": 1,
                "x_size": BLOCK_IN,
            })

            # GEMM 计算
            instructions.append({
                "type": "GEMM",
                "reset_acc": (ic == 0),
                "uop_begin": 0,
                "uop_end": BLOCK_IN * BLOCK_OUT,
                "loop_extent": 1,
            })

        # 激活函数（ALU）
        instructions.append({
            "type": "ALU",
            "alu_type": "MAX",  # ReLU
            "use_imm": True,
            "imm": 0,
            "loop_extent": BLOCK_OUT,
        })

        # 存储结果
        instructions.append({
            "type": "STORE",
            "mem_type": "output",
            "sram_offset": 0,
            "dram_offset": ...,
        })

    return instructions

def vta_simulation(model, input_data):
    """在 CPU 上模拟 VTA 执行"""

    # 1. 加载 VTA 配置
    config = load_vta_config("vta_config.json")

    # 2. 创建模拟器
    simulator = VTASimulator(config)

    # 3. 编译模型为 VTA 指令
    instructions = compile_to_vta_instructions(model)

    # 4. 执行模拟
    for insn in instructions:
        simulator.execute(insn)

    # 5. 获取结果
    output = simulator.get_output()

    return output

class VTASimulator:
    """VTA 软件模拟器"""

    def __init__(self, config):
        self.config = config
        # 片上缓冲区
        self.weight_buffer = np.zeros(
            (config["BLOCK_IN"], config["BLOCK_OUT"]),
            dtype=np.int8
        )
        self.input_buffer = np.zeros(
            (config["BATCH"], config["BLOCK_IN"]),
            dtype=np.int8
        )
        self.acc_buffer = np.zeros(
            (config["BATCH"], config["BLOCK_OUT"]),
            dtype=np.int32
        )

    def execute(self, instruction):
        """执行一条 VTA 指令"""
        if instruction["type"] == "LOAD":
            self.execute_load(instruction)
        elif instruction["type"] == "GEMM":
            self.execute_gemm(instruction)
        elif instruction["type"] == "ALU":
            self.execute_alu(instruction)
        elif instruction["type"] == "STORE":
            self.execute_store(instruction)

    def execute_gemm(self, insn):
        """执行 GEMM 指令"""
        result = np.dot(
            self.input_buffer.astype(np.int32),
            self.weight_buffer.astype(np.int32).T
        )
        if insn.get("reset_acc", False):
            self.acc_buffer = result
        else:
            self.acc_buffer += result
```

    """估算 VTA 配置所需的 FPGA 资源"""

    BLOCK_IN = config["BLOCK_IN"]
    BLOCK_OUT = config["BLOCK_OUT"]
    INP_WIDTH = config["INP_WIDTH"]
    WGT_WIDTH = config["WGT_WIDTH"]
    ACC_WIDTH = config["ACC_WIDTH"]

    # GEMM 单元：BLOCK_IN × BLOCK_OUT 个 MAC
    mac_count = BLOCK_IN * BLOCK_OUT
    dsp_count = mac_count
    lut_count = mac_count * 20

    # 缓冲区
    weight_buffer_size = BLOCK_IN * BLOCK_OUT * WGT_WIDTH / 8
    input_buffer_size = BLOCK_IN * INP_WIDTH / 8
    acc_buffer_size = BLOCK_OUT * ACC_WIDTH / 8
    total_bram = (weight_buffer_size + input_buffer_size + acc_buffer_size) / 1024

    resources = {
        "dsp_slices": dsp_count,
        "lut_estimate": lut_count,
        "bram_kb": total_bram,
        "ff_estimate": lut_count * 2,
    }

    return resources

package vta

import chisel3._
import chisel3.util._

// 128 位指令字段定义
class VTAInstruction extends Bundle {
  val opcode = UInt(8.W)     // 操作码
  val pad = UInt(8.W)        // 保留字段
  val src = UInt(16.W)       // 源地址
  val dst = UInt(16.W)       // 目标地址
  val size = UInt(16.W)      // 数据大小
  val param = UInt(16.W)     // 附加参数
  val immediate = UInt(48.W) // 立即数
}

// 指令编码器
class InstructionEncoder extends Module {
  val io = IO(new Bundle {
    val opcode = Input(UInt(8.W))
    val src = Input(UInt(16.W))
    val dst = Input(UInt(16.W))
    val size = Input(UInt(16.W))
    val param = Input(UInt(16.W))
    val immediate = Input(UInt(48.W))
    val instruction = Output(UInt(128.W))
  })

  // 将字段组装为 128 位指令
  io.instruction := Cat(
    io.immediate,   // [127:80]
    io.param,       // [79:64]
    io.size,        // [63:48]
    io.dst,         // [47:32]
    io.src,         // [31:16]
    0.U(8.W),       // [15:8] pad
    io.opcode       // [7:0]
  )
}

// LOAD 指令编码示例
object LoadInstruction {
  // 从 DRAM 加载输入数据到 INP 缓冲区
  def apply(dram_addr: UInt, sram_addr: UInt,
            size: UInt, pad: Int = 0): UInt = {
    val inst = Wire(new VTAInstruction)
    inst.opcode := "h00".U           // LOAD 操作码
    inst.pad := pad.U
    inst.src := dram_addr             // DRAM 地址
    inst.dst := sram_addr             // SRAM 地址
    inst.size := size                 // 传输大小
    inst.param := "h0004".U           // INP 子类型
    inst.immediate := 0.U             // 无立即数
    Cat(inst.immediate, inst.param, inst.size,
        inst.dst, inst.src, inst.pad, inst.opcode)
  }
}

// GEMM 指令编码示例
object GemmInstruction {
  // 执行矩阵乘法: C[dst:dst+size] += A[src:src+size] * B
  def apply(src: UInt, dst: UInt, size: UInt,
            uop_begin: UInt, uop_end: UInt): UInt = {
    val inst = Wire(new VTAInstruction)
    inst.opcode := "h02".U           // GEMM 操作码
    inst.pad := 0.U
    inst.src := src                   // 输入 SRAM 地址
    inst.dst := dst                   // 累加器 SRAM 地址
    inst.size := size                 // 矩阵维度
    inst.param := Cat(uop_end, uop_begin)  // μOP 范围
    inst.immediate := 0.U
    Cat(inst.immediate, inst.param, inst.size,
        inst.dst, inst.src, inst.pad, inst.opcode)
  }

// 文件: vta/hardware/chisel/src/main/scala/vta/GEMM.scala
// VTA GEMM 单元实现

package vta

import chisel3._
import chisel3.util._

// GEMM 单元配置
case class GEMMConfig(
  batch: Int = 1,           // 批次大小
  inBlock: Int = 16,        // 输入块大小
  outBlock: Int = 16,       // 输出块大小
  bits: Int = 8,            // 数据位宽
  accBits: Int = 32         // 累加器位宽
)

// 单个 MAC 单元
class MACUnit(implicit config: GEMMConfig) extends Module {
  val io = IO(new Bundle {
    val a = Input(SInt(config.bits.W))      // 输入 A
    val b = Input(SInt(config.bits.W))      // 输入 B
    val c = Input(SInt(config.accBits.W))   // 部分和输入
    val out = Output(SInt(config.accBits.W)) // 部分和输出
  })

  // 乘累加: out = a * b + c
  io.out := (io.a * io.b) + io.c
}

// 脉动阵列
class SystolicArray(implicit config: GEMMConfig) extends Module {
  val n = config.outBlock  // 阵列大小 (N×N)
  val m = config.inBlock   // 输入块大小

  val io = IO(new Bundle {
    val a = Input(Vec(n, SInt(config.bits.W)))      // 输入行
    val b = Input(Vec(n, SInt(config.bits.W)))      // 权重列
    val valid = Input(Bool())                         // 有效信号
    val out = Output(Vec(n, SInt(config.accBits.W))) // 输出行
  })

  // 创建 N×N MAC 阵列
  val macs = Seq.fill(n, n)(Module(new MACUnit))

  // 水平连接（输入数据流动）
  for (i <- 0 until n) {
    for (j <- 0 until n) {
      if (j == 0) {
        // 第一列：从输入端口获取数据
        macs(i)(j).io.a := io.a(i)
      } else {
        // 后续列：从前一列获取数据（流水线寄存器）
        macs(i)(j).io.a := RegNext(macs(i)(j-1).io.a)
      }
    }
  }

  // 垂直连接（权重数据流动）
  for (j <- 0 until n) {
    for (i <- 0 until n) {
      if (i == 0) {
        // 第一行：从输入端口获取权重
        macs(i)(j).io.b := io.b(j)
      } else {
        // 后续行：从上一行获取权重（流水线寄存器）
        macs(i)(j).io.b := RegNext(macs(i-1)(j).io.b)
      }
    }
  }

  // 部分和累加（垂直方向）
  for (i <- 0 until n) {
    for (j <- 0 until n) {
      if (i == 0) {
        // 第一行：初始部分和为 0
        macs(i)(j).io.c := 0.S
      } else {
        // 后续行：累加上一行的部分和
        macs(i)(j).io.c := RegNext(macs(i-1)(j).io.out)
      }
    }
  }

  // 输出：最后一行的结果
  for (j <- 0 until n) {
    io.out(j) := macs(n-1)(j).io.out
  }
}

// GEMM 顶层模块
class GEMMUnit(implicit config: GEMMConfig) extends Module {
  val io = IO(new Bundle {
    val uop = new MicroOpIO           // 微操作接口
    val inp = new SRAMReadIO           // 输入 SRAM 读接口
    val wgt = new SRAMReadIO           // 权重 SRAM 读接口
    val acc = new SRAMReadWriteIO      // 累加器 SRAM 读写接口
    val start = Input(Bool())          // 启动信号
    val done = Output(Bool())          // 完成信号
  })

  // 实例化脉动阵列
  val array = Module(new SystolicArray)

  // 控制状态机
  val sIdle :: sLoad :: sCompute :: sDone :: Nil = Enum(4)
  val state = RegInit(sIdle)
  val cycle_counter = RegInit(0.U(log2Ceil(config.inBlock + 1).W))

  // 状态转移
  switch(state) {
    is(sIdle) {
      when(io.start) {
        state := sLoad
      }
    }
    is(sLoad) {
      // 加载数据到脉动阵列
      array.io.valid := true.B
      cycle_counter := cycle_counter + 1.U
      when(cycle_counter === (config.inBlock - 1).U) {
        state := sCompute
        cycle_counter := 0.U
      }
    }
    is(sCompute) {
      // 执行计算
      cycle_counter := cycle_counter + 1.U
      when(cycle_counter === (config.inBlock - 1).U) {
        state := sDone
      }
    }
    is(sDone) {
      io.done := true.B
      state := sIdle
    }
  }
}

def run_vta_fpga_synthesis(board="pynq-z1", clock_mhz=100):
    """VTA FPGA 综合自动化脚本

    参数:
        board: 目标板卡 (pynq-z1, ultra96, etc.)
        clock_mhz: 目标时钟频率 (MHz)
    """

    import subprocess
    import os

    # 板卡配置
    board_configs = {
        "pynq-z1": {
            "part": "xc7z020clg484-1",
            "dram": "512MB",
            "clock_period_ns": 10.0,  # 100MHz
        },
        "ultra96": {
            "part": "xczu3eg-sbva484-1-e",
            "dram": "2GB",
            "clock_period_ns": 5.0,   # 200MHz
        },
    }

    config = board_configs[board]
    config["clock_period_ns"] = 1000.0 / clock_mhz

    # 1. 生成 Chisel Verilog
    print("Step 1: 生成 Verilog...")
    subprocess.run([
        "sbt", "runMain vta.Generator",
        f"--target-dir", "./vta_verilog",
        f"--config", board,
    ], check=True)

    # 2. 运行 Vivado 综合
    print("Step 2: 运行 Vivado 综合...")
    tcl_script = generate_tcl_script(config)
    with open("synthesize.tcl", "w") as f:
        f.write(tcl_script)

    subprocess.run([
        "vivado", "-mode", "batch", "-source", "synthesize.tcl"
    ], check=True)

    # 3. 检查结果
    print("Step 3: 检查综合结果...")
    check_synthesis_results()

    # 4. 生成比特流
    bitstream_path = "./vta_project/vta_project.runs/impl_1/VTATop.bit"

        print(f"综合成功！比特流文件: {bitstream_path}")
        # 复制到部署目录
        os.makedirs("./deploy", exist_ok=True)
        subprocess.run(["cp", bitstream_path, "./deploy/vta.bit"])

    完整流程:
    1. 比特流编程
    2. 加载 VTA 驱动
    3. 配置 TVM 运行时

    import numpy as np

    # 2. 获取 VTA 设备句柄
    vta_dev = overlay.vta

    # 3. 配置 VTA 参数
    # 通过 AXI-Lite 寄存器配置

    remote = rpc.connect("pynq", 9090)

    dev = remote.vta(0)
    runtime = graph_executor.GraphModule(lib["default"](dev))

    # 5. 准备输入数据
    input_data = np.random.randint(
        -128, 127, size=(1, 3, 224, 224), dtype=np.int8
    )

    runtime.run()
    output = runtime.get_output(0).numpy()

    print(f"推理完成，输出形状: {output.shape}")
    print(f"预测类别: {np.argmax(output)}")

    return output
