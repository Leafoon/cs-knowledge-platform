# 第三十三章 DMA方式

DMA（Direct Memory Access）让数据传输不经过CPU，由DMA控制器直接控制总线，大幅提高高速设备的数据传输效率。**DMA的三种传送方式是考研重点**。

---

## 33.1 DMA的基本概念

### 33.1.1 定义

DMA（Direct Memory Access）：数据在**主存和I/O设备之间**直接传输，**不经过CPU**。

### 33.1.2 DMA与中断方式的对比

| 特性 | 中断方式 | DMA方式 |
|------|---------|---------|
| 数据传输 | **经CPU** | **不经CPU** |
| CPU参与 | 每次传输都参与 | 只在开始和结束时 |
| 传输单位 | 字 | **数据块** |
| 适用速度 | 低速设备 | **高速设备** |
| 中断用途 | 每次传输完成 | 一批数据传输完成 |

### 33.1.3 DMA的工作原理

```
1. CPU设置DMA控制器参数（预处理）
2. DMA控制器接管总线
3. 数据在主存和I/O设备之间直接传输
4. 传输完成，DMA向CPU发中断
5. CPU进行后处理
```

### 33.1.5 DMA与CPU、存储器的连接方式

**方式一：DMA共享系统总线**

```
    CPU ════════════╗
                    ║ 系统总线
    DMA ════════════╬══════ 主存
                    ║
    I/O ════════════╝
```

- DMA和CPU分时使用系统总线
- 需要总线仲裁
- 结构简单，成本低

**方式二：DMA专用总线**

```
    CPU ═══════════════════ 主存
                    ║
                    ║ 共享系统总线
                    ║
    DMA ════════════╩══════ 主存
         (专用数据通路)
```

- DMA有独立的数据通路
- CPU和DMA可以并行工作
- 结构复杂，成本高

> [!NOTE]
> 大多数系统采用方式一（共享总线），通过总线仲裁来协调CPU和DMA对总线的访问。

### 33.2.1 DMA控制器中的寄存器

### 33.2.3 各寄存器的详细说明

**主存地址寄存器（AR/MAR）**：

| 字段 | 位数 | 说明 |
|------|------|------|
| 地址值 | 32位（现代系统） | 存放下一个要访问的主存地址 |
| 自动递增 | 硬件逻辑 | 每次传送后自动+1（或+4/8） |
| 初始化 | CPU写入 | 预处理时设置起始地址 |

**字计数器（WC/BC）**：

| 字段 | 位数 | 说明 |
|------|------|------|
| 计数值 | 16~32位 | 剩余要传送的数据单位数 |
| 溢出检测 | 硬件逻辑 | 计数值减到0时产生结束信号 |
| 初始化 | CPU写入 | 预处理时设置传送数据量 |

> [!TIP]
> 字计数器通常使用补码形式存储。例如要传送N个字，初始化时写入-N，每次加1，溢出时表示传送完成。

**控制/状态寄存器（CSR）**：

```
CSR各位定义：
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│  D7  │  D6  │  D5  │  D4  │  D3  │  D2  │  D1  │  D0  │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ 传送 │ DMA  │ 传送 │ 错误 │ 传送 │ 传送 │ 设备 │ DMA  │
│ 完成 │ 允许 │ 方向 │ 标志 │ 类型 │ 速度 │ 选择 │ 请求 │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘

D7: 传送完成标志 (1=完成, 0=未完成)
D6: DMA允许 (1=允许, 0=禁止)
D5: 传送方向 (1=读设备写主存, 0=读主存写设备)
D4: 错误标志 (1=有错误, 0=无错误)
D3-D2: 传送类型 (00=单字, 01=块, 10=请求, 11=级联)
D1-D0: 设备选择
```

**预处理（CPU执行）**：
1. 设置传送方向（读/写）
2. 设置主存起始地址（→AR）
3. 设置传送数据量（→WC）
4. 启动设备

**数据传送（DMA控制器执行）**：
1. 设备准备好数据 → DMA请求
2. DMA向总线仲裁器请求总线
3. 获得总线 → 数据传送
4. 每传一个字：AR+1, WC-1
5. WC溢出（=0）→ 传送结束

**后处理（CPU执行）**：
1. DMA向CPU发中断
2. CPU检查数据
3. 判断是否继续

## 33.3 DMA控制器与总线的连接

### 33.3.1 DMA的总线请求与应答

```
┌──────────────────────────────────────────────────────────────┐
│                    DMA总线请求/应答时序                        │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  时钟 ──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐        │
│          │ │  │ │  │ │  │ │  │ │  │ │  │ │  │ │  │        │
│          └─┘  └─┘  └─┘  └─┘  └─┘  └─┘  └─┘  └─┘  └─┘      │
│                                                              │
│  HRQ    ─────────────────┐                                   │
│  (DMA→CPU)               │ 请求总线                          │
│                          └───────────────────────────────    │
│                                                              │
│  HLDA   ──────────────────────────┐                          │
│  (CPU→DMA)                        │ 授予总线                  │
│                                   └──────────────────────    │
│                                                              │
│  地址    ══════════════════════════╗═════════════════════     │
│  总线                             ║ DMA驱动地址总线           │
│                                   ╚═════════════════════     │
│                                                              │
│  数据    ══════════════════════════╦═════════════════════     │
│  总线                             ║ DMA传送数据               │
│                                   ╚═════════════════════     │
└──────────────────────────────────────────────────────────────┘
```

**信号说明**：

| 信号 | 方向 | 说明 |
|------|------|------|
| HRQ (Hold Request) | DMA→CPU | DMA请求总线控制权 |
| HLDA (Hold Acknowledge) | CPU→DMA | CPU同意让出总线 |
| DREQ | I/O→DMA | I/O设备请求DMA传送 |
| DACK | DMA→I/O | DMA应答设备请求 |

### 33.3.2 DMA与CPU的总线竞争

当CPU和DMA同时需要使用总线时，需要仲裁：

```
┌──────────────────────────────────────────────────────────────┐
│                    总线仲裁逻辑                               │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  CPU请求 ──→┐                                                │
│             │    ┌──────────────┐                            │
│  DMA请求 ──→┼───→│  总线仲裁器   │───→ 总线授予信号            │
│             │    │  (优先级比较) │                            │
│  外设请求 ──→┘    └──────────────┘                            │
│                                                              │
│  仲裁策略：                                                   │
│  ① 固定优先级：DMA > CPU（常用）                               │
│  ② 轮询优先级：轮流服务                                       │
│  ③ 动态优先级：根据紧急程度调整                                │
└──────────────────────────────────────────────────────────────┘
```

**DMA优先于CPU的原因**：
- DMA通常服务于高速设备，延迟敏感
- 磁盘等设备有严格的时间约束
- CPU可以等待，但高速设备数据可能丢失

> [!NOTE]
> 在周期挪用方式中，DMA只在CPU不访存时窃取总线周期，此时不需要显式的总线仲裁。但在停止CPU访问方式中，DMA需要完全接管总线，此时必须有总线仲裁。

**交替访问的硬件要求**：

| 要求 | 说明 |
|------|------|
| 存储器速度 | 必须是CPU访存速度的2倍 |
| 时钟同步 | CPU和DMA的时钟必须同步 |
| 地址锁存 | 需要锁存CPU和DMA的地址 |
| 数据缓冲 | 需要数据缓冲寄存器 |

### 33.4.4 三种传送方式的详细对比

| 对比维度 | 停止CPU访问 | 周期挪用 | 交替访问 |
|---------|------------|---------|---------|
| CPU利用率 | **最低** | 中 | **最高** |
| DMA传送效率 | **最高** | 低 | 高 |
| 传送连续性 | 连续 | **不连续** | 连续 |
| 硬件复杂度 | 低 | 中 | **高** |
| 存储器要求 | 正常 | 正常 | **速度加倍** |
| 实现难度 | 简单 | 中等 | 复杂 |
| 适用场景 | 高速批量传送 | 一般情况 | 高性能系统 |
| 总线占用 | 完全占用 | 按需占用 | 分时共享 |

> [!TIP]
> 考研重点：理解三种方式的本质区别。停止CPU访问是"独占"，周期挪用是"借用"，交替访问是"共享"。

### 33.3.1 停止CPU访问

**原理**：DMA传送期间，CPU**放弃总线控制权**，停止一切访存操作。

```
时间轴：
CPU  |████████████|                              |████████████|
DMA  |            |████████████████████████████|  |
     ↑            ↑                            ↑  ↑
     CPU停止      DMA开始传送                   DMA结束  CPU恢复
```

**优点**：控制简单，传送连续

**缺点**：CPU利用率低（DMA传送期间CPU无法访存）

### 33.3.2 周期挪用（Cycle Stealing）

**原理**：DMA在CPU**不访存时**窃取总线周期。

```
时间轴：
CPU  |IF|ID|EX|MEM|IF|ID|EX|   |IF|ID|EX|MEM|
DMA  |  |  |  |   |  |  |  |DMA|  |  |  |   |
                              ↑
                           窃取一个周期
```

**特点**：
- CPU和DMA交替使用总线
- 如果CPU正在访存，DMA等待
- 如果CPU和DMA同时请求访存，**DMA优先**

**优点**：CPU浪费少

**缺点**：DMA传送不连续，每次传送一个字

### 33.3.3 DMA与CPU交替访问

**原理**：将CPU周期分为两半，一半给CPU，一半给DMA。

```
时间轴：
CPU  |上半周期|上半周期|上半周期|上半周期|
DMA  |下半周期|下半周期|下半周期|下半周期|
```

**优点**：CPU和DMA都不等待

**缺点**：需要CPU周期足够长

### 33.3.4 三种传送方式对比

| 方式 | CPU利用率 | 传送连续性 | 硬件复杂度 | 适用场景 |
|------|----------|-----------|-----------|---------|
| 停止CPU访问 | 低 | 连续 | 低 | 高速大批量 |
| 周期挪用 | 高 | 不连续 | 中 | 一般情况 |
| 交替访问 | 最高 | 连续 | 高 | CPU周期长 |

### 33.5.3 DMA传送的微操作序列

**单次DMA传送的微操作**（以周期挪用方式为例）：

```
T0: 检测DREQ信号
    若DREQ有效，发HRQ信号

T1: 等待HLDA信号

T2: 收到HLDA
    MAR ← AR          // 地址送地址总线
    驱动地址总线

T3: 读操作（设备→主存）：
    DBR ← 设备数据
    M[MAR] ← DBR      // 数据写入主存
    
    写操作（主存→设备）：
    DBR ← M[MAR]      // 从主存读数据
    设备 ← DBR

T4: AR ← AR + 1      // 地址递增
    WC ← WC - 1      // 计数减1

T5: 释放总线
    撤销HRQ信号

T6: 检查WC
    若WC=0，置完成标志，发中断
    若WC≠0，等待下一个DREQ
```

## 33.7 DMA的性能分析

<div data-component="DMAPerformanceCalc"></div>

### 33.4.1 预处理

CPU执行I/O指令，设置DMA控制器：

| 参数 | 内容 |
|------|------|
| 传送方向 | 读（设备→主存）或写（主存→设备） |
| 主存起始地址 | 写入AR |
| 传送数据量 | 写入WC |
| 设备地址 | 写入设备寄存器 |
| 启动 | 设置控制寄存器 |

### 33.4.2 数据传送

```
循环：
1. 设备准备好 → DMA请求
2. DMA请求总线
3. 获得总线：
   - 读操作：设备 → DR → 主存(AR)
   - 写操作：主存(AR) → DR → 设备
4. AR + 1
5. WC - 1
6. WC ≠ 0 → 返回步骤1
7. WC = 0 → 传送结束，发中断
```

### 33.4.3 后处理

CPU响应DMA中断：
- 校验数据（如校验和）
- 检查错误状态
- 判断是否继续传送
- 释放DMA控制器

## 33.5 DMA的性能分析

### 33.5.1 传送时间

$$
\text{DMA传送时间} = \text{数据量} \times \text{存储周期}
$$

**【例 33-1】** 从磁盘读1KB数据到主存，存储周期100ns。

$$
\text{传送时间} = 1024 \times 100\text{ns} = 102.4\mu\text{s}
$$

### 33.5.2 CPU被暂停的时间

**停止CPU访问方式**：

$$
\text{CPU暂停时间} = \text{DMA传送时间}
$$

**周期挪用方式**：

$$
\text{CPU暂停时间} = \text{数据量} \times \text{每个字挪用的周期数} \times \text{时钟周期}
$$

**【例 33-2】** 用周期挪用方式传输1KB数据，存储周期100ns，CPU时钟周期50ns。

- 每个字挪用1个存储周期（100ns = 2个CPU周期）
- CPU暂停时间 = $1024 \times 2 \times 50\text{ns} = 102.4\mu\text{s}$

### 33.5.3 DMA对CPU性能的影响

$$
\text{CPU性能损失} = \frac{\text{DMA暂停时间}}{\text{总执行时间}}
$$

## 33.6 本章典型例题

### 例题1：DMA传送过程

**题目**：描述DMA从磁盘读取4KB数据到主存（起始地址0x1000）的完整过程。

**解答**：

**预处理**：
1. 设置传送方向：读（磁盘→主存）
2. 设置AR = 0x1000
3. 设置WC = 4096
4. 设置磁盘地址
5. 启动DMA

**数据传送**：
1. 磁盘读取数据 → DMA请求
2. DMA获得总线
3. 数据：磁盘 → DR → 主存(0x1000)
4. AR = 0x1001, WC = 4095
5. 重复直到WC = 0

**后处理**：
1. DMA发中断
2. CPU检查数据
3. 确认传送完成

### 例题2：传送方式选择

**题目**：某系统存储周期200ns，CPU时钟周期50ns。需要传输1MB数据。比较三种DMA方式的CPU暂停时间。

**解答**：

**停止CPU访问**：
- CPU暂停时间 = $1\text{MB} / 1\text{B} \times 200\text{ns} = 1048576 \times 200\text{ns} = 209.7\text{ms}$

**周期挪用**：
- 每个字挪用1个存储周期（200ns = 4个CPU周期）
- CPU暂停时间 = $1048576 \times 4 \times 50\text{ns} = 209.7\text{ms}$

**交替访问**：
- CPU周期 = 200ns / 2 = 100ns
- CPU暂停时间 = 0（CPU不等待）

## 33.7 本章常见误区与易错点

1. **DMA不经过CPU**：数据直接在主存和I/O设备之间传输。

2. **DMA只在开始和结束时需要CPU**：预处理和后处理由CPU执行。

3. **周期挪用会暂停CPU**：虽然CPU利用率高，但每次挪用都会暂停CPU一个存储周期。

4. **DMA优先级高于CPU**：在周期挪用方式中，DMA请求优先于CPU访存请求。

5. **DMA传送完成后发中断**：这个中断是通知CPU传送完成，不是每次传输都发中断。

## 33.10 深入理解DMA

### 33.10.1 DMA的工作原理详解

**DMA的核心思想**：数据传输不经过CPU，由DMA控制器直接控制总线。

**DMA与中断方式的对比**：

| 特性 | 中断方式 | DMA方式 |
|------|---------|---------|
| 数据传输 | **经CPU** | **不经CPU** |
| CPU参与 | 每次传输都参与 | 只在开始和结束时 |
| 传输单位 | 字 | **数据块** |
| 适用速度 | 低速设备 | **高速设备** |
| 中断用途 | 每次传输完成 | 一批数据传输完成 |

### 33.10.2 DMA控制器详解

**DMA控制器中的寄存器**：

| 寄存器 | 缩写 | 功能 |
|--------|------|------|
| 主存地址寄存器 | AR/MAR | 存放主存地址 |
| 字计数器 | WC/BC | 存放传送数据量 |
| 数据缓冲寄存器 | DR/DBR | 暂存传输数据 |
| 控制/状态寄存器 | CSR | 控制命令和状态 |
| DMA请求触发器 | — | 设备请求标志 |
| 中断逻辑 | — | 传送完成中断 |

**DMA控制器的工作流程**：

**预处理（CPU执行）**：
1. 设置传送方向（读/写）
2. 设置主存起始地址（→AR）
3. 设置传送数据量（→WC）
4. 启动设备

**数据传送（DMA控制器执行）**：
1. 设备准备好数据 → DMA请求
2. DMA向总线仲裁器请求总线
3. 获得总线 → 数据传送
4. 每传一个字：AR+1, WC-1
5. WC溢出（=0）→ 传送结束

**后处理（CPU执行）**：
1. DMA向CPU发中断
2. CPU检查数据
3. 判断是否继续

### 33.10.3 三种传送方式详解

**停止CPU访问**：

DMA传送期间，CPU放弃总线控制权，停止一切访存操作。

- 优点：控制简单，传送连续
- 缺点：CPU利用率低

**周期挪用**：

DMA在CPU不访存时窃取总线周期。

- 优点：CPU浪费少
- 缺点：DMA传送不连续

**DMA与CPU交替访问**：

将CPU周期分为两半，一半给CPU，一半给DMA。

- 优点：CPU和DMA都不等待
- 缺点：需要CPU周期足够长

### 33.10.4 DMA的性能分析详解

**传送时间**：

$$
\text{DMA传送时间} = \text{数据量} \times \text{存储周期}
$$

**CPU被暂停的时间**：

**停止CPU访问方式**：

$$
\text{CPU暂停时间} = \text{DMA传送时间}
$$

**周期挪用方式**：

$$
\text{CPU暂停时间} = \text{数据量} \times \text{每个字挪用的周期数} \times \text{时钟周期}
$$

## 33.12 DMA与中断方式的详细对比

### 33.12.1 工作原理对比

```
中断方式的数据路径：
    I/O设备 ──→ I/O接口 ──→ CPU（累加器） ──→ 主存
                  ↑              │
                  └──────────────┘
                   每次传输都经过CPU

DMA方式的数据路径：
    I/O设备 ──→ I/O接口 ──→ DMA控制器 ──→ 主存
                                    ↑
                                    │
                                 不经过CPU
```

### 33.12.2 性能对比

**【例 33-5】** 传输1MB数据，存储周期100ns，CPU时钟1GHz，中断开销60周期。

| 指标 | 中断方式 | DMA方式 |
|------|---------|---------|
| 传输单位 | 1字节 | 1MB块 |
| 传输次数 | 1048576次 | 1次（初始化） |
| CPU参与 | 每次都参与 | 只在开始和结束 |
| 传输时间 | 1048576 × (100ns + 60/1GHz) | 1048576 × 100ns |
| CPU开销 | 1048576 × 60/1GHz = 62.9ms | 约几μs |
| CPU利用率损失 | **6.29%** | **≈0%** |

### 33.12.3 综合对比表

| 对比维度 | 中断方式 | DMA方式 |
|---------|---------|---------|
| 数据传输路径 | 经过CPU | **不经过CPU** |
| CPU参与程度 | 每次传输 | 仅开始和结束 |
| 传输单位 | 字/字节 | **数据块** |
| 适用速度 | 低中速 | **高速** |
| 中断频率 | 高（每次传输） | **低（一批完成）** |
| 硬件复杂度 | 中 | 中高 |
| 实时性 | 好 | 差（批量处理） |
| CPU利用率 | 中 | **高** |
| 典型应用 | 键盘、鼠标 | 磁盘、网卡 |

### 33.12.4 何时选择DMA

当满足以下条件时，应选择DMA方式：

1. **数据量大**：需要传输的数据块较大
2. **速度要求高**：设备数据传输率较高
3. **CPU负载重**：CPU需要处理其他任务
4. **实时性要求不高**：可以接受批量处理

> [!TIP]
> 一般规则：当设备数据传输率超过CPU处理能力的10%时，应考虑使用DMA方式。

## 33.13 DMA方式的典型应用

### 33.13.1 磁盘DMA

磁盘是DMA最典型的应用场景：

```
┌──────────────────────────────────────────────────────────────┐
│                    磁盘DMA传送过程                            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. CPU设置DMA控制器                                         │
│     - 主存起始地址：0x10000                                  │
│     - 传送数据量：4KB                                        │
│     - 传送方向：读（磁盘→主存）                               │
│                                                              │
│  2. DMA控制器启动磁盘                                        │
│     - 发送读命令到磁盘控制器                                  │
│     - 指定磁盘扇区地址                                       │
│                                                              │
│  3. 磁盘寻道和旋转                                            │
│     - 磁头移动到目标磁道                                      │
│     - 等待目标扇区旋转到磁头下方                              │
│                                                              │
│  4. 数据传送                                                  │
│     - 磁盘读取数据 → DMA控制器 → 主存                        │
│     - 每个存储周期传送一个字                                  │
│     - 4KB数据需要4096次传送                                  │
│                                                              │
│  5. 传送完成                                                  │
│     - DMA发中断给CPU                                         │
│     - CPU检查数据（校验和）                                   │
│     - 处理完成                                               │
└──────────────────────────────────────────────────────────────┘
```

### 33.13.2 网卡DMA

网络数据包的接收和发送通常使用DMA：

```
接收过程：
    网卡收到数据包 → DMA将数据包写入主存缓冲区 → 发中断给CPU

发送过程：
    CPU将数据包放入主存缓冲区 → DMA将数据包读出到网卡 → 网卡发送
```

### 33.13.3 显卡DMA

图形数据的传输也常使用DMA：

```
图形渲染：
    CPU准备图形数据 → DMA传输到显存 → GPU渲染
```

## 33.14 DMA方式的常见问题

### 33.14.1 Cache一致性问题

当DMA直接写入主存时，Cache中的数据可能过时：

```
问题场景：
    ① CPU读取主存地址X，数据进入Cache
    ② DMA将新数据写入主存地址X
    ③ CPU再次读取地址X，得到的是Cache中的旧数据

解决方案：
    ① 写穿策略：Cache写穿到主存，DMA写入时自动更新Cache
    ② Cache刷新：DMA传送前刷新相关Cache行
    ③ 非Cache区域：将DMA缓冲区标记为非Cache
```

### 33.14.2 地址对齐问题

某些DMA控制器要求数据地址对齐：

| 对齐要求 | 说明 | 影响 |
|---------|------|------|
| 字对齐 | 地址必须是4的倍数 | 简化硬件 |
| 页对齐 | 地址必须是页大小的倍数 | 提高效率 |
| 无对齐要求 | 任意地址 | 硬件复杂 |

### 33.14.3 传送长度限制

| 限制类型 | 说明 |
|---------|------|
| 最大长度 | 单次DMA传送的最大数据量 |
| 对齐长度 | 传送长度必须是字大小的倍数 |
| 跨页问题 | 传送可能跨越页边界 |
