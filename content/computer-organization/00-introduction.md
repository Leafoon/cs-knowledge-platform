# 计算机组成原理

本课程系统讲解计算机硬件系统的基本组成与工作原理，涵盖数据的表示与运算、指令系统、中央处理器、存储系统、总线与输入输出系统六大核心模块。内容参照唐朔飞《计算机组成原理》体系，兼顾考研408大纲要求，配套丰富的交互式可视化组件，帮助深入理解底层硬件机制。

## 二、计算机系统的完整组成框架

### 2.1 计算机系统的层次概念

计算机系统由 **硬件** 和 **软件** 两大部分组成。理解计算机系统需要从多个抽象层次来观察：

| 层次 | 名称 | 说明 |
|------|------|------|
| L6 | 应用语言层 | 面向用户的应用程序 |
| L5 | 高级语言层 | C/C++/Java 等高级语言 |
| L4 | 汇编语言层 | 汇编指令 |
| L3 | 操作系统层 | 机器语言 + 操作系统管理 |
| L2 | 指令集架构层 | ISA（硬件与软件的接口） |
| L1 | 微体系结构层 | 数据通路、控制器的具体实现 |
| L0 | 数字逻辑层 | 门电路、触发器、组合/时序逻辑 |

**计算机组成原理** 主要工作在 L1（微体系结构层）和 L2（指令集架构层），并涉及 L0 的基础概念。

### 2.2 冯·诺依曼体系结构

1945 年，冯·诺依曼（John von Neumann）提出了**存储程序**（Stored Program）的概念，奠定了现代计算机的设计基础。

#### 核心思想

1. **存储程序**：程序（指令）和数据以同等地位存储在同一个存储器中
2. **顺序执行**：计算机按照程序中指令的顺序逐条执行（可被转移指令改变）
3. **二进制表示**：指令和数据均采用二进制编码

#### 冯·诺依曼计算机的五大部件

```
                 ┌───────────────────────────────┐
                 │          控制器 (CU)           │
                 │   ┌───────────────────────┐   │
                 │   │  指令寄存器 IR        │   │
                 │   │  程序计数器 PC        │   │
                 │   │  控制信号生成逻辑     │   │
                 │   └───────────────────────┘   │
                 └──────────────┬────────────────┘
                                │ 控制信号
        ┌───────────────────────┼───────────────────────┐
        │                       │                       │
┌───────▼───────┐     ┌────────▼────────┐     ┌────────▼────────┐
│   运算器 ALU  │◄───►│    主存储器      │◄───►│    输入设备     │
│               │     │   (Main Memory)  │     │   Input Device  │
│  ACC, MQ, X  │     │  MAR, MDR       │     └─────────────────┘
│  ALU 核心    │     │  存储体         │
└───────┬───────┘     └────────┬────────┘     ┌─────────────────┐
        │                      │              │    输出设备      │
        └──────────────────────┴─────────────►│  Output Device  │
                                               └─────────────────┘
```

#### 五大部件功能概览

| 部件 | 核心功能 | 关键寄存器 |
|------|----------|-----------|
| **运算器** | 执行算术运算和逻辑运算 | ACC（累加器）、MQ（乘商寄存器）、X（操作数寄存器）、ALU |

| **输出设备** | 将计算结果转换为人能理解的形式 | — |

### 2.3 现代计算机对冯·诺依曼结构的改进

冯·诺依曼结构虽然奠定了基础，但存在一些固有瓶颈。现代计算机对其进行了多方面的改进：

#### 冯·诺依曼瓶颈

原始结构中，指令和数据共享同一条总线（存储器只有一个端口），CPU 执行指令时必须交替取指和取数，形成"取指—执行"的串行模式，限制了整体吞吐量。

#### 主要改进措施

| 改进方向 | 具体技术 | 说明 |
|----------|---------|------|
| **哈佛结构** | 指令存储器与数据存储器分离 | 嵌入式处理器、DSP 广泛采用，允许同时取指和取数 |
| **高速缓存** | Cache 层次结构 | L1/L2/L3 Cache 弥合 CPU 与主存速度差距 |
| **多总线/交叉存储** | 多端口存储器、多模块交叉编址 | 提升存储器带宽 |
| **流水线** | 指令流水线 | 多条指令重叠执行 |
| **多核/多处理器** | 片上多核 (CMP) | 并行执行多条指令流 |
| **推测执行** | 分支预测、乱序执行 | 提前执行可能需要的指令 |

#### 哈佛结构 vs 冯·诺依曼结构

```
冯·诺依曼结构（普林斯顿结构）           哈佛结构
┌──────────┐                      ┌──────────┐
│    CPU   │                      │    CPU   │
└────┬─────┘                      └──┬───┬───┘
     │                              │   │
     ▼                              ▼   ▼
┌────────────┐               ┌──────┐ ┌──────┐
│  统一存储器  │               │指令  │ │数据  │
│ (指令+数据) │               │存储器│ │存储器│
└────────────┘               └──────┘ └──────┘
     一条总线                    两条独立总线
```

| 对比项 | 冯·诺依曼 | 哈佛 |
|--------|----------|------|
| 存储器 | 指令和数据统一存放 | 指令和数据分开存放 |
| 总线 | 共用一条 | 独立的指令总线和数据总线 |
| 并行性 | 取指和取数不能同时进行 | 可同时取指和取数 |
| 应用 | 通用计算机（PC、服务器） | 嵌入式系统、DSP、单片机 |

## 三、CPU 详细剖析

### 3.1 CPU 的功能定位

CPU（Central Processing Unit，中央处理器）是计算机的**核心**，其本质功能可以概括为一条循环：

```
取指 → 译码 → 执行 → 访存 → 写回 → (重复)
```

CPU 需要完成以下四类基本操作：

1. **指令控制**：保证指令按正确的顺序执行（取指、译码）
2. **操作控制**：产生完成指令所需的各种操作控制信号
3. **时间控制**：对各种操作进行定时控制
4. **数据加工**：执行算术运算、逻辑运算、移位等操作

### 3.2 CPU 的内部结构

CPU 从逻辑上可以分为两大部分：**运算器** 和 **控制器**。

```
┌──────────────────────────────────────────────────────────┐
│                         CPU                              │
│  ┌──────────────────────┐    ┌──────────────────────┐   │
│  │       运算器          │    │       控制器          │   │
│  │  ┌─────┐  ┌──────┐  │    │  ┌─────┐  ┌──────┐  │   │
│  │  │ ALU │  │ ACC  │  │    │  │ PC  │  │  IR  │  │   │
│  │  └─────┘  └──────┘  │    │  └─────┘  └──────┘  │   │
│  │  ┌─────┐  ┌──────┐  │    │  ┌─────┐  ┌──────┐  │   │
│  │  │ MQ  │  │  X   │  │    │  │ MAR │  │  MDR │  │   │
│  │  └─────┘  └──────┘  │    │  └─────┘  └──────┘  │   │
│  │  ┌─────┐  ┌──────┐  │    │  ┌──────────────┐   │   │
│  │  │PSW  │  │通用寄│  │    │  │    CU        │   │   │
│  │  │     │  │存器组│  │    │  │(控制信号生成) │   │   │
│  │  └─────┘  └──────┘  │    │  └──────────────┘   │   │
│  └──────────────────────┘    └──────────────────────┘   │
│                                                          │
│  ┌──────────────────────────────────────────────────┐   │
│  │              内部总线（数据通路）                    │   │
│  └──────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────┘
```

### 3.3 运算器的组成与功能

运算器是数据加工处理的核心部件，以 ALU 为中心，包含多个寄存器。

#### 运算器中的关键寄存器

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **ALU** | Arithmetic Logic Unit | 执行算术运算（加减乘除）和逻辑运算（与或非异或） |
| **ACC** | Accumulator（累加器） | 存放操作数和运算结果，是 ALU 的主要输入/输出 |
| **MQ** | Multiplier-Quotient Register | 乘法时存放乘数/部分积，除法时存放商 |
| **X** | 操作数寄存器 | 存放 ALU 的另一个操作数 |
| **PSW** | Program Status Word | 存放程序状态（进位、溢出、零、符号等标志位） |
| **通用寄存器组** | GPRs | R0~Rn，存放操作数、地址等 |

#### 典型运算的数据流

**加法操作：`ACC ← ACC + X`**

```
ACC ──────┐
          │
          ▼
        ┌─────┐
   X ──►│ ALU │──► ACC
        └─────┘
   (加法)
```

**乘法操作：`ACC:MQ ← MQ × X`**

```
MQ ──────┐
         │
         ▼
  X ──►┌─────┐──► ACC（高位结果）
       │ ALU │──► MQ（低位结果）
       └─────┘
  (乘法)
```

### 3.4 控制器的组成与功能

控制器是计算机的**指挥中心**，负责从存储器中取出指令、分析指令、产生控制信号。

#### 控制器中的关键寄存器

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **PC** | Program Counter | 存放**下一条**要执行的指令地址，具有自动 +1 功能 |
| **IR** | Instruction Register | 存放**当前正在执行**的指令（操作码 + 地址码） |
| **MAR** | Memory Address Register | 存放要访问的存储器地址 |
| **MDR** | Memory Data Register | 存放从存储器读出或要写入的数据 |
| **CU** | Control Unit | 控制信号发生器，根据指令产生各种微操作控制信号 |

#### 指令执行的基本过程

一条指令的执行分为以下阶段：

| 阶段 | 微操作 | 说明 |
|------|--------|------|
| **取指** | `MAR ← (PC)` | 将 PC 中的地址送到 MAR |
| | `M(MAR) → MDR` | 从存储器读取指令到 MDR |
| | `IR ← (MDR)` | 将指令送入 IR |
| | `PC ← (PC) + 1` | PC 指向下一条指令 |
| **译码** | `OP(IR) → CU` | 指令操作码送入 CU 进行译码 |
| **执行** | 取操作数 → 运算 → 存结果 | 具体微操作取决于指令类型 |

#### 控制器的两种实现方式

| 特性 | 硬布线控制器 | 微程序控制器 |
|------|-------------|-------------|
| 实现方式 | 组合逻辑电路 | 微指令存储在控制存储器中 |
| 速度 | 快 | 较慢（需访问控制存储器） |
| 灵活性 | 差，修改困难 | 好，修改微程序即可 |
| 复杂度 | 指令越多越复杂 | 设计规整，易于扩展 |
| 典型应用 | RISC 处理器 | CISC 处理器 |

### 3.5 CPU 的数据通路

数据通路（Datapath）是 CPU 中数据流动的物理路径，包括 ALU、寄存器组、多路选择器、总线等。

#### 单总线数据通路

所有寄存器通过一条公共总线（Bus）连接到 ALU：

```
  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
  │R0 │ │R1 │ │R2 │ │R3 │ │ PC│
  └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘
    │     │     │     │     │
    └─────┴─────┴─────┴─────┘
                  │
              ┌───▼───┐
              │  Bus  │
              └───┬───┘
                  │
           ┌──────▼──────┐
           │    ALU      │
           └──────┬──────┘
                  │
              ┌───▼───┐
              │  Bus  │
              └───┬───┘
                  │
    ┌─────────────┼─────────────┐
    │             │             │
  ┌─▼─┐       ┌──▼──┐      ┌──▼──┐
  │MAR│       │ MDR │      │  Y  │
  └───┘       └─────┘      └─────┘
```

**单总线的瓶颈**：一次只能传输一个数据，多操作数指令需要多个时钟周期。

#### 专用数据通路

为关键路径建立专用通路，避免总线争用：

```
  寄存器组                ALU              结果总线
  ┌──────┐    A总线   ┌─────────┐    ┌──────────┐
  │ R0~Rn│──────────►│  A  B   │───►│结果寄存器 │
  └──────┘    B总线   │ ALU核心 │    └─────┬────┘
              │       └─────────┘          │
              └────────────────────────────┘
                     (反馈回寄存器组)
```

#### 数据通路的性能指标

| 指标 | 含义 | 影响因素 |
|------|------|----------|
| 时钟周期 | 完成一个基本操作的时间 | 最长数据通路延迟 |
| CPI | 每条指令的时钟周期数 | 指令复杂度、数据通路设计 |
| 数据通路宽度 | 一次传输的数据位数 | 总线宽度、寄存器位数 |

### 3.6 CPU 的工作周期

CPU 按照节拍（时钟周期）工作，一条指令的执行可能经历多个机器周期：

| 工作周期 | 标志 | 完成的任务 |
|----------|------|-----------|
| **取指周期** | FE | 从存储器取出指令，PC + 1 |
| **间址周期** | IND | 对于间接寻址，取出操作数的有效地址 |
| **执行周期** | EX | 执行指令规定的操作 |
| **中断周期** | INT | 处理中断请求 |

```
  ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
  │取指  │──►│间址  │──►│执行  │──►│中断  │──► 下一条指令
  │周期  │   │周期  │   │周期  │   │周期  │
  └──────┘   └──────┘   └──────┘   └──────┘
     │           │          │          │
     │     (可能跳过)  (可能跳过)  (可能跳过)
     └──────────────────────────────────┘
```

## 四、存储器层次结构

### 4.1 为什么需要层次结构

存储器的**速度**、**容量**、**价格**三者之间存在固有矛盾：

| 存储层次 | 速度 | 容量 | 价格/位 | 技术 |
|----------|------|------|---------|------|
| 寄存器 | ~1 ns | 几百 B~几 KB | 最高 | 触发器 |
| Cache (L1) | ~1 ns | 32~64 KB | 高 | SRAM |
| Cache (L2) | ~3~10 ns | 256 KB~1 MB | 较高 | SRAM |
| Cache (L3) | ~10~20 ns | 数 MB~数十 MB | 中 | SRAM |
| 主存 | ~50~100 ns | 数 GB~数百 GB | 较低 | DRAM |
| 固态硬盘 SSD | ~100 μs | 数百 GB~数 TB | 低 | Flash |
| 机械硬盘 HDD | ~5~10 ms | 数 TB | 最低 | 磁介质 |

**核心思想**：用层次化结构，在保证大容量的同时获得接近最快层的速度。

<div data-component="MemoryHierarchyInteractive"></div>

### 4.2 存储层次结构全景

```
            速度递减 ─────────────────────────►
            容量递增 ─────────────────────────►

  ┌──────┐  ┌──────────┐  ┌──────────┐  ┌──────┐  ┌──────┐
  │寄存器│─►│   Cache  │─►│   主存   │─►│ 辅存 │─►│网络  │
  │      │  │ L1/L2/L3 │  │  (DRAM)  │  │(磁盘)│  │存储  │
  └──────┘  └──────────┘  └──────────┘  └──────┘  └──────┘
    ↑            ↑              ↑
    │            │              │
  CPU内部     CPU芯片内      主板上

  ◄──── 由硬件自动管理 ────►◄── 由OS管理 ──►
```

### 4.3 寄存器层

寄存器是 CPU 内部速度最快的存储单元，直接参与运算。

#### 典型 CPU 的寄存器组

| 类别 | 寄存器 | 位数 | 用途 |
|------|--------|------|------|
| 通用寄存器 | RAX/RBX/RCX/RDX (x86-64) | 64 位 | 存放操作数和运算结果 |
| 指针寄存器 | RSP（栈指针）、RBP（基址指针） | 64 位 | 栈操作 |
| 索引寄存器 | RSI（源）、RDI（目的） | 64 位 | 字符串操作 |
| 程序计数器 | RIP | 64 位 | 下一条指令地址 |
| 标志寄存器 | RFLAGS | 64 位 | 状态标志（CF/ZF/SF/OF） |
| 段寄存器 | CS/DS/SS/ES/FS/GS | 16 位 | 内存分段 |
| 控制寄存器 | CR0~CR4 | 64 位 | 处理器控制 |
| 浮点寄存器 | ST(0)~ST(7) | 80 位 | 浮点运算 |
| SIMD 寄存器 | XMM0~XMM15 / YMM / ZMM | 128/256/512 位 | 向量运算 |

### 4.4 Cache 层

Cache（高速缓存）利用**程序的局部性原理**，将主存中的热点数据复制到高速 SRAM 中。

#### 局部性原理

| 类型 | 描述 | 例子 |
|------|------|------|
| **时间局部性** | 最近访问的数据很可能再次被访问 | 循环变量 |
| **空间局部性** | 访问某个地址时，附近地址很可能被访问 | 数组遍历 |

#### Cache 的工作原理

```
CPU 请求地址
      │
      ▼
┌───────────┐   命中    ┌──────────┐
│  Cache    │─────────►│ 返回数据  │
│  查找     │           └──────────┘
└─────┬─────┘
      │ 未命中
      ▼
┌───────────┐           ┌──────────┐
│ 主存访问  │─────────►│ 数据送   │
│ +填入Cache│           │ CPU+Cache│
└───────────┘           └──────────┘
```

#### Cache 映射方式

| 映射方式 | 原理 | 优点 | 缺点 |
|----------|------|------|------|
| **直接映射** | 主存块只能映射到 Cache 的固定行 | 简单、速度快 | 冲突率高 |
| **全相联映射** | 主存块可映射到 Cache 的任意行 | 冲突率低 | 硬件复杂、比较器多 |
| **组相联映射** | Cache 分组，主存块映射到固定组、任意行 | 折中方案 | 复杂度适中 |

**直接映射公式**：

$$
\text{Cache 行号} = \text{主存块号} \mod \text{Cache 行数}
$$

**n 路组相联映射公式**：

$$
\text{Cache 组号} = \text{主存块号} \mod \text{Cache 组数}
$$

#### Cache 替换算法

| 算法 | 原理 | 优缺点 |
|------|------|--------|
| **LRU** | 替换最近最少使用的行 | 接近最优，实现需计数器 |
| **FIFO** | 替换最早进入的行 | 实现简单，可能替换热点数据 |
| **LFU** | 替换使用频率最低的行 | 需要计数器，可能有历史偏差 |
| **随机替换** | 随机选择一行替换 | 实现最简单，效果不可控 |

#### Cache 写策略

| 写策略 | 描述 | 适用场景 |
|--------|------|----------|
| **写直达** (Write-Through) | 同时写 Cache 和主存 | 一致性好，但写速度慢 |
| **写回** (Write-Back) | 只写 Cache，脏块被替换时写回主存 | 写速度快，但一致性管理复杂 |
| **写分配** (Write-Allocate) | 写未命中时，先将块调入 Cache 再写 | 通常配合写回使用 |
| **写不分配** (No-Write-Allocate) | 写未命中时，直接写主存 | 通常配合写直达使用 |

#### Cache 性能指标

$$
\text{平均访问时间} = h \times t_c + (1 - h) \times t_m
$$

其中：
- $h$ = Cache 命中率
- $t_c$ = Cache 访问时间
- $t_m$ = 主存访问时间

**例题**：Cache 命中率 95%，Cache 访问时间 10 ns，主存访问时间 200 ns。

$$
T_{avg} = 0.95 \times 10 + 0.05 \times 200 = 9.5 + 10 = 19.5 \text{ ns}
$$

### 4.5 主存层

#### 主存的组成

主存（Main Memory）由半导体存储芯片组成，分为 RAM 和 ROM 两大类：

| 类型 | 全称 | 特点 | 应用 |
|------|------|------|------|
| **SRAM** | Static RAM | 不需刷新、速度快、成本高 | Cache |
| **DRAM** | Dynamic RAM | 需定期刷新、速度较慢、成本低 | 主存 |
| **ROM** | Read-Only Memory | 非易失、只读 | BIOS/固件 |
| **Flash** | Flash Memory | 非易失、可擦写 | SSD、U盘 |

#### 存储器容量扩展

当单个芯片容量不够时，需要进行扩展：

| 扩展方式 | 目的 | 方法 |
|----------|------|------|
| **位扩展** | 增加字长 | 多芯片并联，地址线相同、数据线拼接 |
| **字扩展** | 增加字数 | 多芯片串联，数据线相同、地址线用译码器选择 |
| **字位同时扩展** | 同时增加字长和字数 | 先位扩展再字扩展 |

#### 交叉编址存储器

多模块交叉编址可提升存储器带宽：

| 编址方式 | 地址分配 | 访问特性 |
|----------|----------|----------|
| **顺序编址** | 模块 0: 0,1,2,3; 模块 1: 4,5,6,7 | 连续访问同一模块，带宽低 |
| **交叉编址** | 模块 0: 0,4,8,12; 模块 1: 1,5,9,13 | 连续访问不同模块，并行访问，带宽高 |

### 4.6 辅存层

辅助存储器用于永久存储数据，容量大但速度慢。

| 存储介质 | 读写方式 | 容量 | 访问时间 | 特点 |
|----------|----------|------|----------|------|
| **机械硬盘 HDD** | 磁头寻道 + 盘片旋转 | 数 TB | 5~10 ms | 容量大、成本低 |
| **固态硬盘 SSD** | 电子擦写 | 数百 GB~数 TB | 0.1 ms | 无机械部件、抗振 |
| **光盘** | 激光读写 | 数百 MB~数十 GB | 较慢 | 适合归档 |

#### 磁盘性能参数

| 参数 | 含义 |
|------|------|
| 寻道时间 | 磁头移动到目标磁道的时间 |
| 旋转延迟 | 等待目标扇区旋转到磁头下方的时间 |
| 数据传输时间 | 读写数据的实际时间 |
| 平均存取时间 | 寻道时间 + 旋转延迟 + 传输时间 |

$$
T_{access} = T_{seek} + T_{rotation} + T_{transfer}
$$

## 五、总线系统

### 5.1 总线的基本概念

总线（Bus）是一组能为多个部件**分时共享**的公共信息传送线路。"分时"指同一时刻只能有一个部件发送数据；"共享"指总线上可以挂接多个部件。

### 5.2 总线的分类

#### 按数据传输方式分

| 类型 | 特点 | 代表 |
|------|------|------|
| **并行总线** | 多位数据同时传输 | PCI、内存总线 |
| **串行总线** | 数据逐位传输 | USB、SATA、PCIe（多通道串行） |

#### 按功能分（系统总线）

| 总线类型 | 功能 | 信号 |
|----------|------|------|
| **数据总线 (DB)** | 传输数据 | 双向，宽度决定一次传输的数据量 |
| **地址总线 (AB)** | 传输地址 | 单向（CPU → 存储器/I/O），宽度决定寻址空间 |
| **控制总线 (CB)** | 传输控制信号 | 部分单向、部分双向 |

**地址总线宽度与寻址空间的关系**：

$$
\text{寻址空间} = 2^n \text{ 字节}
$$

| 地址总线宽度 | 寻址空间 |
|-------------|----------|
| 16 位 | 64 KB |
| 20 位 | 1 MB |
| 32 位 | 4 GB |
| 48 位 | 256 TB |
| 64 位 | 16 EB |

### 5.3 总线的结构

#### 单总线结构

```
┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐
│ CPU │──│  M  │──│I/O 1│──│I/O 2│
└─────┘  └─────┘  └─────┘  └─────┘
     所有部件共享一条系统总线
```

**缺点**：总线成为瓶颈，所有通信都经过同一条总线。

#### 多总线结构

```
┌─────┐
│ CPU │
└──┬──┘
   │  CPU 总线
┌──▼──┐
│ 桥  │──── 高速总线 ──── Cache/主存
└──┬──┘
   │
   ├──── 系统总线 ──── I/O 设备
   │
   └──── 扩展总线 ──── 扩展槽
```

现代计算机通常采用 **北桥/南桥**（或平台控制器中枢 PCH）的层次化总线结构。

### 5.4 总线仲裁

当多个设备同时请求使用总线时，需要总线仲裁器来决定谁获得使用权。

| 仲裁方式 | 原理 | 优点 | 缺点 |
|----------|------|------|------|
| **链式查询** | 设备按优先级链式连接，BR→BG→BS | 简单、易于扩展 | 优先级固定，对故障敏感 |
| **计数器定时查询** | 仲裁器用计数器扫描设备号 | 优先级可调 | 需要设备号线 |
| **独立请求** | 每个设备有独立的 BR/BG 线 | 速度快、优先级灵活 | 线数多、复杂度高 |

```
链式查询方式：
  ┌───────────┐
  │ 总线仲裁器 │
  └──┬──┬──┬──┘
     │BR│BG│BS    (BR=请求, BG=授权, BS=忙)
     │  │  │
  ┌──▼──▼──▼──┐
  │  设备 0   │──┐
  └───────────┘  │ BG 串行传递
  ┌───────────┐  │
  │  设备 1   │◄─┘──┐
  └───────────┘     │
  ┌───────────┐     │
  │  设备 2   │◄────┘
  └───────────┘
```

### 5.5 总线通信方式

| 通信方式 | 原理 | 适用场景 |
|----------|------|----------|
| **同步通信** | 收发双方由统一时钟信号控制 | 速度匹配的设备之间 |
| **异步通信** | 通过握手信号协调，无需统一时钟 | 速度差异大的设备 |
| **半同步通信** | 结合同步和异步特点 | 折中方案 |
| **分离式通信** | 将一次传输分为两个独立子周期 | 高性能总线 |

#### 异步通信的握手协议

```
不互锁方式：
  请求方 ──── REQ ────►  应答方
  请求方 ◄─── ACK ────  应答方
  (REQ 撤销不等 ACK)

半互锁方式：
  请求方 ──── REQ ────►  应答方
  请求方 ◄─── ACK ────  应答方
  (等待 ACK 后才撤销 REQ)

全互锁方式：
  请求方 ──── REQ ────►  应答方
  请求方 ◄─── ACK ────  应答方
  (REQ 等 ACK，ACK 等 REQ 撤销)
```

## 六、I/O 系统

### 6.1 I/O 系统概述

I/O 系统负责计算机与外部设备之间的信息交换，是计算机系统的重要组成部分。

#### I/O 系统的基本组成

```
┌──────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ CPU  │◄──►│ I/O 接口 │◄──►│ I/O 控制 │◄──►│ 外部设备 │
│      │    │ (适配器) │    │   器     │    │          │
└──────┘    └──────────┘    └──────────┘    └──────────┘
                │
            ┌───▼───┐
            │ 总线  │
            └───────┘
```

### 6.2 I/O 接口

I/O 接口（又称 I/O 适配器或 I/O 控制器）是 CPU 与外部设备之间的桥梁。

#### I/O 接口的功能

| 功能 | 说明 |
|------|------|
| **数据缓冲** | 用数据缓冲寄存器匹配 CPU 与设备的速度差异 |
| **信号转换** | 电平转换、串并转换等 |
| **设备选择** | 通过地址译码选择目标设备 |
| **命令译码** | 解释 CPU 发来的控制命令 |
| **状态反馈** | 向 CPU 报告设备当前状态（忙/就绪/错误） |
| **中断逻辑** | 向 CPU 发送中断请求 |

#### I/O 接口中的关键寄存器

| 寄存器 | 功能 | 编址方式 |
|--------|------|----------|
| **数据缓冲寄存器 (DBR)** | 暂存输入/输出数据 | 统一编址 / 独立编址 |
| **状态寄存器 (SR)** | 存放设备状态信息 | 同上 |
| **控制/命令寄存器 (CR)** | 存放 CPU 发来的控制命令 | 同上 |

#### I/O 端口的编址方式

| 编址方式 | 原理 | 优点 | 缺点 |
|----------|------|------|------|
| **统一编址** (存储器映射) | I/O 端口与主存统一编址 | 可用访存指令访问 I/O，灵活 | 占用主存地址空间 |
| **独立编址** (I/O 映射) | I/O 端口有独立的地址空间 | 不占用主存地址空间 | 需要专门的 I/O 指令 |

### 6.3 I/O 控制方式

CPU 与外设之间的数据传输控制方式有以下几种，从简单到复杂：

#### 6.3.1 程序查询方式

CPU 主动轮询设备状态，设备就绪后进行数据传输。

```
CPU 执行流程：
  ┌─────────────┐
  │ 启动外设    │
  └──────┬──────┘
         ▼
  ┌─────────────┐     ┌──────────┐
  │ 读取状态   │────►│ 设备就绪?│
  └──────┬──────┘     └────┬─────┘
         │                 │ 否
         │    ┌────────────┘
         │    ▼
         │  ┌─────────────┐
         │  │ 继续轮询... │──┐
         │  └─────────────┘  │
         │                   │
         │    ┌──────────────┘
         │    │ 是
         ▼    ▼
  ┌─────────────┐
  │ 传输数据    │
  └─────────────┘
```

**特点**：CPU 利用率低，CPU 大量时间浪费在等待设备。

#### 6.3.2 程序中断方式

外设就绪后主动向 CPU 发送中断请求，CPU 响应中断后执行中断服务程序进行数据传输。

| 特性 | 说明 |
|------|------|
| 触发条件 | 外设就绪或出错时主动通知 CPU |
| CPU 开销 | 每次中断需要保存/恢复现场 |
| 适用场景 | 低速设备，如键盘、鼠标 |
| 优点 | CPU 不必等待，可并行处理其他任务 |
| 缺点 | 中断处理有开销，不适合高速大量数据传输 |

**中断处理流程**：

```
主程序执行中
      │
      ▼ 中断请求
  ┌──────────────┐
  │ 关中断       │
  │ 保存断点(PC) │
  │ 保存现场     │
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 中断服务程序  │
  │ (数据传输)   │
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 恢复现场     │
  │ 恢复断点(PC) │
  │ 开中断       │
  └──────┬───────┘
         ▼
  继续执行主程序
```

#### 中断优先级与判优

| 判优方式 | 原理 |
|----------|------|
| **硬件排队** | 用优先级编码器硬件实现，速度快 |
| **软件查询** | 按查询顺序确定优先级，灵活但慢 |

**多重中断（中断嵌套）**：高优先级中断可以打断低优先级中断的处理。

#### 6.3.3 DMA 方式

DMA（Direct Memory Access，直接存储器访问）允许外设直接与主存交换数据，不经过 CPU。

| 特性 | 说明 |
|------|------|
| 数据通路 | 外设 ↔ 主存，CPU 不参与数据传送 |
| CPU 角色 | 仅在开始和结束时参与 |
| 传输单位 | 数据块 |
| 适用场景 | 高速设备，如磁盘、网卡 |

**DMA 控制器的组成**：

```
┌─────────────────────────────────────┐
│           DMA 控制器                │
│                                     │
│  ┌─────────┐  ┌─────────────────┐  │
│  │ 控制/   │  │ 传送计数器      │  │
│  │ 状态    │  │ (BC)            │  │
│  │ 寄存器  │  └─────────────────┘  │
│  └─────────┘  ┌─────────────────┐  │
│  ┌─────────┐  │ 主存地址寄存器  │  │
│  │ 数据    │  │ (AR)            │  │
│  │ 缓冲    │  └─────────────────┘  │
│  └─────────┘                       │
│                                     │
│  BR ──► 总线请求                    │
│  BG ◄── 总线授权                    │
└─────────────────────────────────────┘
```

**DMA 传送方式**：

| 方式 | 特点 | CPU 工作 |
|------|------|----------|
| **停止 CPU 访存** | DMA 完全占用总线 | CPU 暂停，直到 DMA 完成 |
| **周期挪用** | DMA 在 CPU 不访存的周期插入传送 | 影响较小，但可能延迟 CPU |
| **交替访问** | CPU 和 DMA 分时使用总线 | 透明传送，无冲突 |

#### 6.3.4 通道方式

通道（Channel）是一种专用处理器，有自己的指令系统，可以独立执行通道程序来管理 I/O 操作。

| 通道类型 | 数据传输单位 | 复杂度 | 适用设备 |
|----------|-------------|--------|----------|
| **字节多路通道** | 字节 | 低 | 低速设备（键盘、鼠标） |
| **选择通道** | 数据块 | 中 | 高速设备（磁盘） |
| **数组多路通道** | 数据块（可交替） | 高 | 高速设备（可多设备交替） |

### 6.4 I/O 控制方式对比

| 对比项 | 程序查询 | 程序中断 | DMA | 通道 |
|--------|---------|---------|-----|------|
| 数据传送 | CPU 控制 | CPU 控制 | DMA 控制器 | 通道处理器 |
| CPU 参与程度 | 全程参与 | 每次传送参与 | 仅首尾参与 | 启动和结束时 |
| 传送单位 | 字/字节 | 字/字节 | 数据块 | 数据块/一组块 |
| 并行能力 | 无 | 有 | 高 | 最高 |
| 适用设备 | 低速 | 低中速 | 高速 | 高速 |
| 硬件复杂度 | 最低 | 低 | 中 | 高 |

## 七、现代计算机的演进与改进

### 7.1 从真空管到超大规模集成电路

| 代 | 时间 | 核心器件 | 代表机器 | 特征 |
|----|------|---------|---------|------|
| 第一代 | 1946~1957 | 真空管 | ENIAC, EDVAC | 体积巨大、可靠性差 |
| 第二代 | 1958~1964 | 晶体管 | IBM 7090 | 体积缩小、速度提升 |
| 第三代 | 1965~1971 | 中小规模集成电路 | IBM 360 | 通用化、系列化 |
| 第四代 | 1972~至今 | 大/超大规模集成电路 | Intel x86 | 微处理器、个人电脑 |
| 现代 | 2000s~ | 多核/异构 | Apple M系列 | 片上系统、AI 加速 |

### 7.2 摩尔定律与后摩尔时代

**摩尔定律**（Gordon Moore, 1965）：集成电路上可容纳的晶体管数量约每 18~24 个月翻一番。

| 指标 | 1971 (4004) | 2000 (Pentium 4) | 2024 (Apple M4) |
|------|------------|------------------|-----------------|
| 晶体管数 | 2,300 | 42,000,000 | 28,000,000,000 |
| 工艺节点 | 10 μm | 180 nm | 3 nm |
| 时钟频率 | 740 kHz | 1.5 GHz | 4.5 GHz |

**后摩尔时代的应对策略**：

1. **多核架构**：单芯片集成多个处理核心
2. **异构计算**：CPU + GPU + NPU + DSP 协同
3. **3D 封装**：芯片垂直堆叠（如 Intel Foveros、AMD 3D V-Cache）
4. **新器件**：量子计算、光子计算、神经形态芯片

### 7.3 指令集架构的演进

| 时间线 | 事件 | 意义 |
|--------|------|------|
| 1978 | Intel 8086 | x86 架构诞生，CISC 代表 |
| 1985 | ARM1 | RISC 架构在嵌入式领域崛起 |
| 1985 | MIPS | 学术界 RISC 典范 |
| 2011 | ARM Cortex-A15 | ARM 进入高性能领域 |
| 2020 | Apple M1 | ARM 架构挑战桌面/笔记本市场 |
| 2022 | RISC-V 全面兴起 | 开源 ISA，学术和产业并进 |

## 八、典型计算机系统案例分析

### 8.1 x86 架构（CISC 典型代表）

x86 是 Intel 和 AMD 主导的指令集架构，从 16 位发展到 64 位（x86-64/AMD64），是 PC 和服务器市场的主流。

#### x86 的关键特征

| 特征 | 说明 |
|------|------|
| 指令长度 | 可变长度（1~15 字节） |
| 寻址方式 | 丰富（立即数、寄存器、直接、间接、基址+变址+偏移等） |
| 寄存器 | 早期较少（8 个通用），x86-64 扩展到 16 个 |
| CISC 特点 | 单条指令可完成复杂操作（如 REP MOVSB） |
| 微操作转换 | 现代 x86 将复杂指令内部译码为类 RISC 微操作 |

#### x86 处理器的内部结构演进

```
8086 (1978)              Pentium (1993)            Core i7 (2008+)
┌──────────┐            ┌───────────────┐         ┌────────────────────┐
│ EU (执行)│            │ U-流水线      │         │ 多核 + 超线程      │
│ BIU(总线)│            │ V-流水线      │         │ 乱序执行           │
└──────────┘            │ 分支预测      │         │ 分支预测 + 推测执行│
  无流水线               │ 2 路超标量    │         │ L1/L2/L3 Cache     │
                        └───────────────┘         │ SIMD (AVX-512)     │
                                                  └────────────────────┘
```

### 8.2 ARM 架构（RISC 典型代表）

ARM（Advanced RISC Machines）以低功耗、高能效著称，广泛用于移动设备、嵌入式系统，近年来进入服务器和笔记本市场。

#### ARM 的关键特征

| 特征 | 说明 |
|------|------|
| 指令长度 | 固定 32 位（ARM 模式）或 16 位（Thumb 模式） |
| 寄存器 | 16 个通用寄存器（R0~R15） |
| 条件执行 | 大部分指令可条件执行，减少分支 |
| Load/Store 架构 | 只有 Load/Store 指令访问内存 |
| 功耗 | 天生低功耗设计 |

#### ARM vs x86 对比

| 对比项 | x86 | ARM |
|--------|-----|-----|
| 设计哲学 | CISC | RISC |
| 指令长度 | 可变 | 固定 |
| 译码复杂度 | 高（需转换为微操作） | 低 |
| 功耗 | 较高 | 低 |
| 生态系统 | Windows/Linux 桌面主导 | 移动/嵌入式主导 |
| 性能策略 | 高频率、深流水线 | 高能效比 |

### 8.3 RISC-V 架构

RISC-V 是 2010 年诞生于 UC Berkeley 的开源指令集架构，具有模块化设计。

#### RISC-V 的关键特征

| 特征 | 说明 |
|------|------|
| 开源 | 完全免费、无专利限制 |
| 模块化 | 基础指令集 + 可选扩展（M/A/F/D/C/V 等） |
| 指令长度 | 基础 32 位，可选 16 位压缩 |
| 寄存器 | 32 个通用寄存器（x0~x31） |
| 简洁性 | 指令数量少，易于学习和实现 |

#### RISC-V 指令集模块

| 模块 | 名称 | 功能 |
|------|------|------|
| RV32I | 基础整数指令集 | 基本整数运算、访存、分支 |
| M | 乘除法扩展 | 硬件乘除法 |
| A | 原子操作扩展 | 多核同步原语 |
| F | 单精度浮点 | 32 位浮点运算 |
| D | 双精度浮点 | 64 位浮点运算 |
| C | 压缩指令 | 16 位短指令，提升代码密度 |
| V | 向量扩展 | SIMD 向量运算 |

#### RISC-V 与其他 RISC 架构对比

| 对比项 | RISC-V | ARM | MIPS |
|--------|--------|-----|------|
| 开源 | 完全开源 | 否 | 部分开源 |
| 授权费用 | 免费 | 需授权 | 部分免费 |
| 指令数 | ~100 (RV32I) | ~200+ | ~200+ |
| 应用领域 | 嵌入式、AI、教学 | 移动、服务器 | 教学、网络 |
| 发展趋势 | 高速增长 | 成熟稳定 | 逐渐式微 |

| CPI | 每条指令的平均时钟周期数 | 越小越好 |
| MIPS | 每秒百万条指令 | 不同 ISA 不宜直接比较 |
| FLOPS | 每秒浮点运算次数 | 衡量浮点性能 |

### 11.2 数据表示速查

| 概念 | 公式/表示 | 说明 |
|------|----------|------|
| 原码 | 正数不变，负数符号位为 1 | $[+5]_原 = 0,0101$；$[-5]_原 = 1,0101$ |
| 反码 | 正数不变，负数取反 | $[-5]_反 = 1,1010$ |
| 补码 | 正数不变，负数取反加 1 | $[-5]_补 = 1,1011$ |
| 移码 | 补码符号位取反 | 用于浮点数阶码 |
| IEEE 754 单精度 | 1+8+23 位 | $(-1)^S \times 1.M \times 2^{E-127}$ |
| IEEE 754 双精度 | 1+11+52 位 | $(-1)^S \times 1.M \times 2^{E-1023}$ |

### 11.3 指令系统速查

| 概念 | 说明 |
|------|------|
| 操作码 (Opcode) | 指定操作类型 |
| 地址码 (Operand) | 指定操作数来源和结果去向 |
| 寻址方式 | 确定操作数有效地址的方法 |
| 有效地址 (EA) | 操作数在主存中的实际地址 |
| 指令格式 | 操作码 + 地址码的编码方式 |

#### 常见寻址方式速查

| 寻址方式 | 有效地址 | 特点 |
|----------|---------|------|
| 立即寻址 | 操作数在指令中 | 速度快，不访存 |
| 直接寻址 | EA = A（地址码） | 简单，寻址空间小 |
| 间接寻址 | EA = (A) | 灵活，需两次访存 |
| 寄存器寻址 | 操作数在寄存器 | 速度最快 |
| 寄存器间接 | EA = (R) | 灵活，一次访存 |
| 基址寻址 | EA = (BR) + A | 适合程序重定位 |
| 变址寻址 | EA = (IX) + A | 适合数组操作 |
| 相对寻址 | EA = (PC) + A | 适合程序内部跳转 |

### 11.4 CPU 设计速查

| 概念 | 说明 |
|------|------|
| 数据通路 | 数据在 CPU 内部的物理传输路径 |
| 控制信号 | 控制器产生的、驱动数据通路各部件操作的信号 |
| 微操作 | CPU 可执行的最基本操作（如 PCout, MARin） |
| 微指令 | 一组同时执行的微操作的编码 |
| 微程序 | 完成一条机器指令的微指令序列 |
| 流水线 | 将指令执行过程分为多个阶段，多条指令重叠执行 |

#### 流水线性能公式

$$
T_{流水线} = n \times \Delta t + (k - 1) \times \Delta t
$$

其中：$n$ = 指令条数，$k$ = 流水线级数，$\Delta t$ = 时钟周期

$$
\text{加速比} S = \frac{n \times k}{n + k - 1}
$$

当 $n \to \infty$ 时，$S \to k$（理论最大加速比等于流水线级数）

### 11.5 存储系统速查

| 概念 | 说明 |
|------|------|
| 存取时间 (Access Time) | 从启动访问到数据被读出的时间 |
| 存储周期 (Cycle Time) | 连续两次访问之间的最小间隔 |
| 带宽 (Bandwidth) | 单位时间传输的数据量 |
| 局部性原理 | 时间局部性 + 空间局部性 |
| 命中率 (Hit Rate) | 访问在 Cache 中找到数据的概率 |
| 缺页率 | 访问的页不在主存中的概率 |

#### Cache 地址划分（直接映射）

```
|<--- 标记(Tag) --->|<- 行号(Index) ->|<- 块内偏移(Offset) ->|
```

**地址位数 = Tag 位数 + Index 位数 + Offset 位数**

### 11.6 I/O 系统速查

| 概念 | 说明 |
|------|------|
| 中断 | 外部事件通知 CPU 的机制 |
| 中断向量 | 中断服务程序的入口地址 |
| 中断向量表 | 存放所有中断向量的表 |
| 中断屏蔽 | 通过屏蔽字禁止某些中断 |
| DMA | 外设直接访问主存，CPU 不参与数据传送 |
| 通道 | 专用处理器，独立执行通道程序管理 I/O |
