# 第四章 定点数的表示

计算机内部一切数值数据都以二进制形式存储。对于有符号数，如何用有限的二进制位表示正负、如何统一加减运算、如何检测溢出，是本章要解决的核心问题。**补码**是现代计算机中最广泛使用的有符号数表示方法，理解其原理对后续学习运算器和CPU至关重要。

> **本章在考研中的地位**：补码的求法、补码加减法、溢出判断是每年必考内容。本章是第5章（定点运算）和第8章（浮点数）的基础。

---

## 4.1 无符号数与有符号数

### 4.1.1 无符号数

**无符号数**（Unsigned Number）的所有二进制位都用来表示数值大小，没有符号位。

对于 $n$ 位二进制数，其表示范围为：

$$
0 \leq N \leq 2^n - 1
$$

**【例 4-1】** 8位无符号数的范围：

- 最小值：$00000000_2 = 0$
- 最大值：$11111111_2 = 2^8 - 1 = 255$
- 范围：$0 \sim 255$，共 $256$ 个数

**【例 4-2】** 16位无符号数的范围：$0 \sim 65535$（$2^{16} - 1$）

### 4.1.2 有符号数

**有符号数**（Signed Number）用**最高位**表示符号：

- **0** 表示正数
- **1** 表示负数

其余位表示数值（具体含义取决于编码方式）。

### 4.1.3 真值与机器数

- **真值**：带有正负号的实际数值，如 $+5$、$-3$
- **机器数**：计算机内部存储的二进制编码，如 $00000101$（+5的原码）、$10000011$（-3的原码）

**机器数的三个特点**：

1. **符号数值化**：用0/1表示正/负
2. **数值位数固定**：受机器字长限制
3. **小数点位置隐含**：定点数中小数点不占存储位

**定点整数**：小数点隐含在最低位之后

```
┌─────┬──────────────────┐
│ 符号 │    数值部分       │
│  0/1│   n-1位          │
└─────┴──────────────────┘
```

**定点小数**：小数点隐含在符号位之后

```
┌─────┬──────────────────┐
│ 符号 │    数值部分       │
│  0/1│   n-1位          │
└─────┴──────────────────┘
```

## 4.2 原码表示法

### 4.2.1 原码的定义

**原码**（Sign-Magnitude）是最直观的有符号数表示法：最高位为符号位，其余位为数值的**绝对值**。

**整数原码**的定义：

$$
[X]_{\text{原}} = \begin{cases} X & 0 \leq X < 2^{n-1} \\ 2^{n-1} + |X| & -2^{n-1} < X \leq 0 \end{cases}
$$

**小数原码**的定义（小数点在符号位之后）：

$$
[X]_{\text{原}} = \begin{cases} X & 0 \leq X < 1 \\ 1 + |X| & -1 < X \leq 0 \end{cases}
$$

### 4.2.2 原码的编码规则

**正数的原码**：符号位为0，数值位为二进制绝对值

**负数的原码**：符号位为1，数值位为二进制绝对值

**【例 4-3】** 求以下真值的8位原码：

| 真值 | 原码 | 计算过程 |
|------|------|---------|
| +0 | `0 0000000` | 符号0 + 数值0 |
| -0 | `1 0000000` | 符号1 + 数值0 |
| +5 | `0 0000101` | 符号0 + $5 = 101_2$ |
| -5 | `1 0000101` | 符号1 + $5 = 101_2$ |
| +127 | `0 1111111` | 符号0 + $127 = 1111111_2$ |
| -127 | `1 1111111` | 符号1 + $127 = 1111111_2$ |
| +1 | `0 0000001` | |
| -1 | `1 0000001` | |

**8位原码的表示范围**：$-(2^7-1) \sim +(2^7-1)$，即 $-127 \sim +127$

**一般公式**：$n$ 位原码的表示范围为 $-(2^{n-1}-1) \sim +(2^{n-1}-1)$

<div data-component="NumberRangeExplorer"></div>

### 4.2.3 原码的数轴表示

```
-127  -126  ...  -1   -0  +0   +1  ...  +126  +127
  ←─────── 负数 ───────→│←→│←─────── 正数 ───────→
                    10000000 00000000
```

注意 $+0$ 和 $-0$ 的编码不同，中间有一个"断裂"。

### 4.2.4 原码的特点

**优点**：
- 表示直观，与真值转换简单
- 正数的原码就是其二进制表示
- 符号位和数值位分离

**缺点**：
- **0 有两种表示**：$+0 = 00000000$，$-0 = 10000000$
- **加减运算复杂**：需要判断符号位决定加还是减
- 硬件实现不友好

**【例 4-4】** 用原码计算 $5 + (-3)$。

不能直接相加！需要：
1. 比较绝对值：$|5| > |-3|$
2. 用大的减小的：$5 - 3 = 2$
3. 结果取大者的符号：正号
4. 结果：$+2$

这需要额外的比较和减法电路，硬件复杂。

> [!WARNING]
> 原码中 **0 有两种表示**（+0 和 -0），这浪费了一个编码位置，也是原码不适合直接用于运算的根本原因。

### 4.2.5 原码的适用场景

- 早期计算机使用
- 浮点数的尾数部分（配合阶码使用）
- 某些特殊用途的定点数

## 4.3 反码表示法

### 4.3.1 反码的定义

**反码**（One's Complement）是原码到补码的过渡形式。

**正数的反码** = 原码本身

**负数的反码** = 符号位不变，数值位**按位取反**

$$
[X]_{\text{反}} = \begin{cases} X & 0 \leq X < 2^{n-1} \\ (2^n - 1) - |X| & -2^{n-1} < X \leq 0 \end{cases}
$$

**另一种理解方式**：对于 $n$ 位二进制数，负数的反码 = $2^n - 1 - |X|$，即用 $n$ 个1组成的数减去绝对值。

### 4.3.2 反码的求法

**方法一：由原码求**
- 正数：反码 = 原码
- 负数：符号位不变，数值位取反

**方法二：由真值直接求**
- 正数：反码 = 二进制表示（高位补0到n位）
- 负数：先求绝对值的二进制，然后所有位取反（包括符号位）

### 4.3.3 反码的编码示例

| 真值 | 原码 | 反码 | 转换过程 |
|------|------|------|----------|
| +5 | `0 0000101` | `0 0000101` | 正数不变 |
| -5 | `1 0000101` | `1 1111010` | 符号位不变，数值位取反 |
| +0 | `0 0000000` | `0 0000000` | — |
| -0 | `1 0000000` | `1 1111111` | — |
| +127 | `0 1111111` | `0 1111111` | — |
| -127 | `1 1111111` | `1 0000000` | — |
| +1 | `0 0000001` | `0 0000001` | — |
| -1 | `1 0000001` | `1 1111110` | — |

**8位反码的表示范围**：$-(2^7-1) \sim +(2^7-1)$，即 $-127 \sim +127$

### 4.3.4 反码中0的两种表示

- $+0 = 00000000$
- $-0 = 11111111$

### 4.3.5 反码的运算

反码运算需要**循环进位**（end-around carry）：最高位的进位要加到最低位。

**【例 4-5】** 用反码计算 $5 + (-3)$。

$[+5]_{\text{反}} = 00000101$，$[-3]_{\text{反}} = 11111100$

```
  00000101
+ 11111100
---------
1 00000001    （最高位有进位）
         +1   （循环进位：加到最低位）
---------
  00000010    = +2 ✓
```

### 4.3.6 反码的特点

- **0 仍有两种表示**：$+0 = 00000000$，$-0 = 11111111$
- 反码运算需要循环进位，增加了硬件复杂度
- 反码主要作为从原码到补码的**中间过渡**，实际计算机中很少直接使用

## 4.4 补码表示法（重点）

### 4.4.1 补码的数学基础——模运算

理解补码的关键在于**模运算**（Modular Arithmetic）。

**生活中的模运算——时钟**：

想象一个只有12小时的时钟：
- 当前时间 3 点，要拨到 10 点：
  - 方法1：顺时针拨 7 小时（$3 + 7 = 10$）
  - 方法2：逆时针拨 5 小时（$3 - 5 = -2$，但时钟显示 $12 - 2 = 10$）

在模 12 的系统中，$-5 \equiv 7 \pmod{12}$，即 $-5$ 和 $7$ 对于模 12 是等价的。

**核心思想**：在模运算系统中，**减去一个数等于加上这个数关于模的补数**。

$$
a - b \equiv a + (M - b) \pmod{M}
$$

其中 $M - b$ 就是 $b$ 的**补数**。

### 4.4.2 二进制补码的推导

对于 $n$ 位二进制数，模为 $2^n$。

设 $X$ 为负数（$-2^{n-1} \leq X < 0$），则 $|X| = -X$。

$$
[X]_{\text{补}} = 2^n + X = 2^n - |X|
$$

**【例 4-6】** 推导 $-5$ 的8位补码。

$$
[-5]_{\text{补}} = 2^8 + (-5) = 256 - 5 = 251 = 11111011_2
$$

验证：$[-5]_{\text{补}} = 11111011$

**另一种推导方式**（取反加1）：

$|5| = 00000101$

取反：$11111010$

加1：$11111011$ ✓

### 4.4.3 补码的正式定义

**整数补码**：

$$
[X]_{\text{补}} = \begin{cases} X & 0 \leq X < 2^{n-1} \\ 2^n + X & -2^{n-1} \leq X < 0 \end{cases}
$$

**小数补码**：

$$
[X]_{\text{补}} = \begin{cases} X & 0 \leq X < 1 \\ 2 + X & -1 \leq X < 0 \end{cases}
$$

> [!NOTE]
> 与原码和反码不同，补码中 $-2^{n-1}$ 是**有编码的**！这是因为补码中0的表示唯一，省出了一个编码位置。

### 4.4.4 补码的求法

**方法一：由定义直接求**

- 正数的补码 = 原码
- 负数的补码 = $2^n + X$（即模减去绝对值）

**方法二：由原码求（取反加1）**

- 正数的补码 = 原码
- 负数的补码 = 符号位不变，数值位**取反加1**

**方法三：由反码求**

- 正数的补码 = 反码 = 原码
- 负数的补码 = 反码 + 1

**方法四：快速求法（从右往左，第一个1不变，其左边各位取反）**

对于负数的补码：
1. 从右往左找到第一个1
2. 这个1及其右边的0保持不变
3. 这个1左边的所有位取反（包括符号位）

**【例 4-7】** 用各种方法求 $-75$ 的8位补码。

**方法一**：$[-75]_{\text{补}} = 2^8 - 75 = 256 - 75 = 181 = 10110101_2$

**方法二**：$75 = 1001011_2$，原码 = `11001011`

取反：`10110100`，加1：`10110101` ✓

**方法三**：原码 `11001011` → 反码 `10110100` → 补码 `10110101`

**方法四**：`11001011` → 从右往左第一个1在第0位 → 不变部分：`...011`，取反部分：`101101` → 结果：`10110101`

### 4.4.5 补码的编码示例

| 真值 | 原码 | 反码 | 补码 | 说明 |
|------|------|------|------|------|
| +0 | `0 0000000` | `0 0000000` | `0 0000000` | |
| -0 | `1 0000000` | `1 1111111` | `0 0000000` | **补码中0唯一！** |
| +5 | `0 0000101` | `0 0000101` | `0 0000101` | |
| -5 | `1 0000101` | `1 1111010` | `1 1111011` | 反码+1 |
| +127 | `0 1111111` | `0 1111111` | `0 1111111` | 最大正数 |
| -127 | `1 1111111` | `1 0000000` | `1 0000001` | |
| +1 | `0 0000001` | `0 0000001` | `0 0000001` | |
| -1 | `1 0000001` | `1 1111110` | `1 1111111` | |
| -128 | 无 | 无 | `1 0000000` | **只有补码能表示！** |

### 4.4.6 -128 的补码（考研重点）

$-128$ 在原码和反码中**没有表示**（8位原码/反码只能表示 $-127 \sim +127$），但在补码中有唯一的表示：

$$
[-128]_{\text{补}} = 2^8 + (-128) = 256 - 128 = 128 = 10000000_2
$$

**验证**：对 $10000000$ 求补（取反加1）：
- 取反：$01111111$
- 加1：$10000000$

即 $10000000$ 的补码是它本身，说明它表示 $-128$。

**另一种理解**：补码 $10000000$ 没有对应的原码和反码，它是补码特有的编码。

> [!TIP]
> **8位补码的表示范围**：$-128 \sim +127$（共 256 个数）。比原码/反码多表示一个数（-128），因为 0 的表示唯一（$+0 = -0 = 00000000$）。

### 4.4.7 补码的数轴表示

<div data-component="TwosComplementVisualizer"></div>

```
-128  -127  ...  -1    0    +1  ...  +126  +127
  │←─────── 负数 ───────→│←─────── 正数 ───────→
10000000              00000000              01111111
```

注意：补码的数轴是**连续的**，没有断裂（0的表示唯一）。

### 4.4.8 补码的核心优势

| 特性 | 原码 | 反码 | 补码 |
|------|------|------|------|
| 0的表示 | +0和-0两种 | +0和-0两种 | **唯一** |
| 表示范围（8位） | -127~+127 | -127~+127 | **-128~+127** |
| 加减运算 | 需判断符号 | 需循环进位 | **统一加法** |
| 硬件实现 | 复杂 | 较复杂 | **简单** |
| 溢出检测 | 简单 | 较简单 | 需要额外逻辑 |

补码的**最大优势**：将减法统一为加法，符号位参与运算，无需额外处理。

**【例 4-8】** 用补码计算 $5 + (-3)$。

$[+5]_{\text{补}} = 00000101$，$[-3]_{\text{补}} = 11111101$

```
  00000101
+ 11111101
---------
1 00000010    （丢弃最高位进位）
```

结果 = $00000010 = +2$ ✓

**【例 4-9】** 用补码计算 $(-5) + (-3)$。

$[-5]_{\text{补}} = 11111011$，$[-3]_{\text{补}} = 11111101$

```
  11111011
+ 11111101
---------
1 11111000    （丢弃最高位进位）
```

结果 = $11111000$，是负数。求真值：取反加1 → $00001000 = 8$，所以结果为 $-8$ ✓（$-5 + (-3) = -8$）

## 4.5 移码表示法

### 4.5.1 移码的定义

**移码**（Excess/Bias Code）是在补码的基础上将**符号位取反**得到的。

$$
[X]_{\text{移}} = [X]_{\text{补}} \text{的符号位取反}
$$

等价定义：$[X]_{\text{移}} = 2^{n-1} + X$

### 4.5.2 移码的编码示例

| 真值 | 补码 | 移码 | 说明 |
|------|------|------|------|
| +127 | `0 1111111` | `1 1111111` | |
| +5 | `0 0000101` | `1 0000101` | |
| +0 | `0 0000000` | `1 0000000` | |
| -0 | `0 0000000` | `1 0000000` | 唯一 |
| -5 | `1 1111011` | `0 1111011` | |
| -127 | `1 0000001` | `0 0000001` | |
| -128 | `1 0000000` | `0 0000000` | |

### 4.5.3 移码的核心特点

1. **保持大小顺序**：移码的二进制编码大小与真值大小一致
   - 真值越大 → 移码的二进制值越大
   - 可以直接用比较器比较两个移码的大小

2. **0 的表示唯一**

3. **用途**：浮点数的**阶码**（exponent）通常用移码表示
   - IEEE 754 单精度浮点数的阶码偏移量为 127（excess-127）
   - 双精度为 1023（excess-1023）

**【例 4-10】** 为什么浮点数阶码用移码？

假设阶码用补码表示：正阶码 `0xxx`，负阶码 `1xxx`。比较两个阶码时需要考虑符号位，硬件复杂。

用移码表示：所有阶码的编码大小与实际大小一致，可以直接用无符号数比较器比较，硬件简单。

> [!NOTE]
> 移码的"移"来自偏移量（bias）。IEEE 754 单精度浮点数中，阶码字段存储的是 $E + 127$，这样即使 $E$ 为负数，存储的值也是正数，便于比较。

### 4.5.4 四种码制的完整对照表（8位）

| 真值 | 原码 | 反码 | 补码 | 移码 |
|------|------|------|------|------|
| +127 | 01111111 | 01111111 | 01111111 | 11111111 |
| +126 | 01111110 | 01111110 | 01111110 | 11111110 |
| ... | ... | ... | ... | ... |
| +1 | 00000001 | 00000001 | 00000001 | 10000001 |
| +0 | 00000000 | 00000000 | 00000000 | 10000000 |
| -0 | 10000000 | 11111111 | 00000000 | 10000000 |
| -1 | 10000001 | 11111110 | 11111111 | 01111111 |
| -2 | 10000010 | 11111101 | 11111110 | 01111110 |
| ... | ... | ... | ... | ... |
| -127 | 11111111 | 10000000 | 10000001 | 00000001 |
| -128 | 无 | 无 | 10000000 | 00000000 |

## 4.6 溢出检测

<div data-component="OverflowDetectionDemo"></div>

### 4.6.1 溢出的概念

**溢出**（Overflow）是指运算结果超出了数据表示范围的情况。

**什么时候会溢出？**

> [!WARNING]
> **只有同号数相加才可能溢出！** 异号数相加永远不会溢出（结果的绝对值一定小于两个操作数中绝对值较大的那个）。

**溢出的两种情况**：
- **正溢出**（上溢）：两个正数相加，结果为负（超出了最大正数）
- **负溢出**（下溢）：两个负数相加，结果为正（超出了最小负数）

**【例 4-11】** 8位补码中，$65 + 64$ 是否溢出？

$[+65]_{\text{补}} = 01000001$，$[+64]_{\text{补}} = 01000000$

```
  01000001
+ 01000000
---------
  10000001
```

结果 = $10000001$，是负数（-127），但 $65 + 64 = 129 > 127$，**正溢出**！

**【例 4-12】** 8位补码中，$(-65) + (-64)$ 是否溢出？

$[-65]_{\text{补}} = 10111111$，$[-64]_{\text{补}} = 11000000$

```
  10111111
+ 11000000
---------
1 01111111
```

丢弃进位，结果 = $01111111 = +127$，但 $-65 + (-64) = -129 < -128$，**负溢出**！

### 4.6.2 一位符号位检测法

只用一个符号位时，通过以下逻辑判断溢出：

$$
V = A_s \cdot B_s \cdot \overline{S_s} + \overline{A_s} \cdot \overline{B_s} \cdot S_s
$$

其中 $A_s$、$B_s$ 为操作数的符号位，$S_s$ 为结果的符号位。

**含义**：
- $A_s = B_s = 1$（两负数）且 $S_s = 0$（结果为正）→ 负溢出
- $A_s = B_s = 0$（两正数）且 $S_s = 1$（结果为负）→ 正溢出

**【例 4-13】** 用一位符号位法判断 $65 + 64$ 是否溢出。

$A_s = 0, B_s = 0, S_s = 1$

$V = 0 \cdot 0 \cdot 0 + 1 \cdot 1 \cdot 1 = 1$ → **溢出！**

### 4.6.3 双符号位检测法（变形补码）

用**两个符号位**表示数据（00正，11负），也称为**模4补码**。

| 双符号位 | 含义 |
|----------|------|
| 00 | 正数（结果正确） |
| 01 | 正溢出 |
| 10 | 负溢出 |
| 11 | 负数（结果正确） |

**溢出判断**：两个符号位**不同**即为溢出。

$$
V = S_{s1} \oplus S_{s2}
$$

**【例 4-14】** 用双符号位补码判断 $65 + 64$ 是否溢出（8位）。

$[+65]_{\text{补}} = 00\;1000001$，$[+64]_{\text{补}} = 00\;1000000$

```
  00 1000001
+ 00 1000000
-----------
  01 0000001
```

双符号位 = `01` → **正溢出！**（$65 + 64 = 129 > 127$）

**【例 4-15】** 用双符号位补码判断 $(-65) + (-64)$ 是否溢出。

$[-65]_{\text{补}} = 11\;0111111$，$[-64]_{\text{补}} = 11\;1000000$

```
  11 0111111
+ 11 1000000
----------
1 10 1111111
```

丢弃最高位进位，双符号位 = `10` → **负溢出！**（$-65 + (-64) = -129 < -128$）

### 4.6.4 进位检测法

利用最高数值位的进位 $C_n$ 和符号位的进位 $C_s$ 判断：

$$
V = C_n \oplus C_s
$$

- $C_n = 1$ 且 $C_s = 0$ → 正溢出
- $C_n = 0$ 且 $C_s = 1$ → 负溢出
- $C_n = C_s$ → 无溢出

**【例 4-16】** 用进位法判断 $65 + 64$ 是否溢出。

```
  0 1000001
+ 0 1000000
---------
  1 0000001
```

- 符号位进位 $C_s = 0$（符号位相加无进位）
- 最高数值位进位 $C_n = 1$（第6位相加有进位）
- $V = C_n \oplus C_s = 1 \oplus 0 = 1$ → **溢出！**

### 4.6.5 三种溢出检测方法对比

| 方法 | 需要的符号位 | 判断逻辑 | 硬件复杂度 | 特点 |
|------|------------|----------|-----------|------|
| 一位符号位 | 1 | $A_s B_s \overline{S_s} + \overline{A_s}\overline{B_s}S_s$ | 较复杂 | 需要与门和或门 |
| 双符号位 | 2 | $S_{s1} \oplus S_{s2}$ | **最简单** | 只需1个异或门 |
| 进位检测 | 1 | $C_n \oplus C_s$ | 简单 | 只需1个异或门 |

> [!TIP]
> **考试建议**：双符号位法和进位法最常用，推荐优先使用。一位符号位法虽然逻辑稍复杂，但理解其含义对做选择题很有帮助。

## 4.8 本章典型例题

### 例题1：各种码制的转换

**题目**：求 $-75$ 的8位原码、反码、补码和移码。

**解答**：

$|-75| = 75 = 1001011_2$

- 原码：`1 1001011`（符号位1 + 绝对值）
- 反码：`1 0110100`（符号位不变，数值位取反）
- 补码：`1 0110101`（反码 + 1）
- 移码：`0 0110101`（补码符号位取反）

**快速验证**：补码取反加1应得到原码。

$10110101 \to 01001010 + 1 = 01001011$... 不对，应该是 $11001011$。

等等，让我重新算。补码 = `10110101`，取反加1（连同符号位）：

$10110101 \to 01001010 + 1 = 01001011$

这得到的是绝对值的补码，说明 $-75$ 的补码确实是 `10110101`。

### 例题2：补码求真值

**题目**：已知 $[X]_{\text{补}} = 10110100$，求 $X$ 的真值。

**解答**：

最高位为1，说明是负数。

**方法**：将补码**取反加1**（连同符号位）得到绝对值的二进制。

$10110100 \to 01001011 + 1 = 01001100 = 76$

所以 $X = -76$

**验证**：$[-76]_{\text{补}} = 2^8 - 76 = 180 = 10110100$ ✓

## 4.9 本章常见误区与易错点

1. **-128 没有原码和反码**：只有补码能表示 -128（`10000000`），这是考研常考的陷阱。

2. **补码取反加1得到的是绝对值**：对负数补码取反加1得到的是其绝对值的二进制，而不是另一个数的补码。

3. **溢出 vs 进位**：溢出是指结果超出表示范围，进位是指最高位产生的进位。两者概念不同，溢出不一定有进位，有进位不一定溢出。

4. **符号位参与运算**：补码运算中，符号位与数值位一起参与加法运算，不需要单独处理。

5. **移码与补码的关系**：移码 = 补码的符号位取反。不要混淆"移码"和"偏移量"。

6. **双符号位的含义**：`00`正、`01`正溢出、`10`负溢出、`11`负。记住：**不同则溢出**。

7. **原码中0有两种表示**：+0和-0的编码不同，浪费了一个编码位置。

8. **反码中0也有两种表示**：+0 = `00000000`，-0 = `11111111`。

9. **补码的范围比原码/反码多一个数**：因为0的表示唯一，省出一个位置给-128。

10. **移码保持大小顺序**：可以直接用无符号数比较器比较移码的大小。

## 4.10 本章总结

### 核心公式

| 公式 | 含义 |
|------|------|
| $n$位补码范围 $-2^{n-1} \sim 2^{n-1}-1$ | 比原码多表示一个数 |
| 负数补码 = 取反加1 | 快速求补码的方法 |
| $V = S_{s1} \oplus S_{s2}$ | 双符号位溢出判断 |
| $V = C_n \oplus C_s$ | 进位法溢出判断 |
| $[X]_{\text{移}} = 2^{n-1} + X$ | 移码的定义 |

### 知识框架

```
定点数表示
├── 无符号数：0 ~ 2^n - 1
└── 有符号数
    ├── 原码：直观，0有两种，运算复杂
    ├── 反码：过渡，0有两种，需循环进位
    ├── 补码：★重点★，0唯一，统一加法
    └── 移码：符号位取反，用于浮点阶码
```
