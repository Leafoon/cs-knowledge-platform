# 第十二章 寻址方式

寻址方式决定了指令中操作数地址的形成方法。不同的寻址方式影响指令的灵活性、寻址范围和执行效率。**有效地址的计算是考研高频考点**。

## 12.1 寻址方式概述

### 12.1.1 有效地址（EA）

**有效地址（EA）**：操作数在存储器中的实际地址。

寻址方式就是从指令中的地址码字段**计算出有效地址**的方法。

### 12.1.2 寻址方式的分类

```
寻址方式
├── 指令寻址
│   ├── 顺序寻址
│   └── 跳跃寻址
└── 数据寻址
    ├── 立即寻址
    ├── 直接寻址
    ├── 间接寻址
    ├── 寄存器寻址
    ├── 寄存器间接寻址
    ├── 基址寻址
    ├── 变址寻址
    ├── 相对寻址
    └── 堆栈寻址
```

---

## 12.2 指令寻址

### 12.2.1 顺序寻址

PC自动递增，指向下一条指令。

```
PC → PC + 1
```

### 12.2.2 跳跃寻址

通过转移指令修改PC的值，实现程序跳转。

```
PC → 目标地址
```

## 12.3 数据寻址方式

### 12.3.1 立即寻址

操作数直接在指令中。

$$
\text{EA} = \text{地址字段本身（就是操作数）}
$$

- 优点：无需访存，速度最快
- 缺点：操作数范围受限于地址字段长度
- 用途：常量赋值、初始化

**【例 12-1】** 指令 `MOV R1, #5`，操作数5直接在指令中。

### 12.3.2 直接寻址

地址字段就是操作数的有效地址。

$$
\text{EA} = A
$$

- 优点：只需一次访存
- 缺点：寻址范围受限于地址字段长度

**【例 12-2】** 指令 `MOV R1, [1000]`，EA = 1000，操作数 = M(1000)。

### 12.3.3 间接寻址

地址字段指向的存储单元存放的是有效地址。

$$
\text{EA} = (A)
$$

- 一次间接：地址字段 → 有效地址
- 多次间接：地址字段 → 地址 → ... → 有效地址
- 优点：寻址范围大
- 缺点：需要多次访存（至少2次）

**【例 12-3】** 指令 `MOV R1, @1000`，M(1000) = 2000，EA = 2000，操作数 = M(2000)。

### 12.3.4 寄存器寻址

操作数在寄存器中。

$$
\text{EA} = R_i
$$

- 优点：无需访存，速度最快
- 缺点：寄存器数量有限

**【例 12-4】** 指令 `MOV R1, R2`，操作数 = R2。



<div data-component="RegisterFileViz"></div>

### 12.3.5 寄存器间接寻址

寄存器中存放的是操作数的地址。

$$
\text{EA} = (R_i)
$$

- 比直接间接寻址快（寄存器访问比存储器快）

**【例 12-5】** 指令 `MOV R1, [R2]`，R2 = 1000，EA = 1000，操作数 = M(1000)。

### 12.3.6 基址寻址

$$
\text{EA} = (\text{BR}) + A
$$

BR为**基址寄存器**（Base Register），A为指令中的地址字段（偏移量/位移量）。

- 用途：程序重定位、存储空间扩展
- 特点：基址寄存器内容由**操作系统**设定，用户不能修改

**【例 12-6】** 指令 `MOV R1, 100[BR]`，BR = 2000，EA = 2000 + 100 = 2100。

### 12.3.7 变址寻址

$$
\text{EA} = (\text{IX}) + A
$$

IX为**变址寄存器**（Index Register），A为指令中的地址字段。

- 用途：数组访问、循环操作
- 特点：变址寄存器内容由**用户程序**修改

**【例 12-7】** 指令 `MOV R1, 1000[IX]`，IX = 50，EA = 1000 + 50 = 1050。

### 12.3.8 基址 vs 变址

| 特性 | 基址寻址 | 变址寻址 |
|------|---------|---------|
| 基址/变址寄存器 | 由OS设定 | 由用户修改 |
| 地址字段 | 偏移量 | 基地址 |
| 主要用途 | 程序重定位 | 数组/循环 |
| 地址字段变化 | 不变 | 变化（循环中） |

<div data-component="IndexVsBaseCompare"></div>

### 12.3.9 相对寻址

$$
\text{EA} = (\text{PC}) + A
$$

以PC为基址，A为偏移量（可正可负）。

- 用途：条件跳转、位置无关代码（PIC）
- 特点：程序可以加载到内存任意位置执行

**【例 12-8】** 指令 `JMP +10`，PC = 1000，EA = 1000 + 10 = 1010。

### 12.3.10 堆栈寻址

操作数隐含在栈顶。

- 硬件堆栈：用寄存器实现（速度快）
- 软件堆栈：用存储器实现（容量大）

## 12.4 寻址方式综合对比

| 寻址方式 | 有效地址 | 访存次数 | 速度 | 适用场景 |
|----------|---------|---------|------|---------|
| 立即寻址 | 操作数在指令中 | 0 | 最快 | 常量 |
| 直接寻址 | EA = A | 1 | 快 | 简单访问 |
| 间接寻址 | EA = (A) | 2+ | 慢 | 间接引用 |
| 寄存器寻址 | EA = R | 0 | 最快 | 频繁使用 |
| 寄存器间接 | EA = (R) | 1 | 较快 | 指针 |
| 基址寻址 | EA = BR+A | 1 | 较快 | 重定位 |
| 变址寻址 | EA = IX+A | 1 | 较快 | 数组 |
| 相对寻址 | EA = PC+A | 1 | 较快 | 跳转 |

## 12.5 本章典型例题

### 例题1：寻址方式分析

**题目**：设R1=2000H，(2000H)=3000H，(3000H)=4000H，指令 `MOV R0, (R1)` 的操作数是什么？

**解答**：

寄存器间接寻址：EA = (R1) = 2000H，操作数 = (2000H) = 3000H

### 例题2：变址寻址计算

**题目**：IX=100H，指令 `LOAD A, 50H[IX]`，求有效地址。

**解答**：

EA = (IX) + A = 100H + 50H = 150H

### 例题3：相对寻址

**题目**：PC=1000H，指令 `JMP -20H`，求目标地址。

**解答**：

EA = (PC) + A = 1000H + (-20H) = 0FE0H

## 12.6 本章常见误区与易错点

1. **基址寻址和变址寻址的区别**：基址寄存器由OS设定，变址寄存器由用户修改。

2. **间接寻址的访存次数**：一次间接需要2次访存（第一次取地址，第二次取数据）。

3. **相对寻址的偏移量**：可以是正数也可以是负数。

4. **立即寻址的操作数在指令中**：不需要访存。

5. **寄存器寻址不需要访存**：操作数在寄存器中。

6. **基址寻址的地址字段是偏移量**：不是基地址。

7. **变址寻址的地址字段是基地址**：不是偏移量。

## 12.9 深入理解各种寻址方式

### 12.9.1 立即寻址的深入分析

**立即寻址**的操作数直接包含在指令中，不需要访问存储器。

**优点**：
- 速度最快（0次访存）
- 指令自包含，不需要额外的地址计算

**缺点**：
- 操作数范围受限于地址字段长度
- 操作数在编译时确定，运行时不能改变

**典型应用**：
- 常量赋值：`MOV R1, #5`
- 初始化：`MOV R1, #0`
- 位操作：`AND R1, #0xFF`

**立即数的范围**：
- n位地址字段：$-2^{n-1}$ 到 $2^{n-1}-1$（有符号）
- n位地址字段：$0$ 到 $2^n-1$（无符号）

### 12.9.2 直接寻址的深入分析

**直接寻址**的地址字段就是操作数的有效地址。

**有效地址**：$EA = A$

**优点**：
- 简单直观
- 只需1次访存

**缺点**：
- 寻址范围受限于地址字段长度
- 地址在编译时确定，运行时不能改变

**典型应用**：
- 全局变量访问：`MOV R1, [global_var]`
- 固定地址访问：`MOV R1, [0x1000]`

### 12.9.3 间接寻址的深入分析

**间接寻址**的地址字段指向的存储单元存放的是有效地址。

**有效地址**：$EA = (A)$

**一次间接**：地址字段 → 有效地址 → 操作数

**多次间接**：地址字段 → 地址1 → 地址2 → ... → 有效地址 → 操作数

**优点**：
- 寻址范围大
- 支持动态地址

**缺点**：
- 需要多次访存（至少2次）
- 速度慢

**典型应用**：
- 指针访问：`MOV R1, @ptr`
- 函数指针：`CALL @func_ptr`

### 12.9.4 寄存器寻址的深入分析

**寄存器寻址**的操作数在寄存器中。

**有效地址**：$EA = R_i$

**优点**：
- 速度最快（0次访存）
- 寄存器访问比存储器访问快得多

**缺点**：
- 寄存器数量有限
- 需要额外的寄存器管理

**典型应用**：
- 变量操作：`ADD R1, R2, R3`
- 临时存储：`MOV R1, R2`

### 12.9.5 寄存器间接寻址的深入分析

**寄存器间接寻址**的寄存器中存放的是操作数的地址。

**有效地址**：$EA = (R_i)$

**优点**：
- 比直接间接寻址快
- 支持动态地址

**缺点**：
- 需要1次访存

**典型应用**：
- 指针访问：`MOV R1, [R2]`
- 数组遍历：通过修改寄存器值访问不同元素

### 12.9.6 基址寻址的深入分析

**基址寻址**的有效地址 = 基址寄存器 + 偏移量。

**有效地址**：$EA = (BR) + A$

**特点**：
- 基址寄存器由**操作系统**设定
- 用户不能修改基址寄存器
- 偏移量在指令中

**典型应用**：
- 程序重定位：程序加载到不同地址时，只需修改基址寄存器
- 存储空间扩展：超出地址字段范围的访问

**【例 12-6】** 程序加载到地址0x1000开始的区域。

基址寄存器BR = 0x1000

指令 `MOV R1, 50[BR]` 访问地址 0x1000 + 50 = 0x1032

### 12.9.7 变址寻址的深入分析

**变址寻址**的有效地址 = 变址寄存器 + 地址字段。

**有效地址**：$EA = (IX) + A$

**特点**：
- 变址寄存器由**用户程序**修改
- 地址字段是基地址
- 变址寄存器是偏移量

**典型应用**：
- 数组访问：`MOV R1, array[IX]`
- 循环操作：通过修改IX访问数组的不同元素

**【例 12-7】** 数组array起始地址1000，访问第5个元素（假设每个元素4字节）。

变址寄存器IX = 5 × 4 = 20

指令 `MOV R1, 1000[IX]` 访问地址 1000 + 20 = 1020

### 12.9.8 基址寻址 vs 变址寻址的深入对比

| 特性 | 基址寻址 | 变址寻址 |
|------|---------|---------|
| 寄存器内容 | 由OS设定 | 由用户修改 |
| 地址字段 | 偏移量 | 基地址 |
| 主要用途 | 程序重定位 | 数组/循环 |
| 地址字段变化 | 不变 | 变化（循环中） |
| 典型场景 | 程序加载到不同地址 | 遍历数组 |

**关键区别**：
- 基址寻址：**基址固定，偏移变化**
- 变址寻址：**基址固定，变址变化**

实际上，两者可以结合使用：$EA = (BR) + (IX) + A$

### 12.9.9 相对寻址的深入分析

**相对寻址**的有效地址 = PC + 偏移量。

**有效地址**：$EA = (PC) + A$

**特点**：
- 以PC为基址
- 偏移量可正可负
- 程序可以加载到内存任意位置执行

**典型应用**：
- 条件跳转：`BEQ R1, R2, label`
- 无条件跳转：`JMP +10`
- 位置无关代码（PIC）

**【例 12-8】** PC = 1000，指令 `JMP -20`。

目标地址 = 1000 + (-20) = 980

### 12.9.10 堆栈寻址的深入分析

**堆栈寻址**的操作数隐含在栈顶。

**硬件堆栈**：
- 用寄存器实现
- 速度快
- 容量有限

**软件堆栈**：
- 用存储器实现
- 容量大
- 速度较慢

**堆栈操作**：
- PUSH：将数据压入栈顶
- POP：从栈顶弹出数据
- 栈指针（SP）自动调整

## 12.10 寻址方式的综合应用

### 12.10.1 数组访问

**方法1：变址寻址**

```assembly
MOV IX, 0          ; 初始化变址寄存器
LOOP: MOV R1, array[IX]  ; 访问array[IX]
      ADD IX, 4          ; IX += 4（假设每个元素4字节）
      CMP IX, 40         ; 比较是否到达数组末尾
      BNE LOOP           ; 未到达则继续循环
```

**方法2：寄存器间接寻址 + 自动增量**

```assembly
MOV R2, array      ; R2指向数组起始地址
LOOP: MOV R1, [R2]+  ; 访问*R2，然后R2++
      CMP R2, array_end
      BNE LOOP
```

### 12.10.2 函数调用

**参数传递**：
- 寄存器传参：参数放在寄存器中
- 栈传参：参数压入栈中

**返回地址**：
- 保存在寄存器中（如MIPS的$ra）
- 保存在栈中（如x86）

### 12.10.3 指针操作

**一级指针**：寄存器间接寻址

```assembly
MOV R2, ptr        ; R2 = ptr的值
MOV R1, [R2]       ; R1 = *ptr
```

**二级指针**：间接寻址

```assembly
MOV R2, ptr        ; R2 = ptr的值
MOV R3, [R2]       ; R3 = *ptr
MOV R1, [R3]       ; R1 = **ptr
```

## 12.11 本章综合练习

### 练习1：有效地址计算

**题目**：设R1=0x1000，R2=0x500，(0x1500)=0x2000，(0x2000)=0x3000。求以下指令的有效地址和操作数：

1. `MOV R0, #100`
2. `MOV R0, [0x1500]`
3. `MOV R0, @0x1500`
4. `MOV R0, R1`
5. `MOV R0, [R1]`
6. `MOV R0, 500[R1]`
7. `MOV R0, [R1+R2]`

**解答**：

1. 立即寻址：操作数 = 100
2. 直接寻址：EA = 0x1500，操作数 = (0x1500) = 0x2000
3. 间接寻址：EA = (0x1500) = 0x2000，操作数 = (0x2000) = 0x3000
4. 寄存器寻址：操作数 = R1 = 0x1000
5. 寄存器间接寻址：EA = (R1) = 0x1000，操作数 = (0x1000)
6. 基址寻址：EA = (R1) + 500 = 0x1000 + 500 = 0x11F4，操作数 = (0x11F4)
7. 变址寻址：EA = (R1) + (R2) = 0x1000 + 0x500 = 0x1500，操作数 = (0x1500) = 0x2000

### 练习2：寻址方式选择

**题目**：以下场景应该使用哪种寻址方式？

1. 访问数组的第i个元素
2. 调用函数
3. 访问全局变量
4. 加载常量
5. 程序重定位

**解答**：

1. 变址寻址：EA = 基地址 + i × 元素大小
2. 相对寻址或寄存器间接寻址
3. 直接寻址
4. 立即寻址
5. 基址寻址

### 练习3：寻址方式综合

**题目**：描述以下C代码的汇编实现：

```c
int array[10];
int sum = 0;
for (int i = 0; i < 10; i++) {
    sum += array[i];
}
```

**解答**：

```assembly
MOV R1, 0          ; sum = 0
MOV R2, 0          ; i = 0
MOV R3, array      ; R3指向array
LOOP: ADD R1, [R3]  ; sum += array[i]
      ADD R3, 4     ; R3指向下一个元素
      ADD R2, 1     ; i++
      CMP R2, 10    ; 比较i和10
      BNE LOOP      ; 如果i != 10，继续循环
```

## 12.13 寻址方式的工程实践

### 12.13.1 现代处理器的寻址方式

**x86的寻址方式**：

| 寻址方式 | 示例 | 有效地址 |
|----------|------|---------|
| 立即数 | MOV EAX, 5 | 操作数=5 |
| 寄存器 | MOV EAX, EBX | 操作数=EBX |
| 直接 | MOV EAX, [1000] | EA=1000 |
| 寄存器间接 | MOV EAX, [EBX] | EA=EBX |
| 基址+偏移 | MOV EAX, [EBX+100] | EA=EBX+100 |
| 基址+变址 | MOV EAX, [EBX+ECX] | EA=EBX+ECX |
| 比例变址 | MOV EAX, [EBX+ECX*4] | EA=EBX+ECX×4 |

**MIPS的寻址方式**：

| 寻址方式 | 示例 | 有效地址 |
|----------|------|---------|
| 立即数 | ADDI $t0, $zero, 5 | 操作数=5 |
| 寄存器 | ADD $t0, $t1, $t2 | 操作数=$t2 |
| 基址 | LW $t0, 4($sp) | EA=$sp+4 |
| PC相对 | BEQ $t0, $t1, label | EA=PC+label |

### 12.13.2 寻址方式的选择

| 场景 | 推荐寻址方式 | 原因 |
|------|------------|------|
| 常量 | 立即寻址 | 无需访存 |
| 局部变量 | 寄存器寻址 | 速度快 |

## 12.15 本章扩展练习

### 练习4：寻址方式综合

**题目**：设R1=0x1000，R2=0x500，(0x1500)=0x2000，(0x2000)=0x3000，PC=0x8000。求以下指令的有效地址和操作数：

1. `MOV R0, #0x1234`
2. `MOV R0, [0x1500]`
3. `MOV R0, @0x1500`
4. `MOV R0, R1`
5. `MOV R0, [R1]`
6. `MOV R0, 500[R1]`
7. `MOV R0, [R1+R2]`
8. `JMP +100`

**解答**：

1. 立即寻址：操作数 = 0x1234
2. 直接寻址：EA = 0x1500，操作数 = (0x1500) = 0x2000
3. 间接寻址：EA = (0x1500) = 0x2000，操作数 = (0x2000) = 0x3000
4. 寄存器寻址：操作数 = R1 = 0x1000
5. 寄存器间接寻址：EA = (R1) = 0x1000，操作数 = (0x1000)
6. 基址寻址：EA = (R1) + 500 = 0x1000 + 500 = 0x11F4，操作数 = (0x11F4)
7. 变址寻址：EA = (R1) + (R2) = 0x1000 + 0x500 = 0x1500，操作数 = (0x1500) = 0x2000
8. 相对寻址：EA = (PC) + 100 = 0x8000 + 100 = 0x8064

### 练习5：寻址方式选择

**题目**：以下C代码应该使用哪种寻址方式？

```c
int array[100];
int sum = 0;
for (int i = 0; i < 100; i++) {
    sum += array[i];
}
```

**解答**：

- `array`：直接寻址（全局变量地址）
- `i`：寄存器寻址（循环变量）
- `array[i]`：变址寻址（EA = array基地址 + i × 4）
- `sum`：寄存器寻址（累加器）

### 练习6：寻址方式性能

**题目**：比较以下两种代码的性能：

**代码A**（使用间接寻址）：
```assembly
MOV R1, [R2]  ; R2指向数据
```

**代码B**（使用直接寻址）：
```assembly
MOV R1, [0x1000]  ; 直接访问地址
```

**解答**：

- 代码A：1次访存（R2 → 数据）
- 代码B：1次访存（地址 → 数据）
- 性能相同，但代码A更灵活（R2可以指向任意地址）

## 12.18 寻址方式的常见错误

### 12.18.1 有效地址计算错误

**错误1**：混淆基址和变址。

**正确**：
- 基址寻址：EA = 基址寄存器 + 偏移量（偏移量在指令中）
- 变址寻址：EA = 变址寄存器 + 基地址（基地址在指令中）

**错误2**：相对寻址的PC值错误。

**正确**：PC是当前指令的下一条指令地址，不是当前指令地址。

**错误3**：间接寻址的访存次数错误。

**正确**：一次间接需要2次访存（第一次取地址，第二次取数据）。

### 12.18.2 寻址方式选择错误

**错误1**：数组访问使用直接寻址。

**正确**：数组访问应使用变址寻址（支持循环）。

**错误2**：常量使用直接寻址。

**正确**：常量应使用立即寻址（无需访存）。

## 12.22 地址计算单元（AGU）的内部结构与工作原理

### 12.22.1 AGU概述

<div data-component="AddressCalculationUnit"></div>

地址生成单元（Address Generation Unit，AGU）是CPU中专门负责计算有效地址的硬件模块。在支持复杂寻址方式的处理器中，AGU与ALU并行工作，避免地址计算占用ALU资源。

**AGU的核心功能**：

| 寻址方式 | AGU计算 | 公式 |
|----------|---------|------|
| 直接寻址 | 无需计算 | EA = A |
| 基址+偏移 | 一次加法 | EA = Base + Offset |
| 基址+变址 | 一次加法 | EA = Base + Index |
| 比例变址 | 移位+加法 | EA = Base + Index × Scale + Disp |
| PC相对 | 一次加法 | EA = PC + Offset |

### 12.22.2 AGU的内部结构

```
┌─────────────────────────────────────────┐
│              AGU 内部结构                 │
│                                          │
│  Base ──→ ┌──────┐                      │
│           │ 加法器 │──→ ┌──────┐        │
│  Index ──→│  A    │    │ 加法器 │──→ EA │
│  Scale ──→│  ×S   │──→│  B    │        │
│           └──────┘    └──────┘        │
│  Disp ──────────────────→ ↑            │
└─────────────────────────────────────────┘
```

x86处理器的AGU需要计算 $EA = Base + Index \times Scale + Displacement$，其中Scale可以是1、2、4或8。

| 组件 | 功能 | 延迟 |
|------|------|------|
| 比例乘法器 | Index × Scale | 1个周期 |
| 第一级加法器 | Base + Index×Scale | 0.5ns |
| 第二级加法器 | 结果 + Displacement | 0.5ns |
| 总延迟 | | 1~2ns |

### 12.22.3 AGU与ALU的关系

在简单处理器中，地址计算和数据运算共用ALU；在高性能处理器中，AGU是独立的功能单元。

| 设计方式 | 优点 | 缺点 | 适用场景 |
|----------|------|------|---------|
| 共用ALU | 硬件简单 | 地址计算与数据运算冲突 | 简单RISC |
| 独立AGU | 并行执行 | 面积增大 | 高性能CISC/RISC |

> [!NOTE]
> MIPS等简单RISC处理器通常没有独立AGU，因为Load/Store指令的地址计算（基址+偏移）可以复用ALU。x86等CISC处理器由于寻址方式复杂，通常配备独立AGU。

## 12.23 地址加法器的设计与性能

### 12.23.1 地址加法器的特殊需求

地址加法器是AGU中的核心运算部件，与普通ALU加法器相比有以下特殊要求：

| 需求 | 说明 |
|------|------|
| 低位进位输入 | 支持PC+Offset（偏移量可正可负） |
| 饱和/截断 | 地址溢出时的处理策略 |
| 速度优先 | 地址计算在关键路径上 |

### 12.23.2 加法器实现方式

| 实现方式 | 延迟 | 面积 | 适用场景 |
|----------|------|------|---------|
| 行波进位加法器 | O(n) | 小 | 低功耗嵌入式 |
| 先行进位加法器 | O(log n) | 中 | 通用处理器 |
| 选择进位加法器 | O(√n) | 中 | 平衡方案 |
| 超前进位加法器 | O(log n) | 大 | 高性能处理器 |

现代高性能处理器的地址加法器通常采用**先行进位（CLA）**或**选择进位（CSA）**设计，在32/64位地址宽度下实现1个周期内的地址计算。

### 12.23.3 地址加法器在数据通路中的位置

```
取指阶段        译码阶段        执行阶段        访存阶段
    │              │              │              │
    │    寄存器读 → │              │              │
    │              │ → 地址加法器 →│→ 访存地址     │
    │              │ → ALU →      │→ 数据运算结果  │
    │              │              │              │
```

地址加法器通常在**执行阶段的前半段**完成计算，以便后半段将有效地址送往存储器接口。

> [!TIP]
> 地址加法器的延迟是Load/Store指令执行延迟的关键因素。在5级流水线中，地址计算必须在一个时钟周期内完成，这要求加法器的延迟不超过半个周期。

## 12.24 寄存器文件的组织与读写端口

### 12.24.1 寄存器文件的基本结构

寄存器文件（Register File）是CPU中速度最快的存储部件，由多个寄存器和多端口读写逻辑组成。

```
         读端口1    读端口2
            ↓          ↓
        ┌──────────────────┐
  R0 ──→│  32×32 寄存器阵列  │
  R1 ──→│  (32个32位寄存器)  │
  ...   │                    │
  R31──→│                    │
        └──────────────────┘
            ↑          ↑
         写端口1    写使能
```

### 12.24.2 读写端口数量与寻址方式的关系

不同的指令格式和寻址方式对寄存器文件的端口数量有不同要求：

| 指令类型 | 读端口需求 | 写端口需求 | 说明 |
|----------|-----------|-----------|------|
| 三地址R型 | 2读 | 1写 | ADD rd, rs, rt |
| Load指令 | 1读 | 1写 | LW rt, offset(rs) |
| Store指令 | 2读 | 0写 | SW rt, offset(rs) |
| 立即数指令 | 1读 | 1写 | ADDI rt, rs, imm |

为支持每周期执行一条R型指令，寄存器文件至少需要**2个读端口和1个写端口**。

### 12.24.3 多端口寄存器文件的实现

| 端口配置 | 面积开销 | 适用场景 |
|----------|---------|---------|
| 1R1W | 基准 | 简单单发射 |
| 2R1W | ~2× | 标准RISC |
| 2R2W | ~3× | 超标量/乱序 |
| 4R2W | ~5× | 高性能乱序 |
| 6R3W | ~8× | 宽发射超长流水线 |

端口数量增加会导致面积和功耗显著增大，因此寄存器文件的端口数量是性能与功耗的重要权衡点。

### 12.24.4 寄存器文件的访问延迟

寄存器文件的访问延迟取决于端口数量和寄存器数量：

| 寄存器数量 | 位宽 | 读延迟 | 写延迟 |
|-----------|------|--------|--------|
| 16 | 32位 | 0.3ns | 0.5ns |
| 32 | 32位 | 0.5ns | 0.7ns |
| 32 | 64位 | 0.6ns | 0.8ns |
| 128(ROB) | 64位 | 1.0ns | 1.2ns |

> [!NOTE]
> RISC处理器通常有32个通用寄存器，寄存器文件的读延迟约为0.5ns，可以在一个流水级内完成。如果寄存器数量过多（如128个），访问延迟增大，可能需要增加流水级数来满足时序要求。

## 12.25 地址计算在数据通路中的位置与延迟

### 12.25.1 地址计算的流水级位置

在经典的5级流水线中，地址计算位于**EX（执行）阶段**：

```
IF      ID       EX       MEM      WB
取指    译码     执行     访存     写回
         │        │        │
         │   ┌────┘        │
         │   │ 地址计算     │
         │   │ Base+Offset │
         │   └────→ EA ────┘
         │              访问D-Cache
```

### 12.25.2 地址计算的延迟分析

Load/Store指令的关键路径：

```
寄存器读(0.5ns) → 地址加法(0.5ns) → Cache访问(1.0ns) → 数据准备
       ID阶段           EX阶段           MEM阶段
```

**总延迟**：约2.0ns（不含取指和写回）

| 流水级 | 操作 | 延迟 | 说明 |
|--------|------|------|------|
| ID | 读基址寄存器 | 0.5ns | 寄存器文件读延迟 |
| EX | Base + Offset | 0.5ns | 地址加法器延迟 |
| MEM | Cache访问 | 1.0ns | 地址译码+数据读取 |

### 12.25.3 地址计算的优化技术

**一、地址旁路（Address Bypassing）**

当基址寄存器的值来自前一条指令的结果时，通过旁路网络直接将结果送入地址加法器，避免等待寄存器写回。

```
ADD R1, R2, R3    ← 产生R1的新值
LW  R4, 0(R1)     ← 使用R1作为基址
      │
      └→ 旁路：直接将ADD的结果送入地址加法器
```

**二、提前地址计算**

在乱序执行处理器中，Load指令的地址计算可以提前到发射阶段执行，以便尽早启动Cache访问。

**三、地址计算与数据运算并行**

独立AGU使得地址计算和ALU数据运算可以同时进行：

```
周期1：LW R1, 0(R2)  ──→ AGU计算地址
       ADD R3, R4, R5 ──→ ALU执行加法
       （两条指令并行执行）
```

> [!TIP]
> 地址计算的延迟是访存指令性能的关键瓶颈。在设计流水线时，应确保地址加法器的延迟与ALU的延迟匹配，避免某一个成为关键路径而拉低整体时钟频率。
