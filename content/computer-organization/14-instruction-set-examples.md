# 第十四章 指令系统设计与实例分析

本章通过MIPS和x86两个经典指令集实例，深入理解指令系统的设计原则和实现细节。**MIPS指令编码是考研常见题型**。

---

## 14.1 MIPS指令集分析

### 14.1.1 MIPS简介

MIPS（Microprocessor without Interlocked Pipeline Stages）是最经典的RISC架构，广泛用于教学和嵌入式系统。

**MIPS的特点**：

| 特性 | 说明 |
|------|------|
| 指令长度 | 固定32位 |
| 寄存器 | 32个通用寄存器 |
| 指令格式 | R/I/J三种 |
| 寻址方式 | 立即数、寄存器、基址、PC相对 |
| 流水线 | 天然适合5级流水线 |

### 14.1.2 MIPS寄存器

| 编号 | 名称 | 用途 | 调用者保存 |
|------|------|------|-----------|
| $0 | $zero | 恒为0 | — |
| $1 | $at | 汇编器保留 | — |
| $2-$3 | $v0-$v1 | 函数返回值 | 否 |
| $4-$7 | $a0-$a3 | 函数参数 | 否 |
| $8-$15 | $t0-$t7 | 临时变量 | 否 |
| $16-$23 | $s0-$s7 | 保存变量 | 是 |
| $24-$25 | $t8-$t9 | 临时变量 | 否 |
| $26-$27 | $k0-$k1 | 内核保留 | — |
| $28 | $gp | 全局指针 | — |
| $29 | $sp | 栈指针 | 是 |
| $30 | $fp | 帧指针 | 是 |
| $31 | $ra | 返回地址 | 否 |

### 14.1.3 MIPS指令格式

**R型（寄存器型）**：

```
┌──────┬─────┬─────┬─────┬──────┬──────┐
│opcode│ rs  │ rt  │ rd  │shamt │func  │
│ 6位  │ 5位 │ 5位 │ 5位 │ 5位  │ 6位  │
└──────┴─────┴─────┴─────┴──────┴──────┘
  31 26 25 21 20 16 15 11 10  6 5    0
```

- opcode = 000000（R型标识）
- rs, rt: 源操作数寄存器
- rd: 目的操作数寄存器
- shamt: 移位量
- func: 功能码（区分不同R型指令）

**I型（立即数型）**：

```
┌──────┬─────┬─────┬────────────────┐
│opcode│ rs  │ rt  │   immediate    │
│ 6位  │ 5位 │ 5位 │     16位       │
└──────┴─────┴─────┴────────────────┘
```

- rs: 源操作数寄存器
- rt: 目的操作数寄存器（或源操作数）
- immediate: 16位立即数

**J型（跳转型）**：

```
┌──────┬────────────────────────────┐
│opcode│        address             │
│ 6位  │          26位              │
└──────┴────────────────────────────┘
```

### 14.1.4 MIPS常用指令

**算术指令**（R型）：

| 指令 | 格式 | 功能 | func |
|------|------|------|------|
| ADD | add rd, rs, rt | rd = rs + rt | 100000 |
| SUB | sub rd, rs, rt | rd = rs - rt | 100010 |
| AND | and rd, rs, rt | rd = rs & rt | 100100 |
| OR | or rd, rs, rt | rd = rs \| rt | 100101 |
| SLT | slt rd, rs, rt | rd = (rs < rt) ? 1 : 0 | 101010 |

**立即数指令**（I型）：

| 指令 | 格式 | 功能 | opcode |
|------|------|------|--------|
| ADDI | addi rt, rs, imm | rt = rs + imm | 001000 |
| ANDI | andi rt, rs, imm | rt = rs & imm | 001100 |
| ORI | ori rt, rs, imm | rt = rs \| imm | 001101 |
| LW | lw rt, offset(rs) | rt = M(rs + offset) | 100011 |
| SW | sw rt, offset(rs) | M(rs + offset) = rt | 101011 |
| BEQ | beq rs, rt, offset | if(rs==rt) PC += offset | 000100 |
| BNE | bne rs, rt, offset | if(rs!=rt) PC += offset | 000101 |

**跳转指令**（J型）：

| 指令 | 格式 | 功能 | opcode |
|------|------|------|--------|
| J | j target | PC = target | 000010 |
| JAL | jal target | $ra = PC+4; PC = target | 000011 |

### 14.1.5 MIPS指令编码示例

**【例 14-1】** 将 `add $t0, $t1, $t2` 编码为机器码。

- opcode = 000000（R型）
- rs = $t1 = 01001
- rt = $t2 = 01010
- rd = $t0 = 01000
- shamt = 00000
- func = 100000（ADD）

机器码：`000000 01001 01010 01000 00000 100000`

十六进制：`0x012A4020`

**【例 14-2】** 将 `lw $t0, 4($sp)` 编码为机器码。

- opcode = 100011（LW）
- rs = $sp = 11101
- rt = $t0 = 01000
- immediate = 0000000000000100

机器码：`100011 11101 01000 0000000000000100`

十六进制：`0x8FA80004`

**【例 14-3】** 将 `beq $t0, $t1, label`（label偏移量为-3）编码为机器码。

- opcode = 000100（BEQ）
- rs = $t0 = 01000
- rt = $t1 = 01001
- immediate = 1111111111111101（-3的补码）

机器码：`000100 01000 01001 1111111111111101`

## 14.2 x86指令集简介

### 14.2.1 x86指令格式

x86指令格式复杂：

```
┌────────┬────────┬────────┬─────┬────────┬────────┐
│Prefix  │Opcode  │ModR/M │ SIB │Displacement│Immediate│

```

### 14.2.2 x86寄存器

| 32位 | 64位 | 用途 |
|------|------|------|
| EAX | RAX | 累加器 |
| EBX | RBX | 基址 |
| ECX | RCX | 计数器 |
| EDX | RDX | 数据 |
| ESP | RSP | 栈指针 |
| EBP | RBP | 帧指针 |
| ESI | RSI | 源索引 |
| EDI | RDI | 目的索引 |

### 14.2.3 x86寻址方式

| 寻址方式 | 示例 | 有效地址 |

| 基址+变址 | MOV EAX, [EBX+ECX] | EA=EBX+ECX |
| 比例变址 | MOV EAX, [EBX+ECX*4] | EA=EBX+ECX×4 |

## 14.3 指令系统设计原则

### 14.3.1 设计原则

| 原则 | 说明 |
|------|------|
| 完备性 | 能表达所有需要的计算 |
| 规整性 | 指令格式和编码规则一致 |
| 对称性 | 操作数位置可互换 |
| 正交性 | 操作码和寻址方式独立 |
| 可扩充性 | 留有扩展空间 |
| 兼容性 | 向后兼容 |

### 14.3.2 正交性

**正交性**：操作码和寻址方式的组合是独立的。

例如：ADD指令可以使用任意寻址方式：
- ADD R1, R2（寄存器寻址）
- ADD R1, [1000]（直接寻址）
- ADD R1, [R2]（寄存器间接寻址）

## 14.5 本章典型例题

### 例题1：MIPS指令编码

**题目**：将 `addi $t0, $zero, 25` 编码为机器码。

**解答**：

- opcode = 001000（ADDI）
- rs = $zero = 00000
- rt = $t0 = 01000
- immediate = 0000000000011001（25）

机器码：`001000 00000 01000 0000000000011001`

十六进制：`0x20080019`

### 例题2：MIPS指令解码

**题目**：机器码 `0x012A4020` 对应什么MIPS指令？

**解答**：

二进制：`000000 01001 01010 01000 00000 100000`

- opcode = 000000 → R型
- rs = 01001 = $t1
- rt = 01010 = $t2
- rd = 01000 = $t0
- shamt = 00000
- func = 100000 → ADD

指令：`add $t0, $t1, $t2`

### 例题3：指令系统设计

**题目**：某指令系统有64条指令，指令字长16位，地址码4位。最多可以有多少条三地址指令？

**解答**：

操作码位数 = 16 - 3 × 4 = 4位

最多可表示 $2^4 = 16$ 条三地址指令

但只有64条指令，所以三地址指令最多64条（如果所有指令都是三地址的话）。

实际上，如果需要其他格式的指令，三地址指令数量会减少。

### 14.8.3 MIPS指令执行流程

<div data-component="InstructionExecutionFlow"></div>

**算术指令**（如 `add $t0, $t1, $t2`）：

1. IF：从存储器取指令
2. ID：译码，读寄存器$t1和$t2
3. EX：ALU执行加法
4. MEM：无操作
5. WB：结果写回$t0

**Load指令**（如 `lw $t0, 4($sp)`）：

1. IF：从存储器取指令
2. ID：译码，读寄存器$sp
3. EX：计算地址 $sp + 4
4. MEM：从存储器读数据
5. WB：数据写回$t0

## 14.10 指令系统设计原则详解

### 14.10.1 完备性

指令系统必须能够表达所有需要的计算：
- 算术运算：加、减、乘、除
- 逻辑运算：与、或、非、异或
- 数据传送：寄存器↔存储器
- 控制流：跳转、分支、调用、返回

### 14.10.2 规整性

指令格式和编码规则应该一致：
- 操作码位置固定
- 寻址方式编码一致
- 寄存器编码一致

### 14.10.3 对称性

操作数位置可以互换：
- `ADD R1, R2, R3` 和 `ADD R1, R3, R2` 应该都能执行
- 减法不满足对称性

### 14.10.4 正交性

操作码和寻址方式的组合是独立的：
- ADD指令可以使用任意寻址方式
- 任意寻址方式可以用于任意指令

### 14.10.5 可扩充性

留有扩展空间：
- 操作码保留编码
- 寻址方式可以增加
- 寄存器可以增加

### 14.10.6 兼容性

向后兼容：
- 旧程序在新处理器上能运行
- 新指令不影响旧指令

## 14.14 MIPS处理器的数据通路设计

### 14.14.1 单周期数据通路

MIPS的单周期数据通路是理解CPU工作原理的基础。每条指令在一个时钟周期内完成。

**R型指令数据通路**（如 `add $rd, $rs, $rt`）：

```
         ┌──────┐    ┌──────────────┐    ┌─────┐
PC ──→   │I-Cache│──→│ 指令寄存器IR  │──→│译码器│
         └──────┘    └──────────────┘    └─────┘
                          │ rs,rt,rd          │
                          ↓                   │
                    ┌──────────┐              │
                    │ 寄存器文件 │←── 控制信号  │
                    │ (32×32位) │              │
                    └──────────┘              │
                     │rs数据  │rt数据          │
                     ↓        ↓               │
                    ┌─────────────┐           │
                    │    ALU      │           │
                    │  (加法器)   │           │
                    └─────────────┘           │
                          │                   │
                          ↓ rd写数据           │
                    ┌──────────┐              │
                    │ 寄存器文件 │←────────────┘
                    │  写端口   │    RegWrite
                    └──────────┘
```

### 14.14.2 Load/Store指令数据通路

**Load指令**（如 `lw $rt, offset($rs)`）：

```
IF: PC → I-Cache → IR
ID: rs → 寄存器读 → 基址数据
EX: 基址数据 + 符号扩展(offset) → 有效地址
MEM: 有效地址 → D-Cache → 读出数据
WB: 读出数据 → 写入rt寄存器
```

**Store指令**（如 `sw $rt, offset($rs)`）：

```
IF: PC → I-Cache → IR
ID: rs,rt → 寄存器读 → 基址数据, 存储数据
EX: 基址数据 + 符号扩展(offset) → 有效地址
MEM: 有效地址, 存储数据 → D-Cache写
WB: 无操作
```

### 14.14.3 分支指令数据通路

**Branch指令**（如 `beq $rs, $rt, offset`）：

```
IF: PC → I-Cache → IR
ID: rs,rt → 寄存器读 → 两个数据
EX: 比较两个数据是否相等
    若相等：新PC = PC + 4 + 符号扩展(offset) × 4
    若不等：新PC = PC + 4
```

### 14.14.4 五级流水线数据通路

```
┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐
│ IF  │→│ ID  │→│ EX  │→│ MEM │→│ WB  │
│     │  │     │  │     │  │     │  │     │
│取指  │  │译码  │  │执行  │  │访存  │  │写回  │
│PC更新│  │寄存器│  │ALU  │  │Cache│  │寄存器│
│I-Cache│ │读    │  │运算  │  │读/写│  │写    │
└─────┘  └─────┘  └─────┘  └─────┘  └─────┘
```

**各流水级的功能与硬件资源**：

| 流水级 | 功能 | 使用的硬件 | 典型延迟 |
|--------|------|-----------|---------|
| IF | 取指令 | PC、I-Cache、加法器(PC+4) | 1.0ns |
| ID | 译码+读寄存器 | 寄存器文件(读)、控制单元 | 0.8ns |
| EX | 执行/地址计算 | ALU、地址加法器 | 0.8ns |
| MEM | 访存 | D-Cache | 1.0ns |
| WB | 写回 | 寄存器文件(写) | 0.5ns |

> [!NOTE]
> MIPS的指令格式设计使得rs和rt字段在所有格式中的位置固定（[25:21]和[20:16]），因此ID级可以直接从IR中提取寄存器编号并读取寄存器文件，无需等待译码完成。这是ISA设计与微架构协同优化的典范。

### 14.14.5 数据冒险与旁路网络

流水线中存在数据冒险（Data Hazard），需要通过旁路（Forwarding）解决：

```
ADD $t0, $t1, $t2   ← 产生$t0的新值（在EX级结束时）
SUB $t3, $t0, $t4   ← 使用$t0（在EX级需要$t0的值）
    │
    └→ 旁路：将ADD的ALU输出直接送到SUB的ALU输入
```

**旁路网络的路径**：

| 旁路源 | 旁路目标 | 说明 |
|--------|---------|------|
| EX/MEM寄存器 | EX级ALU输入 | EX冒险 |
| MEM/WB寄存器 | EX级ALU输入 | MEM冒险 |

### 14.14.6 控制冒险与分支处理

分支指令在EX级才能确定是否跳转，导致控制冒险：

| 处理方式 | 性能损失 | 硬件复杂度 |
|----------|---------|-----------|
| 插入气泡（冻结） | 1~2周期 | 低 |
| 预测不跳转 | 误预测时2周期 | 低 |
| 静态分支预测 | 取决于预测准确率 | 中 |
| 动态分支预测 | 误预测时2~3周期 | 高 |

> [!TIP]
> MIPS早期设计采用了延迟分支（Delayed Branch）技术：分支指令后的一条指令（延迟槽）总是被执行。编译器可以在延迟槽中填入有用的指令，从而隐藏分支延迟。

## 14.15 x86处理器的微操作译码

### 14.15.1 x86指令的复杂性

x86指令格式变长且复杂，一条指令可能涉及：

| 复杂性来源 | 说明 | 示例 |
|-----------|------|------|
| 变长前缀 | 0~4字节前缀 | REP, LOCK, 段超越 |
| 变长操作码 | 1~3字节操作码 | 0F转义前缀 |
| ModR/M字节 | 寻址模式编码 | 基址+变址+偏移 |
| SIB字节 | 比例-索引-基址 | [EBX+ECX*4+100] |
| 变长偏移量 | 0/1/2/4字节 | 地址偏移 |
| 变长立即数 | 0/1/2/4字节 | 常量操作数 |

一条x86指令的长度为1~15字节，译码器需要先确定指令边界，再逐字段解析。

### 14.15.2 微操作译码的过程

现代x86处理器将每条CISC指令译码为1~多个RISC微操作（µop）：

**简单指令**（1个µop）：

```
ADD EAX, EBX
  → µop: ALU_ADD EAX, EBX, EAX
```

**中等指令**（2~3个µop）：

```
ADD [EBX+100], EAX
  → µop1: LOAD  tmp, [EBX+100]    (AGU+Load)
  → µop2: ALU_ADD tmp, tmp, EAX   (ALU)
  → µop3: STORE [EBX+100], tmp    (Store)
```

**复杂指令**（4+个µop或微码）：

```
REP MOVSB
  → 微码序列（由微程序存储器提供）
  → 多个LOAD/STORE + 地址更新 + 计数器更新
```

### 14.15.3 译码器的流水线结构

```
┌──────────────────────────────────────────────────┐
│              x86前端译码流水线                     │
│                                                   │
│  ┌──────┐  ┌──────────┐  ┌────────┐  ┌────────┐ │
│  │取指   │→│ 预解码   │→│ 指令   │→│ 微操作  │ │
│  │(Fetch)│  │(Pre-     │  │ 译码器 │  │ 队列   │ │
│  │       │  │ decode)  │  │(Decode)│  │(µopQ)  │ │
│  └──────┘  └──────────┘  └────────┘  └────────┘ │
│      ↓          ↓             ↓           ↓      │
│   I-Cache   确定指令      CISC→RISC    发射到    │
│   访问      边界/类型     微操作转换    执行核心  │
└──────────────────────────────────────────────────┘
```

### 14.15.4 微操作缓存（DSB）

为了避免重复译码，现代x86处理器配备了微操作缓存（Decoded Stream Buffer，DSB）：

| 特性 | 传统译码路径(MROM) | 微操作缓存(DSB) |
|------|------------------|----------------|
| 输入 | 原始x86字节 | 已译码微操作 |
| 延迟 | 4~5周期 | 1周期 |
| 带宽 | 每周期4~5条指令 | 每周期6个微操作 |
| 容量 | 32~64KB I-Cache | 1500~4000 µops |
| 命中率 | 100%（总是工作） | 80~95% |

当DSB命中时，处理器可以跳过整个译码阶段，直接从DSB获取微操作，显著提高前端吞吐量。

> [!NOTE]
> Intel从Sandy Bridge架构开始引入微操作缓存（称为DSB），AMD从Zen架构开始引入类似设计（称为Op Cache）。这是CISC处理器实现高性能前端的关键技术。

## 14.16 指令系统对CPU硬件设计的约束

### 14.16.1 ISA对数据通路的约束

指令系统的各项特性直接约束CPU数据通路的设计：

| ISA特性 | 硬件约束 | 说明 |
|---------|---------|------|
| 指令字长 | 取指通路宽度 | 定长→固定宽度，变长→可变宽度 |
| 寄存器数量 | 寄存器文件大小和端口数 | 32个→5位编号，128个→7位编号 |
| 操作数个数 | 读端口数量 | 三地址→2读端口 |
| 寻址方式 | AGU复杂度 | 简单→共用ALU，复杂→独立AGU |
| 立即数范围 | 符号扩展逻辑 | 16位→需32位符号扩展 |
| 指令格式 | 译码器复杂度 | 定长→简单译码，变长→复杂译码 |

### 14.16.2 ISA对控制单元的约束

| ISA特性 | 控制单元约束 |
|---------|------------|
| 指令数量 | 控制信号的数量和译码器规模 |
| 指令格式规整性 | 译码逻辑的复杂度 |
| 操作码编码方式 | 定长→硬布线，变长→微程序 |
| 条件码/标志位 | 需要额外的状态寄存器和逻辑 |

### 14.16.3 ISA对存储系统的约束

| ISA特性 | 存储系统约束 |
|---------|------------|
| 地址空间大小 | 地址总线宽度、TLB大小 |
| 对齐要求 | 非对齐访问需要额外逻辑 |
| 访存粒度 | 字节/半字/字访问的支持 |
| 缓存一致性 | 多核系统的缓存协议 |

### 14.16.4 ISA对流水线的约束

| ISA特性 | 流水线约束 |
|---------|-----------|
| 指令格式 | 决定各级流水线的功能划分 |
| 数据冒险 | 需要旁路网络或流水线停顿 |
| 控制冒险 | 需要分支预测或延迟分支 |
| 异常处理 | 需要精确异常支持 |

> [!TIP]
> ISA设计是"一锤子买卖"——一旦确定，后续的微架构设计必须在ISA的约束下进行。因此，ISA设计需要充分考虑未来的技术发展趋势。RISC-V的模块化设计就是对这一教训的回应：基础ISA极简，扩展指令按需添加。

## 14.17 指令发射与执行的硬件支持

### 14.17.1 指令发射策略

指令发射（Instruction Issue）是指将译码后的指令发送到执行单元的过程。

**发射策略**：

| 策略 | 说明 | 特点 |
|------|------|------|
| 顺序发射（In-Order） | 按程序顺序发射 | 简单，性能受限 |
| 乱序发射（Out-of-Order） | 数据就绪即发射 | 复杂，性能高 |

### 14.17.2 顺序发射处理器

```
译码 → 发射队列 → 执行单元 → 写回
  │         │          │
  按顺序    按顺序     按顺序
```

**MIPS的经典5级流水线**就是顺序发射：

| 流水级 | 功能 | 硬件资源 |
|--------|------|---------|
| IF | 取指 | PC、I-Cache |
| ID | 译码+发射 | 译码器、寄存器文件读 |
| EX | 执行 | ALU、AGU |
| MEM | 访存 | D-Cache |
| WB | 写回 | 寄存器文件写 |

### 14.17.3 乱序执行处理器

乱序执行需要以下硬件支持：

**一、寄存器重命名**

将ISA寄存器映射到更大的物理寄存器文件，消除假相关（WAR、WAW冒险）：

```
ISA寄存器（32个） → 物理寄存器（128~256个）
    R1           →    P17
    R2           →    P23
    ...
```

**二、重排序缓冲区（ROB）**

维护指令的程序顺序，确保结果按序提交：

```
┌──────────────────────────────────┐
│         ROB（重排序缓冲区）        │
│ ┌─────┬────┬──────┬──────┐      │
│ │ 序号 │指令│ 结果 │ 状态 │      │
│ ├─────┼────┼──────┼──────┤      │
│ │  1  │ADD │ 42   │ 完成 │      │
│ │  2  │LW  │ 等待 │ 执行中│      │
│ │  3  │SUB │ ?    │ 发射 │      │
│ └─────┴────┴──────┴──────┘      │
└──────────────────────────────────┘
```

**三、保留站（Reservation Station）**

存储等待执行的指令及其操作数：

```
┌──────────────────────────────────┐
│         保留站（RS）              │
│ ┌────┬────┬────┬────┬────┐      │
│ │ 操作 │ Op1│ Op2│ 就绪│ 目标│      │
│ ├────┼────┼────┼────┼────┤      │
│ │ ADD │ P17│ P23│  ✓ │ P31│      │
│ │ MUL │ P5 │ -  │  ✗ │ P12│      │
│ └────┴────┴────┴────┴────┘      │
└──────────────────────────────────┘
```

### 14.17.4 超标量发射

超标量处理器每周期可以发射多条指令：

| 发射宽度 | 典型处理器 | 每周期操作 |
|----------|-----------|-----------|
| 1-wide | MIPS R2000 | 1条指令 |
| 2-wide | ARM Cortex-A7 | 2条指令 |
| 4-wide | Intel Skylake | 4个µop |
| 6-wide | Apple M2 | 6条指令 |
| 8-wide | Apple M3 | 8条指令 |

### 14.17.5 发射槽与功能单元的映射

```
                    发射队列
                 ┌────────────┐
                 │ µop1 (ALU) │──→ ALU0
                 │ µop2 (LOAD)│──→ Load单元
                 │ µop3 (ALU) │──→ ALU1
                 │ µop4 (STORE)│──→ Store单元
                 └────────────┘
```

| 功能单元 | 数量 | 延迟 | 说明 |
|----------|------|------|------|
| ALU | 2~4个 | 1周期 | 整数运算 |
| AGU | 2~3个 | 1周期 | 地址生成 |
| Load单元 | 2~3个 | 3~5周期 | 缓存读 |
| Store单元 | 1~2个 | 1周期 | 缓存写 |
| 乘法器 | 1~2个 | 3~5周期 | 乘法运算 |
| 除法器 | 1个 | 10~20周期 | 除法运算 |
| 分支单元 | 1~2个 | 1周期 | 分支判断 |

> [!NOTE]
> 指令发射宽度和功能单元的数量是处理器性能的关键参数。增加发射宽度和功能单元可以提高指令级并行度（ILP），但也增加了硬件复杂度、面积和功耗。现代高性能处理器通常采用4~8发射宽度的设计，这是性能与复杂度的平衡点。

### 14.17.6 硬件对指令系统设计的反馈

指令系统的设计不仅影响硬件，硬件的约束也反过来影响ISA的设计：

| 硬件约束 | 对ISA的影响 | 示例 |
|----------|-----------|------|
| 译码复杂度 | 简化指令格式 | RISC的定长格式 |
| 寄存器端口数 | 限制操作数个数 | 最多3个寄存器操作数 |
| 流水线深度 | 限制指令复杂度 | 单周期执行的要求 |
| 缓存行大小 | 影响指令对齐 | 32/64字节对齐 |
| 分支预测器容量 | 限制循环大小 | 热点代码大小 |

> [!TIP]
> ISA设计是一个软硬件协同优化的过程。好的ISA应该既方便编译器生成高效代码，又便于硬件高效实现。MIPS的成功正是因为它在ISA层面充分考虑了流水线实现的需求；x86的持久生命力则得益于其在保持兼容性的同时不断演进内部微架构。

## 14.18 本章综合练习（补充）

### 练习7：MIPS数据通路分析

**题目**：画出MIPS指令 `sw $t1, 8($sp)` 的数据通路，标注数据流向。

**解答**：

```
IF: PC → I-Cache → IR（取指令）
ID: rs=$sp → 读寄存器 → 基址数据
    rt=$t1 → 读寄存器 → 存储数据
EX: 基址数据 + 符号扩展(8) → 有效地址
MEM: 有效地址, 存储数据 → D-Cache写
WB: 无操作（Store指令不写回寄存器）
```

关键控制信号：RegDst=X, ALUSrc=1, MemtoReg=X, RegWrite=0, MemRead=0, MemWrite=1, Branch=0

### 练习8：x86微操作译码

**题目**：将以下x86指令译码为微操作序列：

```assembly
ADD [EBX+ECX*4+100], EDX
```

**解答**：

```
µop1: LOAD  tmp, [EBX + ECX*4 + 100]   （AGU计算地址 + 读D-Cache）
µop2: ADD   tmp, tmp, EDX               （ALU加法）
µop3: STORE [EBX + ECX*4 + 100], tmp    （写D-Cache）
```

共3个微操作，需要AGU、Load单元、ALU、Store单元协同工作。

### 练习9：指令发射分析

**题目**：某4发射处理器有2个ALU、1个Load单元、1个Store单元。以下指令序列能否在一个周期内全部发射？

```assembly
ADD  $t0, $t1, $t2
SUB  $t3, $t4, $t5
LW   $t6, 0($sp)
SW   $t7, 4($sp)
```

**解答**：

| 指令 | 类型 | 功能单元 | 能否发射 |
|------|------|---------|---------|
| ADD | ALU | ALU0 | ✓ |
| SUB | ALU | ALU1 | ✓ |
| LW | Load | Load单元 | ✓ |
| SW | Store | Store单元 | ✓ |

4条指令恰好占用4个不同的功能单元，可以**在一个周期内全部发射**。
