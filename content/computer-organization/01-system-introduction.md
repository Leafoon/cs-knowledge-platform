# 第一章 计算机系统概论

本章主要介绍计算机系统的基本概念、组成结构和工作原理。包括:计算机的软硬件概念、系统层次结构、冯·诺依曼计算机的特点、计算机的基本组成部件、工作过程以及主要技术指标。

---

## 1.1 计算机系统简介

### 1.1.1 计算机的软硬件概念

#### 硬件 (Hardware)

**硬件**是指计算机的实体部分,它由看得见摸得着的各种电子元器件、各类光、电、机设备的实物组成。硬件是计算机系统中所有物理装置的总称,是计算机系统赖以工作的物质基础。

硬件主要包括:
- **主机**:中央处理器(CPU)、内存储器(主存)等
- **外部设备**:输入设备(键盘、鼠标、扫描仪等)、输出设备(显示器、打印机等)、外存储器(硬盘、固态硬盘、光盘等)

#### 软件 (Software)

**软件**是指计算机运行所需的程序及相关文档资料,它是用户与硬件之间的接口界面。软件是一系列按照特定顺序组织的计算机数据和指令的集合,包括程序、数据和相关文档的完整集合。没有软件的计算机被称为"裸机",无法完成任何有意义的工作。

软件又可分为**系统软件**和**应用软件**两大类:

**1. 系统软件 (System Software)**

系统软件又称系统程序,用来管理整个计算机系统,监视服务,提高计算机使用效率,扩充系统功能。主要包括:

- **标准库程序**:提供常用功能的函数库,如数学函数库、图形库等
- **语言处理程序**:
  - **汇编程序(Assembler)**:将汇编语言翻译成机器语言
  - **编译程序(Compiler)**:将高级语言(如C、C++、Java)一次性全部翻译成机器语言
  - **解释程序(Interpreter)**:将高级语言(如Python、JavaScript)逐条翻译并执行
- **操作系统**(Operating System, OS):如Windows、Linux、macOS、Unix等,负责管理计算机硬件与软件资源
- **数据库管理系统**(DBMS):如MySQL、Oracle、SQL Server等,用于数据的存储、检索和管理
- **网络软件**:提供网络通信和管理功能,如网络协议栈、网络管理工具等
- **服务性程序**:诊断程序、调试程序、性能监控工具等

**2. 应用软件 (Application Software)**

应用软件又称应用程序,是为解决特定领域的应用问题而编制的程序。例如:

- **办公软件**:文字处理软件(Microsoft Word)、表格处理软件(Excel)、演示软件(PowerPoint)
- **多媒体软件**:图像处理软件(Photoshop)、视频编辑软件(Premiere)、音频处理软件
- **工程软件**:CAD设计软件、MATLAB科学计算软件
- **娱乐软件**:各类游戏软件
- **互联网应用**:浏览器、即时通讯软件、电子邮件客户端等

#### 软硬件的关系

软件和硬件是计算机系统缺一不可的两大组成部分,它们之间相互依存、相辅相成:

- **硬件是基础**:软件必须依赖硬件才能运行
- **软件是灵魂**:硬件的功能需要通过软件来实现和扩展
- **逻辑等价性**:某些功能既可用硬件实现,也可用软件实现,两者在逻辑上等效

### 1.1.2 计算机系统的层次结构

通常由用户用高级语言编写程序(称为**源程序**),然后将它和数据一起送入计算机内,再由计算机将其翻译成机器能识别的机器语言程序(称为**目标程序**),计算机自动运行该机器语言程序。

#### 计算机语言的发展历程

计算机语言经历了从低级到高级的发展过程,每一代语言都使编程变得更加容易:

**1. 机器语言(第一代语言)**

早期计算机只有机器语言,用户必须用二进制代码0、1来编写程序。例如,一条加法指令可能是:`0000 0001 0010 0011`。

- **优点**:计算机能直接识别和执行,执行速度最快
- **缺点**:编程效率极低,容易出错,可读性差,难以理解和维护
- **应用**:现代已基本不直接使用,但所有程序最终都要转换为机器语言执行

**2. 汇编语言(第二代语言)**

汇编语言用符号化的助记符代替二进制代码。用符号ADD、SUB、MUL、DIV等分别表示加、减、乘、除等操作。例如:

```assembly
MOV AX, 5    ; 将5送入AX寄存器
ADD AX, 3    ; AX = AX + 3
```

- **优点**:比机器语言可读性强,编程效率提高,便于修改和维护
- **缺点**:仍然与硬件紧密相关,不同CPU的汇编语言不同,可移植性差
- **翻译方式**:需要通过**汇编程序(Assembler)**翻译成机器语言才能执行

**3. 高级语言(第三代语言)**

高级语言更接近人类自然语言和数学表达式,如C、C++、Java、Python等。例如:

```c
int result = (a + b) * c;
```

- **优点**:具有良好的可读性和可理解性;独立于具体机器,可移植性强;编程效率高,易于学习和使用
- **缺点**:需要翻译成机器语言,执行速度通常比汇编语言慢
- **翻译方式**:通过**编译程序**或**解释程序**翻译成机器语言

#### 编译程序与解释程序的区别

将高级语言翻译成机器语言的程序称为**翻译程序**,主要有两种实现方式:

| 特性 | 编译程序 (Compiler) | 解释程序 (Interpreter) |
|------|---------------------|------------------------|
| **翻译方式** | 一次性翻译全部源程序 | 逐条翻译并执行 |
| **执行过程** | 先翻译后执行,分两个阶段 | 边翻译边执行,一个阶段 |
| **目标代码** | 生成独立的目标程序文件 | 不生成独立的目标程序 |
| **执行速度** | 快(只需翻译一次) | 慢(每次执行都需翻译) |
| **调试** | 错误定位相对困难 | 错误定位容易,可逐行调试 |
| **典型语言** | C、C++、Java(字节码编译) | Python、JavaScript、早期BASIC |

#### 虚拟机的概念

如果把具有翻译功能的汇编程序的计算机看作一台机器M₂,那么可以认为M₂在M₁之上。用户可以利用M₂的翻译功能直接向M₂输入汇编语言程序,而M₂又会将翻译后的机器语言程序输入给M₁,M₁执行后将结果输出。因此M₂并不是一台实际机器,它只是人们感觉存在的一台具有翻译功能的机器,称这类机器为**虚拟机(Virtual Machine)**。

### 1.1.3 计算机组成和计算机体系结构

#### 计算机体系结构 (Computer Architecture)

**计算机体系结构**是指那些能够被程序员所见到的计算机系统的属性,即概念性结构与功能特性。主要包括:

- **指令系统**(指令格式、寻址方式、指令类型)
- **数据类型**(整数、浮点数、字符等)
- **寄存器的定义**(通用寄存器的个数、用途)
- **I/O机制**(I/O编址方式、中断结构)
- **存储器的编址**(字节编址、字编址)
- **数据表示方法**(原码、补码、浮点数格式)

#### 计算机组成 (Computer Organization)

**计算机组成**是指如何实现计算机体系结构所体现的属性,它关注的是具体的硬件实现方式和技术细节。主要包括:

- 具体的硬件实现技术
- 各功能部件的结构(如ALU的实现方式)
- 功能部件之间的连接方式
- 控制信号的设计
- 存储器系统的组织(cache的容量、级数)
- I/O设备的接口设计

| 方面 | 计算机体系结构 | 计算机组成 |
|------|---------------|-----------|
| 关注点 | "做什么" (What) | "怎么做" (How) |
| 抽象层次 | 逻辑层面 | 物理实现层面 |
| 可见性 | 程序员可见 | 程序员不可见 |
| 影响 | 影响软件设计 | 影响硬件性能和成本 |
| 稳定性 | 相对稳定 | 随技术发展不断改进 |
| 典型内容 | 指令集、寻址方式 | ALU实现、cache组织 |

## 1.2 计算机的基本组成

### 1.2.1 冯·诺依曼计算机的特点

现代计算机基本上都基于**冯·诺依曼(Von Neumann)体系结构**。1946年,数学家冯·诺依曼在"EDVAC报告"中提出了存储程序的概念,奠定了现代计算机的基本结构。

**冯·诺依曼计算机的特点:**

1. **计算机由五大部件组成**:运算器(ALU)、存储器(Memory)、控制器(Control Unit)、输入设备、输出设备
2. **指令和数据以同等地位存放于存储器内**,并可按地址寻访(这是"存储程序"的核心思想)
3. **指令和数据均用二进制表示**(简化了硬件设计、提高了可靠性和运算速度)
4. **指令由操作码和地址码组成**(操作码表示操作性质,地址码表示操作数位置)
5. **指令在存储器内按顺序存放**,通常顺序执行(PC = PC + 1),特定条件下可改变执行顺序
6. **早期以运算器为中心**,现代计算机已改为以存储器为中心

---

### 1.2.2 计算机的硬件框图

#### 以运算器为中心的计算机硬件框图

典型的冯·诺依曼计算机是以运算器为中心的。在这种结构中,所有数据传送都要经过运算器,运算器成为数据流动的中心枢纽。缺点是运算器负担重,系统效率低。

<div data-component="VonNeumannArchitecture"></div>

现代计算机已转化为以存储器为中心。存储器成为数据交换的中心,I/O设备可以直接与存储器交换数据(DMA方式),减轻了CPU的负担,提高了系统效率。

#### 各部件功能详解

**1. 运算器 (ALU)**

运算器用来完成算术运算和逻辑运算,并将运算的中间结果暂存在运算器内。主要功能包括:算术运算(加、减、乘、除)、逻辑运算(与、或、非、异或)、移位操作等。

主要组成部件:
- **算术逻辑单元(ALU)**:执行具体的运算
- **累加寄存器(ACC)**:存放操作数和运算结果
- **数据缓冲寄存器(DR)**:暂存操作数
- **状态条件寄存器(PSW)**:保存运算状态(进位、溢出、零、负等标志)

**2. 存储器 (Memory)**

存储器用来存放数据和程序。根据位置不同分为:

- **主存储器(内存)**:直接与CPU交换信息,速度快,容量相对较小(如RAM、ROM)
- **辅助存储器(外存)**:不能与CPU直接交换信息,速度慢,容量大,成本低(如硬盘、固态硬盘)
- **高速缓冲存储器(Cache)**:位于CPU与主存之间,弥补主存与CPU速度差距

**3. 控制器 (Control Unit)**

控制器用来控制、指挥程序和数据的输入、运行以及处理运算结果。它是计算机的指挥中心。主要组成部件:

- **程序计数器(PC)**:存放当前正在执行指令的地址或下一条将要执行指令的地址
- **指令寄存器(IR)**:存放当前正在执行的指令
- **指令译码器(ID)**:对指令进行译码,产生相应的控制信号
- **时序控制逻辑**:产生各种时序信号

**4. 输入/输出设备**

输入设备将人们熟悉的信息形式转换为机器能识别的二进制代码(如键盘、鼠标、扫描仪)。输出设备将机器运算结果转换为人们熟悉的信息形式(如显示器、打印机、音箱)。

#### 现代计算机的简化框图

由于运算器和控制器在逻辑关系和电路结构上联系十分紧密,尤其在大规模集成电路制作工艺出现后,这两大部件往往集成在同一芯片上,因此,通常将它们合起来统称为**中央处理器(CPU)**。把输入设备与输出设备简称为**I/O设备**。

### 1.2.3 计算机的工作步骤

#### 上机前的准备工作

用计算机解决一个实际问题通常包含两大步骤:上机前的各种准备和上机运行。

**(1) 建立数学模型**:将实际问题抽象成数学问题,用数学语言描述问题。

**(2) 确定计算方法**:根据数学模型,设计具体的计算步骤和算法。

**(3) 编制解题程序**:用计算机语言编写程序。程序由**数据**(参与运算的各种数据)和**指令**(对数据进行加工处理的命令)两部分组成。

#### 指令的基本格式

一条指令通常由**操作码**和**地址码**两部分组成:

- **操作码(Opcode)**:表示机器所执行的各种操作(如取数、存数、加法、减法、乘法、除法、停机等)
- **地址码(Operand)**:表示参加运算的数在存储器内的位置(地址)

#### 计算机的工作过程

通过一个简单的例子来说明计算机的工作过程:计算 (A + B) × C,并将结果存入D地址。

<div data-component="InstructionCycleSimulator"></div>

**主存储器的组成:**

- **存储体**:由多个存储单元组成
- **存储单元**:由多个存储元件组成
- **存储元件**:能寄存一位二进制代码0或1的基本单元
- **存储字**:一个存储单元可以存储的一串二进制代码
- **存储字长**:存储字的位数(常见8位、16位、32位、64位)

| 寄存器 | 全称 | 功能 | 位数 |
|--------|------|------|------|
| **MAR** | Memory Address Register<br>(存储器地址寄存器) | 存放欲访问的存储单元的地址 | 对应存储单元的个数(如MAR为10位,则有2¹⁰=1024个存储单元,即1K) |
| **MDR** | Memory Data Register<br>(存储器数据寄存器) | 存放从存储体某单元取出的代码,或准备往某存储单元存入的代码 | 与**存储字长**相等 |

**存储容量 = 2^(MAR位数) × MDR位数(bit)**

**运算器的基本组成:**

| 部件 | 全称 | 功能 |
|------|------|------|
| **ACC** | Accumulator (累加器) | 存放操作数和运算结果,是运算器中使用最频繁的寄存器 |
| **MQ** | Multiplier-Quotient Register (乘商寄存器) | 乘法时存放乘数和部分积,除法时存放商 |
| **X** | Operand Register (操作数寄存器) | 存放另一个操作数(加数、乘数、除数等) |
| **ALU** | Arithmetic Logic Unit (算术逻辑单元) | 执行算术运算和逻辑运算 |

**控制器的基本组成:**

| 部件 | 全称 | 功能 |
|------|------|------|
| **PC** | Program Counter (程序计数器) | 存放当前/下一条指令地址,具有计数功能(PC = PC + 1) |
| **IR** | Instruction Register (指令寄存器) | 存放当前正在执行的指令(包括操作码和地址码) |
| **ID** | Instruction Decoder (指令译码器) | 对IR中的指令进行译码,产生相应的控制信号 |
| **CU** | Control Unit (控制单元) | 根据译码结果产生各种控制信号,控制各部件按指令要求工作 |

**指令执行的基本步骤(指令周期):**

**1. 取指 (Fetch)**
- PC → MAR(将PC内容送MAR)
- M(MAR) → MDR(从存储器读指令到MDR)
- MDR → IR(将指令送IR)
- PC + 1 → PC(PC指向下一条指令)

**2. 译码 (Decode)**
- IR → ID(指令译码器分析指令)

**3. 执行 (Execute)**
- 根据指令类型执行相应操作

**4. 回写 (Write Back)**
- 将结果写回存储器或寄存器

## 1.3 计算机硬件的主要技术指标

### 1.3.1 机器字长

**机器字长**是指计算机一次能处理的二进制数据的位数,通常与CPU的寄存器位数和数据总线宽度一致。

**常见的机器字长:**

- **8位机**:早期个人计算机,如Intel 8080
- **16位机**:如Intel 8086/80286
- **32位机**:如Intel 80386/Pentium系列
- **64位机**:现代主流计算机,如Intel Core、AMD Ryzen

**机器字长的影响:**

- **数据表示范围**:8位(-128~+127)、16位(-32768~+32767)、32位(约±21亿)、64位(约±9.2×10¹⁸)
- **寻址能力**:32位CPU最大寻址4GB,64位CPU最大寻址16EB(理论值)
- **运算精度**:字长越长,数值计算精度越高
- **处理速度**:字长越长,一次能处理的数据越多

---

### 1.3.2 存储容量

**主存容量**是指主存储器能存储信息的最大容量,反映了计算机记忆信息的能力。

| 单位 | 符号 | 换算 | 近似值 |
|------|------|------|--------|
| 位 | bit (b) | — | 最小单位 |
| 字节 | Byte (B) | 1B = 8 bit | — |
| 千字节 | KB | 1KB = 2¹⁰B | ≈ 1000B |
| 兆字节 | MB | 1MB = 2²⁰B | ≈ 100万B |
| 吉字节 | GB | 1GB = 2³⁰B | ≈ 10亿B |
| 太字节 | TB | 1TB = 2⁴⁰B | ≈ 1万亿B |
| 拍字节 | PB | 1PB = 2⁵⁰B | ≈ 1千万亿B |
| 艾字节 | EB | 1EB = 2⁶⁰B | ≈ 100京B |

**辅存容量**是指外部存储设备(硬盘、固态硬盘、光盘等)的存储容量。

| 设备类型 | 典型容量 | 特点 |
|----------|----------|------|
| 机械硬盘(HDD) | 500GB ~ 20TB | 容量大,价格低,速度较慢 |
| 固态硬盘(SSD) | 250GB ~ 8TB | 速度快,抗震,价格较高 |
| U盘 | 8GB ~ 512GB | 便携,容量适中 |
| 光盘 | CD:700MB / DVD:4.7GB / 蓝光:25GB | 成本低,不易修改 |

### 1.3.3 运算速度

#### (1) 主频 (Clock Frequency)

**主频**又称时钟频率,是指CPU内数字脉冲信号振荡的频率。单位:MHz(兆赫兹)、GHz(吉赫兹)。

**时钟周期 T = 1 / 主频 f**

例如:主频为4 GHz的CPU,时钟周期 T = 1/(4×10⁹) ≈ 0.25 ns

注意:主频越高,性能不一定越强,还要看CPU架构、核心数、缓存大小、指令集等因素。

#### (2) 每秒执行指令数 (IPS)

**IPS** (Instructions Per Second) 表示每秒执行多少条指令。

- **KIPS**:千条指令/秒 (10³)
- **MIPS**:百万条指令/秒 (10⁶)
- **GIPS**:十亿条指令/秒 (10⁹)
- **TIPS**:万亿条指令/秒 (10¹²)

**MIPS = 主频 / CPI**

其中,CPI (Cycles Per Instruction) 是平均每条指令所需的时钟周期数。

#### (3) 每秒浮点运算次数 (FLOPS)

**FLOPS** (Floating-point Operations Per Second) 表示每秒执行多少次浮点运算,常用于衡量科学计算性能。

- **MFLOPS**:百万次/秒 (10⁶)
- **GFLOPS**:十亿次/秒 (10⁹)
- **TFLOPS**:万亿次/秒 (10¹²)
- **PFLOPS**:千万亿次/秒 (10¹⁵)
- **EFLOPS**:百亿亿次/秒 (10¹⁸)

#### (4) 基准程序性能测试

由于上述指标都有局限性,现代计算机采用**标准基准程序(Benchmark)**来测试性能。常见基准测试:

- **SPEC**:系统性能评估协会的标准测试
- **LINPACK**:线性方程组求解,超算排名依据
- **Geekbench**:跨平台性能测试
- **3DMark**:图形性能测试
- **PCMark**:综合办公性能测试

**CPI (Cycles Per Instruction):**

**CPI = 执行某程序所需的总时钟周期数 / 程序包含的指令总数**

**CPU执行时间:**

**CPU时间 = 程序的指令数 × CPI × 时钟周期 = (程序的指令数 × CPI) / 主频**
