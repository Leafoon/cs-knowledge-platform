# 第二章 计算机的发展及应用

本章介绍计算机的发展历程、微型计算机的演进、软件技术的兴起、计算机的广泛应用以及未来发展趋势。了解计算机的发展历史有助于理解计算机组成原理的设计思想。

## 2.1 计算机的发展史

### 2.1.1 计算机的产生和发展

**ENIAC**（Electronic Numerical Integrator and Computer）：

1946年诞生于美国宾夕法尼亚大学，是世界上第一台通用电子计算机。

| 参数 | 数值 |
|------|------|
| 重量 | 30吨 |
| 占地面积 | 170平方米 |
| 电子管数量 | 18000个 |
| 功耗 | 150千瓦 |
| 运算速度 | 5000次加法/秒 |
| 内存 | 20个10位寄存器 |

**计算机的四代划分**（按电子器件）：

| 代次 | 时间 | 电子器件 | 存储器 | 运算速度 | 软件 |
|------|------|---------|--------|---------|------|
| 第1代 | 1946~1958 | 电子管 | 汞延迟线 | 数千次/秒 | 机器语言、汇编语言 |
| 第2代 | 1958~1964 | 晶体管 | 磁芯存储器 | 数百万次/秒 | FORTRAN、COBOL |
| 第3代 | 1964~1971 | SSI/MSI集成电路 | 半导体存储器 | 数千万次/秒 | 操作系统、结构化编程 |
| 第4代 | 1971~至今 | LSI/VLSI集成电路 | 半导体存储器 | 数十亿次/秒 | 数据库、网络、AI |

**各代计算机的特点**：

**第1代（电子管时代）**：
- 体积大、功耗高、可靠性差
- 使用机器语言编程
- 主要用于科学计算
- 代表：ENIAC、EDVAC、IBM 701

**第2代（晶体管时代）**：
- 体积缩小、功耗降低、可靠性提高
- 出现高级语言（FORTRAN、COBOL）
- 应用扩展到数据处理
- 代表：IBM 7090、CDC 1604

**第3代（集成电路时代）**：
- 体积进一步缩小
- 出现操作系统
- 出现小型计算机（PDP-8）
- 代表：IBM 360/370、PDP系列

**第4代（超大规模集成电路时代）**：
- 微处理器出现
- 个人计算机普及
- 互联网兴起
- 代表：Intel系列、Apple、IBM PC

### 2.1.2 摩尔定律

**摩尔定律**（Moore's Law）：集成电路上的晶体管数量大约每18~24个月翻一倍。

1965年由Intel联合创始人戈登·摩尔提出。

**摩尔定律的含义**：
- 晶体管密度：每18~24个月翻倍
- 性能：每18~24个月翻倍
- 成本：每18~24个月减半
- 功耗：持续降低

**摩尔定律的演进**：

<div data-component="TransistorEvolutionChart"></div>

| 年份 | 处理器 | 晶体管数 | 制程 |
|------|--------|---------|------|
| 1971 | Intel 4004 | 2300 | 10μm |
| 1978 | Intel 8086 | 29000 | 3μm |
| 1989 | Intel 486 | 1200000 | 1μm |
| 1993 | Pentium | 3100000 | 0.8μm |
| 1999 | Pentium III | 9500000 | 250nm |
| 2004 | Pentium 4 | 125000000 | 90nm |
| 2012 | Core i7 | 1400000000 | 22nm |
| 2020 | Core i9 | ~10000000000 | 10nm |
| 2023 | Core Ultra | ~15000000000 | 7nm |

**摩尔定律的极限**：

<div data-component="MooreLawVisualizer"></div>

- 物理极限：晶体管尺寸接近原子级别
- 功耗极限：功耗密度无法持续增加
- 经济极限：制造成本越来越高
- 量子效应：在纳米尺度下量子效应显著

### 2.1.3 集成电路制造工艺发展

集成电路制造工艺是推动摩尔定律持续演进的核心技术。制程节点从微米级缩小到纳米级，每一次突破都伴随着光刻、材料和器件结构的创新。

#### 光刻技术演进

光刻是集成电路制造中最关键的工序，决定了晶体管的最小特征尺寸。

| 时代 | 光刻技术 | 光源波长 | 分辨率 | 典型制程 |
|------|---------|---------|--------|---------|
| 1980s | g-line光刻 | 436nm | ~0.5μm | 1μm~0.5μm |
| 1990s | i-line光刻 | 365nm | ~0.35μm | 0.5μm~0.25μm |
| 2000s | DUV（深紫外） | 248nm (KrF) | ~0.15μm | 0.25μm~90nm |
| 2000s~2010s | DUV浸没式 | 193nm (ArF) | ~38nm | 65nm~7nm |
| 2019~至今 | EUV（极紫外） | 13.5nm | ~13nm | 7nm~3nm |

**DUV（Deep Ultraviolet）光刻**：
- 使用KrF（248nm）或ArF（193nm）准分子激光器作为光源
- 浸没式光刻（Immersion Lithography）在镜头和晶圆之间填充纯水（折射率1.44），等效提高数值孔径
- 通过多重图案化技术（Multiple Patterning）突破单次曝光分辨率极限
- 193nm浸没式光刻支撑了从65nm到7nm共6个制程节点

**EUV（Extreme Ultraviolet）光刻**：
- 使用13.5nm波长的极紫外光，由高功率CO₂激光轰击锡滴等离子体产生
- 光源功率从最初的10W发展到250W以上，晶圆产率达到每小时100片以上
- 无需多重图案化，单次曝光即可实现13nm以下分辨率
- 需要全真空环境（EUV光被几乎所有材料吸收）
- 反射式掩模版（而非传统透射式），由多层Mo/Si薄膜构成
- ASML的EUV光刻机单价超过1.5亿美元，是人类制造的最复杂精密设备之一

> [!NOTE]
> EUV光刻机是当前半导体制造的"卡脖子"技术。全球仅ASML一家公司能够生产，其光源、光学系统和精密机械的制造难度代表了人类工程能力的极限。

#### 晶体管结构创新

随着制程微缩，传统平面MOSFET结构面临严重的短沟道效应（漏电流急剧增加），晶体管结构经历了从平面到三维的根本性变革。

| 晶体管结构 | 时代 | 核心创新 | 首次应用 |
|-----------|------|---------|---------|
| 平面MOSFET | ~2000年以前 | 标准平面结构 | 通用 |
| 应变硅（Strained Si） | ~2003年 | 提高载流子迁移率 | Intel 90nm |
| 高K金属栅（HKMG） | ~2007年 | 替代SiO₂栅介质 | Intel 45nm |
| FinFET | ~2011年 | 三维鳍片结构 | Intel 22nm |
| GAA（Gate-All-Around） | ~2023年 | 全包围栅极 | Samsung 3nm |

**FinFET（鳍式场效应晶体管）**：
- 栅极三面包裹沟道（鳍片），形成三维结构
- 相比平面MOSFET，有效沟道宽度更大，驱动电流提高20%~30%
- 漏电流降低至平面结构的1/10以下
- 2011年Intel在22nm制程首次商用，此后成为主流结构直至5nm
- 台积电、三星、Intel的FinFET实现细节有所不同

**GAA（Gate-All-Around，全环绕栅极）**：
- 栅极完全包围沟道（四面），进一步增强栅控能力
- 采用纳米片（Nanosheet）或纳米线（Nanowire）作为沟道
- 可通过调整纳米片宽度灵活调节驱动电流
- 三星在3nm率先采用GAA，台积电在2nm（N2）引入
- Intel的RibbonFET也是GAA的一种实现

#### 后摩尔时代技术路径

当传统等比例缩小（Dennard Scaling）失效后，产业界探索出多条技术路径延续性能提升。

**Chiplet（小芯片）架构**：
- 将大型SoC拆分为多个功能独立的小芯片（Chiplet）
- 不同Chiplet可采用不同制程工艺（核心用先进制程，I/O用成熟制程）
- 通过先进封装技术互连，实现"异构集成"
- 优势：提高良率、降低成本、灵活组合、突破光罩面积限制
- 代表：AMD EPYC（Zen架构）、Intel Ponte Vecchio、Apple M系列

**3D封装技术**：
- 晶圆级封装（Wafer Level Packaging）
- 硅通孔技术（TSV，Through-Silicon Via）：在芯片垂直方向制作导电通道
- 混合键合（Hybrid Bonding）：铜-铜直接键合，间距可达1μm以下
- 代表：台积电SoIC、Intel Foveros、AMD 3D V-Cache

**其他后摩尔技术路径**：
- 新型材料：二维材料（石墨烯、MoS₂）、碳纳米管
- 新型器件：自旋电子器件、量子点器件
- 光子互连：用光信号替代电信号进行芯片间通信
- 近存储计算（Near-Memory Computing）：将计算逻辑靠近存储单元
- 存内计算（In-Memory Computing）：直接在存储阵列中完成计算

> [!TIP]
> 后摩尔时代的核心思想是：不再单纯依赖晶体管尺寸缩小，而是通过架构创新、封装技术和新型器件来延续性能提升。Chiplet和3D封装是目前最接近大规模商用的技术路径。

### 2.1.5 存储器技术发展史

存储器是计算机系统的核心组成部分，其技术演进直接影响了计算机的性能、容量和应用范围。从最早的汞延迟线到现代的NVMe SSD，存储器经历了多次革命性变革。

#### 存储器技术演进总览

| 时代 | 存储技术 | 存取方式 | 典型容量 | 访问时间 | 主要用途 |
|------|---------|---------|---------|---------|---------|
| 1940s~1950s | 汞延迟线 | 串行 | 数KB | 数ms | 主存 |
| 1950s~1970s | 磁芯存储器 | 随机 | 数KB~数MB | 数μs | 主存 |
| 1960s~至今 | SRAM | 随机 | 数KB~数MB | 1~10ns | Cache |
| 1970s~至今 | DRAM | 随机 | 数GB~数TB | 10~100ns | 主存 |
| 1980s~至今 | 磁盘（HDD） | 半随机 | 数TB | 数ms | 辅存 |
| 2000s~至今 | 闪存（SSD） | 随机 | 数TB | 10~100μs | 辅存 |
| 2010s~至今 | NVMe SSD | 随机 | 数TB | 数μs | 高速辅存 |

#### 汞延迟线存储器

汞延迟线（Mercury Delay Line）是最早使用的存储器技术，利用声波在汞（水银）中的传播延迟来存储数据。

**工作原理**：
1. 数据以电信号形式输入
2. 电信号通过换能器转换为声波
3. 声波在汞管中传播（传播速度约1450 m/s）
4. 在汞管另一端，声波被换能器重新转换为电信号
5. 电信号经放大和整形后重新输入，形成循环存储

**特点**：
- 串行存取：必须等待声波传播到目标位置
- 访问时间取决于数据在管中的位置（平均等待时间约1ms）
- 容量受管长限制
- 代表：ENIAC的汞延迟线存储器（20个10位字）

#### 磁芯存储器

磁芯存储器（Core Memory）使用微小的铁氧体磁环（磁芯）作为存储单元，是1950s~1970s的主流主存技术。

**工作原理**：
1. 每个磁芯可存储1位数据（磁化方向代表0或1）
2. 通过在X和Y方向的导线中通电流来选择特定磁芯
3. 写入：通过电流方向设定磁化方向
4. 读取：检测磁芯的磁化方向（破坏性读取，读后需要重写）

**特点**：
- 随机存取：可通过行列地址直接访问任意存储单元
- 非易失性：断电后磁化状态保持
- 可靠性高：无机械运动部件
- 访问时间约1μs
- 典型容量：数KB~数MB

> [!NOTE]
> 磁芯存储器是"核心存储器"（Core Memory）一词的来源，现代CPU中的"Core"（核心）一词就源于此。虽然磁芯存储器已被淘汰，但这个术语保留了下来。

#### SRAM的工作原理

SRAM（Static Random-Access Memory，静态随机存取存储器）使用双稳态触发器电路存储每个比特。

**6管SRAM单元结构**：
```
       VDD
        │
   ┌────┼────┐
   │    │    │
  M2   M4    │
   │    │    │
   ├───BL────┤
   │    │    │
  M1   M3    │
   │    │    │
   └────┼────┘
       GND

M1/M2: 交叉耦合反相器（存储节点）
M3/M4: 负载管
M5/M6: 访问管（由字线控制）
```

- 由6个晶体管组成（4个构成两个交叉耦合的反相器，2个为访问管）
- 两个稳定状态分别表示0和1
- 只要供电，数据就保持不变（无需刷新）
- 读取是非破坏性的

#### DRAM的工作原理

DRAM（Dynamic Random-Access Memory，动态随机存取存储器）使用电容存储每个比特。

**1管DRAM单元结构**：
```
    字线 (Word Line)
      │
      M1（访问管）
      │
  ┌───┤
  │   │
 BL   Cs（存储电容）
  │   │
  └───┤
      │
     GND
```

- 由1个晶体管和1个电容组成
- 电容充电表示1，放电表示0
- 电容会缓慢漏电，需要定期刷新（典型64ms刷新一次）
- 读取是破坏性的（读取时电容放电），读后需要重写
- 刷新操作由DRAM控制器自动完成

#### SRAM vs DRAM 设计权衡

| 特性 | SRAM | DRAM | 权衡分析 |
|------|------|------|---------|
| 存储单元结构 | 6个晶体管 | 1个晶体管+1个电容 | SRAM面积是DRAM的5~8倍 |
| 速度 | 1~10ns | 10~100ns | SRAM快10倍左右 |
| 密度 | 低 | 高 | DRAM单位面积容量远大于SRAM |
| 成本/位 | 高 | 低 | SRAM成本约为DRAM的10~100倍 |
| 功耗（待机） | 较高（静态电流） | 较低（仅漏电） | SRAM需持续供电维持状态 |
| 功耗（工作） | 较低（无需刷新） | 较高（需刷新） | DRAM刷新消耗额外功耗 |
| 刷新 | 不需要 | 需要（64ms周期） | DRAM控制器增加系统复杂度 |
| 读取方式 | 非破坏性 | 破坏性（读后重写） | DRAM读取延迟更高 |
| 典型用途 | Cache（L1/L2/L3） | 主存 | 速度与成本的权衡 |
| 工艺节点 | 与CPU同制程 | 专用制程 | DRAM优化密度，SRAM优化速度 |

**设计权衡总结**：
- SRAM：面积大、成本高、速度快、无需刷新 → 适合小容量、高速缓存
- DRAM：面积小、成本低、速度慢、需要刷新 → 适合大容量主存
- 现代处理器通常集成数MB的SRAM作为Cache，搭配数GB~数TB的DRAM作为主存

#### 存储器层次结构

现代计算机采用多级存储层次结构，其根本原因是**存储器的速度、容量和成本三者之间存在不可调和的矛盾**。

**层次结构的形成原因**：

| 层次 | 速度 | 容量 | 成本/位 | 技术 | 典型大小 |
|------|------|------|---------|------|---------|
| 寄存器 | <1ns | 数百B~数KB | 极高 | 触发器 | 1~2KB |
| L1 Cache | 1~2ns | 数十KB~数百KB | 很高 | SRAM | 32~128KB |
| L2 Cache | 3~10ns | 数百KB~数MB | 高 | SRAM | 256KB~数MB |
| L3 Cache | 10~30ns | 数MB~数百MB | 中 | SRAM | 数MB~数百MB |
| 主存 | 50~100ns | 数GB~数TB | 低 | DRAM | 数GB~数TB |
| SSD | 10~100μs | 数百GB~数TB | 很低 | NAND Flash | 数百GB~数TB |
| HDD | 5~10ms | 数TB~数十TB | 极低 | 磁记录 | 数TB~数十TB |

**层次结构的运作机制**：
- 利用程序的**局部性原理**（时间局部性和空间局部性）
- 频繁访问的数据放在靠近CPU的高速小容量存储器中
- 不常用的数据放在远离CPU的低速大容量存储器中
- 由硬件（Cache控制器）和软件（虚拟内存管理）共同管理

> [!TIP]
> 存储器层次结构是理解计算机性能的关键。程序的实际运行速度往往取决于Cache命中率，而非CPU峰值性能。一个好的算法应该具有良好的数据局部性，充分利用Cache。

#### 现代存储器的关键性能指标

**容量指标**：
- SRAM Cache：L1通常32~128KB/核，L2通常256KB~数MB/核，L3可达数百MB
- DRAM主存：典型配置8~128GB，服务器可达数TB
- SSD/HDD：消费级512GB~8TB，企业级可达数十TB

**速度指标**：
- SRAM访问延迟：0.5~2ns（工艺和容量相关）
- DRAM访问延迟：10~100ns（取决于CAS延迟和频率）
- SSD随机读延迟：10~100μs
- HDD寻道时间：5~10ms

**带宽指标**：
- DDR5-6400：单通道51.2GB/s，双通道102.4GB/s
- HBM3：单堆栈可达819GB/s
- NVMe SSD（PCIe 4.0 x4）：顺序读约7GB/s
- SATA SSD：顺序读约550MB/s

**成本指标**（2024年参考价格）：
- SRAM：~$1000/GB（片上集成）
- DRAM：~$2~5/GB（DDR5模组）
- NAND Flash SSD：~$0.05~0.10/GB
- HDD：~$0.01~0.03/GB

**功耗指标**：
- SRAM：静态功耗随工艺缩小而增加（漏电流问题），动态功耗较低
- DDR5 DIMM：典型3~8W
- NVMe SSD：典型3~10W
- HDD：典型6~10W（含电机）

### 2.1.6 CPU微架构发展深度剖析

CPU微架构（Microarchitecture）是指令集架构（ISA）的具体硬件实现。相同ISA可以有截然不同的微架构，性能差异可达数十倍。

#### 微架构演进总览

| 阶段 | 时代 | 核心技术 | 性能提升来源 | 代表处理器 |
|------|------|---------|------------|-----------|
| 简单流水线 | 1980s | 5级流水线 | 指令级并行（ILP） | Intel 486、MIPS R3000 |
| 超标量 | 1990s初 | 多发射 | 多条指令并行执行 | Pentium、MIPS R4000 |
| 乱序执行 | 1995~ | 保留站、ROB | 消除数据冒险 | Pentium Pro、Alpha 21264 |
| 超线程 | 2002~ | SMT | 利用空闲执行单元 | Pentium 4 HT |
| 多核 | 2005~ | 多核集成 | 线程级并行（TLP） | Core 2 Duo、Athlon X2 |
| 异构多核 | 2011~ | 大小核 | 能效优化 | ARM big.LITTLE、Intel Alder Lake |

#### 简单流水线

**5级经典流水线**（RISC风格）：
```
IF → ID → EX → MEM → WB
取指  译码  执行  访存  写回
```

- 每条指令被分为5个阶段，每个阶段由独立的硬件完成
- 理想情况下每个时钟周期完成一条指令（CPI=1）
- 流水线冒险：结构冒险、数据冒险、控制冒险
- 解决方法：数据前递（Forwarding）、分支预测、插入气泡（Stall）

**数据通路**：
- 取指阶段（IF）：从I-Cache取指令，PC更新
- 译码阶段（ID）：指令译码，读寄存器堆
- 执行阶段（EX）：ALU运算，地址计算
- 访存阶段（MEM）：D-Cache读写
- 写回阶段（WB）：结果写回寄存器堆

#### 超标量处理器

超标量（Superscalar）处理器每个时钟周期可以发射（Issue）多条指令。

**关键技术**：
- 多端口寄存器堆：支持同时读写多个操作数
- 多个功能单元：整数ALU、浮点单元、Load/Store单元可并行工作
- 指令分派逻辑：判断哪些指令可以同时发射
- 依赖检测：检查指令间的数据依赖关系

**发射策略**：
- 顺序发射、顺序完成（In-Order Issue, In-Order Completion）
- 顺序发射、乱序完成（In-Order Issue, Out-of-Order Completion）
- 乱序发射、乱序完成（Out-of-Order Issue, Out-of-Order Completion）

#### 乱序执行处理器

乱序执行（Out-of-Order Execution, OoOE）是现代高性能处理器的核心技术。

**核心组件**：
- **重排序缓冲区（ROB, Reorder Buffer）**：记录指令的程序顺序，保证最终结果按序提交
- **保留站（Reservation Station）**：缓存已发射但未执行完的指令，等待操作数就绪
- **寄存器重命名（Register Renaming）**：消除假相关（WAR、WAW），将逻辑寄存器映射到物理寄存器
- **重命名寄存器文件（PRF, Physical Register File）**：比逻辑寄存器更多的物理寄存器

**执行流程**：
1. **取指（Fetch）**：从I-Cache取指令
2. **译码（Decode）**：指令译码
3. **重命名（Rename）**：寄存器重命名，消除假相关
4. **分派（Dispatch）**：将指令送入保留站和ROB
5. **发射（Issue）**：保留站检测操作数就绪，发射到执行单元
6. **执行（Execute）**：功能单元执行运算
7. **写回（Writeback）**：结果写回重命名寄存器
8. **提交（Commit/Retire）**：按ROB顺序提交结果到架构状态

#### Intel微架构代际分析

**P6微架构（1995年，Pentium Pro）**：
- 首次引入乱序执行和寄存器重命名
- 3路超标量，12级流水线
- μop（微操作）译码：将复杂x86指令分解为简单μop
- 分支目标缓冲区（BTB）和分支预测器
- 奠定了此后20年Intel处理器的微架构基础

**NetBurst微架构（2000年，Pentium 4）**：
- 超长流水线设计（20~31级），追求高主频
- Trace Cache：缓存已译码的μop序列
- 快速执行引擎（Rapid Execution Engine）：ALU以2倍核心频率运行
- 超线程技术（Hyper-Threading）首次引入
- **教训**：超长流水线导致分支预测失败代价极高，功耗大幅上升，最终被放弃

**Core微架构（2006年，Core 2 Duo）**：
- 回归短流水线设计（14级），强调能效比
- 宽发射（4发射），深度乱序执行
- 宏操作融合（Macro-Fusion）：将cmp+jcc融合为单个μop
- 微操作融合（Micro-Fusion）：将load+op融合
- 双核设计，共享L2 Cache

**Nehalem微架构（2008年，Core i7）**：
- 集成内存控制器（IMC）：减少内存访问延迟
- 三级Cache层次（L1/L2私有，L3共享）
- 超线程技术回归
- Turbo Boost动态频率调节
- QuickPath Interconnect（QPI）总线

**Sandy Bridge微架构（2011年）**：
- 环形总线（Ring Bus）连接多核和L3 Cache
- 集成GPU（核芯显卡）
- AVX（Advanced Vector Extensions）指令集：256位SIMD
- 微操作缓存（μop Cache）：L1级μop缓存，减少译码功耗
- 分支预测器升级（TAGE预测器）

**Alder Lake微架构（2021年）**：
- **混合架构（Hybrid Architecture）**：首次在x86桌面处理器引入大小核设计
  - Performance Core（P-Core）：Golden Cove微架构，高性能，支持超线程
  - Efficient Core（E-Core）：Gracemont微架构，高能效，不支持超线程
- 硬件线程调度器（Thread Director）：与操作系统协作，智能分配任务到P-Core或E-Core
- 支持DDR5内存和PCIe 5.0
- 体现了与ARM big.LITTLE类似的设计哲学

#### ARM架构的设计哲学与x86对比

ARM和x86代表了两种截然不同的处理器设计哲学。

| 方面 | ARM | x86 |
|------|-----|-----|
| 指令集类型 | RISC（精简指令集） | CISC（复杂指令集） |
| 指令长度 | 固定长度（32位/16位） | 可变长度（1~15字节） |
| 指令数量 | 较少（~1000条） | 较多（~3000条以上） |
| 寻址模式 | 较少 | 丰富 |
| 译码复杂度 | 低（固定格式） | 高（需μop分解） |
| 能效比 | 高 | 相对较低 |
| 设计哲学 | 能效优先，按需定制 | 性能优先，向后兼容 |
| 生态系统 | 授权模式，多家厂商 | Intel/AMD垄断 |
| 典型功耗 | 数瓦~数十瓦 | 数十瓦~数百瓦 |
| 主要市场 | 移动、嵌入式、服务器 | 桌面、服务器、笔记本 |

**ARM的设计优势**：
- 简单的指令译码器：功耗低，面积小
- 固定指令长度：简化取指和译码逻辑
- Load/Store架构：只有Load/Store指令访问内存
- 条件执行：减少分支指令，提高流水线效率
- Thumb/Thumb-2指令集：16位压缩指令，提高代码密度

**Apple M系列的启示**：
- Apple M1/M2/M3芯片证明ARM架构可以达到极高性能
- 统一内存架构（UMA）：CPU、GPU、NPU共享内存
- 超宽执行引擎（8发射以上），深度乱序执行
- 高带宽低延迟的片上互连

#### RISC-V开放指令集架构

RISC-V是一种开源的精简指令集架构，2010年诞生于加州大学伯克利分校。

**核心特点**：
1. **完全开源**：无授权费用，任何人可自由使用、修改和扩展
2. **模块化设计**：
   - RV32I：基础整数指令集（47条指令）
   - M扩展：乘除法
   - A扩展：原子操作
   - F/D扩展：单/双精度浮点
   - C扩展：压缩指令（16位）
   - V扩展：向量运算
3. **简洁高效**：基础指令集仅47条，译码逻辑极其简单
4. **自定义扩展**：预留大量操作码空间用于用户自定义指令

**三大开放架构对比**：

| 方面 | RISC-V | ARM | x86 |
|------|--------|-----|-----|
| 授权模式 | 开源免费 | 授权付费 | 授权付费 |
| 指令集复杂度 | 低（基础47条） | 中（~1000条） | 高（~3000条以上） |
| 主要应用 | 嵌入式、IoT、教育 | 移动、服务器 | 桌面、服务器 |
| 自定义扩展 | 完全支持 | 有限支持 | 不支持 |
| 生态成熟度 | 快速成长 | 非常成熟 | 非常成熟 |
| 代表芯片 | SiFive、平头哥 | Apple M系列、骁龙 | Intel Core、AMD Ryzen |

> [!TIP]
> RISC-V的开源特性使其成为教学、科研和定制化芯片设计的理想选择。在IoT和嵌入式领域，RISC-V正在快速替代ARM。但在高性能计算领域，RISC-V的生态成熟度仍有差距。

### 2.1.7 总线技术发展

总线（Bus）是计算机各部件之间传输信息的公共通道。总线技术的发展直接影响了系统的带宽、延迟和扩展能力。

#### 总线技术演进

| 总线标准 | 年代 | 类型 | 数据宽度 | 时钟频率 | 峰值带宽 | 特点 |
|---------|------|------|---------|---------|---------|------|
| ISA | 1981 | 并行共享 | 16位 | 8MHz | 16 MB/s | IBM PC标准总线 |
| EISA | 1988 | 并行共享 | 32位 | 8.33MHz | 33 MB/s | ISA的32位扩展 |
| VESA Local Bus | 1992 | 并行共享 | 32位 | 40MHz | 160 MB/s | 视频专用 |
| PCI | 1992 | 并行共享 | 32/64位 | 33/66MHz | 533 MB/s | 即插即用，独立于CPU |
| AGP | 1997 | 点对点 | 32位 | 66MHz | 2.1 GB/s | 图形专用 |
| PCI-X | 1998 | 并行共享 | 64位 | 133MHz | 8.5 GB/s | 服务器领域 |
| PCIe 1.0 | 2003 | 串行点对点 | x1~x32 | 2.5 GT/s/通道 | 8 GB/s (x16) | 串行革命 |
| PCIe 2.0 | 2007 | 串行点对点 | x1~x32 | 5 GT/s/通道 | 16 GB/s (x16) | 带宽翻倍 |
| PCIe 3.0 | 2010 | 串行点对点 | x1~x32 | 8 GT/s/通道 | 32 GB/s (x16) | 128b/130b编码 |
| PCIe 4.0 | 2017 | 串行点对点 | x1~x32 | 16 GT/s/通道 | 64 GB/s (x16) | — |
| PCIe 5.0 | 2019 | 串行点对点 | x1~x32 | 32 GT/s/通道 | 128 GB/s (x16) | — |
| PCIe 6.0 | 2022 | 串行点对点 | x1~x32 | 64 GT/s/通道 | 256 GB/s (x16) | PAM4编码 |

#### PCI到PCIe的范式转变

**PCI总线的问题**：
- 并行共享总线：所有设备共享带宽
- 时钟频率受限于信号完整性（并行线间串扰）
- 总线仲裁增加延迟
- 最大带宽受限于总线宽度×时钟频率

**PCIe的创新**：
- **串行点对点**：每条链路独立，不共享带宽
- **差分信号**：使用一对导线传输一个信号，抗干扰能力强
- **多通道（Lane）**：x1、x4、x8、x16灵活配置
- **分层协议**：事务层（TLP）、数据链路层（DLLP）、物理层
- **带宽可扩展**：增加通道数即可线性增加带宽

**PCIe带宽计算**：
```
每通道有效带宽 = 传输速率 × 编码效率 ÷ 8

PCIe 3.0：8 GT/s × (128/130) ÷ 8 = 984.6 MB/s
PCIe 4.0：16 GT/s × (128/130) ÷ 8 = 1969.2 MB/s
PCIe 5.0：32 GT/s × (128/130) ÷ 8 = 3938.5 MB/s
```

#### 片上总线（AMBA/AXI）

片上总线用于SoC内部各模块之间的互连。

**AMBA（Advanced Microcontroller Bus Architecture）**是ARM公司提出的片上总线标准。

| 总线 | 定位 | 特点 | 典型用途 |
|------|------|------|---------|
| APB | 低速外设总线 | 简单、低功耗 | UART、SPI、GPIO |
| AHB | 系统总线 | 流水线、突发传输 | CPU、DMA、SRAM |
| AXI | 高性能总线 | 读写通道独立、乱序完成 | 高速外设、DDR控制器 |

**AXI（Advanced eXtensible Interface）总线**：
- 读写通道独立：可同时进行读写操作
- 支持突发传输（Burst Transfer）：一次地址传输，多次数据传输
- 支持乱序完成：低延迟事务可先于高延迟事务完成
- 支持多个未完成事务（Outstanding Transactions）
- AXI4、AXI4-Lite、AXI4-Stream三种变体

> [!NOTE]
> 从PCI到PCIe的转变体现了"串行取代并行"的设计趋势。当信号频率很高时，并行线间的串扰和时钟偏移（Skew）成为瓶颈，而串行差分信号可以工作在更高的频率上。类似的趋势也发生在存储器接口（SATA取代PATA）和USB的发展中。

### 2.1.8 I/O技术发展

I/O（输入/输出）技术是计算机与外部设备交互的桥梁。I/O技术的发展直接影响了系统的整体性能和用户体验。

#### I/O控制方式演进

| 阶段 | I/O方式 | 年代 | CPU参与度 | 并行度 | 适用场景 |
|------|--------|------|---------|--------|---------|
| 1 | 程序查询方式 | 1950s~ | 完全参与 | 无 | 低速简单设备 |
| 2 | 中断方式 | 1960s~ | 部分参与 | 有限 | 中低速设备 |
| 3 | DMA方式 | 1970s~ | 仅初始化和结束 | 高 | 高速块设备 |
| 4 | 通道方式 | 1970s~ | 仅发出命令 | 很高 | 大型机I/O |

#### 程序查询方式

程序查询（Programmed I/O，PIO）是最简单的I/O控制方式。

**工作原理**：
1. CPU执行I/O指令，向设备控制器发送命令
2. CPU不断查询设备状态寄存器（轮询）
3. 设备就绪后，CPU读取/写入数据
4. 重复步骤2~3直到传输完成

**特点**：
- CPU完全参与数据传输的每一个字节
- CPU大部分时间在等待设备就绪（浪费计算资源）
- 实现简单，无需额外硬件
- 不支持CPU与I/O设备并行工作

#### 中断方式

中断驱动I/O在设备就绪时主动通知CPU，避免了无效的轮询。

**工作原理**：
1. CPU向设备发出I/O命令后继续执行其他程序
2. 设备完成操作后，向CPU发送中断请求
3. CPU响应中断，执行中断服务程序
4. 中断服务程序完成数据传输
5. 返回原程序继续执行

**特点**：
- CPU与I/O设备可以部分并行工作
- 每次中断仍需CPU执行数据传输（每次传输一个字/字节）
- 中断响应和上下文切换有开销
- 适用于中低速设备

#### DMA方式

DMA（Direct Memory Access，直接内存访问）允许设备直接与内存交换数据，无需CPU干预。

**DMA控制器的组成**：
- 内存地址寄存器（MAR）：存放内存地址
- 字计数器（WC）：存放剩余传输字节数
- 数据缓冲寄存器（DBR）：暂存传输数据
- 控制/状态寄存器（CSR）：存放控制命令和状态信息

**DMA传输过程**：
1. CPU初始化DMA控制器：设置MAR、WC、传输方向
2. CPU启动I/O设备，继续执行其他程序
3. 设备准备好数据，向DMA发送DREQ（DMA请求）
4. DMA向CPU发送HRQ（总线请求）
5. CPU完成当前总线周期后，发出HLDA（总线响应），释放总线
6. DMA接管总线，控制数据在设备和内存之间直接传输
7. 每传输一个字，MAR加1，WC减1
8. WC溢出（传输完成），DMA释放总线，向CPU发送中断
9. CPU执行中断服务程序，处理传输完成事件

**DMA的三种工作模式**：
| 模式 | 说明 | CPU利用率 |
|------|------|----------|
| 停止CPU访问 | DMA独占总线直到传输完成 | 最低 |
| 周期挪用 | DMA在CPU不使用总线时传输 | 中等 |
| 交替访问 | DMA与CPU交替使用总线 | 最高 |

#### 通道方式

通道（Channel）是一种专用的I/O处理器，可以独立执行通道程序。

**工作原理**：
1. CPU执行I/O指令，启动通道并指定通道程序
2. 通道从内存取出通道指令，独立控制I/O操作
3. 通道完成I/O操作后，向CPU发送中断
4. CPU只需发出启动命令和处理结束中断

**通道类型**：
| 类型 | 数据传输单位 | 适用设备 | 复杂度 |
|------|------------|---------|--------|
| 字节多路通道 | 字节 | 低速设备（键盘、打印机） | 低 |
| 数组多路通道 | 数据块 | 高速设备（磁盘） | 中 |
| 选择通道 | 数据块 | 高速设备（磁带） | 高 |

**I/O方式性能对比**：

| 方面 | 程序查询 | 中断 | DMA | 通道 |
|------|---------|------|-----|------|
| CPU干预频率 | 每字节 | 每字/块 | 仅首尾 | 仅首尾 |
| 数据传输 | CPU执行程序 | CPU执行程序 | DMA控制器 | 通道处理器 |
| CPU利用率 | 最低 | 较低 | 较高 | 最高 |
| 系统开销 | 低 | 中（上下文切换） | 中（初始化） | 高（通道程序） |
| 并行度 | 无 | 有限 | 高 | 很高 |
| 适用场景 | 简单嵌入式 | 中低速设备 | 高速块设备 | 大型机 |
| 硬件复杂度 | 最低 | 低 | 中 | 高 |

#### I/O接口标准发展

| 接口 | 年代 | 类型 | 最大速率 | 特点 | 典型用途 |
|------|------|------|---------|------|---------|
| RS-232 | 1960 | 串行 | 20 kbps | 点对点 | 串口通信 |
| 并行端口 | 1970 | 并行 | 2 MB/s | Centronics | 打印机 |
| SCSI | 1986 | 并行 | 640 MB/s | 多设备 | 服务器存储 |
| USB 1.0 | 1996 | 串行 | 12 Mbps | 即插即用 | 外设 |
| USB 2.0 | 2000 | 串行 | 480 Mbps | 高速 | 外设、存储 |
| SATA 1.0 | 2003 | 串行 | 1.5 Gb/s | 点对点 | 存储设备 |
| USB 3.0 | 2008 | 串行 | 5 Gbps | 超高速 | 高速外设 |
| SATA 3.0 | 2009 | 串行 | 6 Gb/s | — | 存储设备 |
| USB 3.2 | 2017 | 串行 | 20 Gbps | 双通道 | 高速外设 |
| USB4 | 2019 | 串行 | 40 Gbps | Thunderbolt兼容 | 通用 |
| USB4 v2 | 2022 | 串行 | 80 Gbps | — | 通用 |

**NVMe（Non-Volatile Memory Express）**：
- 专为SSD设计的存储接口协议
- 通过PCIe总线直接连接CPU，绕过传统SATA/AHCI瓶颈
- 支持多达65535个命令队列，每队列65536个命令（AHCI仅1个队列32个命令）
- 低延迟：命令处理延迟从AHCI的~6μs降低到~2.8μs
- NVMe over Fabrics（NVMe-oF）：支持远程SSD访问

#### 现代I/O子系统架构

**现代I/O子系统的关键特征**：
1. **PCIe作为核心互连**：NVMe SSD、GPU、网卡等高速设备都通过PCIe连接
2. **设备直连CPU**：高性能设备（NVMe、GPU）通过PCIe直连CPU，减少延迟
3. **芯片组作为I/O集线器**：低速设备（USB、SATA、以太网）通过芯片组连接
4. **统一内存架构（UMA）**：部分架构中CPU和GPU共享内存，减少数据拷贝
5. **IOMMU**：I/O内存管理单元，支持设备直接虚拟地址访问，提高虚拟化性能

> [!TIP]
> I/O技术发展的核心趋势是减少CPU参与度、增加并行度、降低延迟。从程序查询到DMA，CPU从每个字节都参与变为仅在首尾干预。NVMe的出现则是将存储I/O的延迟从毫秒级降低到微秒级，使SSD的性能潜力得到充分发挥。

#### 例题2：摩尔定律

**题目**：解释摩尔定律的含义，并分析其面临的主要挑战。

**解答**：

摩尔定律：集成电路上的晶体管数量大约每18~24个月翻一倍。

主要挑战：
1. **物理极限**：晶体管尺寸接近原子级别，量子隧穿效应导致漏电
2. **功耗极限**：功耗密度无法持续增加（功耗墙）
3. **经济极限**：先进制程的研发和制造成本急剧上升
4. **散热极限**：芯片功率密度接近核反应堆水平

#### 例题3：计算机应用

**题目**：列举计算机在人工智能领域的三个应用。

**解答**：

1. **自然语言处理**：机器翻译、语音识别、文本生成
2. **计算机视觉**：图像识别、人脸识别、自动驾驶
3. **推荐系统**：电商推荐、内容推荐

#### 例题4：存储器性能分析

**题目**：某计算机系统中，L1 Cache命中率为95%，L2 Cache命中率为90%（在L1未命中时），主存访问时间为100ns，L1 Cache访问时间为1ns，L2 Cache访问时间为5ns。求CPU访存的平均访问时间。

**解答**：

**各级命中情况**：
- L1命中率 = 95%，访问时间 = 1ns
- L1未命中但L2命中率 = (1-95%) × 90% = 4.5%
- L1和L2都未命中率 = (1-95%) × (1-90%) = 0.5%

**平均访问时间**：
$$T_{avg} = 0.95 \times 1 + 0.045 \times (1+5) + 0.005 \times (1+5+100)$$
$$T_{avg} = 0.95 + 0.27 + 0.53 = 1.75\text{ns}$$

> [!TIP]
> 计算多级Cache的平均访问时间时，每一级的访问时间是**累加**的。L2命中时，总访问时间 = L1时间 + L2时间。这是串行查找的模型。

#### 例题5：DMA传送计算

**题目**：某系统以DMA方式从磁盘读取数据到内存。磁盘数据传输速率为100MB/s，CPU时钟频率为1GHz，总线宽度为32位，DMA采用周期挪用方式。每个总线周期为1个时钟周期。计算DMA传送对CPU的影响。

**解答**：

**数据传输率**：100MB/s = 100 × 10⁶ B/s

**每次DMA传送**：32位 = 4字节

**DMA传送频率**：100 × 10⁶ ÷ 4 = 25 × 10⁶ 次/秒

**每次DMA挪用周期数**：1个总线周期

**CPU时钟频率**：1GHz = 10⁹ 周期/秒

**DMA挪用的CPU时间比例**：25 × 10⁶ ÷ 10⁹ = **2.5%**

CPU仍能使用约97.5%的时间执行程序，DMA开销很小。

#### 例题6：PCIe带宽计算

**题目**：一块PCIe 3.0 x4的NVMe SSD，其理论最大顺序读取带宽是多少？若协议效率为80%，实际可用带宽是多少？

**解答**：

PCIe 3.0每通道带宽 = 8 GT/s × (128/130) ÷ 8 = 984.6 MB/s

x4配置的理论带宽 = 984.6 MB/s × 4 = **3938.4 MB/s ≈ 3.94 GB/s**

实际可用带宽 = 3938.4 × 80% = **3150.7 MB/s ≈ 3.15 GB/s**

#### 例题7：流水线性能计算

**题目**：一个5级流水线处理器，时钟频率为2GHz，执行1000条指令。假设无流水线冒险，求流水线加速比和吞吐率。

**解答**：

**非流水线执行时间**：
- 每条指令需要5个时钟周期
- 总时间 = 1000 × 5 × (1/2GHz) = 2.5μs

**流水线执行时间**：
- 第一条指令需要5个周期
- 后续每条指令1个周期
- 总时间 = (5 + 999) × (1/2GHz) = 502ns

**加速比** = 2.5μs / 502ns ≈ **4.98**（接近理想值5）

**吞吐率** = 1000 / 502ns ≈ **1.99 GIPS**（接近理想值2 GIPS）

#### 例题10：I/O方式对比分析

**题目**：某系统需要从高速数据采集卡读取数据，数据速率为200MB/s，CPU时钟为2GHz。分别分析使用中断方式和DMA方式时CPU的负担。

**解答**：

**中断方式**：
- 假设每次中断传输4字节数据
- 中断频率 = 200 × 10⁶ ÷ 4 = 50 × 10⁶ 次/秒
- 假设每次中断处理需要100个时钟周期
- CPU开销 = 50 × 10⁶ × 100 ÷ (2 × 10⁹) = **250%**（超过100%，无法实现）

**DMA方式**：
- DMA采用32位总线，每次传送4字节
- DMA传送频率 = 50 × 10⁶ 次/秒
- DMA采用周期挪用，每次挪用1个周期
- CPU开销 = 50 × 10⁶ × 1 ÷ (2 × 10⁹) = **2.5%**

**结论**：高速数据传输必须使用DMA方式，中断方式的CPU开销过大，无法满足要求。

### 例题4：存储器层次结构

**题目**：为什么现代计算机采用多级存储层次结构？各级存储器的选择依据是什么？

**解答**：

- CPU速度增长远快于存储器速度（处理器-内存差距）

|------|---------|---------|
| L1 Cache | 最快、最小 | SRAM |
| L2 Cache | 速度与容量平衡 | SRAM |
| 主存 | 大容量、可接受速度 | DRAM |
| 辅助存储 | 最大容量、最低成本 | SSD/HDD |

| 特性 | SRAM | DRAM |
|------|------|------|
| 存储单元 | 6个晶体管 | 1个晶体管+1个电容 |
| 速度 | 快（1~10ns） | 慢（10~100ns） |
| 密度 | 低 | 高 |

| 刷新 | 不需要 | 需要（64ms） |
| 用途 | Cache、寄存器 | 主存 |

### 例题6：DMA传输过程

**题目**：描述DMA方式传输数据的完整过程，并与中断方式进行比较。

**解答**：

**DMA传输过程**：
1. CPU初始化DMA控制器（设置内存地址、传输字节数、传输方向）
2. 外设向DMA发出DMA请求（DREQ）
3. DMA向CPU发出总线请求（HRQ）
4. CPU释放总线，发出总线响应（HLDA）
5. DMA控制数据在内存和外设之间直接传输
6. 传输完成后，DMA向CPU发出中断请求

**DMA vs 中断比较**：

| 方面 | 中断方式 | DMA方式 |
|------|---------|---------|
| 数据传输 | CPU执行程序传输 | DMA控制器直接传输 |
| CPU参与度 | 每次传输都参与 | 仅初始化和结束时参与 |
| 适用场景 | 低速、少量数据 | 高速、大量数据 |
| 系统开销 | 较大（上下文切换） | 较小 |
| 传输单位 | 字节/字 | 数据块 |

## 2.6 本章常见误区与易错点

1. **ENIAC不是第一台存储程序计算机**：EDVAC才是（冯·诺依曼设计）。

2. **摩尔定律不是物理定律**：是一个经验性的产业规律。

3. **第4代计算机的标志是微处理器**：不是集成电路（那是第3代）。

4. **计算机发展不只看硬件**：软件技术同样重要。

5. **量子计算机不会完全取代经典计算机**：只在特定问题上有优势。

6. **SRAM不需要刷新≠非易失性**：SRAM不需要刷新是因为双稳态电路可以保持状态，但断电后数据仍会丢失。

7. **DMA不是完全不需要CPU**：DMA传输前需要CPU初始化，传输完成后需要CPU处理。

8. **流水线不能减少单条指令的执行时间**：流水线提高的是吞吐率，单条指令延迟不变甚至增加（流水线寄存器开销）。

9. **超线程不等于双核**：超线程是逻辑上的并行，共享执行资源，性能提升有限。

10. **PCIe是串行总线，不是并行总线**：虽然"总线"一词暗示并行，但PCIe使用串行差分信号传输。

11. **摩尔定律已"失效"**：更准确地说是节奏放缓，从18个月延长到2~3年，但仍在继续。

12. **ARM架构不等于低性能**：Apple M系列芯片证明ARM架构可以达到极高性能。

13. **EUV光刻不是简单的波长缩短**：EUV需要全新的光学系统（反射式而非透射式）、全真空环境、全新掩模版技术，是完全不同的技术体系。

14. **FinFET的"鳍片"不是为了增加表面积**：鳍片结构的目的是让栅极从三面包围沟道，增强栅极对沟道的控制能力，减少漏电流。

15. **Chiplet不是简单的"拼芯片"**：Chiplet之间的互连需要先进封装技术（如硅桥、硅中介层、混合键合），互连带宽和延迟是关键挑战。

16. **DRAM需要刷新≠SRAM是非易失性的**：SRAM不需要刷新是因为双稳态电路可以自保持，但断电后两者都会丢失数据（都是易失性存储器）。

17. **PCIe的"串行"不意味着比"并行"慢**：PCIe使用高速差分信号，单通道频率远高于并行总线，多通道聚合后带宽远超PCI。

18. **DMA不是"零CPU开销"**：DMA传输前需要CPU初始化控制器，传输完成后需要CPU处理中断和数据，且周期挪用模式下仍会占用少量总线周期。

19. **乱序执行不会改变程序结果**：乱序执行只是改变了指令的执行顺序，但通过ROB（重排序缓冲区）保证最终结果按程序顺序提交，对程序员透明。
