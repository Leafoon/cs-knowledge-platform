# 第十五章 CPU概述与数据通路

CPU是计算机的核心，负责指令的获取、译码和执行。本章讲解CPU的功能、组成和数据通路设计，为后续控制器和流水线章节奠定基础。

---

## 15.1 CPU的功能与组成

### 15.1.1 CPU的核心功能

| 功能 | 说明 |
|------|------|
| **指令控制** | 控制程序的执行顺序（取指、译码） |
| **操作控制** | 产生各种操作的控制信号 |
| **时间控制** | 对各种操作实施时间上的控制 |
| **数据加工** | 对数据进行算术和逻辑运算 |

## 15.2 CPU中的寄存器

### 15.2.1 用户可见寄存器

| 寄存器 | 用途 | 可编程访问 |
|--------|------|-----------|
| 通用寄存器 | 存放操作数和结果 | 是 |
| PC | 存放下一条指令地址 | 间接（跳转指令） |
| PSW | 存放标志位和状态 | 部分 |

### 15.2.2 用户不可见寄存器

<div data-component="RegisterFileExplorer"></div>

| 寄存器 | 用途 |
|--------|------|
| MAR | 存放要访问的存储器地址 |
| MDR | 存放从存储器读出/写入的数据 |
| IR | 存放当前正在执行的指令 |
| 暂存寄存器 | 存放中间运算结果 |

### 15.2.3 存储器相关寄存器

$$
\text{存储容量} = 2^{\text{MAR位数}} \times \text{MDR位数}
$$

**【例 15-1】** MAR为16位，MDR为8位。

$$
\text{容量} = 2^{16} \times 8 = 65536 \times 8 = 64\text{KB}
$$

## 15.3 数据通路的基本概念

### 15.3.1 什么是数据通路

**数据通路**（Data Path）是CPU中数据流动的路径，包括：
- **功能部件**：ALU、寄存器、存储器接口
- **传输线路**：总线、多路选择器
- **控制信号**：选择数据来源、控制数据流向

### 15.3.2 数据通路的基本元素

| 元素 | 功能 |
|------|------|
| 寄存器 | 暂存数据 |
| ALU | 执行运算 |
| MUX（多路选择器） | 选择数据来源 |
| 总线 | 传输数据 |
| 三态门 | 控制数据是否上总线 |

---

## 15.4 数据通路的组织方式

### 15.4.1 CPU内部总线方式

所有寄存器通过内部总线连接，ALU通过总线接收操作数。

**单总线结构**：

```
寄存器A ──┐
寄存器B ──┤
寄存器C ──┼── 内部总线 ── ALU
MAR ──────┤              │
MDR ──────┘              │
                    结果 → 目标寄存器
```

- 一个时钟周期只能传输一个数据
- 需要多个时钟周期完成一次运算

**双总线结构**：

```
寄存器 ── 总线1 ── ALU输入A
寄存器 ── 总线2 ── ALU输入B
                ALU输出 → 目标寄存器
```

- 可以同时传输两个操作数
- 一个时钟周期完成一次运算

**三总线结构**：

```
总线1 ── ALU输入A
总线2 ── ALU输入B
        ALU ── 总线3 ── 目标寄存器
```

- 三个总线独立，效率最高

### 15.4.2 专用数据通路方式

寄存器之间有专用的直连通路，无需通过总线。

- 速度快，但硬件复杂
- 现代处理器通常采用专用数据通路

### 15.4.3 总线方式对比

| 结构 | 每周期传输数据量 | 硬件复杂度 | 速度 |
|------|----------------|-----------|------|
| 单总线 | 1个 | 低 | 慢 |
| 双总线 | 2个 | 中 | 中 |
| 三总线 | 3个 | 高 | 快 |
| 专用通路 | 多个 | 最高 | 最快 |

## 15.5 典型数据通路分析

### 15.5.1 取指令的数据通路

```
PC → MAR
1 → R（读命令）
M(MAR) → MDR
MDR → IR
OP(IR) → ID
PC + 1 → PC
```

### 15.5.2 ALU运算的数据通路（加法）

```
寄存器A → ALU输入1
寄存器B → ALU输入2
ALU执行加法
ALU结果 → 目标寄存器
```

### 15.5.3 访存的数据通路（Load）

```
IR的地址字段 → MAR
1 → R（读命令）
M(MAR) → MDR
MDR → 目标寄存器
```

### 15.5.4 访存的数据通路（Store）

```
IR的地址字段 → MAR
源寄存器 → MDR
0 → R（写命令）
MDR → M(MAR)
```

## 15.6 CPU的工作过程

### 15.6.1 指令执行的基本流程

```
1. 取指：PC → MAR → 存储器 → MDR → IR
2. 译码：IR的操作码 → ID → 产生控制信号
3. 执行：根据指令类型执行相应操作
4. 写回：结果写入目标寄存器
5. 更新PC：PC + 1（或跳转地址）
```

### 15.6.2 不同指令的数据通路

| 指令类型 | 数据通路 |
|----------|---------|
| 寄存器传送 | 寄存器 → 内部总线 → 目标寄存器 |
| ALU运算 | 寄存器 → ALU → 结果 → 寄存器 |
| Load | 地址 → MAR → 存储器 → MDR → 寄存器 |
| Store | 寄存器 → MDR，地址 → MAR → 存储器 |
| 跳转 | 目标地址 → PC |

## 15.7 本章典型例题

### 例题1：CPU组成分析

**题目**：某CPU有16根地址线、8根数据线，MAR和MDR各需要多少位？

**解答**：

- MAR = 16位（与地址线数相同）
- MDR = 8位（与数据线数相同）
- 存储容量 = $2^{16} \times 8 = 64\text{KB}$

### 例题2：数据通路分析

**题目**：描述执行 `ADD R1, R2, R3`（R1 = R2 + R3）的数据通路。

**解答**：

1. R2 → ALU输入A
2. R3 → ALU输入B
3. ALU执行加法
4. ALU结果 → R1

### 例题3：寄存器功能

**题目**：PC、IR、MAR、MDR分别有什么功能？

**解答**：

- PC（程序计数器）：存放下一条指令的地址
- IR（指令寄存器）：存放当前正在执行的指令
- MAR（存储器地址寄存器）：存放要访问的存储器地址
- MDR（存储器数据寄存器）：存放从存储器读出/写入的数据

## 15.10 深入理解CPU的组成

### 15.10.1 运算器的详细结构

运算器是CPU的执行部件，负责算术运算和逻辑运算。

**核心部件详解**：

| 部件 | 全称 | 功能 | 位数 |
|------|------|------|------|
| ALU | Arithmetic Logic Unit | 执行算术和逻辑运算 | 与字长相同 |
| ACC | Accumulator | 存放运算结果 | 与字长相同 |
| MQ | Multiplier-Quotient | 乘除法时存放乘数/商 | 与字长相同 |
| X | — | 存放一个操作数 | 与字长相同 |
| PSW | Program Status Word | 存放状态标志位 | 16~64位 |

**ALU的功能**：
- 算术运算：加、减、乘、除
- 逻辑运算：与、或、非、异或
- 移位运算：左移、右移
- 比较运算：大于、小于、等于

### 15.10.2 控制器的详细结构

控制器是CPU的指挥中心，负责控制程序的执行。

**核心部件详解**：

| 部件 | 全称 | 功能 | 说明 |
|------|------|------|------|
| PC | Program Counter | 存放下一条指令地址 | 自动递增 |
| IR | Instruction Register | 存放当前指令 | 从存储器读取 |
| ID | Instruction Decoder | 译码操作码 | 产生控制信号 |
| CU | Control Unit | 产生控制信号 | 协调各部件 |
| 时序发生器 | — | 产生时序信号 | 节拍控制 |

**CU的功能**：
- 根据指令操作码产生控制信号
- 根据时序信号协调各部件
- 处理中断和异常

### 15.10.3 CPU的内部总线

CPU内部总线连接各部件：

```
┌─────────────────────────────────────┐
│               CPU                    │
│                                     │
│  ┌─────┐   ┌─────┐   ┌─────┐      │
│  │寄存器│──→│ ALU │──→│结果 │      │
│  │  组  │   │     │   │寄存器│      │
│  └─────┘   └─────┘   └─────┘      │
│      ↑         ↑           │       │
│      └─────────┴───────────┘       │
│           内部总线                   │
│                │                    │
│         ┌──────┴──────┐            │
│         │   控制器     │            │
│         └─────────────┘            │
└─────────────────────────────────────┘
```

## 15.11 CPU的工作过程详解

### 15.11.1 指令执行的基本流程

```
1. 取指：PC → MAR → 存储器 → MDR → IR
2. 译码：IR的操作码 → ID → 产生控制信号
3. 执行：根据指令类型执行相应操作
4. 写回：结果写入目标寄存器
5. 更新PC：PC + 1（或跳转地址）
```

### 15.11.2 不同指令的数据通路

**算术指令**（如 `ADD R1, R2, R3`）：

```
R2 → ALU输入A
R3 → ALU输入B
ALU执行加法
ALU结果 → R1
```

**Load指令**（如 `LW R1, 4(R2)`）：

```
R2 → ALU输入A
4 → ALU输入B
ALU计算地址 R2+4 → MAR
存储器[MAR] → MDR
MDR → R1
```

**Store指令**（如 `SW R1, 4(R2)`）：

```
R2 → ALU输入A
4 → ALU输入B
ALU计算地址 R2+4 → MAR
R1 → MDR
MDR → 存储器[MAR]
```

**Branch指令**（如 `BEQ R1, R2, label`）：

```
R1 → ALU输入A
R2 → ALU输入B
ALU比较 R1-R2
如果ZF=1（相等）：PC+label → PC
否则：PC+1 → PC
```

## 15.12 CPU的性能指标

### 15.12.1 时钟频率

时钟频率是CPU的基本性能指标：

$$
T = \frac{1}{f}
$$

其中 $T$ 为时钟周期，$f$ 为时钟频率。

### 15.12.2 CPI

CPI（Cycles Per Instruction）是每条指令的平均时钟周期数：

$$
\text{CPI} = \frac{\text{总时钟周期数}}{\text{指令数}}
$$

### 15.12.3 MIPS

MIPS（Million Instructions Per Second）是每秒百万指令数：

$$
\text{MIPS} = \frac{\text{指令数}}{\text{执行时间} \times 10^6}
$$

### 15.12.4 CPU执行时间

$$
\text{CPU时间} = \frac{\text{指令数} \times \text{CPI}}{\text{时钟频率}}
$$

## 15.15 CPU内部寄存器组的详细结构

### 15.15.1 通用寄存器组

通用寄存器组（General Purpose Register File）是CPU中用于暂存操作数和运算结果的一组高速存储单元。

**典型RISC处理器的寄存器文件**：

| 处理器 | 通用寄存器数量 | 位数 | 特点 |
|--------|--------------|------|------|
| MIPS | 32（$0~$31） | 32位 | $0恒为0 |
| ARM | 16（R0~R15） | 32位 | R13=SP, R14=LR, R15=PC |
| RISC-V | 32（x0~x31） | 32/64位 | x0恒为0 |
| x86-64 | 16（RAX~R15） | 64位 | 兼容32/16/8位访问 |

**MIPS寄存器文件的详细分配**：

| 寄存器 | 编号 | 用途 | 调用约定 |
|--------|------|------|---------|
| $zero | $0 | 恒为0 | 硬件连线 |
| $at | $1 | 汇编器保留 | — |
| $v0-$v1 | $2-$3 | 函数返回值 | caller保存 |
| $a0-$a3 | $4-$7 | 函数参数 | caller保存 |
| $t0-$t7 | $8-$15 | 临时寄存器 | caller保存 |
| $s0-$s7 | $16-$23 | 保存寄存器 | callee保存 |
| $t8-$t9 | $24-$25 | 临时寄存器 | caller保存 |
| $k0-$k1 | $26-$27 | 内核保留 | — |
| $gp | $28 | 全局指针 | — |
| $sp | $29 | 栈指针 | callee保存 |
| $fp | $30 | 帧指针 | callee保存 |
| $ra | $31 | 返回地址 | callee保存 |

### 15.15.2 专用寄存器

专用寄存器用于CPU内部控制，程序员通常不可直接访问。

| 寄存器 | 全称 | 位数 | 功能 | 访问方式 |
|--------|------|------|------|---------|
| PC | Program Counter | 32/64 | 存放下条指令地址 | 跳转指令间接修改 |
| IR | Instruction Register | 32/64 | 存放当前指令 | 不可直接访问 |
| MAR | Memory Address Register | 与地址线同宽 | 存放访存地址 | 不可直接访问 |
| MDR | Memory Data Register | 与数据线同宽 | 存放读写数据 | 不可直接访问 |
| PSW | Program Status Word | 16~64 | 存放状态标志 | 部分可读写 |
| SP | Stack Pointer | 32/64 | 栈顶指针 | 可编程 |
| PSR | Processor Status Register | 32 | 处理器状态 | 特权指令访问 |

**PSW（程序状态字）的典型格式**：

```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│ 15 │ 14 │ 13 │ 12 │ 11 │ 10 │  9 │  8 │
│    │    │    │    │ OF │ DF │ IF │ TF │
├────┼────┼────┼────┼────┼────┼────┼────┤
│  7 │  6 │  5 │  4 │  3 │  2 │  1 │  0 │
│ SF │ ZF │    │ AF │    │ PF │    │ CF │
└────┴────┴────┴────┴────┴────┴────┴────┘
```

| 标志位 | 名称 | 含义 |
|--------|------|------|
| CF | Carry Flag | 进位/借位标志 |
| PF | Parity Flag | 奇偶标志 |
| AF | Auxiliary Carry Flag | 辅助进位标志 |
| ZF | Zero Flag | 零标志 |
| SF | Sign Flag | 符号标志 |
| OF | Overflow Flag | 溢出标志 |
| TF | Trap Flag | 陷阱标志（单步调试） |
| IF | Interrupt Enable Flag | 中断允许标志 |
| DF | Direction Flag | 方向标志（字符串操作） |

### 15.15.3 寄存器文件的多端口设计

寄存器文件需要支持同时读写多个寄存器，因此采用多端口设计。

**单端口寄存器文件**：

```
        ┌─────────────┐
地址 ──→│             │──→ 读出数据
        │  寄存器文件   │
数据 ──→│             │
        └─────────────┘
        一次只能读或写
```

**双端口寄存器文件（1读1写）**：

```
        ┌─────────────┐
读地址 ─→│             │──→ 读出数据
        │  寄存器文件   │
写地址 ─→│             │←── 写入数据
        └─────────────┘
        可同时读和写
```

**三端口寄存器文件（2读1写）**：

```
        ┌─────────────┐
读地址1 ─→│             │──→ 读出数据1
        │             │
读地址2 ─→│  寄存器文件   │──→ 读出数据2
        │             │
写地址 ─→│             │←── 写入数据
        └─────────────┘
        可同时读两个寄存器并写一个寄存器
```

> [!NOTE]
> RISC处理器（如MIPS）的寄存器文件通常采用**三端口设计**（2读1写），因为R型指令需要同时读取两个源寄存器并写入一个目的寄存器。

**寄存器文件的硬件实现**：

```
              读地址A    读地址B    写地址
                 │          │         │
                 ▼          ▼         ▼
```

| 步骤 | 微操作 | 时钟周期 | 说明 |
|------|--------|---------|------|
| 1 | R1_out, Y_in | T1 | R1送到总线，写入暂存器Y |

| 5 | Read, MDR_in | T5 | 读数据到MDR |
| 6 | MDR_out, R1_in | T6 | MDR送R1 |

> 需要 **6个时钟周期**

**结构图**：

```

│  │ R0~Rn │   │ R0~Rn │   │ R0~Rn │                 │
│  │       │   │       │   │       │                 │
│  └───┬───┘   └───┬───┘   └───┬───┘                 │

│      │     ┌─────┴─────┐     │                      │
│      │     │           │     │                      │
│      └────→│   ALU     │────→┘                      │
│            │           │                            │
│            └───────────┘                            │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**三总线执行 `ADD R1, R2` 的微操作序列**：

| 步骤 | 微操作 | 时钟周期 | 说明 |
|------|--------|---------|------|
| 1 | R1_out_A, R2_out_B, ALU_add, R1_in_C | T1 | 一步完成 |

> 需要 **1个时钟周期**（最快）

### 15.16.4 三种总线结构的性能对比

| 指标 | 单总线 | 双总线 | 三总线 |
|------|--------|--------|--------|
| ADD指令周期数 | 3 | 2 | 1 |
| LOAD指令周期数 | 6 | 4 | 3 |
| 每周期传输数据量 | 1个 | 2个 | 3个 |
| 总线数量 | 1 | 2 | 3 |
| MUX数量 | 少 | 中 | 多 |
| 硬件复杂度 | 低 | 中 | 高 |
| 控制复杂度 | 简单 | 中等 | 复杂 |
| 典型应用 | 教学模型 | 早期处理器 | 高性能处理器 |

**性能分析公式**：

设指令数为 $N$，单总线CPI为 $C_1$，双总线CPI为 $C_2$，三总线CPI为 $C_3$：

$$
\text{加速比}_{2/1} = \frac{C_1}{C_2}, \quad \text{加速比}_{3/1} = \frac{C_1}{C_3}
$$

> [!TIP]
> 三总线结构的理论加速比最高，但硬件成本也最高。实际处理器通常在双总线和三总线之间取平衡，或采用专用数据通路。

```
│  │ 指令Cache│  │ 立即数扩展│  │ 移位器   │                  │

│  └──────────┘  └──────────┘  └──────────┘                  │
│                                      │                      │
│                                      ▼                      │
│                              ┌──────────┐  ┌──────────┐    │
│                              │ 访存单元  │  │ 写回单元  │    │

│                              │          │  │          │    │
```

## 15.20 RISC vs CISC的数据通路差异

### 15.20.1 设计理念对比

| 特性 | RISC | CISC |
|------|------|------|
| 指令数量 | 少（~100条） | 多（~300条） |
| 指令长度 | 固定 | 可变 |
| 寻址方式 | 少（3~5种） | 多（10~20种） |
| 访存方式 | Load/Store | 可直接运算 |
| 流水线 | 易于实现 | 难于实现 |
| 控制器 | 硬布线 | 微程序 |

### 15.20.2 数据通路差异

**RISC数据通路特征**：

```
┌─────────────────────────────────────┐
│            RISC CPU                  │
│                                     │
│  ┌─────────────────────────────┐   │
│  │      通用寄存器文件           │   │
│  │   (32个，3端口)              │   │
│  └────────────┬────────────────┘   │
│               │                     │
│  ┌────────────┴────────────────┐   │
│  │      ALU / 移位器            │   │
│  └────────────┬────────────────┘   │
│               │                     │
│  ┌────────────┴────────────────┐   │
│  │    Load/Store单元            │   │
│  │    (仅访存指令访问存储器)     │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

**RISC特点**：
- 所有指令长度相同（32位）
- 只有Load/Store指令访问存储器
- 三地址格式：`ADD R1, R2, R3`
- 寄存器数量多（32个）
- 硬布线控制器

**CISC数据通路特征**：

```
┌─────────────────────────────────────┐
│            CISC CPU                  │
│                                     │
│  ┌─────────────────────────────┐   │
│  │      通用寄存器文件           │   │
│  │   (8个，多用途)              │   │
│  └────────────┬────────────────┘   │
│               │                     │
│  ┌────────────┴────────────────┐   │
│  │      ALU                    │   │
│  │   (支持复杂运算)             │   │
│  └────────────┬────────────────┘   │
│               │                     │
│  ┌────────────┴────────────────┐   │
│  │    存储器接口                 │   │
│  │    (运算指令可直接访存)       │   │
│  └─────────────────────────────┘   │
│                                     │
│  ┌─────────────────────────────┐   │
│  │    微程序控制器               │   │
│  │    (控制存储器)              │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

**CISC特点**：
- 指令长度可变（1~15字节）
- 运算指令可直接操作存储器
- 二地址格式：`ADD [mem], reg`
- 寄存器数量少（8个）
- 微程序控制器

### 15.20.3 指令执行对比

**RISC执行 `ADD R1, R2, R3`**：

```
单周期完成：
R2 + R3 → R1
```

**CISC执行 `ADD [addr], R1`**（存储器操作数加法）：

```
多周期完成：
1. 取指：从存储器取指令
2. 译码：解析指令
3. 取操作数：从存储器读取[addr]
4. 执行：[addr] + R1
5. 写回：结果写回存储器[addr]
```

### 15.20.4 流水线效率对比

| 指标 | RISC | CISC |
|------|------|------|
| 流水线级数 | 5~15级 | 10~20+级 |
| 流水线冒险 | 少 | 多 |
| 分支预测 | 简单 | 复杂 |
| CPI | 1~2 | 2~20 |
| 时钟频率 | 高 | 中等 |

> [!NOTE]
> 现代x86处理器（CISC）内部采用RISC核心，将CISC指令翻译为RISC微操作（μops），兼具CISC兼容性和RISC高性能。

### 15.20.5 RISC与CISC的融合趋势

```
┌─────────────────────────────────────────────┐
│           现代x86处理器内部                    │
│                                             │
│  CISC指令 ──→ 译码器 ──→ RISC微操作(μops)   │
│                              │              │
│                              ▼              │
│                    ┌─────────────────┐      │
│                    │  RISC执行核心    │      │
│                    │  (超标量/乱序)   │      │
│                    └─────────────────┘      │
└─────────────────────────────────────────────┘
```

## 15.21 CPU性能的深入分析

### 15.21.1 Amdahl定律

$$
\text{加速比} = \frac{1}{(1-f) + \frac{f}{S}}
$$

其中 $f$ 为可加速部分的比例，$S$ 为可加速部分的加速倍数。

### 15.21.2 CPU性能公式详解

$$
\text{CPU时间} = \text{指令数} \times \text{CPI} \times \text{时钟周期}
$$

展开为：

$$
\text{CPU时间} = \sum_{i=1}^{n}(IC_i \times CPI_i) \times T_{clk}
$$

其中 $IC_i$ 为第 $i$ 类指令的数量，$CPI_i$ 为第 $i$ 类指令的CPI。

**【例 15-2】** 某程序有100条指令，其中ALU指令50条（CPI=1），Load指令30条（CPI=2），Store指令10条（CPI=2），Branch指令10条（CPI=3）。时钟频率1GHz。求CPU执行时间。

$$
\text{总周期数} = 50 \times 1 + 30 \times 2 + 10 \times 2 + 10 \times 3 = 50 + 60 + 20 + 30 = 160
$$

$$
\text{CPU时间} = \frac{160}{1 \times 10^9} = 160\text{ns}
$$

### 15.21.3 影响CPI的因素

| 因素 | 影响 |
|------|------|
| Cache命中率 | Cache miss增加CPI |
| 流水线冒险 | 数据/控制冒险增加CPI |
| 分支预测准确率 | 预测错误增加CPI |
| 中断频率 | 中断处理增加CPI |
| 指令组合 | 复杂指令CPI高 |

**多级Cache对CPI的影响**：

$$
\text{CPI}_{\text{实际}} = \text{CPI}_{\text{理想}} + \text{Miss Rate} \times \text{Miss Penalty}
$$

## 15.22 本章补充例题

### 例题4：寄存器文件端口设计

**题目**：某CPU需要支持以下操作：同时读取两个源寄存器并写入一个目的寄存器。寄存器文件至少需要几个端口？

**解答**：

- 读端口：2个（读取两个源操作数）
- 写端口：1个（写入结果）
- 总共需要 **3个端口**

### 例题5：数据通路性能比较

**题目**：比较单总线和三总线CPU执行 `ADD R1, R2, R3` 所需的时钟周期数。

**解答**：

**单总线**：
1. R2 → Y（T1）
2. R3 + Y → Z（T2）
3. Z → R1（T3）
共需 **3个时钟周期**

**三总线**：
1. R2 + R3 → R1（T1）
共需 **1个时钟周期**

### 例题6：RISC vs CISC

**题目**：为什么RISC处理器通常采用硬布线控制器？

**解答**：

RISC指令集简单、规整，指令数量少且格式统一，硬布线逻辑的复杂度可控。硬布线控制器无需访问控制存储器，产生控制信号的速度更快，适合追求高性能的RISC设计。而CISC指令集复杂，硬布线实现困难且不灵活，因此多采用微程序控制器。
