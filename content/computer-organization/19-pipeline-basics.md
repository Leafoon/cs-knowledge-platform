# 第十九章 流水线技术 — 基本原理

流水线是提高CPU性能的关键技术，通过将指令执行过程分为多个阶段并行处理，实现"每个时钟周期完成一条指令"的理想状态。**本章是考研重点章节**，需要熟练掌握流水线的性能指标计算和时空图绘制。

## 19.1 流水线的基本概念

### 19.1.1 工厂流水线类比

**汽车装配线**：
- 4个工位：安装底盘 → 安装发动机 → 安装车身 → 喷漆
- 每个工位耗时1小时
- 非流水线：每辆车需要4小时，4辆车需要16小时
- 流水线：第1辆车4小时，之后每小时完成1辆，4辆车只需7小时

### 19.1.2 指令流水线

将指令执行过程分为多个阶段，多条指令同时处于不同阶段。

**经典5级流水线**：

| 阶段 | 名称 | 功能 |
|------|------|------|
| IF | 取指（Instruction Fetch） | 从存储器取出指令 |
| ID | 译码（Instruction Decode） | 译码操作码，读寄存器 |
| EX | 执行（Execute） | ALU运算或地址计算 |
| MEM | 访存（Memory Access） | 读写存储器 |
| WB | 写回（Write Back） | 结果写回寄存器 |

<div data-component="PipelineStageViz"></div>

### 19.1.3 流水线的工作原理

**非流水线执行**：

```
指令  | 时间
I1    | IF ID EX MEM WB
I2    |                IF ID EX MEM WB
I3    |                                IF ID EX MEM WB
```

每条指令5个时钟周期，3条指令需要15个时钟周期。

**流水线执行**：

```
指令  | 时间
      | T1 T2 T3 T4 T5 T6 T7
I1    | IF ID EX MEM WB
I2    |    IF ID EX  MEM WB
I3    |       IF ID  EX  MEM WB
```

3条指令需要7个时钟周期（而非15个）。

## 19.2 流水线的时空图

### 19.2.1 时空图的绘制

- **横轴**：时间（时钟周期 T1, T2, T3, ...）
- **纵轴**：空间（流水线各段 IF, ID, EX, MEM, WB）
- 每个格子表示某个指令在某个时钟周期处于哪个阶段

**【例 19-1】** 绘制5级流水线执行5条指令的时空图。

```
阶段  | T1  T2  T3  T4  T5  T6  T7  T8  T9
WB    |         I1  I2  I3  I4  I5
MEM   |     I1  I2  I3  I4  I5
EX    | I1  I2  I3  I4  I5
ID    | I1  I2  I3  I4  I5
IF    | I1  I2  I3  I4  I5
```

每个时钟周期都有一条指令完成（从T5开始），理想情况下吞吐率为1条/周期。

## 19.3 流水线的性能指标

<div data-component="ThroughputEfficiencyChart"></div>

### 19.3.1 吞吐率（Throughput, TP）

**定义**：单位时间内完成的指令数。

$$
TP = \frac{n}{T_{\text{流水线}}}
$$

其中 $n$ 为指令数，$T_{\text{流水线}}$ 为流水线执行时间。

**理想情况**（$k$ 段流水线，$n$ 条指令）：

$$
T_{\text{流水线}} = (k + n - 1) \Delta t
$$

其中 $k$ 为流水线段数，$\Delta t$ 为时钟周期。

$$
TP = \frac{n}{(k + n - 1) \Delta t}
$$

当 $n \to \infty$ 时，$TP \to \frac{1}{\Delta t}$（每个时钟周期完成一条指令）

### 19.3.2 加速比（Speedup, S）

**定义**：非流水线执行时间与流水线执行时间的比值。

$$
S = \frac{T_{\text{非流水线}}}{T_{\text{流水线}}} = \frac{nk \Delta t}{(k + n - 1) \Delta t} = \frac{nk}{k + n - 1}
$$

当 $n \to \infty$ 时，$S \to k$（理想加速比等于段数）

### 19.3.3 效率（Efficiency, E）

**定义**：流水线各段的实际使用时间占比。

$$
E = \frac{S}{k} = \frac{n}{k + n - 1}
$$

当 $n \to \infty$ 时，$E \to 1$

### 19.3.4 性能计算示例

**【例 19-2】** 5级流水线，时钟周期100ns，执行100条指令。求吞吐率、加速比、效率。

**流水线执行时间**：

$$
T_{\text{流水线}} = (5 + 100 - 1) \times 100 = 10400 \text{ ns}
$$

**非流水线执行时间**：

$$
T_{\text{非流水线}} = 100 \times 5 \times 100 = 50000 \text{ ns}
$$

**吞吐率**：

$$
TP = \frac{100}{10400 \text{ ns}} = 9.62 \times 10^6 \text{ 条/秒}
$$

**加速比**：

$$
S = \frac{50000}{10400} = 4.81
$$

**效率**：

$$
E = \frac{4.81}{5} = 96.2\%
$$

> [!TIP]
> **关键观察**：100条指令时，加速比已经达到4.81（接近理想的5），效率96.2%。指令数越多，性能越接近理想值。

**【例 19-3】** 4级流水线，各段时间分别为90ns、80ns、100ns、70ns。求流水线时钟周期和吞吐率。

**流水线时钟周期**由最慢的段决定：

$$
\Delta t = \max(90, 80, 100, 70) = 100 \text{ ns}
$$

**吞吐率**（$n$ 条指令）：

$$
TP = \frac{n}{(4 + n - 1) \times 100}
$$

> [!WARNING]
> 流水线的时钟周期由**最慢的段**决定！这是流水线的一个重要限制。如果某一段特别慢，整个流水线都会被拖慢。

## 19.5 非线性流水线

### 19.5.1 概念

存在**反馈回路**的流水线，不能简单地每个时钟周期启动一条新指令。

### 19.5.2 预约表（Reservation Table）

用于分析非线性流水线的冲突和确定最小启动间隔。

**示例**：

| 时间 | T1 | T2 | T3 | T4 | T5 |
|------|----|----|----|----|-----|
| S1   | X  |    |    | X  |     |
| S2   |    | X  |    |    | X   |
| S3   |    |    | X  |    |     |

X表示该时间段该段被占用。

**禁止间隔**：从预约表中找出同一行中两个X之间的距离。

## 19.6 流水线的局限性

### 19.6.1 流水线不能无限细分

- 每段之间需要**锁存器**（latch），增加延迟
- 段数过多时，锁存器开销抵消流水线收益

### 19.6.2 流水线不能消除所有延迟

- **结构冒险**：资源冲突
- **数据冒险**：数据相关
- **控制冒险**：分支跳转

这些"冒险"（Hazard）会在第20章详细讲解。

## 19.7 本章典型例题

### 例题1：基本性能计算

**题目**：某流水线有4段，各段执行时间分别为2ns、3ns、4ns、1ns。求：
（1）流水线时钟周期
（2）执行100条指令的加速比

**解答**：

（1）时钟周期 = $\max(2, 3, 4, 1) = 4$ ns

（2）非流水线执行时间 = $100 \times (2+3+4+1) = 1000$ ns

流水线执行时间 = $(4 + 100 - 1) \times 4 = 412$ ns

加速比 = $1000 / 412 = 2.43$

### 例题2：时空图绘制

**题目**：4级流水线（IF/ID/EX/WB），执行4条指令，无冒险。绘制时空图并计算性能指标。

**解答**：

```
阶段 | T1  T2  T3  T4  T5  T6  T7
WB   |         I1  I2  I3  I4
EX   |     I1  I2  I3  I4
ID   | I1  I2  I3  I4
IF   | I1  I2  I3  I4
```

- 执行时间 = $(4 + 4 - 1) \times \Delta t = 7\Delta t$
- 吞吐率 = $4 / (7\Delta t)$
- 加速比 = $(4 \times 4) / 7 = 16/7 = 2.29$
- 效率 = $2.29 / 4 = 57.1\%$

### 例题3：实际加速比

**题目**：某程序有1000条指令，5级流水线，每段1ns。其中有20%的指令是Load指令，Load指令后面紧跟的指令需要暂停1个周期（Load-Use冒险）。求实际加速比。

**解答**：

理想执行时间 = $(5 + 1000 - 1) \times 1 = 1004$ ns

Load-Use暂停：$1000 \times 20\% = 200$ 条Load指令，每条后面暂停1周期

实际执行时间 = $1004 + 200 = 1204$ ns

非流水线执行时间 = $1000 \times 5 = 5000$ ns

实际加速比 = $5000 / 1204 = 4.15$

理想加速比 = $5000 / 1004 = 4.98$

性能损失 = $(4.98 - 4.15) / 4.98 = 16.7\%$

## 19.8 本章常见误区与易错点

1. **流水线执行时间公式**：$T = (k + n - 1)\Delta t$，不是 $n \times k \times \Delta t$。第一个指令需要 $k$ 个周期，之后每条指令只需1个周期。

2. **时钟周期由最慢的段决定**：如果各段时间不同，时钟周期 = $\max$（各段时间）。

3. **加速比的分母是非流水线执行时间**：不是流水线执行时间。

4. **效率 = 加速比 / 段数**：不要忘记除以段数。

5. **指令数越多，性能越接近理想值**：少量指令时流水线的效果不明显。

## 19.9 本章总结

### 关键公式

| 公式 | 含义 |
|------|------|
| $T = (k + n - 1)\Delta t$ | 流水线执行时间 |
| $TP = n / T$ | 吞吐率 |
| $S = nk / (k + n - 1)$ | 加速比 |
| $E = S / k$ | 效率 |
| $\Delta t = \max(t_1, t_2, ..., t_k)$ | 时钟周期 |

### 性能指标极限值

| 指标 | 公式 | 极限值（$n \to \infty$） |
|------|------|------------------------|
| 吞吐率 | $n/(k+n-1)\Delta t$ | $1/\Delta t$ |
| 加速比 | $nk/(k+n-1)$ | $k$ |
| 效率 | $n/(k+n-1)$ | $1$ |

## 19.11 深入理解流水线

### 19.11.1 流水线的原理详解

**工厂流水线类比**：

假设汽车装配线有4个工位：
1. 安装底盘（1小时）
2. 安装发动机（1小时）
3. 安装车身（1小时）
4. 喷漆（1小时）

**非流水线**：每辆车需要4小时，4辆车需要16小时。

**流水线**：第1辆车4小时，之后每小时完成1辆，4辆车只需7小时。

**指令流水线类比**：

5级流水线：
1. 取指（IF）：从存储器取出指令
2. 译码（ID）：译码操作码，读寄存器
3. 执行（EX）：ALU运算或地址计算
4. 访存（MEM）：读写存储器
5. 写回（WB）：结果写回寄存器

### 19.11.2 时空图详解

**时空图的绘制方法**：
- 横轴：时间（时钟周期 T1, T2, T3, ...）
- 纵轴：空间（流水线各段 IF, ID, EX, MEM, WB）
- 每个格子表示某个指令在某个时钟周期处于哪个阶段

**【例 19-4】** 绘制5级流水线执行5条指令的时空图。

```
阶段  | T1  T2  T3  T4  T5  T6  T7  T8  T9
WB    |         I1  I2  I3  I4  I5
MEM   |     I1  I2  I3  I4  I5
EX    | I1  I2  I3  I4  I5
ID    | I1  I2  I3  I4  I5
IF    | I1  I2  I3  I4  I5
```

**分析**：
- 第1条指令在T1~T5完成
- 第2条指令在T2~T6完成
- ...
- 第5条指令在T5~T9完成
- 总共需要9个时钟周期

### 19.11.3 流水线的性能详解

**吞吐率**（Throughput）：

$$
TP = \frac{n}{(k + n - 1) \Delta t}
$$

**含义**：单位时间内完成的指令数。

**加速比**（Speedup）：

$$
S = \frac{nk}{k + n - 1}
$$

**含义**：非流水线执行时间与流水线执行时间的比值。

**效率**（Efficiency）：

$$
E = \frac{S}{k} = \frac{n}{k + n - 1}
$$

**含义**：流水线各段的实际使用时间占比。

**【例 19-5】** 5级流水线，时钟周期100ns，执行100条指令。求吞吐率、加速比、效率。

**解答**：

流水线执行时间 = $(5 + 100 - 1) \times 100 = 10400$ ns

非流水线执行时间 = $100 \times 5 \times 100 = 50000$ ns

吞吐率 = $100 / 10400\text{ns} = 9.62 \times 10^6$ 条/秒

加速比 = $50000 / 10400 = 4.81$

效率 = $4.81 / 5 = 96.2\%$

## 19.12 流水线的深入分析

### 19.12.1 流水线的限制因素

**结构冒险**：多条指令同时需要同一资源。

**数据冒险**：后续指令需要前面指令的结果。

**控制冒险**：分支指令改变程序执行顺序。

### 19.12.2 流水线的深度

**流水线深度**：流水线的阶段数。

| 深度 | 特点 | 典型 |
|------|------|------|
| 浅（3~5级） | 简单，每级时间长 | 早期处理器 |
| 中（8~15级） | 平衡 | 现代处理器 |
| 深（20+级） | 复杂，每级时间短 | Pentium 4 |

**权衡**：
- 更深的流水线 → 更高的时钟频率
- 更深的流水线 → 更多的冒险
- 更深的流水线 → 更大的功耗

### 19.12.3 流水线的利用率

**理想情况**：每个时钟周期都有一条指令完成。

**实际情况**：由于冒险和停顿，流水线利用率低于100%。

$$
\text{利用率} = \frac{\text{实际吞吐率}}{\text{理想吞吐率}} \times 100\%
$$

## 19.13 运算流水线详解

### 19.13.1 浮点加法流水线

浮点加法可以分为4个阶段：

```
对阶 → 尾数加 → 规格化 → 舍入
```

**每个阶段**：
- 对阶：比较阶码，小阶向大阶看齐
- 尾数加：尾数相加
- 规格化：调整结果使尾数在规定范围内
- 舍入：处理精度丢失

### 19.13.2 乘法流水线

乘法可以分为3个阶段：

```
部分积生成 → 部分积累加 → 结果规范化
```

### 19.13.3 运算流水线的性能

**运算流水线的吞吐率**：

$$
TP = \frac{1}{\Delta t}
$$

每个时钟周期完成一次运算。

## 19.17 流水线寄存器的硬件结构与功能

### 19.17.1 为什么需要流水线寄存器

在非流水线设计中，指令的执行是串行的，中间结果直接在组合逻辑中传递。而在流水线设计中，多条指令同时处于不同阶段执行，必须在相邻两级之间设置**流水线寄存器**（Pipeline Register），以实现以下目的：

1. **隔离各级**：防止不同指令之间的数据相互干扰
2. **暂存中间结果**：在时钟边沿到来时，将当前级的输出锁存到寄存器中
3. **同步控制**：确保所有信号在同一时钟边沿更新，维持流水线节奏

> [!NOTE]
> 流水线寄存器是流水线能否正确工作的**硬件基础**。没有流水线寄存器，流水线就无法实现指令间的隔离，会导致数据冒险和控制冒险无法管理。

### 19.17.2 四级流水线寄存器的命名与位置

经典五级流水线包含四个流水线寄存器，位于相邻两级之间：

```
┌──────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌──────┐
│  IF  │→│ IF/ID   │→│  ID    │→│ ID/EX   │→│  EX    │→│EX/MEM│
│ 取指 │  │ 流水线  │  │ 译码   │  │ 流水线  │  │ 执行   │  │流水线 │
│      │  │ 寄存器  │  │       │  │ 寄存器  │  │       │  │寄存器 │
└──────┘  └────────┘  └────────┘  └────────┘  └────────┘  └──────┘
                                                              │
                                                              ↓
┌──────┐  ┌────────┐  ┌──────┐
│  WB  │←│ MEM/WB  │←│ MEM  │
│ 写回  │  │ 流水线  │  │ 访存  │
│      │  │ 寄存器  │  │      │
└──────┘  └────────┘  └──────┘
```

### 19.17.3 各流水线寄存器存储的字段

#### IF/ID 寄存器

| 字段 | 位宽 | 说明 |
|------|------|------|
| `Instr` | 32 bit | 从指令存储器取出的指令 |
| `PC+4` | 32 bit | 下一条指令地址（用于计算分支目标） |

#### ID/EX 寄存器

| 字段 | 位宽 | 说明 |
|------|------|------|
| `PC+4` | 32 bit | 传递过来的 PC+4 |
| `ReadData1` | 32 bit | 寄存器堆读端口 1 的数据 |
| `ReadData2` | 32 bit | 寄存器堆读端口 2 的数据 |
| `SignExtImm` | 32 bit | 符号扩展后的立即数 |
| `Rs` | 5 bit | 源寄存器 1 编号 |
| `Rt` | 5 bit | 源寄存器 2 编号 |
| `Rd` | 5 bit | 目的寄存器编号 |
| `Funct` | 6 bit | 功能码（R-type 指令） |
| `Control Signals` | — | EX、MEM、WB 级的控制信号 |

#### EX/MEM 寄存器

| 字段 | 位宽 | 说明 |
|------|------|------|
| `ALUResult` | 32 bit | ALU 运算结果 |
| `WriteData` | 32 bit | 要写入数据存储器的数据（store 指令） |
| `WriteReg` | 5 bit | 写回目标寄存器编号 |
| `Zero` | 1 bit | ALU 零标志（用于分支判断） |
| `BranchTarget` | 32 bit | 分支目标地址 |
| `Control Signals` | — | MEM、WB 级的控制信号 |

#### MEM/WB 寄存器

| 字段 | 位宽 | 说明 |
|------|------|------|
| `ALUResult` | 32 bit | ALU 运算结果 |
| `ReadData` | 32 bit | 从数据存储器读出的数据（load 指令） |
| `WriteReg` | 5 bit | 写回目标寄存器编号 |
| `Control Signals` | — | WB 级的控制信号 |

### 19.17.4 流水线寄存器的时钟行为

```
         ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐
CLK      │   │   │   │   │   │   │   │   │   │
    ─────┘   └───┘   └───┘   └───┘   └───┘   └───
         ↑       ↑       ↑       ↑       ↑
       采样边沿  采样边沿  采样边沿  采样边沿  采样边沿
```

- **建立时间（Setup Time）**：数据必须在时钟上升沿到来前 $t_{setup}$ 时间内稳定
- **保持时间（Hold Time）**：数据必须在时钟上升沿后保持稳定 $t_{hold}$ 时间
- **时钟到输出延迟（$t_{clk \to q}$）**：时钟边沿后，寄存器输出更新所需的时间

> [!TIP]
> **建立时间约束**决定了流水线的最小时钟周期：
> $$T_{min} = t_{clk \to q} + t_{最长组合逻辑级} + t_{setup}$$
> 这是流水线设计中**最关键的时序约束**之一。

### 19.17.5 流水线寄存器的设计要点

1. **宽度**：根据所经过级的信号总量确定，通常为 100~200 位
2. **写入控制**：所有寄存器在每个时钟边沿都写入（正常情况）；遇到 stall 时需要**冻结**
3. **清零控制**：遇到 flush（冲刷）时，将寄存器内容清零，插入气泡（NOP）

```
流水线寄存器内部结构示意：
┌─────────────────────────────────────┐
│  数据字段: [Instr | PC+4 | Data1 |  │
│            Data2 | Imm | Rs | Rt |  │
│            Rd | Funct | ...]        │
│                                     │
│  控制字段: [EX_ctrl | MEM_ctrl |    │
│            WB_ctrl]                 │
│                                     │
│  控制信号:                          │
│    Write Enable (冻结/使能)         │
│    Flush (冲刷/清零)               │
│    CLK (时钟输入)                   │
└─────────────────────────────────────┘
```

> [!NOTE]
> 在考试中，流水线寄存器的字段大小经常作为计算题出现。需要根据具体指令集（如 MIPS）的字段宽度精确计算每个流水线寄存器的总位数。

## 19.18 流水线控制逻辑的设计

### 19.18.1 控制信号概述

流水线处理器的控制逻辑与单周期处理器类似，但控制信号需要**随指令一起在流水线中传播**。每个控制信号只在特定的级起作用：

| 控制信号 | 作用级 | 功能 |
|----------|--------|------|
| `RegDst` | EX | 选择写回寄存器（Rd vs Rt） |
| `ALUSrc` | EX | 选择 ALU 第二操作数（寄存器 vs 立即数） |
| `ALUOp` | EX | ALU 操作类型编码 |
| `Branch` | MEM | 是否为分支指令 |
| `MemRead` | MEM | 是否读数据存储器 |
| `MemWrite` | MEM | 是否写数据存储器 |
| `MemtoReg` | WB | 写回数据来源（ALU结果 vs 存储器数据） |
| `RegWrite` | WB | 是否写寄存器堆 |

### 19.18.2 控制信号的传播方式

控制信号在 **ID 级（译码级）** 由控制单元生成，然后随指令数据一起写入 ID/EX 流水线寄存器。后续各级从流水线寄存器中读取属于自己的控制信号：

```
控制信号传播流程：

ID 级 (控制单元生成)
    │
    ▼
┌─────────────────────────────────┐
│         ID/EX 寄存器            │
│  ┌─────┐ ┌──────┐ ┌──────┐    │
│  │EX   │ │MEM   │ │WB    │    │
│  │ctrl │ │ctrl  │ │ctrl  │    │
│  └──┬──┘ └──┬───┘ └──┬───┘    │
└─────┼───────┼────────┼────────┘
      │       │        │
      ▼       ▼        ▼
   EX 级    MEM 级    WB 级
  (使用EX   (使用MEM  (使用WB
   控制信号)  控制信号)  控制信号)
      │       │        │
      ▼       ▼        │
┌─────────────────┐    │
│  EX/MEM 寄存器  │    │
│  ┌──────┐┌────┐│    │
│  │MEM   ││WB  ││    │
│  │ctrl  ││ctrl││    │
│  └──┬───┘└─┬──┘│    │
└─────┼──────┼───┘    │
      │      │        │
      ▼      ▼        │
   MEM 级  WB 级      │

      ▼      │        │

│  ┌─────┐   │        │
│  │WB   │   │        │
│  │ctrl │   │        │
│  └──┬──┘   │        │
└─────┼──────┘        │
      ▼               │
    WB 级 ─────────────┘
```

### 19.18.3 各类指令的控制信号

| 指令 | RegDst | ALUSrc | ALUOp | Branch | MemRead | MemWrite | MemtoReg | RegWrite |
|------|--------|--------|-------|--------|---------|----------|----------|----------|
| R-type | 1 | 0 | Funct | 0 | 0 | 0 | 0 | 1 |
| lw | 0 | 1 | Add | 0 | 1 | 0 | 1 | 1 |
| sw | × | 1 | Add | 0 | 0 | 1 | × | 0 |
| beq | × | 0 | Sub | 1 | 0 | 0 | × | 0 |
| addi | 0 | 1 | Add | 0 | 0 | 0 | 0 | 1 |

> [!TIP]
> **考试高频考点**：「×」表示不关心（don't care），即该控制信号的值不影响指令执行结果。在考试中，填写控制信号表时，注意区分 0、1 和 ×。

### 19.18.4 冒险对控制信号的影响

当发生冒险时，需要对控制信号进行特殊处理：

1. **数据冒险（stall）**：将 ID/EX 寄存器的控制信号清零，相当于插入 NOP
2. **控制冒险（flush）**：将 IF/ID 和 ID/EX 寄存器冲刷
3. **数据冒险（forwarding）**：控制信号不变，但需要额外的 MUX 选择转发数据

```
Stall 时控制信号处理：

正常路径：ID ──→ ID/EX ──→ EX ──→ ...
                      │
Stall 时：             ▼
              ┌──────────────┐
              │ ID/EX 控制信号 │
              │  全部清零      │
              │  (插入气泡)    │
              └──────────────┘
                      │
                      ▼
              EX 级执行 NOP（什么都不做）
```

### 19.18.5 控制信号的设计决策

> [!NOTE]
> 流水线控制信号设计有两种策略：
>
> 1. **集中式控制**：控制单元在 ID 级集中生成所有信号，然后传播 → 简单但延迟大
> 2. **分布式控制**：每级有自己的局部控制单元 → 复杂但延迟小
>
> 经典五级流水线通常采用**集中式控制**，因为 ID 级的组合逻辑延迟不是瓶颈。

## 19.19 五级经典流水线每级的详细硬件组成

### 19.19.1 IF 级（取指级）

**组成部件**：
- **PC 寄存器**：存储当前指令地址，每个时钟周期更新
- **指令存储器（Instruction Memory）**：以 PC 为地址读取 32 位指令
- **加法器**：计算 PC+4

```
IF 级数据通路：
                ┌────────┐
          ┌─────│   PC   │─────┐
          │     └────────┘     │
          │          │         │
          │          ▼         │
          │   ┌────────────┐   │
          │   │ 指令存储器  │   │
          │   │  (I-Mem)   │   │
          │   └─────┬──────┘   │
          │         │          │
          │         ▼          │
          │    ┌─────────┐     │
          ├────│  加法器  │     │
          │    │ (PC + 4) │     │
          │    └────┬────┘     │
          │         │          │
          └─────────┼──────────┘
                    │
                    ▼
             ┌─────────────┐
             │  IF/ID 寄存器 │
             │ [Instr|PC+4] │
             └─────────────┘
```

**关键时序**：PC 更新 → 指令存储器读取 → 写入 IF/ID 寄存器，全部在一个时钟周期内完成。

### 19.19.2 ID 级（译码级）

**组成部件**：
- **寄存器堆（Register File）**：两个读端口，读取 Rs 和 Rt 的数据
- **符号扩展单元（Sign Extend）**：将 16 位立即数扩展为 32 位
- **控制单元（Control Unit）**：根据指令 opcode/funct 生成控制信号
- **分支目标加法器**：计算分支目标地址 PC+4 + SignExt(Imm)<<2

```
ID 级数据通路：
    ┌─────────────┐
    │  IF/ID 寄存器 │
    │ [Instr|PC+4] │
    └──────┬──────┘
           │
    ┌──────┼──────────────────┐
    │      │                  │
    ▼      ▼                  ▼
┌───────┐ ┌──────────┐  ┌──────────┐
│ 控制  │ │ 寄存器堆  │  │ 符号扩展  │
│ 单元  │ │ (RF)     │  │(SignExt) │
└───────┘ │ Rd1 │ Rd2│  └────┬─────┘
    │     └──┬─────┬┘       │
    │        │     │         │
    │        ▼     ▼         ▼
    │  ┌─────────────────────────┐
    │  │       ID/EX 寄存器       │
    └─→│ [Ctrl|PC+4|Rd1|Rd2|Imm| │
       │  Rs|Rt|Rd|Funct]        │
       └─────────────────────────┘
```

### 19.19.3 EX 级（执行级）

**组成部件**：
- **ALU**：执行算术逻辑运算
- **ALU 输入 MUX**：选择第二操作数（寄存器数据 vs 立即数）
- **ALU 控制单元**：根据 ALUOp 和 funct 生成 ALU 操作码
- **转发 MUX**（Forwarding MUX）：解决数据冒险

```
EX 级数据通路：
    ┌─────────────────────────────────┐
    │          ID/EX 寄存器            │
    │ [PC+4|Rd1|Rd2|Imm|Rs|Rt|Rd|Fn] │
    └────┬────┬────┬────┬──┬──┬──┬───┘
         │    │    │    │  │  │  │
         │    │    │    │  │  │  │
         │    │    ▼    │  │  │  │
         │    │ ┌─────┐ │  │  │  │
         │    │ │ MUX │←┼──┤  │  │  ← ALUSrc
         │    │ └──┬──┘ │  │  │  │
         │    │    │    │  │  │  │
         │    ▼    ▼    │  │  │  │
         │  ┌─────────┐ │  │  │  │
         │  │   ALU   │ │  │  │  │
         │  └────┬────┘ │  │  │  │
         │       │      │  │  │  │
         │    Zero│      │  │  │  │
         │       ▼      │  ▼  │  │
         │  ┌─────────┐ │MUX │  │  ← RegDst
         │  │  加法器  │ │    │  │
         │  │(分支地址)│ └──┬─┘  │
         │  └────┬────┘    │    │

```

## 19.20 流水线的性能瓶颈分析

### 19.20.1 流水线 CPI 分析

理想流水线的 CPI = 1，但实际中由于各种开销，CPI 会增大：

$$CPI_{pipeline} = CPI_{ideal} + CPI_{stall} + CPI_{flush}$$

其中：

$$CPI_{ideal} = 1$$

$$CPI_{stall} = \sum_{i} (\text{第 } i \text{ 类冒险的频率} \times \text{停顿周期数})$$

$$CPI_{flush} = \text{分支指令比例} \times \text{分支代价（周期数）}$$

**示例计算**：

假设某程序中：
- Load 指令占 25%，其中 40% 产生数据冒险，每次停顿 1 个周期
- 分支指令占 15%，分支预测准确率 80%，预测错误代价 2 个周期

$$CPI = 1 + (0.25 \times 0.40 \times 1) + (0.15 \times 0.20 \times 2) = 1 + 0.10 + 0.06 = 1.16$$

### 19.20.2 Amdahl 定律在流水线中的应用

Amdahl 定律：

$$S = \frac{1}{(1 - f) + \frac{f}{k}}$$

其中 $f$ 为可流水化的比例，$k$ 为流水线级数。

**流水线深度的边际收益分析**：

| 流水线级数 $k$ | 理想加速比 | 实际加速比（考虑开销） | 边际收益 |
|----------------|-----------|----------------------|----------|
| 1（无流水线） | 1 | 1 | — |
| 5（经典） | 5 | ~3.5~4.0 | 高 |
| 10 | 10 | ~5.0~6.0 | 中 |
| 15 | 15 | ~5.5~6.5 | 低 |
| 20 | 20 | ~5.5~6.5 | 极低 |

> [!NOTE]
> 当流水线级数增加到一定程度后，由于流水线寄存器延迟、冒险增加等因素，**实际加速比趋于饱和**。这就是为什么现代处理器的流水线深度通常在 10~20 级之间，而不是无限增加。

### 19.20.3 流水线深度与频率的关系

最小时钟周期受限于最慢的一级：

$$T_{clk} = \max(T_{IF}, T_{ID}, T_{EX}, T_{MEM}, T_{WB}) + t_{reg}$$

其中 $t_{reg}$ 为流水线寄存器的延迟。

**增加级数的影响**：

```
频率提升 vs 流水线深度：

频率(MHz)
    │
6000│                              ┌──── 饱和区
    │                         ┌────┘
4000│                    ┌────┘
    │               ┌────┘
2000│          ┌────┘
    │     ┌────┘
    │ ┌───┘
    └─┴──────────────────────────────── 流水线级数
      1    5    10    15    20    25
```

**深度流水线的代价**：

| 因素 | 影响 |
|------|------|
| 流水线寄存器延迟占比增大 | 每级有效计算时间减少 |
| 分支预测错误代价增大 | 需要冲刷更多级 |
| 数据冒险频率增加 | 相关指令间间隔变短 |
| 功耗增加 | 更多寄存器、更多级间缓冲 |
| 设计复杂度增加 | 验证和调试难度上升 |

### 19.20.4 功耗考虑

流水线对功耗的影响主要体现在：

1. **动态功耗**：流水线寄存器每个时钟周期都在翻转，增加动态功耗
   $$P_{dynamic} = \alpha C V_{dd}^2 f$$
   其中 $\alpha$ 为翻转率，$C$ 为负载电容，$V_{dd}$ 为供电电压，$f$ 为时钟频率

2. **短路功耗**：流水线级数增加导致更多的短路电流

3. **泄漏功耗**：更多的晶体管意味着更大的静态功耗

> [!TIP]
> **考试常见问题**：「增加流水线级数一定能提高性能吗？」答案是否定的。需要综合考虑冒险增加、功耗增大、设计复杂度等因素。当级数过多时，实际性能可能反而下降。

## 19.21 非线性流水线的调度（详细）

### 19.21.1 预约表分析

**预约表**（Reservation Table）是非线性流水线调度的基础工具。它展示了单条指令在各功能段上的使用时间。

**示例**：某 4 段非线性流水线的预约表

| 功能段 \ 时间 | $t_1$ | $t_2$ | $t_3$ | $t_4$ | $t_5$ | $t_6$ | $t_7$ |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| $S_1$ | ✓ | | | | | | ✓ |
| $S_2$ | | ✓ | | | ✓ | | |
| $S_3$ | | | ✓ | | | | |
| $S_4$ | | | | ✓ | | | |

解读：一条指令在 $t_1$ 时刻使用 $S_1$，$t_2$ 时刻使用 $S_2$，依此类推。

### 19.21.2 禁止间隔（Forbidden Latency）

**禁止间隔**是指两条相邻指令之间**不能**使用的时间间隔，否则会导致功能段冲突。

**推导方法**：观察预约表中每一行，找出同一行中有两个 ✓ 的所有时间差。

对于上述预约表：
- $S_1$ 行：✓ 在 $t_1, t_7$ → 间隔 $7 - 1 = 6$
- $S_2$ 行：✓ 在 $t_2, t_5$ → 间隔 $5 - 2 = 3$
- $S_3$ 行：只有 1 个 ✓ → 无禁止间隔
- $S_4$ 行：只有 1 个 ✓ → 无禁止间隔

**禁止间隔集合**：$F = \{3, 6\}$

> [!NOTE]
> 禁止间隔的含义：如果两条指令的启动间隔恰好等于禁止间隔中的某个值，就会发生功能段冲突（collision）。

### 19.21.3 禁止向量（Collision Vector）

禁止向量是禁止间隔的二进制表示。设预约表的最小禁止间隔为 $f_{min}$，最大禁止间隔为 $f_{max}$。

禁止向量的位数为 $f_{max}$，从低位到高位编号 $1, 2, \ldots, f_{max}$。如果间隔 $i$ 在禁止间隔集合中，则第 $i$ 位为 1，否则为 0。

对于 $F = \{3, 6\}$：

$$\text{Collision Vector} = C = (100100)_2$$

位置：$\underbrace{1}_{6}\ \underbrace{0}_{5}\ \underbrace{0}_{4}\ \underbrace{1}_{3}\ \underbrace{0}_{2}\ \underbrace{0}_{1}$

> [!TIP]
> **构造禁止向量的步骤**：
> 1. 找出预约表中每行所有 ✓ 对之间的时间差
> 2. 收集所有时间差得到禁止间隔集合 $F$
> 3. 构造 $f_{max}$ 位的二进制数，$F$ 中有的位置 1，没有的置 0

### 19.21.4 状态转移图

状态转移图描述了流水线调度的所有可能状态和转移。

**初始状态** = 禁止向量 $C = (100100)_2$

**状态转移规则**：
1. 对于当前状态 $C$，找出所有可以安全启动新指令的时间间隔（$C$ 中为 0 的位）
2. 对于每个可启动间隔 $i$：
   - 将 $C$ 右移 $i$ 位
   - 与原始禁止向量 $C_0$ 做 OR 运算
   - 得到新状态：$C_{new} = (C \gg i) \lor C_0$

**计算过程**：

初始状态 $C_0 = (100100)_2$

**从 $C_0$ 出发**：

右移 1 位后 OR：$C_1 = (010010) \lor (100100) = (110110)_2$ ← 间隔 1 可行
右移 2 位后 OR：$C_2 = (001001) \lor (100100) = (101101)_2$ ← 间隔 2 可行
右移 4 位后 OR：$C_4 = (000100) \lor (100100) = (100100)_2$ ← 间隔 4 可行（回到初始状态）
右移 5 位后 OR：$C_5 = (000010) \lor (100100) = (100110)_2$ ← 间隔 5 可行

```
状态转移图：

     ┌─────────────────┐
     │  100100 (S₀)    │
     │   初始状态       │
     └──┬──┬──┬──┬─────┘
        │  │  │  │
      ① │  │  │  │ ⑤
        │  │  │  │
        ▼  │  │  ▼
  110110   │  │  100110 (S₃)
   (S₁)    │  │
    │   ②  │  │ ④
    │      │  │
    │      ▼  │
    │  101101 │
    │   (S₂) │
    │        │
    └────────┘  (④ 从 S₀ 回到 S₀)
```

### 19.21.5 最小平均延迟计算

**平均启动间隔**（Average Latency）：

$$\bar{L} = \frac{\text{一个调度周期的总时间}}{\text{一个调度周期内启动的指令数}}$$

**目标**：找到一个**恒定循环调度**（Constant-Cycle Schedule），使得平均启动间隔最小。

**分析各状态的可行间隔**：

| 状态 | 可行间隔 | 最小间隔 |
|------|----------|----------|
| $S_0 = (100100)$ | 1, 2, 4, 5 | 1 |
| $S_1 = (110110)$ | 4 | 4 |
| $S_2 = (101101)$ | 3 | 3 |
| $S_3 = (100110)$ | 1, 4 | 1 |

**恒定循环调度**：

选择在每个状态都使用最小可行间隔：

$$S_0 \xrightarrow{1} S_1 \xrightarrow{4} S_0$$

周期长度 = $1 + 4 = 5$，启动 2 条指令

$$\bar{L} = \frac{5}{2} = 2.5$$

**另一种调度**：

$$S_0 \xrightarrow{2} S_2 \xrightarrow{3} S_0$$

周期长度 = $2 + 3 = 5$，启动 2 条指令

$$\bar{L} = \frac{5}{2} = 2.5$$

**最优调度**：

$$S_0 \xrightarrow{4} S_0$$

周期长度 = 4，启动 1 条指令

$$\bar{L} = \frac{4}{1} = 4$$

平均延迟 = 4，但吞吐率低。实际上 $\bar{L} = 2.5$ 的调度更优。

### 19.21.6 完整例题

**题目**：某非线性流水线有 3 个功能段，预约表如下：

| 功能段 \ 时间 | $t_1$ | $t_2$ | $t_3$ | $t_4$ | $t_5$ |
|:---:|:---:|:---:|:---:|:---:|:---:|
| $S_1$ | ✓ | | | | ✓ |
| $S_2$ | | ✓ | ✓ | | |
| $S_3$ | | | | ✓ | |

**步骤 1：找禁止间隔**
- $S_1$：$t_5 - t_1 = 4$
- $S_2$：$t_3 - t_2 = 1$
- $S_3$：无

禁止间隔集合：$F = \{1, 4\}$

**步骤 2：构造禁止向量**

$$C = (10010)_2$$

（注意：最大禁止间隔为 4，所以需要 4 位；但按惯例用 $f_{max}$ 位）

实际上 $C = (10010)_2$（5 位，位置 1~5）

$$C = (10010)_2$$

位置：$\underbrace{1}_{5}\ \underbrace{0}_{4}\ \underbrace{0}_{3}\ \underbrace{1}_{2}\ \underbrace{0}_{1}$

**步骤 3：状态转移**

$C_0 = (10010)_2$

从 $C_0$ 可行的间隔：位置 1（0），位置 3（0），位置 4（0）

- 间隔 1：$C_1 = (01001) \lor (10010) = (11011)_2$
- 间隔 3：$C_3 = (00100) \lor (10010) = (10110)_2$
- 间隔 4：$C_4 = (00010) \lor (10010) = (10010)_2 = C_0$（回到初始）

**步骤 4：画状态转移图**

```
      ┌──────────────┐
      │ 10010 (S₀)  │
      └─┬───┬───┬────┘
        │   │   │
      ① │ ③ │ ④ │
        │   │   │
        ▼   │   │
    11011   │   │
     (S₁)   ▼   │
        │ 10110 │
        │ (S₂)  │
        │   │   │
        └───┴───┘  (④ 回到 S₀)
```

**步骤 5：找最优调度**

- $S_0 \xrightarrow{4} S_0$：$\bar{L} = 4$
- $S_0 \xrightarrow{1} S_1 \xrightarrow{?} S_0$：$S_1$ 只有间隔 4 可行，$S_1 \xrightarrow{4} C' = (00001) \lor (10010) = (10011)_2$，继续分析...
- $S_0 \xrightarrow{3} S_2 \xrightarrow{?} S_0$：$S_2$ 的可行间隔分析...

最终最优调度：$S_0 \xrightarrow{3} S_2 \xrightarrow{?} \ldots$

需要检查 $S_2 = (10110)$ 的可行间隔（位置 1 和 3 为 0）：
- 间隔 1：$(01011) \lor (10010) = (11011) = S_1$ → 然后 $S_1$ 只有间隔 4 → 回到 $S_0$
  调度：$3, 1, 4$，周期 $= 3 + 1 + 4 = 8$，3 条指令，$\bar{L} = 8/3 ≈ 2.67$

最优恒定调度：$S_0 \xrightarrow{3} S_2 \xrightarrow{3} S_0$

检查：$S_2 = (10110)$，间隔 3 → $(00010) \lor (10010) = (10010) = C_0$ ✓

$\bar{L} = \frac{3 + 3}{2} = 3$

> [!TIP]
> **非线性流水线调度解题步骤**：
> 1. 从预约表找禁止间隔
> 2. 构造禁止向量
> 3. 画状态转移图
> 4. 在图中找包含所有状态的**最小平均延迟循环**
> 5. 答案就是这个循环的平均延迟

## 19.22 流水线与数据通路的关系

### 19.22.1 三种数据通路实现方式

#### 单周期数据通路（Single-Cycle）

每条指令在一个时钟周期内完成，时钟周期由**最慢的指令**决定。

$$T_{clk} = T_{lw} = T_{IF} + T_{ID} + T_{EX} + T_{MEM} + T_{WB}$$

#### 多周期数据通路（Multi-Cycle）

每条指令被分解为多个步骤，每步一个时钟周期。不同指令使用不同数量的周期。

$$T_{clk} = \max(T_{IF}, T_{ID}, T_{EX}, T_{MEM}, T_{WB})$$

#### 流水线数据通路（Pipelined）

多条指令重叠执行，每条指令仍需多个周期完成，但吞吐率大幅提升。

$$T_{clk} = \max(T_{IF}, T_{ID}, T_{EX}, T_{MEM}, T_{WB}) + t_{reg}$$

### 19.22.2 三种方式的详细对比

| 特性 | 单周期 | 多周期 | 流水线 |
|------|--------|--------|--------|
| **时钟周期** | 最慢指令的延迟 | 最慢级的延迟 | 最慢级的延迟 + 寄存器延迟 |
| **CPI** | 1 | 1.0~5.0（指令相关） | 理想 1.0，实际 1.1~1.5 |
| **指令吞吐率** | 低 | 中 | 高 |
| **硬件利用率** | 低（每周期大部分空闲） | 中 | 高 |
| **控制复杂度** | 简单 | 中等 | 复杂（冒险处理） |
| **功能单元数量** | 每类 1 个 | 每类 1 个 | 每类 1 个 |
| **寄存器数量** | 少 | 中（多周期状态寄存器） | 多（流水线寄存器） |
| **适用场景** | 教学/简单系统 | 中等性能需求 | 高性能通用处理器 |

### 19.22.3 从单周期到流水线的改造过程

**改造核心思想**：在单周期数据通路的各级之间**插入流水线寄存器**。

```
单周期数据通路：
┌────────────────────────────────────────────────────┐
│  PC → I-Mem → RF → ALU → D-Mem → RF(写)           │
│        (全部组合逻辑，一个周期完成)                   │
└────────────────────────────────────────────────────┘

改造步骤：
Step 1: 识别各级之间的自然分割点
        PC → [IF/ID] → RF → [ID/EX] → ALU → [EX/MEM] → D-Mem → [MEM/WB] → RF(写)

Step 2: 在每个分割点插入流水线寄存器
        ┌────┐ ┌────────┐ ┌────┐ ┌────────┐ ┌────┐ ┌────────┐ ┌────┐ ┌────────┐
        │ PC │→│ IF/ID   │→│ RF │→│ ID/EX  │→│ALU │→│ EX/MEM │→│DMem│→│ MEM/WB │→RF(写)
        └────┘ └────────┘ └────┘ └────────┘ └────┘ └────────┘ └────┘ └────────┘

Step 3: 调整功能单元的连接
        - IF 级：PC 只在 IF 级使用
        - ID 级：寄存器堆读在 ID 级，写在 WB 级（需要两个端口独立）
        - EX 级：ALU 只在 EX 级使用
        - MEM 级：数据存储器只在 MEM 级使用
        - WB 级：写回数据通过 MEM/WB 寄存器传回

Step 4: 添加冒险处理硬件
        - 数据冒险：转发单元 + 转发 MUX
        - 控制冒险：分支预测逻辑
        - 结构冒险：确保硬件资源不冲突
```

### 19.22.4 关键硬件修改

从单周期到流水线，需要以下关键修改：

| 修改项 | 原因 | 具体改动 |
|--------|------|----------|
| 插入流水线寄存器 | 隔离各级 | 在 IF/ID、ID/EX、EX/MEM、MEM/WB 处插入寄存器 |
| 分离寄存器堆读写 | 读在 ID，写在 WB | 写端口需在 WB 级独立操作 |
| 增加转发通路 | 解决数据冒险 | 从 EX/MEM、MEM/WB 到 EX 级的 MUX |
| 增加冒险检测单元 | 检测 load-use 冒险 | 比较 ID/EX 的 Rs/Rt 与 EX/MEM 的目标寄存器 |
| 增加分支处理逻辑 | 解决控制冒险 | 提前计算分支目标、分支预测 |
| PC 选择 MUX | 分支/jump 时选择正确 PC | 在 IF 级增加 MUX 选择 PC+4 或分支目标 |

### 19.22.5 流水线对硬件资源的要求

> [!NOTE]
> 流水线要求**每个功能单元在每个周期都被使用**（理想情况下），这与单周期设计形成鲜明对比。单周期中，功能单元每个周期只使用一次然后等待下一条指令；流水线中，功能单元每个周期都在处理不同指令的不同阶段。

**结构冒险的典型场景**：

```
场景：指令存储器和数据存储器共用一个存储器

周期1: lw 指令在 MEM 级访问数据存储器
周期1: 新指令在 IF 级访问指令存储器
       → 冲突！（单端口存储器无法同时读取）

解决方案 1：使用分离的 I-Cache 和 D-Cache（哈佛结构）
解决方案 2：使用双端口存储器
解决方案 3：在结构冒险时插入 stall
```

> [!TIP]
> **考试重点**：理解三种数据通路实现方式的区别是高频考点。特别是：
> 1. 时钟周期的计算公式
> 2. CPI 的对比
> 3. 从单周期到流水线需要做哪些硬件修改
> 4. 流水线引入的新问题（冒险）及其解决方案
