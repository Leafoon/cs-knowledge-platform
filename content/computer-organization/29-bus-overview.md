# 第二十九章 总线概述与总线仲裁

总线是连接计算机各部件的公共传输通道。本章讲解总线的分类、性能指标、总线结构和仲裁方式。**总线仲裁方式的对比是考研重点**。

## 29.1 总线的基本概念

### 29.1.1 什么是总线

**总线**（Bus）：多个部件共享的**公共传输通道**。

**特点**：**分时共享**——同一时刻只能有一个主设备使用总线发送数据，但可以有多个从设备接收数据。

### 29.1.2 总线的分类

**按数据传输格式**：

| 类型 | 特点 | 优点 | 缺点 |
|------|------|------|------|
| 并行总线 | 多位同时传输 | 速度快 | 线多、干扰大 |
| 串行总线 | 逐位传输 | 线少、距离远 | 速度慢 |

**按总线功能**：

| 类型 | 功能 | 说明 |
|------|------|------|
| 数据总线（DB） | 传输数据 | 宽度决定一次传输的数据量 |
| 地址总线（AB） | 传输地址 | 宽度决定寻址范围 |
| 控制总线（CB） | 传输控制信号 | R/W、CS、中断等 |

**按总线位置**：

| 类型 | 位置 | 典型 |
|------|------|------|
| 片内总线 | CPU内部 | 寄存器之间 |
| 系统总线 | CPU、主存、I/O之间 | 前端总线、内存总线 |
| I/O总线 | I/O设备之间 | PCI、PCIe |
| 通信总线 | 系统之间 | USB、以太网 |

## 29.2 总线的性能指标

### 29.2.1 总线宽度

一次传输的数据位数。

### 29.2.2 总线频率

总线的工作频率（MHz/GHz）。

### 29.2.3 总线带宽

单位时间内总线传输的数据量。

$$
\boxed{\text{总线带宽} = \text{总线宽度} \times \text{总线频率}}
$$

**【例 29-1】** 总线宽度64位，频率800MHz。

$$
\text{带宽} = 64 \times 800\text{MHz} = 51200\text{Mbit/s} = 6.4\text{GB/s}
$$

### 29.2.4 总线复用

同一组信号线在不同时刻传输不同类型的信息（如地址/数据复用）。

### 29.2.5 信号线数

总线中信号线的总数。

## 29.3 总线的结构

### 29.3.1 单总线结构

所有部件挂在一条总线上。

```
CPU ←→ 总线 ←→ 主存
               ←→ I/O设备1
               ←→ I/O设备2
```

- 优点：简单、易于扩展
- 缺点：带宽瓶颈、总线争用

### 29.3.2 双总线结构

```
CPU ←→ 主存总线 ←→ 主存
  ↓
I/O总线 ←→ I/O设备1
         ←→ I/O设备2
```

- 主存总线：CPU与主存之间
- I/O总线：CPU与I/O设备之间
- 通过通道或I/O处理器连接

### 29.3.3 三总线结构

```
CPU ←→ 局部总线 ←→ Cache
  ↓
系统总线 ←→ 主存
  ↓
I/O总线 ←→ I/O设备
```

- 局部总线：CPU与Cache之间
- 系统总线：连接主存
- I/O总线：连接I/O设备

### 29.3.4 现代多总线结构

```
CPU ←→ 前端总线(FSB) ←→ 北桥 ←→ 内存总线 ←→ 主存
                          ↓
                        PCI总线 ←→ 南桥 ←→ I/O总线
```

## 29.4 总线仲裁

### 29.4.1 为什么需要仲裁

总线上有多个**主设备**（CPU、DMA控制器等），同一时刻只能有一个主设备使用总线。仲裁机制决定谁获得总线使用权。

### 29.4.2 集中仲裁方式

<div data-component="BusArbitrationDemo"></div>

**一、链式查询（菊花链）**

```
BR（总线请求）→ 所有设备
BG（总线允许）→ 设备1 → 设备2 → 设备3 → ...
BS（总线忙）
```

**工作过程**：
1. 设备发出BR请求
2. 仲裁器发出BG信号
3. BG信号从设备1开始依次传递
4. 第一个有请求的设备获得总线，停止传递BG
5. 该设备置BS=1，开始传输

**优先级**：离仲裁器越近优先级越高。

**优点**：简单，只需3根线

**缺点**：
- 不公平（设备1总是优先）
- 对电路故障敏感（BG链断裂）
- 速度慢（BG串行传递）

**二、计数器定时查询**

```
BR（总线请求）→ 仲裁器
设备地址线（log2 n根）→ 所有设备
BS（总线忙）
```

**工作过程**：
1. 设备发出BR请求
2. 仲裁器从某个初始值开始计数
3. 设备地址线广播计数值
4. 有请求且匹配计数值的设备获得总线

**优先级**：
- 初始值固定 → 固定优先级
- 初始值为上次获得总线的设备+1 → 循环优先级（更公平）

**优点**：优先级可设置，较公平

**缺点**：需要额外的地址线，速度中等

**三、独立请求**

```
设备1: BR1, BG1
设备2: BR2, BG2
设备3: BR3, BG3
...
```

**工作过程**：
1. 每个设备有独立的BR和BG线
2. 仲裁器根据优先级直接选择
3. 直接向选中的设备发出BG信号

**优先级**：由仲裁器内部逻辑决定，可编程。

**优点**：速度快（并行判断），优先级灵活

**缺点**：线数多（2n+1根），扩展困难

### 29.4.3 分布式仲裁方式

**自举分布式仲裁**：

每个设备监听比自己优先级高的设备的BR线。如果高优先级设备没有请求，则自己获得总线。

**冲突检测**：

设备发送数据后监听总线，如果发现数据冲突则停止，随机等待后重试（类似以太网CSMA/CD）。

### 29.4.4 三种集中仲裁方式对比

| 方式 | 线数 | 优先级 | 公平性 | 速度 | 复杂度 |
|------|------|--------|--------|------|--------|
| 链式查询 | 3 | 固定（设备1最高） | 差 | 慢 | 低 |
| 计数器查询 | 3+log₂n | 可设置 | 中 | 中 | 中 |
| 独立请求 | 2n+1 | 可编程 | 好 | 快 | 高 |

## 29.5 本章典型例题

### 例题1：总线带宽计算

**题目**：某总线宽度32位，频率66MHz，求总线带宽。

**解答**：

$$
\text{带宽} = 32 \times 66\text{MHz} = 2112\text{Mbit/s} = 264\text{MB/s}
$$

### 例题2：仲裁方式分析

**题目**：某系统有8个设备，采用计数器定时查询仲裁。需要几根设备地址线？如果从设备4开始计数，设备0、2、5同时请求，哪个先获得总线？

**解答**：

- 设备地址线：$\log_2 8 = 3$ 根
- 从设备4开始计数：4, 5, 6, 7, 0, 1, 2, 3
- 设备4：无请求
- 设备5：有请求 → **设备5先获得总线**

### 例题3：仲裁方式对比

**题目**：比较链式查询和独立请求仲裁方式的优缺点。

**解答**：

| 特性 | 链式查询 | 独立请求 |
|------|---------|---------|
| 线数 | 3根 | 2n+1根 |
| 优先级 | 固定 | 可编程 |
| 公平性 | 差 | 好 |
| 速度 | 慢 | 快 |
| 复杂度 | 低 | 高 |

## 29.12 总线的物理结构

### 29.12.1 信号线分组

总线由多根信号线组成，按功能可分为以下几组：

| 信号线类别 | 功能 | 典型信号 | 数量 |
|-----------|------|---------|------|
| 数据线 | 传输数据 | D0~D63 | 与数据宽度相同 |
| 地址线 | 传输地址 | A0~A31 | 与地址宽度相同 |
| 控制线 | 传输控制信号 | R/W、CS、CLK、RST | 若干根 |
| 电源线 | 供电 | VCC、GND | 若干根 |
| 备用线 | 预留扩展 | — | 若干根 |

### 29.12.2 电气特性

总线信号的电气特性决定了总线的传输质量和速度：

| 参数 | 说明 | 典型值 |
|------|------|--------|
| 逻辑电平 | 高/低电平的电压范围 | TTL: 0~0.8V低, 2.4~5V高 |
| 驱动能力 | 能驱动的负载数 | 通常8~16个 |
| 传输延迟 | 信号从发送到接收的延迟 | 纳秒级 |
| 信号反射 | 阻抗不匹配导致的反射 | 需要终端电阻 |
| 串扰 | 相邻信号线之间的干扰 | 需要屏蔽或地线隔离 |

### 29.12.3 总线的物理连接

```
┌─────────┐                    ┌─────────┐
│  主设备   │═════════════════════│  从设备   │
│  (CPU)   │   数据线 D0~D63     │  (内存)   │
│          │   地址线 A0~A31     │          │
│          │   控制线 R/W,CS    │          │
│          │   时钟线 CLK       │          │
│          │   电源线 VCC,GND   │          │
└─────────┘                    └─────────┘
         │                          │
         └──────── 总线 ────────────┘
```

> [!NOTE]
> 并行总线的信号线之间需要地线隔离，以减少串扰。高速总线还需要终端匹配电阻来防止信号反射。

### 29.12.4 总线的拓扑结构

| 拓扑类型 | 结构 | 优点 | 缺点 |
|---------|------|------|------|
| 星型 | 所有设备连接到中心节点 | 故障隔离好 | 中心节点瓶颈 |
| 总线型 | 所有设备共享一条总线 | 简单、成本低 | 故障影响全局 |
| 环型 | 设备首尾相连成环 | 无冲突 | 一个故障影响全环 |
| 树型 | 层次化连接 | 可扩展性好 | 层间延迟 |

## 29.13 总线控制器的硬件结构

### 29.13.1 总线控制器的组成

总线控制器（Bus Controller）是管理总线仲裁和数据传输的核心硬件：

```
┌──────────────────────────────────────────┐
│              总线控制器                    │
│                                          │
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ 仲裁逻辑  │  │ 时序控制  │  │ 状态机  │ │
│  │          │  │          │  │        │ │
│  │ BR→优先级 │  │ CLK分频  │  │ IDLE   │ │
│  │ BG产生   │  │ 时序生成  │  │ ARBIT  │ │
│  └──────────┘  └──────────┘  │ TRANS  │ │
│                               └────────┘ │
│  ┌──────────┐  ┌──────────┐              │
│  │ 地址锁存  │  │ 数据缓冲  │              │
│  │          │  │          │              │
│  │ 锁存地址  │  │ 缓冲数据  │              │
│  └──────────┘  └──────────┘              │
└──────────────────────────────────────────┘
```

### 29.13.2 总线控制器的状态机

总线控制器的工作过程可以用状态机描述：

```
         BR有效
  ┌──────┐──────→┌──────────┐
  │ IDLE │       │ ARBITRATE │
  │ 空闲  │←──────│  仲裁中    │
  └──────┘ 传输完成└────┬─────┘
      ↑                │ 仲裁完成
      │                ↓
      │         ┌──────────┐
      └─────────│ TRANSFER  │
         传输完成│  传输中    │
                └──────────┘
```

| 状态 | 描述 | 动作 |
|------|------|------|
| IDLE | 总线空闲 | 监听BR信号 |
| ARBITRATE | 仲裁中 | 执行仲裁算法，产生BG信号 |
| TRANSFER | 传输中 | 控制数据传输时序 |

## 29.14 三种集中仲裁方式的详细电路实现

### 29.14.1 链式查询的电路实现

链式查询（Daisy Chain）仲裁的硬件电路：

```
                    ┌─────────────────────────────────────────┐
                    │            总线仲裁器                     │
  ┌─────┐          │  ┌───┐                                  │
  │设备1 │──BR1──→  │  │   │    BG                            │
  ├─────┤          │  │ OR├──→────→ 设备1 ──→ 设备2 ──→ 设备3 │
  │设备2 │──BR2──→  │  │   │        │          │          │   │
  ├─────┤          │  └───┘        ↓          ↓          ↓   │
  │设备3 │──BR3──→  │            BG1_in     BG2_in     BG3_in│
  └─────┘          │                                          │
                    │  ┌───┐                                  │
                    │  │   │←──BS─────────────────────────────│
                    │  │AND│                                  │
                    │  └───┘                                  │
                    └─────────────────────────────────────────┘
```

**链式查询的关键电路逻辑**：

每个设备内部的BG传递逻辑：

```
设备i内部：
BG_out = BG_in AND (NOT BRi)
BS信号 = BG_in AND BRi（本设备有请求且收到BG）
```

**工作时序**：

```
时钟    ─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─
        ─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─
BR      ─────────┐                   ┌───
                 └───────────────────┘
BG      ─────────┐
                 └───────────────────
BS      ─────────────────┐
                         └───────────
数据    ───────────────── │ ═════════ │ ──
                 ↑        ↑           ↑
              请求      授权        传输
```

### 29.14.2 计数器定时查询的电路实现

```
                    ┌──────────────────────────────────────┐
                    │            总线仲裁器                  │
  ┌─────┐          │  ┌───┐   ┌─────┐   ┌──────────────┐ │
  │设备0 │──BR0──→  │  │   │   │计数器│   │ 设备地址译码器│ │
  ├─────┤          │  │ OR├──→│     ├──→│              │ │
  │设备1 │──BR1──→  │  │   │   │     │   └──────┬───────┘ │
  ├─────┤          │  └───┘   └─────┘          │         │
  │设备2 │──BR2──→  │                        设备地址线    │
  ├─────┤          │                          (log₂n根)   │
  │设备3 │──BR3──→  │                                      │
  └─────┘          │  ┌───┐                              │
                    │  │   │←──BS────────────────────────│
                    │  │AND│                              │
                    │  └───┘                              │
                    └──────────────────────────────────────┘
```

**计数器查询的关键电路**：

```
仲裁器内部：
计数器模块：
  if (BR有效 AND BS=0) then
    计数值 → 设备地址线
    if (匹配设备有BR请求) then
      向该设备发送BG
      置BS=1
    else
      计数器+1
    end if
  end if

设备内部匹配逻辑：
  匹配 = (设备地址线 == 本设备地址)
  获得总线 = 匹配 AND BRi
```

**地址匹配电路（3-8译码器示例）**：

```
        A2  A1  A0      设备选择
        0   0   0   →   设备0
        0   0   1   →   设备1
        0   1   0   →   设备2
        0   1   1   →   设备3
        1   0   0   →   设备4
        1   0   1   →   设备5
        1   1   0   →   设备6
        1   1   1   →   设备7
```

### 29.14.3 独立请求的电路实现

```
                    ┌──────────────────────────────────────┐
                    │            总线仲裁器                  │
  ┌─────┐          │                                      │
  │设备0 │──BR0──→  │  ┌───────────────────────┐          │
  ├─────┤          │  │    优先级编码器         │          │
  │设备1 │──BR1──→  │  │                       │          │
  ├─────┤          │  │  BR3>BR2>BR1>BR0       │          │
  │设备2 │──BR2──→  │  │  (可编程优先级)        │          │
  ├─────┤          │  └───────────┬───────────┘          │
  │设备3 │──BR3──→  │              │                      │
  └─────┘          │  ┌───────────┴───────────┐          │
                    │  │    BG译码器            │          │
                    │  │                       │          │
                    │  │  BG0 ←── 选中设备0     │          │
                    │  │  BG1 ←── 选中设备1     │          │
                    │  │  BG2 ←── 选中设备2     │          │
                    │  │  BG3 ←── 选中设备3     │          │
                    │  └───────────────────────┘          │
                    └──────────────────────────────────────┘
```

**独立请求仲裁器的核心电路**：

```
优先级编码器逻辑（4设备示例）：

选中设备 = 优先级最高的有请求设备

if (BR3=1) then 选中=3
elsif (BR2=1) then 选中=2
elsif (BR1=1) then 选中=1
elsif (BR0=1) then 选中=0
else 无请求

BG译码逻辑：
BG0 = (选中==0) AND 仲裁有效
BG1 = (选中==1) AND 仲裁有效
BG2 = (选中==2) AND 仲裁有效
BG3 = (选中==3) AND 仲裁有效
```

**可编程优先级实现**：

```
优先级寄存器（4位）：
  PR[3:0] = 1010  → 设备3优先级最高，设备1次之

优先级比较逻辑：
  有效BR[i] = BRi AND PR[i]
  选中 = 有效BR中编号最大者
```

> [!TIP]
> 独立请求方式的仲裁器可以使用**可编程逻辑器件（PLD）**或**FPGA**实现，优先级可以通过软件配置寄存器来动态调整。

## 29.15 分布式仲裁的实现

### 29.15.1 自举分布式仲裁电路

```
设备0（最低优先级）：
  BR_总 = BR0
  获得总线 = BR0 AND (NOT BR1) AND (NOT BR2) AND (NOT BR3)

设备1：
  BR_总 = BR1
  获得总线 = BR1 AND (NOT BR2) AND (NOT BR3)

设备2：
  BR_总 = BR2
  获得总线 = BR2 AND (NOT BR3)

设备3（最高优先级）：
  BR_总 = BR3
  获得总线 = BR3
```

**自举分布式仲裁电路图**：

```
设备0  设备1  设备2  设备3
  │      │      │      │
  BR0    BR1    BR2    BR3
  │      │      │      │
  ├──────┼──────┼──────┤
  │      │      │      │
  ↓      ↓      ↓      ↓
┌──────────────────────────────┐
│  设备0: BR0·BR1'·BR2'·BR3'   │
│  设备1: BR1·BR2'·BR3'         │
│  设备2: BR2·BR3'              │
│  设备3: BR3                   │
└──────────────────────────────┘
```

### 29.15.2 冲突检测与CSMA/CD

**CSMA/CD（载波监听多路访问/冲突检测）工作流程**：

```
┌──────────┐
│ 监听总线  │
└────┬─────┘
     ↓
┌──────────┐    忙
│ 总线空闲？ │───→ 等待
└────┬─────┘
     │ 空闲
     ↓
┌──────────┐
│ 发送数据  │
└────┬─────┘
     ↓
┌──────────┐    冲突
│ 检测冲突？ │───→ 停止发送
└────┬─────┘      ↓
     │ 无冲突   ┌──────────┐
     ↓          │ 发送阻塞信号│
┌──────────┐    └────┬─────┘
│ 传输完成  │         ↓
└──────────┘    ┌──────────┐
                │ 随机退避   │
                │ (指数退避) │
                └────┬─────┘
                     ↓
                返回监听总线
```

**指数退避算法**：

$$
\text{等待时间} = \text{随机数} \times 2^n \times \text{基本退避时间}
$$

其中 $n$ 为冲突次数（有上限）。

## 29.16 总线的时序分析

### 29.16.1 总线传输时序参数

<div data-component="BusTimingDiagram"></div>

| 参数 | 符号 | 说明 |
|------|------|------|
| 建立时间 | $t_{setup}$ | 数据在时钟边沿前必须稳定的最小时间 |
| 保持时间 | $t_{hold}$ | 数据在时钟边沿后必须保持的最小时间 |
| 传输延迟 | $t_{delay}$ | 信号从发送端到接收端的传播延迟 |
| 时钟周期 | $T_{clk}$ | 一个完整时钟周期的时间 |
| 总线周期 | $T_{bus}$ | 一次完整总线传输的时间 |

### 29.16.2 同步总线时序分析

**读操作时序**：

```
时钟    ─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─
        ─┘ └─┘ └─┘ └─┘ └─┘ └─
        │ T1  │ T2  │ T3  │ T4  │
地址    ──X═══════════════════════
CS      ──┐                 ┌────
        ──┘                 └────
R/W     ──┐ (读)
        ──┘
数据    ──────────── X═══════════
                 ↑
              数据有效
```

**总线周期计算**：

$$
T_{bus} = T_{addr\_setup} + T_{access} + T_{data\_setup} + T_{hold}
$$

### 29.16.3 异步总线时序分析

**全互锁异步读操作时序**：

```
主设备  ──┐                                 ┌──
(REQ)   ──┘                                 └──
             ↑                               ↑
从设备  ────────┐                     ┌───────
(ACK)          └─────────────────────┘
                  ↑                 ↑
数据    ────────────── X══════════════──────
                    ↑              ↑
                 REQ→ACK延迟    ACK→REQ撤除
```

**异步传输时间计算**：

$$
T_{async} = t_{req\_to\_ack} + t_{ack\_to\_data} + t_{data\_valid} + t_{req\_remove}
$$

> [!NOTE]
> 异步总线的传输时间取决于握手延迟，通常比同步总线慢，但能适应不同速度的设备。

## 29.17 总线带宽与利用率计算

### 29.17.1 理论带宽与实际带宽

<div data-component="BusBandwidthCalc"></div>

**理论最大带宽**：

$$
BW_{理论} = W \times f
$$

其中 $W$ 为总线宽度（字节），$f$ 为总线频率。

**实际带宽**：

$$
BW_{实际} = BW_{理论} \times \eta
$$

其中 $\eta$ 为总线利用率。

### 29.17.2 总线利用率的影响因素

| 因素 | 影响 | 典型损失 |
|------|------|---------|
| 仲裁开销 | 仲裁需要额外时钟周期 | 1~5个周期 |
| 地址阶段 | 传输地址需要时间 | 1~2个周期 |
| 等待状态 | 慢速设备插入等待周期 | 0~N个周期 |
| 传输间隔 | 两次传输之间的间隔 | 1~2个周期 |

**总线利用率公式**：

$$
\eta = \frac{T_{有效传输}}{T_{总线周期}} = \frac{T_{数据传输}}{T_{仲裁} + T_{地址} + T_{数据} + T_{间隔}}
$$

### 29.17.3 计算实例

**【例 29-3】** 某总线宽度32位，频率100MHz，每次传输需要：
- 仲裁：2个周期
- 地址：1个周期
- 数据：1个周期
- 间隔：1个周期

**计算**：

$$
\eta = \frac{1}{2 + 1 + 1 + 1} = \frac{1}{5} = 20\%
$$

$$
BW_{实际} = 4 \times 100\text{MHz} \times 20\% = 80\text{MB/s}
$$

**【例 29-4】** 突发传输（Burst）模式：首地址后连续传输4个数据。

$$
\eta_{burst} = \frac{4}{2 + 1 + 4 + 1} = \frac{4}{8} = 50\%
$$

$$
BW_{burst} = 4 \times 100\text{MHz} \times 50\% = 200\text{MB/s}
$$

> [!TIP]
> 突发传输可以显著提高总线利用率，因为只需要一次仲裁和地址传输，就可以连续传输多个数据。PCI总线支持突发传输模式。

## 29.18 总线冲突与解决机制

### 29.18.1 总线冲突的类型

| 冲突类型 | 原因 | 后果 |
|---------|------|------|
| 主设备争用 | 多个主设备同时请求总线 | 数据损坏 |
| 地址冲突 | 多个设备响应同一地址 | 数据错误 |
| 数据冲突 | 多个设备同时驱动数据线 | 电平冲突 |
| 时序冲突 | 时序不满足建立/保持时间 | 数据采样错误 |

### 29.18.2 冲突解决机制

**主设备争用解决**：

通过总线仲裁机制解决，同一时刻只允许一个主设备使用总线。

**地址冲突解决**：

```
地址译码逻辑：
  设备使能 = (地址匹配) AND (CS有效)
  
  硬件保证：每个设备的地址范围不重叠
  例外：总线错误信号（BERR#）指示地址无响应
```

**数据冲突解决**：

```
三态缓冲器控制：
  OE（输出使能）= 仲裁成功 AND 传输阶段
  
  未获得总线的设备：OE=0，输出高阻态
  获得总线的设备：OE=1，输出有效数据
```

### 29.18.3 总线错误处理

| 错误类型 | 检测方式 | 处理方法 |
|---------|---------|---------|
| 超时错误 | 看门狗定时器 | 终止传输，报告错误 |
| 奇偶错误 | 奇偶校验位 | 请求重传 |
| 地址错误 | 地址译码无响应 | 总线错误中断 |
| 仲裁超时 | 仲裁定时器 | 释放总线，重新请求 |

## 29.19 现代总线架构

### 29.19.1 前端总线（FSB）

<div data-component="ModernBusArchViz"></div>

前端总线是CPU与北桥芯片之间的连接：

```
CPU ←→ FSB ←→ 北桥 ←→ 内存总线 ←→ 内存
                  ↓
                PCI总线
```

**FSB的典型参数**：

| 参数 | 典型值 |
|------|--------|
| 宽度 | 64位 |
| 频率 | 200~1600MHz |
| 带宽 | 1.6~12.8 GB/s |
| 传输模式 | QDR（四倍速率） |

**FSB带宽计算**：

$$
BW_{FSB} = \text{宽度} \times \text{前端频率} \times 4 \text{（QDR）}
$$

### 29.19.2 QPI（QuickPath Interconnect）

Intel推出的点对点高速互连：

```
CPU0 ←→ QPI ←→ CPU1
  ↕                ↕
内存0            内存1
```

**QPI的特性**：

| 参数 | 典型值 |
|------|--------|
| 通道宽度 | 20位（每方向） |
| 速率 | 4.8~9.6 GT/s |
| 编码 | 8b/10b |
| 带宽 | 19.2~38.4 GB/s（双向） |
| 拓扑 | 点对点、环形、网状 |

### 29.19.3 UPI（Ultra Path Interconnect）

Intel新一代互连，替代QPI：

```
CPU0 ←→ UPI ←→ CPU1 ←→ UPI ←→ CPU2
  ↕                ↕                ↕
内存0            内存1            内存2
```

**UPI vs QPI对比**：

| 特性 | QPI | UPI |
|------|-----|-----|
| 速率 | 4.8~9.6 GT/s | 10.4 GT/s |
| 编码 | 8b/10b | 8b/9b |
| 效率 | 80% | 88.9% |
| 功耗 | 较高 | 较低 |
| 扩展性 | 中等 | 更好 |

### 29.19.4 现代总线架构演进

```
传统架构（FSB）：
CPU ←→ FSB ←→ 北桥 ←→ 内存
              ↓
            南桥 ←→ I/O

现代架构（点对点）：
CPU ←→ UPI ←→ CPU
  ↕              ↕
内存            内存
  ↕
PCIe ←→ I/O设备
```

> [!NOTE]
> 现代处理器已经取消了传统的前端总线（FSB），改用点对点互连（QPI/UPI），并将内存控制器集成到CPU中，大大减少了内存访问延迟。

## 29.20 扩展例题

### 例题4：总线利用率计算

**题目**：某总线宽度64位，频率200MHz，采用同步通信。每次传输包含：仲裁3个周期，地址2个周期，数据4个周期（突发传输），间隔1个周期。求总线利用率和实际带宽。

**解答**：

$$
\eta = \frac{4}{3 + 2 + 4 + 1} = \frac{4}{10} = 40\%
$$

$$
BW_{理论} = 8 \times 200\text{MHz} = 1600\text{MB/s}
$$

$$
BW_{实际} = 1600 \times 40\% = 640\text{MB/s}
$$

### 例题5：仲裁方式设计

**题目**：某系统有16个设备，要求：（1）支持可变优先级；（2）仲裁速度快；（3）线数尽量少。应选择哪种仲裁方式？

**解答**：

| 要求 | 链式查询 | 计数器查询 | 独立请求 |
|------|---------|-----------|---------|
| 可变优先级 | 不支持 | 支持 | 支持 |
| 仲裁速度 | 慢 | 中 | 快 |
| 线数 | 3根 | 7根 | 33根 |

**推荐**：计数器查询方式。

- 线数：3 + log₂16 = 7根（较少）
- 优先级：支持循环优先级
- 速度：中等，满足要求

### 例题6：总线冲突分析

**题目**：某系统采用链式查询仲裁，设备1~4依次连接。如果设备2的BG传递电路故障（BG始终为低），分析对系统的影响。

**解答**：

- 设备1：正常工作（不受影响）
- 设备2：BG_in始终为低 → 无法获得总线 → **设备2失效**
- 设备3：BG_in = 设备2的BG_out = 0 → 无法获得总线 → **设备3失效**
- 设备4：同理 → **设备4失效**

**结论**：链式查询对电路故障敏感，一个设备的BG链故障会导致其后所有设备失效。

### 例题7：分布式仲裁优先级

**题目**：某系统采用自举分布式仲裁，有4个设备（设备0~3，3优先级最高）。如果设备0和设备2同时请求总线，哪个获得总线？

**解答**：

```
设备0：获得总线 = BR0 AND (NOT BR1) AND (NOT BR2) AND (NOT BR3)
      = 1 AND 1 AND 0 AND 1 = 0 → 不获得

设备2：获得总线 = BR2 AND (NOT BR3)
      = 1 AND 1 = 1 → 获得总线
```

**答案**：设备2获得总线（优先级更高）。

### 例题8：QPI带宽计算

**题目**：某CPU的QPI链路宽度20位，速率6.4GT/s，采用8b/10b编码。求单向和双向带宽。

**解答**：

$$
\text{有效带宽} = 20 \times 6.4\text{GT/s} \times \frac{8}{10} = 102.4\text{Gbps}
$$

$$
\text{单向带宽} = 102.4\text{Gbps} / 8 = 12.8\text{GB/s}
$$

$$
\text{双向带宽} = 12.8 \times 2 = 25.6\text{GB/s}
$$

### 练习3：总线利用率优化

某总线当前利用率30%，瓶颈为仲裁开销（5个周期）。如果采用突发传输（每次传8个数据，数据传输1个周期/个），其他开销不变，求新的利用率。

**解答**：

原：$\eta = \frac{1}{5+1+1+1} = 12.5\%$

新：$\eta = \frac{8}{5+1+8+1} = \frac{8}{15} = 53.3\%$

利用率提升：$53.3\% / 12.5\% = 4.26$ 倍
