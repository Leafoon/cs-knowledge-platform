# 第二十三章 存储器概述与主存储器

存储器是计算机的重要组成部分，其速度和容量直接影响系统性能。本章讲解存储器的分类、层次结构、SRAM和DRAM的工作原理，以及ROM的类型。

## 23.1 存储器的分类

### 23.1.1 按存储介质分类

| 类型 | 存储介质 | 特点 | 典型应用 |
|------|---------|------|---------|
| 半导体存储器 | 触发器/电容 | 速度快、集成度高 | Cache、主存 |
| 磁表面存储器 | 磁性材料 | 容量大、成本低 | 硬盘、磁带 |
| 光存储器 | 光学材料 | 容量大、可移动 | CD、DVD、蓝光 |

### 23.1.2 按存取方式分类

| 类型 | 存取方式 | 特点 | 典型应用 |
|------|---------|------|---------|
| RAM（随机存取） | 随机访问任意地址 | 速度均匀 | 主存 |
| SAM（顺序存取） | 按顺序访问 | 速度不均匀 | 磁带 |
| DAM（直接存取） | 先定位区域再顺序 | 介于两者之间 | 硬盘 |

### 23.1.3 按读写功能分类

| 类型 | 功能 | 特点 |
|------|------|------|
| ROM（只读存储器） | 只能读出 | 断电不丢失 |
| RAM（随机读写存储器） | 可读可写 | 断电丢失（易失性） |

### 23.1.4 按信息易失性分类

| 类型 | 断电后 | 典型应用 |
|------|--------|---------|
| 易失性存储器 | 数据丢失 | SRAM、DRAM |
| 非易失性存储器 | 数据保持 | ROM、Flash、磁盘 |

### 23.1.5 存储器分类综合表

| 分类维度 | 类型 |
|----------|------|
| 按介质 | 半导体/磁/光 |
| 按存取方式 | RAM/SAM/DAM |
| 按读写 | ROM/RAM |
| 按易失性 | 易失/非易失 |
| 按作用 | 主存/Cache/辅存 |

## 23.2 存储器的层次结构

### 23.2.1 存储层次金字塔

```
        ┌─────────┐
        │ 寄存器   │ ← 速度最快(<1ns)，容量最小(字节级)
        ├─────────┤
        │  L1 Cache│ ← 1~2ns，KB级
        ├─────────┤
        │  L2 Cache│ ← 5~10ns，MB级
        ├─────────┤
        │  主存    │ ← 50~100ns，GB级
        ├─────────┤
        │  SSD     │ ← 0.1ms，TB级
        ├─────────┤
        │  HDD     │ ← 10ms，TB级
        ├─────────┤
        │ 磁带/光盘│ ← 秒级，PB级
        └─────────┘
```

### 23.2.2 各层次的速度、容量和成本

| 层次 | 速度 | 容量 | 成本/GB | 位置 |
|------|------|------|---------|------|
| 寄存器 | <1ns | B~KB | 极高 | CPU内部 |
| L1 Cache | 1~2ns | 32~64KB | 很高 | CPU内部 |
| L2 Cache | 5~10ns | 256KB~1MB | 高 | CPU内部 |
| 主存 | 50~100ns | 4~64GB | 中 | 主板 |
| SSD | 0.1ms | 256GB~4TB | 低 | 外部 |
| HDD | 10ms | 1~20TB | 最低 | 外部 |

### 23.2.3 局部性原理

**时间局部性**（Temporal Locality）：
- 最近访问的数据**很可能再次被访问**
- 例：循环中的变量、频繁调用的函数

**空间局部性**（Spatial Locality）：
- 相邻地址的数据**很可能被连续访问**
- 例：数组遍历、顺序执行的指令

**局部性原理是Cache和虚拟存储器能够有效工作的理论基础。**

**【例 23-1】** 分析以下程序的局部性：

```c
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += a[i];
}
```

- `sum`：时间局部性（每次循环都访问）
- `a[i]`：空间局部性（连续访问数组元素）
- `i`：时间局部性（每次循环都访问）

## 23.3 SRAM（静态随机存取存储器）

### 23.3.1 基本存储单元——6管CMOS触发器

<div data-component="SRAMCellViz"></div>

SRAM使用**6管CMOS触发器**存储1位数据，电路由两个交叉耦合的CMOS反相器和两个NMOS访问管组成。

**电路结构**：

```
          VDD                    VDD
           │                      │
      ┌────┴────┐            ┌────┴────┐
      │  PMOS   │            │  PMOS   │
      │  (T3)   │            │  (T4)   │
      └────┬────┘            └────┬────┘
           │                      │
     Q ────┼──────────────────────┤──── Q̄
           │                      │
      ┌────┴────┐            ┌────┴────┐
      │  NMOS   │            │  NMOS   │
      │  (T1)   │            │  (T2)   │
      └────┬────┘            └────┬────┘
           │                      │
          GND                    GND

           │                      │
      ┌────┴────┐            ┌────┴────┐
      │  NMOS   │            │  NMOS   │
      │  (T5)   │            │  (T6)   │
      └────┬────┘            └────┬────┘
           │                      │
     BL ───┘                BL̄ ───┘

           └────── WL ──────────┘
```

**六管功能划分**：

| 晶体管 | 类型 | 功能 |
|--------|------|------|
| T1 | NMOS | 左反相器下拉管 |
| T2 | NMOS | 右反相器下拉管 |
| T3 | PMOS | 左反相器上拉管 |
| T4 | PMOS | 右反相器上拉管 |
| T5 | NMOS | 左访问管（字线控制） |
| T6 | NMOS | 右访问管（字线控制） |

**存储原理**：

- T3+T1构成左反相器，T4+T2构成右反相器
- 两个反相器交叉耦合：左反相器输出接右反相器输入，右反相器输出接左反相器输入
- 形成双稳态电路，有两个稳定状态：

| 状态 | Q | Q̄ | T1 | T2 | T3 | T4 |
|------|---|---|----|----|----|----|
| 存储"0" | 低 | 高 | 截止 | 导通 | 导通 | 截止 |
| 存储"1" | 高 | 低 | 导通 | 截止 | 截止 | 导通 |

> [!NOTE]
> 只要VDD持续供电，交叉耦合的反相器就会一直保持当前状态，因此SRAM**不需要刷新**。这是SRAM与DRAM的本质区别。

**读操作详细过程**：

1. 字线WL升为高电平，T5、T6导通
2. 位线BL和BL̄预充电至VDD/2
3. 假设存储"0"（Q低、Q̄高）：
   - T5导通后，BL通过T5→Q→T1放电，BL电压下降
   - T6导通后，BL̄通过T6→Q̄保持高电平
4. 灵敏放大器检测BL与BL̄之间的电压差
5. 放大差值并输出数据

**写操作详细过程**：

1. 字线WL升为高电平，T5、T6导通
2. 假设要写入"1"：BL驱动为高，BL̄驱动为低
3. BL通过T5强制将Q拉高，BL̄通过T6强制将Q̄拉低
4. 正反馈使触发器翻转到新状态
5. 字线撤除后，新状态保持

> [!TIP]
> 写操作需要**强制翻转**触发器，因此驱动能力要大于反相器的保持能力。读操作是**非破坏性**的，触发器状态不会改变。

### 23.3.2 SRAM的特点

| 特性 | 说明 |
|------|------|
| 速度 | **快**（1~10ns） |
| 集成度 | **低**（6个晶体管/位） |
| 成本 | **高** |
| 功耗 | 较低（静态功耗） |
| 刷新 | **不需要** |
| 用途 | **Cache** |
| 破坏性读出 | **否** |
| 电源要求 | 需持续供电 |

### 23.3.3 SRAM的读写时序

**读时序参数**：

```
        ┌─────────────────────────────────────────┐
        │           读周期时序图                    │
        ├─────────────────────────────────────────┤
  地址   ═══════╤═══════════════════════════════════
                │◄── tAA ──►│
  CS̄    ────────┐           │
                │◄─ tACS ──►│
                └───────────┘
  OĒ    ────────┐           │
                │◄─ tOE ──►│
                └───────────┘
  数据   ═══════╤═══════════╤═════════════════════
                │  有效数据  │
                ◄── tOH ───►│
```

| 时序参数 | 含义 | 典型值 |
|----------|------|--------|
| $t_{RC}$ | 读周期时间 | 10~15ns |
| $t_{AA}$ | 地址存取时间 | 8~12ns |
| $t_{ACS}$ | 片选存取时间 | 8~12ns |
| $t_{OE}$ | 输出使能时间 | 5~8ns |
| $t_{OH}$ | 输出保持时间 | 2~5ns |

**写时序参数**：

```
        ┌─────────────────────────────────────────┐
        │           写周期时序图                    │
        ├─────────────────────────────────────────┤
  地址   ═══════╤═══════════════════════════════════
                │
  CS̄    ────────┐                 ┌───────────────
                │◄─── tWP ───────►│
  WĒ    ────────┐                 ┌───────────────
                │◄─── tWP ───────►│
  数据   ═══════╤═════════════════╤═══════════════
                │   写入数据       │
                ◄─ tDS ─►         ◄─ tDH ─►
```

| 时序参数 | 含义 | 典型值 |
|----------|------|--------|
| $t_{WC}$ | 写周期时间 | 10~15ns |
| $t_{WP}$ | 写脉冲宽度 | 8~10ns |
| $t_{DS}$ | 数据建立时间 | 5~8ns |
| $t_{DH}$ | 数据保持时间 | 2~5ns |

## 23.4 DRAM（动态随机存取存储器）

### 23.4.1 基本存储单元——1管1电容

DRAM使用**1个NMOS晶体管 + 1个电容**存储1位数据，这是最简单的存储单元结构。

**电路结构**：

```
     字线 WL
       │
       │
  ┌────┴────┐
  │  NMOS   │
  │  (T)    │
  └────┬────┘
       │
       ├──── 位线 BL
       │
  ┌────┴────┐
  │         │
  │    Cs   │  ← 存储电容（约30~50fF）
  │         │
  └────┬────┘
       │
      GND
```

**存储原理**：

| 存储值 | 电容状态 | 位线电压 |
|--------|---------|---------|
| "1" | Cs充电至高电平 | 高 |
| "0" | Cs放电至低电平 | 低 |

**读操作（破坏性读出）**：

1. 位线BL预充电至参考电压 $V_{REF} = V_{DD}/2$
2. 字线WL升为高电平，T导通
3. 存储电容Cs与位线BL共享电荷
4. 位线电压发生微小变化 $\Delta V$

$$
\Delta V = \frac{C_s}{C_s + C_{BL}} \times (V_{stored} - V_{REF})
$$

其中 $C_{BL}$ 为位线寄生电容（远大于 $C_s$，通常 $C_{BL}/C_s \approx 10$）。

5. 灵敏放大器检测 $\Delta V$ 并放大至 full-swing
6. **读出后电容电荷被破坏，必须再生（写回）**

> [!NOTE]
> DRAM读操作是**破坏性读出**：每次读取后必须将数据重新写回存储电容。这是因为读取时电荷被位线分流，电容上的电压会降低。

**写操作**：

1. 字线WL升为高电平，T导通
2. 位线BL驱动至目标电压（高=VDD写"1"，低=GND写"0"）
3. 电容Cs充电或放电至目标值
4. 字线撤除，T截止，电荷保存在Cs中

**DRAM单元的关键参数**：

| 参数 | 含义 | 典型值 |
|------|------|--------|
| $C_s$ | 存储电容 | 30~50 fF |
| $C_{BL}$ | 位线寄生电容 | 200~500 fF |
| $t_{REF}$ | 刷新周期 | 64ms（标准）/ 32ms（高温） |
| $V_{th}$ | 阈值电压 | 0.3~0.5V |
| $I_{leak}$ | 漏电流 | ~fA级 |

**DRAM存储阵列结构**：

```
     WL0    WL1    WL2    WL3
      │      │      │      │
      T      T      T      T
      │      │      │      │
BL0──┼──────┼──────┼──────┼── 灵敏放大器 ── DOUT
      │      │      │      │
     Cs     Cs     Cs     Cs
      │      │      │      │
     GND    GND    GND    GND
      │      │      │      │
      T      T      T      T
      │      │      │      │
BL1──┼──────┼──────┼──────┼── 灵敏放大器 ── DOUT
      │      │      │      │
     Cs     Cs     Cs     Cs
      │      │      │      │
     GND    GND    GND    GND
```

> [!TIP]
> DRAM的位线通常采用**差分布线**结构（BL和BL̄），与灵敏放大器配合使用，可以提高读出灵敏度和抗噪声能力。

### 23.4.2 DRAM的特点

| 特性 | 说明 |
|------|------|
| 速度 | **较慢**（50~100ns） |
| 集成度 | **高**（1个晶体管+1个电容/位） |
| 成本 | **低** |
| 功耗 | 较高（需要刷新） |
| 刷新 | **需要** |
| 用途 | **主存** |

### 23.4.3 DRAM的读写过程

**读操作**（破坏性读出）：
1. 字线激活，晶体管导通
2. 电容与位线共享电荷
3. 感应放大器检测位线上的电压变化
4. **读出后电容电荷被破坏，需要再生（写回）**

**写操作**：
1. 字线激活
2. 位线上施加要写入的值
3. 电容充电或放电

### 23.4.4 DRAM的刷新机制详细时序

<div data-component="DRAMRefreshDemo"></div>

由于电容漏电，DRAM需要定期刷新以保持数据。

**刷新基本概念**：

| 概念 | 说明 |
|------|------|
| 刷新周期 | 整个存储器所有行刷新一遍的时间（通常64ms） |
| 刷新行数 | 存储阵列的总行数（如4096行、8192行） |
| 刷新间隔 | 两次刷新操作之间的时间间隔 |
| 刷新时间 | 完成一次行刷新所需时间（等于存取时间） |
| 死时间 | 刷新期间CPU不能访问存储器的时间 |

**刷新方式一：集中刷新（Burst Refresh）**

在刷新周期的末尾，集中一段时间刷新所有行。

```
 ◄─────────────────── 64ms 刷新周期 ───────────────────►
 ┌─────────────────────────────┬───────────────────────┐
 │      正常读写时间            │  集中刷新所有行         │
 │      (约63.5ms)             │  (约0.5ms)             │
 └─────────────────────────────┴───────────────────────┘
                               ◄── 死时间 ──►
```

**计算示例**：

假设DRAM有4096行，每行刷新需要50ns：

$$
\text{集中刷新时间} = 4096 \times 50\text{ns} = 204.8\mu\text{s}
$$

$$
\text{死时间比例} = \frac{204.8\mu\text{s}}{64\text{ms}} = 0.32\%
$$

| 优点 | 缺点 |
|------|------|
| 控制简单 | 存在集中死时间 |
| 读写期间无刷新干扰 | 死时间内CPU完全无法访问 |

**刷新方式二：分散刷新（Distributed Refresh）**

每个读写周期后插入一行刷新操作。

```
 ◄── 存储周期 ──►◄── 存储周期 ──►◄── 存储周期 ──►
 ┌─────┬─────────┬─────┬─────────┬─────┬─────────┐
 │读写 │  刷新行0 │读写  │  刷新行1 │读写  │  刷新行2 │
 └─────┴─────────┴─────┴─────────┴─────┴─────────┘
 ◄─────►
  tMC   ← 读写周期 = tMC + tREF
```

**计算示例**：

假设 $t_{MC} = 100\text{ns}$（存储周期），$t_{REF} = 50\text{ns}$（刷新时间）：

$$
\text{实际存储周期} = t_{MC} + t_{REF} = 100 + 50 = 150\text{ns}
$$

$$
\text{所有行刷新时间} = 4096 \times 150\text{ns} = 614.4\mu\text{s}
$$

| 优点 | 缺点 |
|------|------|
| 无集中死时间 | 每个存储周期变长 |
| 刷新均匀分布 | 存储器速度降低50% |

> [!NOTE]
> 分散刷新的缺点是每个存储周期都插入刷新，导致存储器的有效访问速度降低。因此实际中很少单独使用。

**刷新方式三：异步刷新（Asynchronous/Transparent Refresh）**

在刷新周期内均匀安排刷新操作，刷新与读写可并行。

```
 ◄──────────────── 64ms ────────────────►
 │15.6μs│15.6μs│15.6μs│ ... │15.6μs│15.6μs│
 ┌──────┬──────┬──────┬─────┬──────┬──────┐
 │刷新0 │刷新1 │刷新2 │ ... │刷新4094│刷新4095│
 └──────┴──────┴──────┴─────┴──────┴──────┘
```

**计算示例**：

$$
\text{刷新间隔} = \frac{64\text{ms}}{4096} = 15.625\mu\text{s}
$$

在每个 $15.625\mu\text{s}$ 的开始，用一个刷新周期（50ns）刷新一行：

$$
\text{死时间比例} = \frac{50\text{ns}}{15.625\mu\text{s}} = 0.32\%
$$

| 优点 | 缺点 |
|------|------|
| 死时间最小且分散 | 控制逻辑较复杂 |
| 对CPU访问影响最小 | 需要定时器支持 |

**三种刷新方式综合对比**：

| 刷新方式 | 死时间分布 | 平均死时间 | 控制复杂度 | 实际应用 |
|----------|-----------|-----------|-----------|---------|
| 集中刷新 | 集中在末尾 | 0.32% | 低 | 简单系统 |
| 分散刷新 | 每周期都有 | 33%（速度降低） | 低 | 很少使用 |
| 异步刷新 | 均匀分散 | 0.32% | 中 | **最常用** |

**刷新操作的实现机制**：

```
┌──────────────────────────────────────────────────┐
│                  DRAM刷新控制器                    │
├──────────────────────────────────────────────────┤
│                                                  │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐   │
│  │ 定时器    │───►│ 行计数器  │───►│ 刷新地址  │   │
│  │(15.6μs)  │    │ (0~4095) │    │  多路器   │   │
│  └──────────┘    └──────────┘    └─────┬────┘   │
│                                        │        │
│  ┌──────────┐    ┌──────────┐         │        │
│  │ 仲裁器    │◄──►│ CPU请求   │    行地址 ──►│ 译码器 │
│  └─────┬────┘    └──────────┘         │        │
│        │                              │        │
│        ▼                              ▼        │
│  ┌──────────┐              ┌──────────────┐    │
│  │ RAS/CAS  │─────────────►│  存储阵列     │    │
│  │  控制器   │              │              │    │

```

### 23.4.5 SDRAM与DDR SDRAM

| 类型 | 特点 | 说明 |
|------|------|------|
| SDRAM | 同步DRAM | 与系统时钟同步 |
| DDR | 双倍数据率 | 上升沿和下降沿都传数据 |

### 23.5.2 地址译码器

地址译码器将二进制地址转换为选中信号，选择特定的存储单元。

**线性译码器（适用于小容量）**：

```
  A1  A0
   │   │
   ▼   ▼
┌──────────────┐
│   2-4 译码器  │
├──────────────┤
│ Y0 = A1̄·A0̄  │──► 选中第0行
│ Y1 = A1̄·A0   │──► 选中第1行
│ Y2 = A1·A0̄   │──► 选中第2行
│ Y3 = A1·A0   │──► 选中第3行
└──────────────┘
```

**复合译码器（适用于大容量）**：

大容量存储器采用**行列复合译码**结构，将地址分为行地址和列地址两部分：

- 行地址：选择存储阵列的一行（字线）
- 列地址：从选中行中选择特定位线

$$
\text{总地址} = \text{行地址} \times \text{列数} + \text{列地址}
$$

**优势**：减少译码器的输入数，降低硬件复杂度。

| 地址位数 | 直接译码输出数 | 行列译码输出数 |
|----------|---------------|---------------|
| 10位 | 1024 | 32 × 32 = 1024 |
| 12位 | 4096 | 64 × 64 = 4096 |
| 16位 | 65536 | 256 × 256 = 65536 |

### 23.5.3 读写电路

**读出电路**：

```
  位线BL ────┬──── 灵敏放大器 ──── 输出缓冲器 ──── DOUT
              │         ▲
  位线BL̄ ────┘         │
                       │
                    参考电压
                    VREF=VDD/2
```

| 电路模块 | 功能 |
|----------|------|
| 预充电电路 | 读操作前将位线预充至VDD/2 |
| 灵敏放大器 | 检测BL与BL̄之间的微小电压差并放大 |
| 输出缓冲器 | 驱动数据总线 |
| 写入驱动器 | 将数据总线的值写入存储单元 |

### 23.5.4 片选逻辑

片选信号CS̄（Chip Select）决定芯片是否被选中：

```
  ┌───────────┐
  │  存储芯片  │
  ├───────────┤
  │ A(n-1)~A0 │◄─── 地址线
  │ D(m-1)~D0 │◄──► 数据线
  │ CS̄        │◄─── 片选（低电平有效）
  │ R/W̄       │◄─── 读/写控制
  │ OĒ        │◄─── 输出使能
  │ VDD       │◄─── 电源
  │ GND       │◄─── 地
  └───────────┘
```

> [!TIP]
> 片选信号CS̄通常由高位地址线经过译码器产生，低电平有效表示该芯片被选中。未选中的芯片数据线呈高阻态，不影响总线。

## 23.8 ROM（只读存储器）

### 23.8.1 ROM的类型

| 类型 | 全称 | 擦除方式 | 特点 |
|------|------|---------|------|
| ROM | Read-Only Memory | 不可擦除 | 出厂时写入 |
| PROM | Programmable ROM | 一次性编程 | 用户可编程一次 |
| EPROM | Erasable PROM | 紫外线擦除 | 整片擦除 |
| EEPROM | Electrically EPROM | 电擦除 | 按字节擦除 |
| Flash | Flash Memory | 电擦除 | 按块擦除，速度快 |

### 23.8.2 PROM的内部结构

PROM（可编程ROM）出厂时所有位均为"1"，用户通过**熔断**特定位置的熔丝来写入"0"。

**PROM存储单元结构**：

```
     字线 WL
       │
  ┌────┴────┐
  │ 晶体管  │
  │  (T)    │
  └────┬────┘
       │
  ┌────┴────┐
  │  熔丝    │ ← 出厂时连通（存"1"）
  │  (Fuse) │    编程时熔断（存"0"）
  └────┬────┘
       │
     位线 BL
```

| 编程前 | 编程后 |
|--------|--------|
| 熔丝连通 → 读出"1" | 熔丝熔断 → 读出"0" |

> [!NOTE]
> PROM的编程是**不可逆**的，熔丝一旦熔断就无法恢复。因此PROM只能编程一次。

### 23.8.3 EPROM的内部结构

EPROM使用**浮栅晶体管**（Floating Gate Transistor）存储数据。

**浮栅晶体管结构**：

```
        控制栅（Control Gate）
              │
        ┌─────┴─────┐
        │  绝缘层    │
        ├───────────┤
        │  浮栅      │ ← 注入电子后阈值电压升高
        │ (Floating  │
        │  Gate)     │
        ├───────────┤
        │  绝缘层    │
        ├───────────┤
        │  沟道      │
        └─────┬─────┘
              │
     源极 ────┘──── 漏极
```

**编程原理**：

| 状态 | 浮栅状态 | 阈值电压 | 读出值 |
|------|---------|---------|--------|
| 擦除态 | 无电子 | 低 | "1" |
| 编程态 | 有电子 | 高 | "0" |

**擦除方式**：紫外线透过石英窗口照射，使浮栅中的电子获得能量逃逸，所有单元恢复为"1"。

### 23.8.4 EEPROM的内部结构

EEPROM（电可擦除PROM）使用**双层浮栅**或**隧穿氧化层**实现电擦除。

**隧道效应编程**：

- 在控制栅和浮栅之间施加高电压
- 电子通过**Fowler-Nordheim隧穿**穿过薄氧化层
- 可按字节擦除和编程

| 特性 | EPROM | EEPROM |
|------|-------|--------|
| 擦除方式 | 紫外线整片擦除 | 电擦除，按字节 |
| 擦除时间 | 15~20分钟 | 5~10ms |
| 编程电压 | 12.5V | 5V（内部升压） |
| 擦写次数 | ~1000次 | ~10万次 |
| 封装 | 需要石英窗口 | 普通封装 |

### 23.8.5 Flash存储器的内部结构

Flash存储器结合了EPROM的高密度和EEPROM的电擦除能力。

**NOR Flash结构**：

```
  字线 WL ──────────────────────────────────────
       │              │              │
  ┌────┴────┐    ┌────┴────┐    ┌────┴────┐
  │ 浮栅管  │    │ 浮栅管  │    │ 浮栅管  │
  │  (T1)   │    │  (T2)   │    │  (T3)   │
  └────┬────┘    └────┬────┘    └────┬────┘
       │              │              │
  ─────┴──────────────┴──────────────┴───── 位线
       │              │              │
      GND            GND            GND
```

- 单元并联连接到位线
- 可按**字节随机读取**
- 读速度快，写/擦除慢

**NAND Flash结构**：

```
  字线 WL0 ───────┐    ┌─────── 字线 WL1
                   │    │
              ┌────┴────┴────┐
              │   串联单元    │
              │  T1 - T2 - T3 - T4  │
              └────┬────┬────┘
                   │    │
  位线 ────────────┘    └──────── 位线
       │                        │
      GND                      GND
```

- 单元串联连接
- 按页读取，按块擦除
- 容量大，成本低

**NOR Flash vs NAND Flash对比**：

| 特性 | NOR Flash | NAND Flash |
|------|-----------|------------|
| 连接方式 | 并联 | 串联 |
| 读取方式 | 按字节随机 | 按页顺序 |
| 读速度 | 快（~50ns） | 较慢（~25μs） |
| 写速度 | 慢（~10μs） | 较快（~200μs） |
| 擦除速度 | 慢（~1s） | 快（~2ms） |
| 容量 | 小（~256Mb） | 大（~1Tb） |
| 应用 | BIOS、固件 | SSD、U盘 |

### 23.8.6 ROM的特点

| 特性 | 说明 |
|------|------|
| 非易失性 | 断电数据不丢失 |
| 只读/有限写 | 大多数只能读出 |
| 用途 | 固件、BIOS、查找表 |

## 23.9 存储器的主要技术指标

### 23.7.1 存储容量

存储器能够存储的二进制数据总量。

$$
\text{存储容量} = 2^n \times m
$$

其中 $n$ 为地址线数，$m$ 为数据线数（位数）。

**【例 23-2】** 某存储器有16根地址线和8根数据线。

$$
\text{容量} = 2^{16} \times 8 = 65536 \times 8 = 64\text{KB}
$$

**单位换算**：

| 单位 | 符号 | 大小 |
|------|------|------|
| 位 | bit | 1位 |
| 字节 | Byte | 8位 |
| 千字节 | KB | $2^{10}$ B = 1024 B |
| 兆字节 | MB | $2^{20}$ B |
| 吉字节 | GB | $2^{30}$ B |
| 太字节 | TB | $2^{40}$ B |

### 23.7.2 存取时间

从CPU发出读/写命令到数据传输完成的时间。

### 23.7.3 存储周期

连续两次访问的最小时间间隔。

$$
\text{存储周期} = \text{存取时间} + \text{恢复时间}
$$

### 23.7.4 存储器带宽

单位时间内存储器传输的数据量。

$$
\text{带宽} = \frac{\text{数据宽度（位）}}{\text{存储周期}}
$$

**【例 23-3】** 存储器数据宽度64位，存储周期100ns。

$$
\text{带宽} = \frac{64\text{位}}{100\text{ns}} = 640\text{MB/s}
$$

### 23.7.5 存储器的性能指标汇总

| 指标 | 定义 | 影响因素 |
|------|------|---------|
| 存储容量 | 总位数 | 地址线数、数据线数 |
| 存取时间 | 一次读写的时间 | 存储介质、制造工艺 |
| 存储周期 | 连续访问的最小间隔 | 存取时间+恢复时间 |
| 存储带宽 | 单位时间传输量 | 数据宽度/存储周期 |

## 23.10 存储器层次结构的性能分析

### 23.12.1 命中率与失效率

**命中率（Hit Rate）**：访问的数据在当前层次中找到的概率。

$$
h = \frac{\text{命中次数}}{\text{总访问次数}}
$$

**失效率（Miss Rate）**：访问的数据不在当前层次中的概率。

$$
m = 1 - h
$$

**典型命中率**：

| 存储层次 | 典型命中率 | 说明 |
|----------|-----------|------|
| L1 Cache | 95~99% | 最接近CPU |
| L2 Cache | 80~95% | 中间层次 |
| TLB | 99~99.9% | 地址转换缓存 |
| 虚拟存储器 | ~100% | 虚拟地址转换 |

### 23.12.2 平均访问时间

**两级存储层次的平均访问时间**：

$$
T_{avg} = h \times T_{hit} + (1-h) \times T_{miss}
$$

其中：
- $h$ = 命中率
- $T_{hit}$ = 命中时的访问时间（即较快存储器的访问时间）
- $T_{miss}$ = 失效时的访问时间（即较慢存储器的访问时间）

**【例 23-4】** L1 Cache命中率95%，访问时间2ns；主存访问时间100ns。求平均访问时间。

$$
T_{avg} = 0.95 \times 2 + 0.05 \times 100 = 1.9 + 5 = 6.9\text{ns}
$$

**【例 23-5】** 两级Cache系统：L1命中率96%、访问时间1ns；L2命中率90%（在L1失效时）、访问时间10ns；主存100ns。求平均访问时间。

$$
T_{avg} = h_1 \times T_1 + (1-h_1) \times [h_2 \times T_2 + (1-h_2) \times T_m]
$$

$$
T_{avg} = 0.96 \times 1 + 0.04 \times [0.90 \times 10 + 0.10 \times 100]
$$

$$
T_{avg} = 0.96 + 0.04 \times [9 + 10] = 0.96 + 0.76 = 1.72\text{ns}
$$

> [!NOTE]
> 多级存储层次的平均访问时间需要逐级计算：先计算L2层的有效访问时间，再计算L1层的平均访问时间。

### 23.12.3 存储层次的性能指标汇总

| 指标 | 定义 | 影响因素 |
|------|------|---------|
| 命中率 $h$ | 数据在当前层次找到的概率 | 容量、替换算法、访问模式 |
| 失效率 $m$ | 数据不在当前层次的概率 | $m = 1 - h$ |
| 命中时间 $T_{hit}$ | 命中时的访问延迟 | 存储器速度、地址译码 |
| 失效代价 $T_{miss}$ | 失效时的访问延迟 | 下级存储器速度、传输带宽 |
| 平均访问时间 $T_{avg}$ | 加权平均访问延迟 | 命中率、命中时间、失效代价 |
| 存储带宽 | 单位时间数据传输量 | 数据宽度、访问周期 |

### 23.12.4 性能优化策略

**提高命中率的方法**：

| 策略 | 说明 |
|------|------|
| 增大Cache容量 | 减少容量失效 |
| 提高关联度 | 减少冲突失效 |
| 优化替换算法 | LRU优于FIFO |
| 硬件预取 | 利用空间局部性 |
| 编译优化 | 改善数据访问模式 |

**降低失效代价的方法**：

| 策略 | 说明 |
|------|------|
| 多级Cache | L2作为L1的后援 |
| 关键字优先 | 先传需要的字 |
| 非阻塞Cache | 允许失效时继续服务 |
| 写合并 | 减少写回次数 |

### 23.12.5 存储层次设计的权衡

$$
\text{总成本} = \sum_{i=1}^{n} C_i \times S_i
$$

$$
\text{总容量} = S_n \quad (\text{最大层次的容量})
$$

$$
\text{有效速度} \approx T_1 \quad (\text{当命中率足够高时})
$$

| 设计目标 | 权衡 |
|----------|------|
| 高速度 | 使用小容量快速存储器（SRAM） |
| 大容量 | 使用大容量慢速存储器（DRAM） |
| 低成本 | 使用廉价存储介质（磁盘） |
| 综合 | 多级层次结构，兼顾速度和容量 |

## 23.11 本章典型例题

### 例题1：存储器参数计算

**题目**：某存储器有20根地址线和16根数据线，求其容量和带宽（存储周期200ns）。

**解答**：

容量 = $2^{20} \times 16 = 1\text{M} \times 16\text{位} = 2\text{MB}$

带宽 = $16 / 200\text{ns} = 80\text{MB/s}$

### 例题2：刷新计算

**题目**：某DRAM有1024行，刷新周期2ms，每行刷新需要100ns。求集中刷新的死时间。

**解答**：

集中刷新时间 = $1024 \times 100\text{ns} = 102.4\mu\text{s}$

死时间比例 = $102.4\mu\text{s} / 2\text{ms} = 5.12\%$

## 23.14 深入理解存储器

### 23.11.1 存储器的层次结构详解

**存储层次金字塔**：

```
        ┌─────────┐
        │ 寄存器   │ ← 速度最快(<1ns)，容量最小(字节级)
        ├─────────┤
        │  L1 Cache│ ← 1~2ns，KB级
        ├─────────┤
        │  L2 Cache│ ← 5~10ns，MB级
        ├─────────┤
        │  主存    │ ← 50~100ns，GB级
        ├─────────┤
        │  SSD     │ ← 0.1ms，TB级
        ├─────────┤
        │  HDD     │ ← 10ms，TB级
        ├─────────┤
        │ 磁带/光盘│ ← 秒级，PB级
        └─────────┘
```

**局部性原理**：

- **时间局部性**：最近访问的数据可能再次被访问
- **空间局部性**：相邻地址的数据可能被连续访问

### 23.11.2 SRAM详解

**SRAM的基本单元**：6管触发器

**特点**：
- 速度快（1~10ns）
- 集成度低（6个晶体管/位）
- 成本高
- 不需要刷新
- 用途：Cache

**SRAM的读写过程**：

**读操作**：
1. 字线激活（选中该行）
2. 位线上的感应放大器检测存储单元的状态
3. 输出数据

**写操作**：
1. 字线激活
2. 位线上施加要写入的值
3. 触发器翻转到新状态

### 23.11.3 DRAM详解

**DRAM的基本单元**：1管+电容

**特点**：
- 速度较慢（50~100ns）
- 集成度高（1个晶体管+1个电容/位）
- 成本低
- 需要刷新
- 用途：主存

**DRAM的刷新**：

| 刷新方式 | 特点 | 死时间 | 说明 |
|----------|------|--------|------|
| 集中刷新 | 集中一段时间刷新所有行 | 有 | 刷新期间不能读写 |
| 分散刷新 | 每个读写周期后刷新一行 | 分散 | 每行刷新频率更高 |
| 异步刷新 | 在刷新周期内均匀安排刷新 | 最小 | 最常用 |

### 23.11.4 ROM详解

| 类型 | 全称 | 擦除方式 | 特点 |
|------|------|---------|------|
| ROM | Read-Only Memory | 不可擦除 | 出厂时写入 |
| PROM | Programmable ROM | 一次性编程 | 用户可编程一次 |
| EPROM | Erasable PROM | 紫外线擦除 | 整片擦除 |
| EEPROM | Electrically EPROM | 电擦除 | 按字节擦除 |
| Flash | Flash Memory | 电擦除 | 按块擦除，速度快 |

## 23.15 存储器的性能指标详解

### 23.12.1 存储容量

**存储容量**：存储器能存储的二进制数据总量。

$$
\text{存储容量} = 2^{\text{地址线数}} \times \text{数据线数}
$$

**单位换算**：

| 单位 | 符号 | 大小 |
|------|------|------|
| 位 | bit | 1位 |
| 字节 | Byte | 8位 |
| 千字节 | KB | $2^{10}$ B = 1024 B |
| 兆字节 | MB | $2^{20}$ B |
| 吉字节 | GB | $2^{30}$ B |
| 太字节 | TB | $2^{40}$ B |

### 23.12.2 存取时间

**存取时间**：从CPU发出读/写命令到数据传输完成的时间。

### 23.12.3 存储周期

**存储周期**：连续两次访问的最小时间间隔。

$$
\text{存储周期} = \text{存取时间} + \text{恢复时间}
$$

### 23.12.4 存储器带宽

**存储器带宽**：单位时间内存储器传输的数据量。

$$
\text{带宽} = \frac{\text{数据宽度（位）}}{\text{存储周期}}
$$

**【例 23-3】** 存储器数据宽度64位，存储周期100ns。

$$
\text{带宽} = \frac{64\text{位}}{100\text{ns}} = 640\text{MB/s}
$$
