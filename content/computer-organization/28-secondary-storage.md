# 第二十八章 辅助存储器与存储系统综合

本章讲解磁盘、SSD等辅助存储器，以及存储系统的综合性能分析。

---

## 28.1 磁盘存储器

### 28.1.1 磁盘的物理结构

<div data-component="DiskStructureViz"></div>

```
        ┌─────────────────────┐
        │      盘面(Platter)   │
        │    ┌─────────────┐  │
        │    │  磁道(Track) │  │
        │    │  ┌───────┐  │  │
        │    │  │扇区   │  │  │
        │    │  │(Sector)│  │  │
        │    │  └───────┘  │  │
        │    └─────────────┘  │
        └─────────────────────┘
```

| 概念 | 说明 |
|------|------|
| 盘面（Platter） | 磁盘的一个面，涂有磁性材料 |
| 磁道（Track） | 盘面上的同心圆 |
| 扇区（Sector） | 磁道上的一个弧段，最小读写单位（通常512B） |
| 柱面（Cylinder） | 所有盘面上同一磁道的集合 |

### 28.1.2 磁盘的性能参数

| 参数 | 说明 |
|------|------|
| 容量 | 盘面数 × 每面磁道数 × 每磁道扇区数 × 每扇区字节数 |
| 转速 | 每分钟转数（RPM），如7200RPM |
| 寻道时间 | 磁头移动到目标磁道的时间 |
| 旋转延迟 | 等待目标扇区旋转到磁头下的时间 |
| 数据传输时间 | 读写数据的时间 |

### 28.1.3 磁盘访问时间

$$
\boxed{\text{访问时间} = \text{寻道时间} + \text{旋转延迟} + \text{数据传输时间}}
$$

**寻道时间**：磁头从当前磁道移动到目标磁道的时间。

- 最大寻道时间：从最内磁道到最外磁道
- 平均寻道时间：通常为最大寻道时间的1/3

**旋转延迟**：等待目标扇区旋转到磁头下的时间。

$$
\text{平均旋转延迟} = \frac{1}{2 \times \text{转速}} = \frac{60}{2 \times \text{RPM}} \text{ 秒}
$$

**数据传输时间**：

$$
\text{传输时间} = \frac{\text{数据量}}{\text{磁盘传输率}}
$$

$$
\text{磁盘传输率} = \text{每磁道扇区数} \times \text{每扇区字节数} \times \text{转速}
$$

### 28.1.4 磁盘访问时间计算示例

**【例 28-1】** 磁盘转速7200RPM，平均寻道时间8ms，每磁道500个扇区，每扇区512B。

- 平均旋转延迟 = $60 / (2 \times 7200) = 4.17$ ms
- 磁盘传输率 = $500 \times 512 \times 7200/60 = 30\text{MB/s}$
- 传输1个扇区的时间 = $512 / 30\text{MB/s} = 0.017$ ms
- 平均访问时间 = $8 + 4.17 + 0.017 = 12.19$ ms

### 28.1.5 磁盘调度算法

<div data-component="DiskSchedulingDemo"></div>

| 算法 | 策略 | 特点 |
|------|------|------|
| FCFS | 先来先服务 | 公平但寻道时间长 |
| SSTF | 最短寻道时间优先 | 速度快但可能饥饿 |
| SCAN | 电梯算法 | 公平且高效 |
| C-SCAN | 循环扫描 | 更均匀的等待时间 |

## 28.2 RAID（独立磁盘冗余阵列）

<div data-component="RAIDComparisonViz"></div>

### 28.2.1 RAID的基本思想

将多个磁盘组合成一个阵列，通过**数据条带化**提高性能，通过**冗余**提高可靠性。

### 28.2.2 RAID级别

| 级别 | 特点 | 冗余 | 最少磁盘数 | 磁盘利用率 |
|------|------|------|-----------|-----------|
| RAID 0 | 条带化 | **无** | 2 | 100% |
| RAID 1 | 镜像 | 100% | 2 | 50% |
| RAID 3 | 位交叉+专用校验盘 | 1块盘 | 3 | (n-1)/n |
| RAID 4 | 块交叉+专用校验盘 | 1块盘 | 3 | (n-1)/n |
| RAID 5 | 块交叉+分布式校验 | 1块盘 | 3 | (n-1)/n |
| RAID 6 | 双校验 | 2块盘 | 4 | (n-2)/n |
| RAID 10 | RAID1+RAID0 | 镜像+条带 | 4 | 50% |

### 28.2.3 RAID 0（条带化）

数据分散存储在多个磁盘上。

- 优点：读写性能最好
- 缺点：**无冗余**，一个磁盘故障全部数据丢失

### 28.2.4 RAID 1（镜像）

数据在两个磁盘上各存一份。

- 优点：可靠性最高
- 缺点：磁盘利用率只有50%

### 28.2.5 RAID 5（分布式校验）

数据和校验信息分散存储在所有磁盘上。

- 优点：性能和可靠性平衡
- 缺点：写性能较差（需要计算校验）

## 28.3 固态硬盘（SSD）

<div data-component="SSDStructureViz"></div>

### 28.3.1 基本原理

基于NAND Flash存储，无机械部件。

### 28.3.2 SSD的特点

| 特性 | 说明 |
|------|------|
| 随机读 | 极快（~0.1ms） |
| 写入 | 有擦除限制 |
| 功耗 | 低 |
| 抗震 | 好 |
| 噪音 | 无 |

### 28.3.3 HDD vs SSD

| 特性 | HDD | SSD |
|------|-----|-----|
| 随机读 | 慢（~10ms） | **快（~0.1ms）** |
| 顺序读 | 中等 | 快 |
| 写入寿命 | **无限** | 有限 |
| 功耗 | 高 | **低** |
| 抗震性 | 差 | **好** |
| 成本 | **低** | 高 |
| 容量 | 大 | 中等 |

## 28.4 存储系统综合性能分析

<div data-component="StoragePerformanceCalc"></div>

### 28.4.1 存储层次汇总

| 层次 | 速度 | 容量 | 成本/GB |
|------|------|------|---------|
| 寄存器 | <1ns | B~KB | 极高 |
| L1 Cache | 1~2ns | 32~64KB | 很高 |
| L2 Cache | 5~10ns | 256KB~1MB | 高 |
| 主存 | 50~100ns | GB级 | 中 |
| SSD | 0.1ms | TB级 | 低 |
| HDD | 10ms | TB级 | 最低 |

### 28.4.2 端到端访存性能

从CPU到磁盘的完整访存路径：

```
CPU → L1 Cache → L2 Cache → L3 Cache → 主存 → SSD/HDD
```

每层的命中率和访问时间共同决定总体性能。

## 28.5 本章典型例题

### 例题1：磁盘访问时间

**题目**：磁盘转速10000RPM，平均寻道时间6ms，每磁道400个扇区，每扇区512B。求平均访问时间和传输1MB数据的总时间。

**解答**：

- 平均旋转延迟 = $60 / (2 \times 10000) = 3$ ms
- 磁盘传输率 = $400 \times 512 \times 10000/60 = 33.3\text{MB/s}$
- 平均访问时间 = $6 + 3 + 0.015 = 9.015$ ms
- 传输1MB时间 = $1\text{MB} / 33.3\text{MB/s} = 30$ ms
- 总时间 = $9.015 + 30 = 39.015$ ms

### 例题2：RAID对比

**题目**：有5块磁盘，比较RAID 0、RAID 1、RAID 5的磁盘利用率和容错能力。

**解答**：

| RAID | 利用率 | 容错 |
|------|--------|------|
| RAID 0 | 100% | 0块（无容错） |
| RAID 1 | 50%（2.5块） | 2块 |
| RAID 5 | 80%（4块） | 1块 |

## 28.11 磁盘存储器的详细结构

### 28.11.1 磁头的结构和读写原理

磁头是磁盘读写数据的核心部件，悬浮在盘片表面上方约数纳米处飞行。

**写入原理——磁感应写入**：写入电流通过磁头线圈产生磁场，使盘片表面的磁性颗粒按特定方向磁化，分别表示二进制的0和1。

**读取原理**：

| 技术 | 原理 | 时代 |
|------|------|------|
| 感应式磁头 | 磁通变化产生感应电流 | 早期 |
| MR磁头 | 磁阻效应导致电阻变化 | 1990s |
| GMR巨磁阻磁头 | 多层薄膜结构，磁阻变化更大 | 2000s |
| TMR隧道磁阻磁头 | 隧道结效应，灵敏度更高 | 现代 |

```
写入过程：                     读取过程：
电流 → 线圈 → 磁场 → 盘片      盘片磁化 → 磁阻变化 → 电阻变化 → 电信号

    ┌──────────┐                 ┌──────────┐
    │  写入线圈  │                 │  GMR/TMR  │
    │ ════════ │                 │  传感器   │
    └────┬─────┘                 └────┬─────┘
         ▼ 磁场                      ▼ 电阻变化
    ┌──────────┐                 ┌──────────┐
    │ 磁性涂层  │                 │ 磁性涂层  │
    └──────────┘                 └──────────┘
```

### 28.11.2 盘片的物理结构

盘片由多层材料构成，从内到外依次为：

| 层次 | 材料 | 作用 |
|------|------|------|
| 基板 | 铝合金或玻璃 | 支撑结构（玻璃基板更平更硬） |
| 磁性记录层 | 钴铂合金薄膜 | 存储数据的磁性颗粒 |
| 碳保护层 | 类金刚石碳 | 防氧化、防磨损 |
| 润滑层 | 全氟聚醚 | 减少磁头与盘片的摩擦 |

> [!NOTE]
> 现代磁盘采用垂直磁记录（PMR）技术，磁性颗粒垂直于盘面排列，相比传统纵向记录（LMR）大幅提高了面密度。当前面密度已达 ~1 Tbit/in² 量级，HAMR（热辅助磁记录）和 MAMR（微波辅助磁记录）技术有望将其推至 4~5 Tbit/in²。

### 28.11.3 磁道的组织——区位记录（ZBR）

现代磁盘采用**区位记录**（Zone Bit Recording）：将盘面从内到外划分为多个区（Zone），每个区的每条磁道拥有不同数量的扇区。

```
          外圈磁道（Zone 0）──→ 每道扇区数最多
        ┌──────────────────────┐
        │  ┌──────────────┐    │
        │  │  Zone 1      │    │    数据传输率
        │  │  ┌────────┐  │    │    从外到内递减
        │  │  │ Zone 2  │  │    │
        │  │  │  ┌───┐  │  │    │    外圈 ≫ 内圈
        │  │  │  │   │  │  │    │
        │  │  │  └───┘  │  │    │
        │  │  └────────┘  │    │
        │  └──────────────┘    │
        └──────────────────────┘
          内圈磁道 ──→ 每道扇区数最少
```

| 区域 | 每道扇区数（示例） | 相对传输率 |
|------|-------------------|-----------|
| 外圈（Zone 0） | 1000 | 100% |
| 中间区 | 750 | 75% |
| 内圈（Zone N） | 500 | 50% |

### 28.11.4 扇区的结构详解

每个扇区包含以下字段：

```
┌────────┬──────────┬────────┬────────────┬────────┬─────┐
│同步字段 │地址标记AM │ID字段  │  数据字段   │ECC校验 │间隙 │
│(Sync)  │(AM)      │(C/H/S)│ (512B/4KB) │(CRC)   │(Gap)│
└────────┴──────────┴────────┴────────────┴────────┴─────┘
```

| 字段 | 长度 | 作用 |
|------|------|------|
| 同步字段（Sync） | ~15B | 时钟同步，锁定读取频率 |
| 地址标记（AM） | ~2B | 标识扇区起始位置 |
| ID字段 | ~6B | 记录柱面号、磁头号、扇区号 |
| 数据字段 | 512B 或 4KB | 实际存储的数据 |
| ECC校验 | ~50B | 纠错码，用于检测和纠正读取错误 |
| 间隙（Gap） | 可变 | 扇区之间的间隔 |

### 28.11.5 柱面的概念和优化意义

**柱面（Cylinder）**：所有盘面上同一编号磁道的集合。例如，第100号柱面 = 所有盘面的第100号磁道。

**优化意义**：当数据分布在同一柱面的不同盘面上时，切换磁头（~0磁道移动）远快于移动磁头臂（需要寻道）。因此，操作系统和控制器优先将连续数据放在同一柱面内。

```
        柱面 100
        ┌─────────────┐
盘面 0  │  磁道 100    │ ← 磁头 0
        ├─────────────┤
盘面 1  │  磁道 100    │ ← 磁头 1     切换磁头只需 ~0.1ms
        ├─────────────┤                 移动磁头臂需数 ms
盘面 2  │  磁道 100    │ ← 磁头 2
        ├─────────────┤
盘面 3  │  磁道 100    │ ← 磁头 3
        └─────────────┘
```

## 28.12 磁盘的性能参数计算详解

### 28.12.1 寻道时间的详细组成

寻道时间并非单一值，其内部由多个阶段组成：

| 阶段 | 说明 | 典型占比 |
|------|------|---------|
| 启动时间 | 磁头臂从静止开始移动 | ~0.5ms |
| 加速阶段 | 磁头臂加速到最大速度 | ~10% |
| 匀速飞行阶段 | 磁头臂匀速运动 | ~60% |
| 减速稳定阶段 | 减速并精确定位到目标磁道 | ~30% |

**寻道距离与寻道时间的关系**：

$$
T_s = a + b \times \sqrt{n}
$$

其中 $n$ 为跨越的磁道数，$a$ 为启动/稳定时间常数，$b$ 为比例系数。

> [!NOTE]
> 实际寻道时间与跨越磁道数近似呈平方根关系，而非线性关系。这是因为磁头臂需要加速、匀速、减速的完整过程。平均寻道时间通常为全程寻道时间的 $\frac{1}{3}$。

### 28.12.2 旋转延迟的详细分析

$$
T_{rot}(\theta) = \frac{\theta}{360°} \times \frac{60}{\text{RPM}}
$$

| 情况 | 旋转角度 | 延迟 |
|------|---------|------|
| 最佳情况 | 0° | 0（目标扇区刚好在磁头下） |
| 平均情况 | 180° | 半圈时间 $= \frac{60}{2 \times \text{RPM}}$ |
| 最坏情况 | 360° | 一圈时间 $= \frac{60}{\text{RPM}}$ |

### 28.12.3 内部传输率 vs 外部传输率

| 参数 | 定义 | 计算公式 |
|------|------|---------|
| 内部传输率 | 盘片到磁头的实际数据传输速率 | $R_{inner} = \text{每道扇区数} \times \text{扇区大小} \times \text{RPM}/60$ |
| 外部传输率 | 控制器到主机的接口传输速率 | 由接口决定（SATA 6Gbps 等） |

实际传输速率 = min(内部传输率, 外部传输率)，通常瓶颈在内部传输率。

### 28.12.4 连续数据传输 vs 随机数据传输

| 模式 | 特点 | 每次I/O开销 |
|------|------|-----------|
| 连续（顺序）传输 | 数据在相邻扇区 | 仅第一次需要寻道+旋转 |
| 随机传输 | 数据分散在不同位置 | 每次都需要完整寻道+旋转 |

$$
\text{随机读 N 个扇区的总时间} = N \times (T_{seek} + T_{rot} + T_{transfer})
$$

$$
\text{顺序读 N 个扇区的总时间} = T_{seek} + T_{rot} + N \times T_{transfer}
$$

### 28.12.5 磁盘性能计算例题

**【例 28-3】** 某磁盘参数：转速 10000 RPM，平均寻道时间 6ms，每磁道 600 个扇区，每扇区 512B。计算：

（1）平均旋转延迟和内部传输率；

（2）读取分散在不同磁道上的 10 个扇区的总时间；

（3）读取同一磁道上连续 10 个扇区的总时间。

**解答**：

（1）平均旋转延迟：

$$
T_{rot} = \frac{60}{2 \times 10000} = 3 \text{ ms}
$$

内部传输率：

$$
R = 600 \times 512 \times \frac{10000}{60} = 51{,}200{,}000 \text{ B/s} \approx 48.8 \text{ MB/s}
$$

（2）分散读取（每次完整寻道+旋转+传输）：

$$
T_{transfer} = \frac{512}{48.8 \times 10^6} \approx 0.0105 \text{ ms}
$$

$$
T_{total} = 10 \times (6 + 3 + 0.0105) = 10 \times 9.01 \approx 90.1 \text{ ms}
$$

（3）顺序读取（仅一次寻道+旋转，然后连续传输）：

$$
T_{total} = 6 + 3 + 10 \times 0.0105 = 9.105 \text{ ms}
$$

> [!TIP]
> 本例清晰展示了顺序I/O与随机I/O的巨大性能差异：顺序读取仅需 9.1ms，而随机读取需要 90.1ms，相差约 **10倍**。这正是操作系统进行I/O调度和文件系统优化的核心原因。

**【例 28-4】** 某数据库系统每天产生 100 万次随机磁盘I/O，每次I/O读取 4KB 数据。磁盘参数同【例 28-3】。若换用 SSD（随机读延迟 0.1ms），性能提升多少？

**解答**：

HDD 随机 IOPS：

$$
IOPS_{HDD} = \frac{1}{0.006 + 0.003 + 0.000082} \approx 111 \text{ IOPS}
$$

SSD 随机 IOPS（忽略传输时间）：

$$
IOPS_{SSD} = \frac{1}{0.0001} = 10{,}000 \text{ IOPS}
$$

性能提升：$10{,}000 / 111 \approx 90$ 倍。

HDD 处理 100 万次 I/O 需要：$10^6 / 111 \approx 9009$ 秒 ≈ 2.5 小时。

SSD 处理 100 万次 I/O 需要：$10^6 / 10{,}000 = 100$ 秒 ≈ 1.7 分钟。

**【例 28-5】** 计算磁盘容量。某磁盘有 4 个盘面（8 个记录面），每面 20000 条磁道，每磁道 500 个扇区，每扇区 512B。

**解答**：

$$
\text{容量} = 8 \times 20000 \times 500 \times 512 = 40{,}960{,}000{,}000 \text{ B} = 40 \text{ GB}
$$

若采用 ZBR，外圈磁道扇区更多，实际格式化容量会略高于此计算值。

## 28.13 RAID 各级的详细实现与性能对比

### 28.13.1 RAID 0 条带化详解

RAID 0 将数据按条带（Stripe）分散到所有磁盘上，无冗余。

```
条带(Stripe)
├── 条带单元0 ──→ 磁盘 0
├── 条带单元1 ──→ 磁盘 1
├── 条带单元2 ──→ 磁盘 0
├── 条带单元3 ──→ 磁盘 1
└── ...
```

**条带大小选择对性能的影响**：

| 条带大小 | 适用场景 | 原因 |
|---------|---------|------|
| 小（4~16KB） | 大量小I/O请求 | 单次I/O可利用多盘并行 |
| 大（64~256KB） | 大块顺序传输 | 减少管理开销，顺序吞吐高 |

### 28.13.2 RAID 1 镜像详解

**写入策略**：

| 策略 | 说明 | 可靠性 | 性能 |
|------|------|--------|------|
| 同时写 | 两块盘同时写入 | 最高 | 需等待慢盘 |
| 先主后备 | 先写主盘，再写备盘 | 较高 | 以主盘速度为准 |
| 脏位管理 | 标记未同步的数据块 | 便于恢复 | 需额外元数据 |

### 28.13.3 RAID 3 位交叉+专用校验盘

数据按**位/字节**交叉分散到数据盘，校验信息写入专用校验盘。

- **适合**：大块顺序传输（如视频编辑、科学计算），所有盘同时读/写
- **不适合**：小块随机I/O，每次请求都需要访问所有盘（校验盘成为瓶颈）

### 28.13.4 RAID 4 块交叉+专用校验盘

与 RAID 3 类似，但按**块**（Block）交叉而非位交叉。

**写惩罚问题**：每次写入一个数据块，需要更新校验块：

$$
\text{新校验} = \text{旧校验} \oplus \text{旧数据} \oplus \text{新数据}
$$

一次小写操作的步骤（2读2写）：

```
1. 读旧数据块    ──→ 磁盘
2. 读旧校验块    ──→ 校验盘
3. 计算新校验    ──→ 控制器
4. 写新数据块    ──→ 磁盘
5. 写新校验块    ──→ 校验盘
```

校验盘成为写操作的瓶颈（所有写都集中到校验盘），RAID 5 解决了这个问题。

### 28.13.5 RAID 5 分布式校验详解

校验信息分散存储在所有磁盘上，消除校验盘瓶颈。

**常见的校验分布方式**：

| 布局方式 | 特点 |
|---------|------|
| 左对称（Left-Symmetric） | 校验块按对角线分布，数据块连续 |
| 右对称（Right-Symmetric） | 类似左对称，方向相反 |
| 左异或（Left-Asymmetric） | 简单但数据分布不均匀 |

**RAID 5 校验分布示意图（4块盘）**：

```
条带    磁盘0    磁盘1    磁盘2    磁盘3
  0      D0       D1       D2       P0    ← P0 = D0⊕D1⊕D2
  1      D3       D4       P1       D5    ← P1 = D3⊕D4⊕D5
  2      D6       P2       D7       D8    ← P2 = D6⊕D7⊕D8
  3      P3       D9       D10      D11   ← P3 = D9⊕D10⊕D11
```

### 28.13.6 RAID 6 双校验详解

使用两组校验信息（P 和 Q），可容忍任意 **2 块盘同时故障**。

- **P校验**：简单的异或校验（同 RAID 5）
- **Q校验**：基于 Reed-Solomon 编码的伽罗瓦域运算

$$
P = D_0 \oplus D_1 \oplus D_2 \oplus \cdots
$$

$$
Q = D_0 \times \alpha^0 + D_1 \times \alpha^1 + D_2 \times \alpha^2 + \cdots \quad (\text{GF}(2^8))
$$

RAID 6 的写惩罚比 RAID 5 更严重：每次写需要 **3读3写**（读旧数据、旧P、旧Q；写新数据、新P、新Q）。

### 28.13.7 RAID 10 vs RAID 0+1

| 特性 | RAID 10（先镜像后条带） | RAID 0+1（先条带后镜像） |
|------|----------------------|------------------------|
| 结构 | 先做镜像组，再条带化 | 先做条带组，再镜像 |
| 最少磁盘数 | 4 | 4 |
| 可容忍故障 | 每个镜像组可坏1块 | 一个条带组全坏则故障 |
| 可靠性 | **更高** | 较低 |
| 重建速度 | 快（仅重建单个镜像） | 慢（需重建整个条带组） |

```
RAID 10:                      RAID 0+1:

  镜像组0      镜像组1         条带组0       条带组1
 ┌───┬───┐  ┌───┬───┐      ┌───┬───┐    ┌───┬───┐
 │A0 │A0'│  │A1 │A1'│      │A0 │A1 │    │A0'│A1'│
 │B0 │B0'│  │B1 │B1'│      │B0 │B1 │    │B0'│B1'│
 └───┴───┘  └───┴───┘      └───┴───┘    └───┴───┘
   ↓ 条带化 ↓                  ↓ 镜像 ↓
```

### 28.13.8 RAID 各级综合性能对比

| RAID级别 | 随机读IOPS | 随机写IOPS | 连续读带宽 | 连续写带宽 | 可靠性 | 利用率 |
|---------|-----------|-----------|-----------|-----------|--------|--------|
| RAID 0 | $n \times$ | $n \times$ | $n \times$ | $n \times$ | 无 | 100% |
| RAID 1 | $2 \times$ | $1 \times$ | $2 \times$ | $1 \times$ | 最高 | 50% |
| RAID 5 | $(n-1) \times$ | $(n-1)/4 \times$ | $(n-1) \times$ | $(n-1) \times$ | 容1盘 | (n-1)/n |
| RAID 6 | $(n-2) \times$ | $(n-2)/6 \times$ | $(n-2) \times$ | $(n-2) \times$ | 容2盘 | (n-2)/n |
| RAID 10 | $n \times$ | $n/2 \times$ | $n \times$ | $n/2 \times$ | 每组容1盘 | 50% |

> [!TIP]
> **如何选择 RAID 级别**：
> - 追求极致性能、不在乎数据安全 → RAID 0
> - 追求最高可靠性（如 OS 盘） → RAID 1 或 RAID 10
> - 读多写少、性价比优先 → RAID 5
> - 要求高可靠性（如金融数据） → RAID 6
> - 高性能 + 高可靠性（如数据库） → RAID 10

## 28.14 SSD 的内部结构详解

### 28.14.1 NAND Flash 芯片结构层次

```
SSD
├── 闪存控制器
│   ├── 主机接口（SATA/NVMe）
│   ├── Flash 通道（通常 4~16 通道）
│   ├── ECC 纠错引擎
│   ├── FTL（闪存转换层）
│   └── DRAM 缓存
└── NAND Flash 芯片（多颗）
    └── Die（晶粒）
        └── Plane（平面）
            └── Block（擦除块，通常 256~512 页）
                └── Page（读写页，通常 4~16KB）
                    └── Cell（存储单元）
```

### 28.14.2 SLC/MLC/TLC/QLC 详细对比

| 类型 | 每单元比特数 | 电压级别 | P/E 次数 | 读延迟 | 写延迟 | 成本 |
|------|------------|---------|---------|--------|--------|------|
| SLC | 1 | 2 | ~100,000 | ~25μs | ~200μs | 最高 |
| MLC | 2 | 4 | ~10,000 | ~50μs | ~600μs | 高 |
| TLC | 3 | 8 | ~3,000 | ~75μs | ~900μs | 中 |
| QLC | 4 | 16 | ~1,000 | ~100μs | ~1500μs | 最低 |

> [!NOTE]
> 每个存储单元存储的比特数越多，区分不同电压级别的难度越大，导致耐久性和速度下降。3D NAND 通过纵向堆叠层数（目前可达 200+ 层）来提高容量，缓解了平面缩小带来的可靠性问题。

### 28.14.3 FTL 地址映射机制

FTL（Flash Translation Layer）负责将主机的逻辑地址映射到 Flash 的物理地址。

| 映射方式 | 映射粒度 | 映射表大小 | 适用场景 |
|---------|---------|-----------|---------|
| 页级映射 | 每页一个映射项 | 大（需大 DRAM） | 性能最优 |
| 块级映射 | 每块一个映射项 | 小 | 写放大严重 |
| 混合映射 | 热数据页级，冷数据块级 | 中等 | 折中方案 |

### 28.14.4 写入放大（Write Amplification）

$$
\text{写入放大因子 (WAF)} = \frac{\text{实际写入 Flash 的数据量}}{\text{主机写入的数据量}}
$$

写入放大的原因：
1. **擦除-写入不对等**：Flash 必须先擦除整个块（128~512 页）才能写入单页
2. **垃圾回收**：回收有效页需要搬移数据
3. **磨损均衡**：将数据从磨损严重的块搬移到磨损较少的块

理想 WAF = 1，实际 WAF 通常在 2~5 之间。

### 28.14.5 磨损均衡（Wear Leveling）

| 类型 | 策略 | 特点 |
|------|------|------|
| 动态均衡 | 新数据只写入擦写次数少的块 | 简单但冷数据所在块磨损不均 |
| 静态均衡 | 定期搬移冷数据到高磨损块 | 效果好但开销大 |

### 28.14.6 垃圾回收（Garbage Collection）

```
回收前：                     回收后：
块 A: [有效][无效][有效][有效]  块 A: [擦除完成，可重新写入]
块 B: [有效][有效][无效][有效]  块 B: [有效][有效][有效][有效]
                                ← 从块A搬移来的有效页
步骤：
1. 选择包含最多无效页的块
2. 将块中的有效页搬移到其他块
3. 擦除整个块
4. 标记为可用
```

### 28.14.7 TRIM 命令

当操作系统删除文件时，TRIM 命令通知 SSD 哪些页面已不再使用。SSD 可以提前标记这些页面为无效，使垃圾回收更高效，避免不必要的数据搬移。

> [!NOTE]
> **SSD 的性能特点**：读取速度快（随机读 ~0.1ms），但写入需要先擦除，且擦除以块为单位。这导致：
> - 随机写性能受垃圾回收影响
> - 写入放大降低实际吞吐量
> - 长时间使用后性能可能下降（需要 TRIM 和垃圾回收维护）

## 28.15 存储系统综合性能分析

### 28.15.1 IOPS 的定义和计算

**IOPS**（Input/Output Operations Per Second）：每秒完成的 I/O 操作次数，是衡量存储设备性能的关键指标。

| 指标 | 说明 |
|------|------|
| 随机读 IOPS | 随机位置读取小块数据的速率 |
| 随机写 IOPS | 随机位置写入小块数据的速率 |
| 混合 IOPS | 读写混合负载下的 IOPS |
| 连续带宽 | 顺序读/写大块数据的吞吐量（MB/s） |

### 28.15.2 HDD 的 IOPS 计算

$$
\boxed{IOPS_{HDD} = \frac{1}{T_{seek} + T_{rot} + T_{transfer}}}
$$

其中：
- $T_{seek}$：平均寻道时间（通常 3~12ms）
- $T_{rot}$：平均旋转延迟（转速的一半周期）
- $T_{transfer}$：数据传输时间（通常可忽略不计）

**典型 HDD IOPS 估算**（7200 RPM，平均寻道 8ms）：

$$
IOPS = \frac{1}{8 + 4.17 + 0.05} \approx 82 \text{ IOPS}
$$

### 28.15.3 SSD 的 IOPS 特征

SSD 没有机械部件，IOPS 主要受 Flash 通道数、控制器性能和接口限制：

| 指标 | SATA SSD | NVMe SSD |
|------|----------|----------|
| 随机读 IOPS | ~90,000 | ~500,000+ |
| 随机写 IOPS | ~80,000 | ~400,000+ |
| 连续读带宽 | ~550 MB/s | ~7,000 MB/s |
| 连续写带宽 | ~520 MB/s | ~5,000 MB/s |

### 28.15.4 吞吐量 vs 延迟的权衡

| 场景 | 关键指标 | 优化方向 |
|------|---------|---------|
| 数据库查询 | **延迟** | SSD、减少寻道 |
| 视频流 | **吞吐量** | 大块顺序读、高带宽 |
| Web 服务器 | **IOPS** | 多盘并行、缓存 |
| 大数据分析 | **吞吐量 + IOPS** | RAID + SSD 混合 |

### 28.15.5 存储系统瓶颈识别方法

1. **检查 I/O 等待时间**：`iostat` 查看 `%util` 和 `await`
2. **分析 I/O 模式**：顺序 vs 随机、读 vs 写比例
3. **检查队列深度**：队列深度 > 1 时可利用并行性
4. **检查接口带宽**：确认是否达到 SATA/NVMe 带宽上限

### 28.15.6 存储性能计算例题

**【例 28-6】** 比较 HDD 和 SSD 的随机 IOPS。HDD：7200 RPM，平均寻道 8ms。SSD：随机读延迟 0.1ms。计算各自随机 4KB 读的 IOPS。

**解答**：

HDD：

$$
IOPS_{HDD} = \frac{1}{0.008 + 0.00417 + 0.00004} \approx 82 \text{ IOPS}
$$

SSD（假设 4 通道并行）：

$$
IOPS_{SSD} = \frac{4}{0.0001} = 40{,}000 \text{ IOPS}
$$

比值：$40{,}000 / 82 \approx 488$ 倍。SSD 在随机 I/O 场景下具有压倒性优势。

**【例 28-7】** 存储系统端到端延迟分析。CPU 发出一次磁盘读请求，经过以下路径：Cache 未命中 → 页表查找 100ns → SSD 读取 0.1ms → 数据传输到主存 5μs → Cache 填充 50ns。求总延迟。

**解答**：

$$
T_{total} = 100 \times 10^{-9} + 0.1 \times 10^{-3} + 5 \times 10^{-6} + 50 \times 10^{-9}
$$

$$
T_{total} = 0.0001 + 0.1 + 0.005 + 0.00005 = 0.10515 \text{ ms} \approx 105.15 \text{ μs}
$$

瓶颈在 SSD 读取（占 95.1%），进一步优化需引入预取或多级缓存。

## 28.16 各级存储器的协同工作机制

### 28.16.1 Cache-主存层次的协同

- **管理方式**：硬件自动管理（对程序员完全透明）
- **数据单位**：Cache 行（通常 64B）
- **替换策略**：LRU、LFU、随机替换
- **一致性**：通过写穿（Write-Through）或写回（Write-Back）策略维护

| 写策略 | 优点 | 缺点 |
|--------|------|------|
| 写穿 | 主存始终最新 | 每次写都要访问主存，带宽压力大 |
| 写回 | 减少主存访问 | 主存数据可能过时，需要脏位管理 |

### 28.16.2 主存-辅存层次的协同

- **管理方式**：操作系统管理（虚拟存储器）
- **数据单位**：页（通常 4KB）
- **替换策略**：LRU 及其近似算法（Clock算法）
- **地址转换**：通过页表（Page Table）和 TLB 实现虚实地址转换

### 28.16.3 虚拟存储器的桥梁角色

```
┌─────────────────────────────────────────────────────────────┐
│                        存储层次全景                           │
│                                                             │
│  ┌─────────┐  ┌──────────┐  ┌──────────┐  ┌──────────────┐ │
│  │  CPU    │  │  Cache   │  │   主存    │  │   辅存(SSD)   │ │
│  │ 寄存器  │→ │ (硬件管理)│→ │ (OS管理) │→ │  (OS管理)    │ │
│  │         │  │          │  │          │  │              │ │
│  │ <1ns   │  │ 1~10ns   │  │ 50~100ns │  │ 0.1~10ms    │ │
│  └─────────┘  └──────────┘  └──────────┘  └──────────────┘ │
│       ↕            ↕              ↕              ↕          │
│    透明边界     Cache行替换    虚拟存储器      文件系统       │
│   （编译器）   （硬件完成）   （缺页处理）   （磁盘调度）     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

虚拟存储器连接了主存和辅存，使得程序可以使用比物理内存更大的地址空间。当访问的页面不在主存中时，触发**缺页中断**，OS 从辅存调入所需页面。

### 28.16.4 数据一致性问题

| 层次 | 一致性机制 | 说明 |
|------|-----------|------|
| Cache ↔ 主存 | 写穿/写回 | 由硬件维护 |
| 主存 ↔ 辅存 | 写回 | 由 OS 脏页管理维护 |
| 多级 Cache 间 | 包含性/独占性协议 | MESI 协议等 |

### 28.16.5 缺页对 Cache 的影响

当发生缺页时：
1. CPU 切换到 OS 处理缺页中断
2. OS 从磁盘读入所需页面到主存
3. 更新页表项
4. 刷新 TLB（可能需要）
5. 重新执行引发缺页的指令

缺页处理后，TLB 被刷新可能导致 Cache 中的地址映射失效（对于使用物理地址索引的 Cache），需要进行 Cache 刷新或重填。

> [!TIP]
> **理解存储层次的关键**：每一层都是上一层的缓存。Cache 是主存的缓存，主存是辅存的缓存。每一层利用**局部性原理**（时间局部性和空间局部性）来提升性能。访问模式越局部，层次化存储的优势越大。

### 28.16.6 综合存储系统性能计算

**【例 28-8】** 某系统存储层次如下：

| 层次 | 访问时间 | 命中率 |
|------|---------|--------|
| L1 Cache | 1ns | 95% |
| L2 Cache | 10ns | 90%（在L1未命中的情况下） |
| 主存 | 100ns | 99%（在L2未命中的情况下） |
| SSD | 100μs | 100% |

计算平均访存时间。

**解答**：

$$
T_{avg} = T_{L1\_hit} \times H_{L1} + (1 - H_{L1}) \times T_{L1\_miss}
$$

其中 L1 未命中时的平均访问时间：

$$
T_{L1\_miss} = T_{L2\_hit} \times H_{L2} + (1 - H_{L2}) \times T_{L2\_miss}
$$

$$
T_{L2\_miss} = T_{mem} \times H_{mem} + (1 - H_{mem}) \times T_{SSD}
$$

$$
T_{L2\_miss} = 100 \times 0.99 + (100{,}000) \times 0.01 = 99 + 1000 = 1099 \text{ ns}
$$

$$
T_{L1\_miss} = 10 \times 0.9 + 1099 \times 0.1 = 9 + 109.9 = 118.9 \text{ ns}
$$

$$
T_{avg} = 1 \times 0.95 + 118.9 \times 0.05 = 0.95 + 5.945 = 6.895 \text{ ns}
$$

与直接访问 SSD 相比（100μs = 100,000ns），加速比约为：

$$
\frac{100{,}000}{6.895} \approx 14{,}503 \text{ 倍}
$$

> [!NOTE]
> 即使 SSD 命中率仅为 99%，由于缺页代价极高（100μs vs 主存 100ns），整体平均访存时间仍然受到显著影响。这说明在存储层次中，**越底层的未命中代价越大**，优化每一层的命中率都至关重要。
