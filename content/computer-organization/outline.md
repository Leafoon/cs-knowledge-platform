**文件**: `03-number-system-and-coding.md`
**前置知识**: 第1章（计算机系统概述）
**关键词**: 进制转换、BCD码、校验码、ASCII

#### 3.1 进位计数制
- 十进制、二进制、八进制、十六进制的定义与表示
- 各进制之间的相互转换方法
  - 十进制→二进制：整数除2取余、小数乘2取整

- **易错点**: 小数部分转换的精度丢失问题

#### 3.2 BCD码
- 8421码（最常用）：每4位二进制表示1位十进制
- 余3码：8421码 + 0011
- 2421码：权值为2-4-2-1
- BCD码的加法运算与修正（加6修正）
- **表格**: 0~9的各BCD码表示对比
- **例题**: BCD码加法运算（含进位修正）
- **易错点**: BCD码加法的修正条件（和>9或产生进位时加6）

#### 3.3 字符编码
- ASCII码（7位，128个字符）
- 扩展ASCII码（8位，256个字符）
- Unicode与UTF-8简介
- **表格**: 常用字符的ASCII码值（0~9、A~Z、a~z）

#### 3.4 校验码
- **3.4.1 奇偶校验码**
  - 原理：增加1位校验位使1的个数为奇/偶数
  - 检错能力：只能检测奇数位错误，不能纠错
  - **例题**: 给定数据，求奇校验码和偶校验码

- **3.4.2 海明码（汉明码）**
  - 原理：r个校验位可表示2^r种状态，需满足 2^r >= n+r+1
  - 校验位的确定：位号为2的幂次方的位置
  - 校验组的划分：每个数据位被哪些校验位覆盖
  - 编码过程与检错/纠错过程
  - **公式**: 校验位数 r 的确定公式
  - **例题**: 完整的海明码编码与纠错过程（4位数据→7位海明码）
  - **图示**: 校验位覆盖关系图
  - **易错点**: 位号从1开始计数（非0开始）

- **3.4.3 循环冗余校验（CRC）码**
  - 模2运算（模2加/模2除）
  - CRC编码过程：数据后补r个0，除以生成多项式
  - CRC检错：余数为0则无错
  - **例题**: 给定数据和生成多项式，求CRC码
  - **易错点**: 模2运算与普通算术运算的区别

#### 3.5 本章总结与复习建议

- 考研重点：海明码和CRC码的计算
- 典型综合题：给定原始数据，完成BCD编码→海明校验的全流程

### 第4章 定点数的表示

**文件**: `04-fixed-point-representation.md`
**前置知识**: 第3章（数制与编码）
**关键词**: 原码、反码、补码、移码、溢出检测
**可复用组件**: `IntegerMemoryLayout`

#### 4.1 无符号数与有符号数
- 无符号数：全部位表示数值，范围 [0, 2^n - 1]
- 有符号数：最高位为符号位，0正1负
- 真值与机器数的概念
- **表格**: 8位无符号数范围 vs 有符号数范围

#### 4.2 原码表示法
- 定义：符号位 + 数值的绝对值
- 8位原码范围：-(2^7-1) ~ +(2^7-1)，即 -127 ~ +127
- 原码的特点：+0和-0的表示不同（00000000 和 10000000）
- **图示**: 原码的数轴分布（可复用 `IntegerMemoryLayout`）
- **易错点**: 原码中0有两种表示

#### 4.3 反码表示法
- 正数的反码 = 原码
- 负数的反码 = 符号位不变，数值位取反
- 8位反码范围：-(2^7-1) ~ +(2^7-1)
- 反码中0也有两种表示（00000000 和 11111111）
- **例题**: 给定真值，求原码、反码

#### 4.4 补码表示法（重点）
- 补码的数学定义：模运算
- 正数的补码 = 原码
- 负数的补码 = 反码 + 1（取反加1）
- 8位补码范围：-128 ~ +127（注意：-128没有原码和反码）
- 补码的优点：0的表示唯一，运算统一
- **[-128]补 = 10000000** 的推导
- **公式**: n位补码范围 -2^(n-1) ~ 2^(n-1)-1
- **图示**: 补码的数轴分布与模运算示意
- **例题**: 给定真值，求补码；给定补码，求真值
- **易错点**: -128只有补码没有原码和反码、补码取反加1的细节

#### 4.5 移码表示法
- 移码 = 补码的符号位取反
- 用途：浮点数的阶码
- 移码的特点：保持数据的大小顺序（便于比较）
- **表格**: 8位数的原码/反码/补码/移码完整对照表（-128~+127）

#### 4.6 溢出检测
- 溢出的概念：运算结果超出表示范围
- 一位符号位检测法（同号相加异号不溢出）
- 双符号位检测法（变形补码/模4补码）
  - 00正溢出，01正数，10负数，11负溢出
- 进位检测法（最高数值位进位与符号位进位异或）
- **图示**: 溢出检测的逻辑电路
- **例题**: 给定两个补码数，判断加法是否溢出
- **易错点**: 只有同号相加才可能溢出、异号相加永远不溢出

#### 4.7 本章总结与复习建议
- 四种码制的核心区别对照表
- 从真值到各码制的完整转换流程图
- 典型综合题：给定运算，判断溢出并求结果

**关键词**: 原码乘法、补码乘法、Booth算法、阵列乘法器

#### 6.1 原码一位乘法
- 手算乘法的过程分析
- 符号位单独处理（异或），数值部分相乘
- 逐位乘法→移位→累加的硬件实现
- 运算过程：n次加法+n次右移
- **图示**: 原码一位乘法的数据通路
- **例题**: 完整的原码一位乘法过程（4位×4位）
- **易错点**: 部分积的初始值、移位方向

#### 6.2 补码一位乘法 — Booth算法
- Booth算法的原理：相邻位之差决定操作
  - 0→0或1→1：加0（右移）
  - 0→1：减被乘数（右移）
  - 1→0：加被乘数（右移）
- 算法的优势：统一处理正负数，自动识别连续的1
- **图示**: Booth算法的状态转换图
- **例题**: 完整的Booth乘法过程（含详细每步操作）
- **易错点**: Booth算法的附加位初始为0、循环次数为n

#### 6.3 补码两位乘法 — 改进Booth算法（基4 Booth）
- 一次检查3位（2位当前+1位前次），确定操作
- 乘法次数减半：n/2次
- 操作：±0、±1倍、±2倍被乘数
- **表格**: 基4 Booth编码表（8种情况→5种操作）
- **例题**: 基4 Booth乘法过程

#### 6.4 阵列乘法器
- 纯组合逻辑实现：所有部分积同时生成，用加法器阵列求和
- 速度最快，但硬件开销大
- **图示**: 4×4阵列乘法器结构
- Wallace树压缩简介

#### 6.5 乘法运算的性能分析
- 各种乘法方法的延迟与硬件开销对比
- **表格**: 原码一位/Booth/基4 Booth/阵列乘法器对比

### 第8章 浮点数的表示与运算

**文件**: `08-floating-point.md`
**前置知识**: 第4章（定点数表示）、第5~7章（定点运算）
**关键词**: IEEE 754、规格化、舍入、浮点加减法
**可复用组件**: `FloatFormatBitLayout`、`FloatFormatComparison`

#### 8.1 浮点数的基本概念
- 浮点数的表示：N = M × r^E
  - M：尾数（mantissa/significand）
  - E：阶码（exponent）
  - r：基数（radix）
- 浮点数的机器表示格式：阶符+阶码+数符+尾数
- **图示**: 浮点数的位域划分（可复用 `FloatFormatBitLayout`）

#### 8.2 浮点数的规格化
- 规格化的目的：保证精度，表示唯一
- 左规（尾数左移，阶码减1）：尾数太小时
- 右规（尾数右移，阶码加1）：尾数溢出时
- **例题**: 给定浮点数，判断是否规格化，未规格化的进行规格化

#### 8.3 IEEE 754 标准
- **8.3.1 单精度（32位）**
  - 1位符号 + 8位阶码 + 23位尾数
  - 阶码偏移值：127（excess-127）
  - 隐含的整数位1（1.xxx格式）
  - 特殊值：±0、±∞、NaN、非规格化数

- **8.3.2 双精度（64位）**
  - 1位符号 + 11位阶码 + 52位尾数
  - 阶码偏移值：1023

- **8.3.3 特殊值的编码**
  - 阶码全0 + 尾数全0：±0
  - 阶码全0 + 尾数非0：非规格化数（denormalized）
  - 阶码全1 + 尾数全0：±∞
  - 阶码全1 + 尾数非0：NaN

- **表格**: IEEE 754 单/双精度参数对比（可复用 `FloatFormatComparison`）
- **例题**: 给定十进制数，求IEEE 754单精度编码（完整过程）
- **例题**: 给定IEEE 754编码，求对应的十进制值
- **易错点**: 隐含的整数位1、阶码偏移值的使用、非规格化数的特殊处理

#### 8.4 浮点加减法运算
- 完整的5步流程：
  1. 对阶（小阶向大阶看齐，尾数右移）
  2. 尾数加减
  3. 规格化（左规/右规）
  4. 舍入处理

- **图示**: 浮点加减法流程图
- **例题**: 完整的浮点加法运算过程（含每步的中间结果）
- **易错点**: 对阶时是小阶向大阶看齐（不是大阶向小阶）、右规可能需要舍入

#### 8.5 浮点乘除法运算
- 乘法：阶码相加、尾数相乘
- 除法：阶码相减、尾数相除
- 规格化和舍入

#### 8.6 浮点运算的精度问题
- 舍入模式：向零、向最近偶数、向上、向下
- 精度损失的来源
- **例题**: 展示浮点运算中精度丢失的典型例子

### 第12章 寻址方式

**文件**: `12-addressing-modes.md`
**前置知识**: 第11章（指令格式）
**关键词**: 立即寻址、直接寻址、间接寻址、寄存器寻址、变址寻址、基址寻址

#### 12.1 寻址方式概述
- 为什么需要多种寻址方式
- 有效地址（EA）的概念
- 寻址方式的分类框架

#### 12.2 指令寻址
- 顺序寻址：PC自动递增
- 跳跃寻址：通过转移指令修改PC
- **图示**: 指令寻址示意图

#### 12.3 数据寻址方式
- **12.3.1 立即寻址**
  - 操作数直接在指令中
  - EA = 指令中的地址字段（就是操作数本身）
  - 特点：速度最快，但操作数范围受限

- **12.3.2 直接寻址**
  - 指令中的地址字段就是操作数的有效地址
  - EA = A
  - 特点：简单，但寻址范围受限于地址字段长度

- **12.3.3 间接寻址**
  - 指令中的地址字段指向的存储单元存放的是有效地址
  - EA = (A)
  - 一次间接 vs 多次间接
  - 特点：灵活，但需要多次访存

- **12.3.4 寄存器寻址**
  - 操作数在寄存器中
  - EA = R_i
  - 特点：速度最快，无需访存

- **12.3.5 寄存器间接寻址**
  - 寄存器中存放的是操作数的地址
  - EA = (R_i)
  - 特点：比直接间接寻址快

- **12.3.6 偏移寻址**
  - **基址寻址**: EA = (BR) + A，BR为基址寄存器
    - 用途：程序重定位、存储空间扩展
  - **变址寻址**: EA = (IX) + A，IX为变址寄存器
    - 用途：数组访问、循环操作
  - **相对寻址**: EA = (PC) + A
    - 用途：程序跳转、位置无关代码
  - **表格**: 三种偏移寻址的对比
  - **例题**: 分析数组遍历中变址寻址的应用

- **12.3.7 堆栈寻址**
  - 硬件堆栈（寄存器堆栈）vs 软件堆栈（存储器堆栈）
  - 自动压栈/弹栈

#### 12.4 寻址方式综合对比
- **大型表格**: 所有寻址方式的有效地址计算公式、访存次数、特点、适用场景
- **例题**: 给定指令和内存/寄存器状态，逐条分析指令的执行过程

### 第16章 指令周期与数据通路操作

**文件**: `16-instruction-cycle.md`
**前置知识**: 第15章（CPU数据通路）
**关键词**: 取指周期、间址周期、执行周期、中断周期

#### 16.1 指令周期的基本概念
- 指令周期 = 取指周期 + 间址周期 + 执行周期 + 中断周期

#### 16.2 取指周期（FETCH）
- 完整的取指操作步骤：
  1. PC → MAR（将PC内容送到MAR）
  2. 1 → R（读命令）
  3. M(MAR) → MDR（从存储器读指令到MDR）
  4. MDR → IR（指令送入IR）
  5. OP(IR) → ID（操作码送入指令译码器）
  6. PC + 1 → PC（PC自增）
- **图示**: 取指周期的数据通路与控制信号
- **表格**: 取指周期的微操作序列

#### 16.3 间址周期（INDIRECT）
- 判断是否需要间址周期
- 间址的操作步骤：
  1. Ad(IR) → MAR
  2. 1 → R
  3. M(MAR) → MDR
  4. MDR → Ad(IR)
- **图示**: 间址周期的数据通路

#### 16.4 执行周期（EXECUTE）
- 不同指令的执行周期差异很大
- **分析典型指令的执行周期**：
  - 加法指令（ADD X）：取操作数→ALU运算→存结果
  - 存数指令（STA X）：数据→MAR→MDR→存储器
  - 取数指令（LDA X）：从存储器取数到寄存器
  - 跳转指令（JMP X）：将目标地址送PC
  - 条件跳转指令（BAN X）：判断条件后决定是否跳转
- **表格**: 典型指令的执行周期微操作序列

#### 16.5 中断周期（INTERRUPT）
- 中断周期的操作：
  1. 保存断点（将PC内容压栈或存入特定寄存器）
  2. 形成中断服务程序入口地址
  3. 关中断
- **图示**: 中断周期的数据通路

#### 16.6 指令流水线的基本概念（引入）
- 将指令周期划分为更细的阶段
- 取指→译码→执行→访存→写回的5级流水线
- **图示**: 指令流水线的时空图（初步引入）

#### 16.7 本章总结与复习建议
- 微操作序列是控制单元设计的基础
- 需要能够为每类指令写出完整的微操作序列

### 第19章 流水线技术 — 基本原理

**关键词**: 流水线、吞吐率、加速比、效率、时空图

#### 19.1 流水线的基本概念
- 流水线的思想来源：工厂流水线
- 指令流水线：将指令执行过程分为多个阶段
- 经典5级流水线：取指(IF)→译码(ID)→执行(EX)→访存(MEM)→写回(WB)
- **图示**: 5级指令流水线结构图

#### 19.2 流水线的时空图
- 横轴：时间（时钟周期）
- 纵轴：空间（流水线各段）
- 时空图的绘制方法
- **图示**: 5级流水线的时空图（多条指令的执行过程）

#### 19.3 流水线的性能指标
- **19.3.1 吞吐率（TP）**
  - 定义：单位时间内完成的指令数
  - 理想情况：TP = 1/Δt（每个时钟周期完成一条指令）
  - 实际情况受各种因素影响
  - **公式**: TP = n / (k + n - 1) × Δt

- **19.3.2 加速比（S）**
  - 定义：非流水线执行时间 / 流水线执行时间
  - 理想加速比 = k（流水线段数）
  - **公式**: S = n × k / (k + n - 1)
  - 当 n→∞ 时，S→k

- **19.3.3 效率（E）**
  - 定义：流水线各段的实际使用时间占比
  - **公式**: E = S / k = n / (k + n - 1)

- **例题**: 给定流水线段数和时钟周期，计算完成n条指令的吞吐率、加速比、效率
- **易错点**: 加速比的分母是非流水线执行时间（不是串行执行时间的简单相加）

#### 19.4 流水线的分类
- 部件级流水线（运算流水线）

### 第20章 流水线的冒险与处理

**文件**: `20-pipeline-hazards.md`
**前置知识**: 第19章（流水线基本原理）
**关键词**: 结构冒险、数据冒险、控制冒险、前递/旁路、分支预测

#### 20.1 流水线冒险概述
- 冒险（Hazard）的定义：流水线中阻止下一条指令在预定时钟周期执行的情况
- 冒险的分类：结构冒险、数据冒险、控制冒险
- 冒险导致的流水线暂停（stall/bubble）

#### 20.2 结构冒险（资源冲突）
- **定义**: 多条指令同时需要同一硬件资源
- **典型场景**: 单端口存储器同时取指和访存
- **解决方案**:
  - 资源复制（分离指令存储器和数据存储器/使用独立Cache）
  - 流水线暂停
- **图示**: 结构冒险的时空图（含暂停气泡）

#### 20.3 数据冒险（数据相关）
- **20.3.1 数据冒险的类型**
  - 写后读（RAW）：真数据相关，最常见
  - 读后写（WAR）：反相关，在乱序执行中出现
  - 写后写（WAW）：输出相关，在乱序执行中出现
  - **表格**: 三种数据冒险的对比

- **20.3.2 RAW冒险的检测**
  - 判断条件：源寄存器与前面指令的目的寄存器匹配
  - 间隔1条、2条、3条指令时的冒险情况

- **20.3.3 解决方案 — 软件方法**
  - 插入NOP指令
  - 编译器调度（指令重排）

- **20.3.4 解决方案 — 前递/旁路（Forwarding/Bypassing）**
  - 核心思想：将ALU的结果直接旁路到下一条指令的ALU输入
  - EX/MEM → EX 的前递
  - MEM/WB → EX 的前递
  - **图示**: 前递数据通路
  - **例题**: 分析具体指令序列中的前递过程
  - **易错点**: 前递不能解决所有数据冒险（Load-Use冒险）

- **20.3.5 Load-Use冒险**
  - Load指令在MEM阶段才得到数据，但下一条指令在EX阶段就需要
  - 必须暂停1个时钟周期（即使有前递）
  - **图示**: Load-Use冒险的时空图

#### 20.4 控制冒险（控制相关）
- **20.4.1 控制冒险的来源**
  - 分支指令（条件跳转）：目标地址和是否跳转在EX/MEM阶段才确定
  - 无条件跳转：目标地址可能在ID阶段确定

- **20.4.2 解决方案 — 暂停**
  - 等待分支结果确定后再取下一条指令
  - 代价大：暂停多个周期

- **20.4.3 解决方案 — 延迟分支**
  - 分支指令之后的一条指令总是被执行（延迟槽）
  - 编译器在延迟槽中填入有用的指令
  - 适用于RISC

- **20.4.4 解决方案 — 分支预测**
  - **静态分支预测**
    - 总是预测不跳转
    - 总是预测跳转
    - 根据方向预测（后向跳转预测为跳转，前向预测为不跳转）

  - **动态分支预测**
    - 1位预测器：简单但错误率高
    - 2位饱和计数器：需要连续两次错误才改变预测
    - **图示**: 2位预测器的状态转换图
    - 相关预测器简介
    - 分支目标缓冲（BTB）简介

- **20.4.5 分支预测的性能分析**
  - 预测准确率对流水线性能的影响
  - **公式**: 实际CPI = 理想CPI + 分支频率 × 分支代价 × (1 - 预测准确率)
  - **例题**: 给定分支频率和预测准确率，计算实际CPI

#### 20.5 流水线冒险的综合案例
- **例题**: 给定一段指令序列，完整分析其中的各类冒险及处理方式
- **图示**: 包含前递和暂停的完整时空图

#### 20.6 本章总结与复习建议
- 数据冒险和控制冒险是考研高频考点
- 需要能够画出包含前递和暂停的流水线时空图
- 2位预测器的状态转换是常见题型

### 第25章 高速缓冲存储器（Cache）

- **25.2.1 直接映射（Direct Mapped）**
  - 映射关系：主存块号 mod Cache行数 = Cache行号
  - 地址划分：标记(Tag) + 行索引(Index) + 块内偏移(Offset)

  - 主存块可映射到Cache的任意行
  - 地址划分：标记(Tag) + 块内偏移(Offset)
  - 优点：冲突率最低
  - 缺点：需要比较所有行的标记，硬件复杂
  - **图示**: 全相联映射的工作过程

- **25.2.3 组相联映射（Set Associative）**
  - n路组相联：Cache分为若干组，每组n行
  - 映射关系：主存块号 mod 组数 = 组号
  - 地址划分：标记(Tag) + 组索引(Set Index) + 块内偏移(Offset)
  - 直接映射是1路组相联，全相联是只有1组的组相联
  - **图示**: 2路组相联映射的工作过程
  - **例题**: 给定地址和Cache参数，确定映射位置

- **25.2.4 地址划分的计算**
  - 块内偏移位数 = log2(块大小)
  - 组索引位数 = log2(组数)
  - 标记位数 = 地址总位数 - 组索引位数 - 块内偏移位数
  - **公式**: 详细推导过程
  - **例题**: 给定地址空间、Cache容量、块大小、组相联路数，求地址各位的含义

#### 25.3 Cache的替换算法
- **25.3.1 随机替换（Random）**
  - 随机选择一行替换
  - 实现简单，但效果不稳定

- **25.3.2 先进先出（FIFO）**
  - 替换最早进入Cache的行
  - 可能替换掉频繁使用的行
  - Belady异常现象

- **25.3.3 最近最少使用（LRU）**
  - 替换最长时间未被访问的行
  - 实现方式：计数器法、堆栈法
  - 近似LRU：时钟算法（Clock Algorithm）
  - **图示**: LRU替换过程（可复用 `LRUCacheSim`）
  - **例题**: 给定访问序列，模拟LRU替换过程

- **25.3.4 非最近最少使用（NMRU/Not Recently Used）**

- **表格**: 各替换算法的对比（实现代价、命中率、适用场景）

#### 25.4 Cache的写策略
- **25.4.1 写命中**
  - 写直达（Write-Through）：同时写Cache和主存
    - 优点：一致性好
    - 缺点：写操作慢
    - 写缓冲（Write Buffer）优化
  - 写回（Write-Back）：只写Cache，替换时写回主存
    - 优点：写操作快
    - 缺点：需要脏位（Dirty Bit），一致性维护复杂
  - **表格**: 写直达 vs 写回对比

- **25.4.2 写不命中**
  - 写分配（Write-Allocate）：先将块调入Cache，再写
  - 非写分配（No-Write-Allocate）：直接写主存，不调入Cache
  - 常见组合：写回+写分配、写直达+非写分配

- 地址划分的计算必须熟练掌握

### 第26章 Cache性能分析与优化

#### 26.1 Cache性能的基本公式
- CPU时间 = CPU执行时间 + 访存停顿时间
- 访存停顿时间 = 访存次数 × 缺失率 × 缺失代价
- **公式**: 实际CPI = 理想CPI + 每条指令访存次数 × 缺失率 × 缺失代价 / 时钟周期

#### 26.2 Cache缺失的分类（3C模型）
- **强制缺失（Compulsory/Cold）**
  - 首次访问某个块时的缺失
  - 无法避免，可以通过预取优化
- **容量缺失（Capacity）**
  - Cache容量不足以容纳工作集
  - 可通过增大Cache减少
- **冲突缺失（Conflict）**
  - 直接映射或组相联映射导致的冲突
  - 可通过增大相联度减少
- **表格**: 3C模型的对比
- **易错点**: 全相联Cache只有强制缺失和容量缺失，没有冲突缺失

#### 26.3 降低缺失率的技术
- 增大Cache块大小（利用空间局部性）
- 增大Cache容量
- 增大相联度
- 编译器优化（循环交换、分块）
- **图示**: 各优化技术对缺失率的影响曲线

#### 26.4 降低缺失代价的技术
- 多级Cache（L1→L2→L3）
- 关键字优先（Critical Word First）
- 提前重启（Early Restart）
- 合并写缓冲（Write Buffer Merging）
- 非阻塞Cache（Non-blocking Cache）

#### 26.5 多级Cache的性能分析

  - 仲裁决定谁获得总线使用权

- **29.5.2 集中仲裁方式**

  - **链式查询（菊花链）**

  - **计数器定时查询**

    - 优先级：可通过设置计数器初始值改变
    - **图示**: 计数器定时查询仲裁

  - **独立请求**
    - 每个设备有独立的BRi和BGi线
    - 仲裁器根据优先级直接选择
    - 优点：速度快，可灵活设置优先级
    - 缺点：线数多，扩展困难

  - 优点：指令丰富，编程灵活
  - 缺点：占用存储器地址空间

- **31.2.2 独立编址（I/O映射I/O）**
  - I/O端口有独立的地址空间
  - 使用专用的I/O指令（如x86的IN/OUT）
  - 优点：不占用存储器地址空间
  - 缺点：指令受限

- **表格**: 两种编址方式的对比

#### 31.3 I/O接口
- I/O接口的功能
  - 数据缓冲
  - 状态寄存
  - 控制寄存
  - 地址译码
  - 数据格式转换
  - 信号电平转换
- I/O接口的基本结构
  - 数据缓冲寄存器（DBR）
  - 状态寄存器（SR）
  - 控制寄存器（CR）
  - 地址译码逻辑
- **图示**: I/O接口的内部结构
- 接口的分类（并行/串行、通用/专用）

#### 31.4 程序查询方式
- 工作原理：CPU主动查询设备状态
- 查询流程：
  1. CPU向设备发送查询命令
  2. 读取设备状态寄存器
  3. 判断设备是否就绪
  4. 未就绪→返回步骤2
  5. 就绪→进行数据传输
  6. 传输完成→处理下一个设备
- **图示**: 程序查询方式的流程图
- **优点**: 实现简单，不需要额外硬件
- **缺点**: CPU利用率低（忙等待），实时性差，不能处理多个设备
- **适用场景**: 简单嵌入式系统、低速设备

#### 31.5 本章总结与复习建议
- I/O编址方式是基础概念
- 程序查询方式是最简单的I/O方式，理解其缺点有助于理解后续中断和DMA方式

### 第32章 程序中断方式

**文件**: `32-interrupt-io.md`
**前置知识**: 第22章（中断系统）、第31章（I/O概述）
**关键词**: 中断驱动I/O、中断优先级、中断屏蔽、中断响应
**可复用组件**: `InterruptHandlingFlow`、`InterruptClassificationDemo`、`InterruptOptimizationComparison`、`IDTStructureExplorer`

#### 32.1 中断驱动I/O的基本原理
- CPU不再忙等待，而是去做其他工作
- 设备准备好后主动向CPU发中断请求
- CPU在指令执行完毕后响应中断
- **图示**: 中断驱动I/O的工作流程
- **与程序查询方式的对比**: CPU利用率大幅提高

#### 32.2 中断驱动I/O的完整过程
- **32.2.1 CPU发送I/O命令**
  - CPU向I/O接口写入控制字
  - CPU继续执行其他指令

- **32.2.2 设备准备数据**
  - 设备独立工作，CPU无需等待
  - 设备完成操作后置就绪标志

- **32.2.3 中断请求**
  - 设备通过中断请求线向CPU发送中断请求
  - 中断请求寄存器记录请求

- **32.2.4 中断判优与响应**
  - CPU在指令执行完毕后检查中断
  - 判优逻辑确定最高优先级的中断
  - 响应中断（可复用 `InterruptHandlingFlow`）

- **32.2.5 中断处理**
  - 保护断点（硬件自动）
  - 保护现场（软件）
  - 执行中断服务程序（数据传输）
  - 恢复现场
  - 中断返回

- **图示**: 中断驱动I/O的完整时序图

#### 32.3 中断的性能分析
- 中断响应时间
- 中断处理开销
- **公式**: 有效数据传输率 = 数据量 / (传输时间 + 中断开销)
- **例题**: 对比程序查询和中断方式的有效数据传输率

#### 32.4 多设备中断处理
- 中断优先级的设置
- 中断嵌套
- 中断屏蔽字的设置与分析
- **例题**: 给定多个设备的中断优先级和处理时间，分析中断处理过程
- **易错点**: 嵌套中断中保护断点和现场的层次关系

#### 32.5 中断方式的局限性
- 每次数据传输都需要CPU参与
- 高速设备频繁中断会降低CPU效率
- 这些局限性引出了DMA方式

#### 32.6 本章总结与复习建议
- 中断驱动I/O是程序查询方式的重要改进
- 中断处理的完整流程需要熟练掌握
- 中断优先级和屏蔽字的分析是考研重点

### 第33章 DMA方式

**文件**: `33-dma.md`
**前置知识**: 第32章（中断I/O）
**关键词**: DMA控制器、周期挪用、DMA传送方式
**可复用组件**: `InterruptHandlingFlow`

#### 33.1 DMA的基本概念
- DMA（Direct Memory Access）：直接存储器访问
- 数据传输不经过CPU，由DMA控制器直接控制总线
- 适用场景：高速设备（如磁盘、网卡）的大批量数据传输

#### 33.2 DMA与中断方式的对比
- **表格**: DMA vs 中断方式的详细对比
  - 数据传送：DMA不经CPU vs 中断经CPU
  - 响应时间：DMA在存取周期末 vs 中断在指令周期末
  - 适用场景：DMA适合高速大批量 vs 中断适合低速小批量
  - CPU参与：DMA只在开始和结束时参与 vs 中断每传输一次都参与

#### 33.3 DMA控制器的组成
- DMA控制器中的寄存器：
  - 主存地址寄存器（AR/MAR）：存放主存地址
  - 字计数器（WC/BC）：存放传送数据量
  - 数据缓冲寄存器（BR/DBR）：暂存传输数据
  - 控制/状态寄存器（CSR）：存放控制字和状态
  - DMA请求触发器
  - 中断逻辑
- **图示**: DMA控制器的内部结构

#### 33.4 DMA的传送方式
- **33.4.1 停止CPU访问**
  - DMA传送期间，CPU放弃总线控制权
  - CPU停止一切访存操作
  - 优点：控制简单
  - 缺点：CPU利用率低
  - **图示**: 停止CPU访问的时序

- **33.4.2 周期挪用（Cycle Stealing）**
  - DMA在CPU不访存时窃取总线周期
  - 若CPU正在访存，则DMA等待
  - 若CPU和DMA同时请求访存，DMA优先
  - 优点：CPU浪费少
  - 缺点：DMA传送不连续
  - **图示**: 周期挪用的时序

- **33.4.3 DMA与CPU交替访问**
  - 将CPU周期分为两部分：一部分给CPU，一部分给DMA
  - 类似于透明DMA
  - 优点：CPU和DMA都不等待
  - 缺点：需要CPU周期足够长
  - **图示**: DMA与CPU交替访问的时序

- **表格**: 三种DMA传送方式的对比

#### 33.5 DMA的传送过程
- **33.5.1 预处理（初始化）**
  - CPU设置DMA控制器的参数：
    - 传送方向（读/写）
    - 主存起始地址
    - 传送数据量
    - 设备地址
  - 启动设备

- **33.5.2 数据传送**
  - 设备准备好数据后发DMA请求
  - DMA控制器向总线仲裁器请求总线
  - 获得总线后进行数据传送
  - 每传一个字，地址寄存器+1，字计数器-1
  - 字计数器溢出（为0）时，传送结束

- **33.5.3 后处理**
  - DMA向CPU发中断请求
  - CPU响应中断，进行后处理：
    - 校验数据
    - 判断是否继续传送
    - 检查错误

- **图示**: DMA传送的完整流程图
- **例题**: DMA传送过程中的信号时序分析

#### 33.6 DMA传送的性能分析
- **公式**: DMA传送时间 = 数据量 × 存储周期 / 总线宽度
- **公式**: CPU被暂停的时间
- **例题**: 对比DMA与中断方式的传送效率

#### 33.7 本章总结与复习建议
- DMA是CPU与高速外设之间的高效数据传输方式
- 三种传送方式的对比是考研重点
- 理解DMA控制器的寄存器组成
