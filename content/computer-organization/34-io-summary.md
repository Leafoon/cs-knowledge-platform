# 第三十四章 通道方式与I/O系统综合

通道是比DMA更高级的I/O方式。本章讲解通道的原理，并对整个I/O系统进行综合对比，最后给出课程总结和考研复习建议。

---

## 34.1 通道的基本概念

### 34.1.1 什么是通道

通道是**专用的I/O处理器**，可以执行通道指令（通道程序），独立控制I/O操作。

- CPU只需**启动通道**，数据传输由通道独立完成
- 通道完成后向CPU发中断

### 34.1.2 通道与DMA的区别

| 特性 | DMA | 通道 |
|------|-----|------|
| 灵活性 | 固定传送方式 | **可执行通道程序** |
| CPU参与 | 预处理和后处理 | **只需启动** |
| 复杂度 | 中 | 高 |
| 适用 | 单一设备 | **多种设备** |
| 指令系统 | 无 | **有专用指令** |
| 并行度 | 低（一次一个设备） | **高（可多设备）** |
| 独立性 | 需CPU预处理 | **高度独立** |

## 34.2 通道的类型

### 34.2.1 选择通道

一次**只服务一台设备**，传输完后再服务下一台。

- 适用于高速设备（如磁盘）
- 传输率高

### 34.2.2 数组多路通道

以**数据块**为单位交替传输。

- 适用于中高速设备
- 兼顾效率和共享

### 34.2.3 字节多路通道

以**字节**为单位交替传输。

- 适用于低速设备（如键盘、鼠标）
- 共享性好

### 34.2.4 三种通道对比

<div data-component="ChannelTypeComparison"></div>

| 类型 | 传输单位 | 适用设备 | 特点 |
|------|---------|---------|------|
| 选择通道 | 数据块 | 高速设备 | 传输率高 |
| 数组多路通道 | 数据块 | 中高速设备 | 兼顾 |
| 字节多路通道 | 字节 | 低速设备 | 共享性好 |

## 34.3 通道的工作过程

```
1. CPU执行I/O指令 → 启动通道
2. 通道从内存取出通道程序
3. 通道执行通道指令 → 控制设备传输数据
4. 传送完成 → 通道向CPU发中断
5. CPU进行后处理
```

## 34.6 通道的性能分析

### 34.6.1 通道的最大数据传输率

通道的最大数据传输率取决于通道类型和设备速度：

**选择通道**：

$$
R_{max} = \frac{1}{T_s + T_d}
$$

其中 $T_s$ 是设备选择时间，$T_d$ 是数据传送时间。

**数组多路通道**：

$$
R_{max} = \frac{n}{n \times T_s + T_d}
$$

其中 $n$ 是数据块大小。

**字节多路通道**：

$$
R_{max} = \frac{1}{T_s + T_{byte}}
$$

其中 $T_{byte}$ 是一个字节的传送时间。

### 34.6.2 通道的吞吐量分析

**【例 34-1】** 某系统有1个选择通道，连接4台设备。设备数据传输率如下：

| 设备 | 数据传输率 |
|------|-----------|
| 设备0 | 500KB/s |
| 设备1 | 300KB/s |
| 设备2 | 200KB/s |
| 设备3 | 100KB/s |

选择通道最大传输率 = 1MB/s

总需求 = 500 + 300 + 200 + 100 = 1100KB/s > 1MB/s

**结论**：通道无法满足所有设备的传输需求，需要使用多个通道或降低设备传输率。

│                    通道与CPU的负载均衡                         │
├──────────────────────────────────────────────────────────────┤

## 34.10 深入理解I/O系统

### 34.10.1 I/O方式综合对比详解

**程序查询方式**：
- CPU主动查询设备状态
- CPU忙等待，效率低
- 适用于低速设备

**中断方式**：
- 设备准备好后发中断
- CPU不忙等待，效率高
- 每次传输都需要CPU参与

**DMA方式**：
- 数据传输不经过CPU
- CPU只在开始和结束时参与
- 适用于高速设备

**通道方式**：
- 专用I/O处理器
- CPU只需启动通道
- 适用于多种设备

### 34.10.2 各方式的CPU参与程度

```
程序查询 > 中断 > DMA > 通道
   ↑        ↑      ↑      ↑
  最高      中     低    最低
```

### 34.10.3 各方式的有效传输率

| 方式 | 有效传输率 |
|------|-----------|
| 程序查询 | $\frac{数据量}{查询时间 + 传输时间}$ |
| 中断 | $\frac{数据量}{传输时间 + 中断开销}$ |
| DMA | $\frac{数据量}{传输时间}$ |
| 通道 | $\frac{数据量}{传输时间}$（通道独立） |

### 34.10.4 I/O方式选择

| 条件 | 推荐方式 |
|------|---------|
| 低速设备、简单系统 | 程序查询 |
| 中低速设备 | 中断 |
| 高速设备、大批量数据 | DMA |
| 多种设备、高性能需求 | 通道 |

| 通道与DMA | 通道可以执行程序，DMA不能 |
| 周期挪用 | DMA优先级通常高于CPU |
