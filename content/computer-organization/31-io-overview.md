# 第三十一章 I/O系统概述与程序查询方式

I/O系统负责CPU与外部设备之间的数据交换。本章讲解I/O系统的基本概念、I/O编址方式、I/O接口和最简单的I/O方式——程序查询。

---

## 31.1 I/O系统概述

### 31.1.1 I/O系统的组成

| 组件 | 功能 |
|------|------|
| **I/O设备** | 实际的输入输出设备 |
| **I/O接口** | 连接设备和总线的桥梁 |
| **I/O软件** | 驱动程序、中断处理程序 |

### 31.1.2 I/O设备的分类

| 类型 | 设备 | 特点 |
|------|------|------|
| 输入设备 | 键盘、鼠标、扫描仪 | 向CPU发送数据 |
| 输出设备 | 显示器、打印机 | 接收CPU数据 |
| 输入/输出设备 | 磁盘、网卡、触摸屏 | 双向数据传输 |

### 31.1.3 I/O设备的主要特性

| 特性 | 说明 |
|------|------|
| 数据速率 | 设备的数据传输速度 |
| 数据格式 | 并行/串行、字长 |
| 错误率 | 传输错误的概率 |
| 编码方式 | ASCII、二进制等 |

## 31.2 I/O编址方式

### 31.2.1 统一编址（存储器映射I/O）

I/O端口和存储器**共享地址空间**。

```
地址空间：
0x0000~0x7FFF：存储器
0x8000~0xFFFF：I/O端口
```

- 使用**访存指令**访问I/O端口
- 优点：指令丰富，编程灵活
- 缺点：占用存储器地址空间

### 31.2.2 独立编址（I/O映射I/O）

I/O端口有**独立的地址空间**。

```
存储器地址空间：0x0000~0xFFFF
I/O地址空间：0x00~0xFF
```

- 使用**专用I/O指令**（如x86的IN/OUT）
- 优点：不占用存储器地址空间
- 缺点：指令受限

### 31.2.3 两种编址方式对比

| 特性 | 统一编址 | 独立编址 |
|------|---------|---------|
| 地址空间 | 共享 | 独立 |
| 访问指令 | 访存指令 | 专用I/O指令 |
| 指令丰富度 | 高 | 低 |
| 地址空间占用 | 占用 | 不占用 |
| 典型处理器 | ARM、MIPS | x86 |

## 31.4 程序查询方式

### 31.4.1 工作原理

<div data-component="ProgramQueryFlow"></div>

CPU**主动查询**设备状态，就绪则传输数据。

```
循环：
1. CPU向设备发查询命令
2. 读取状态寄存器
3. 判断是否就绪
   - 未就绪 → 返回步骤2（忙等待）
   - 就绪 → 进行数据传输
4. 传输完成 → 处理下一个设备
```

### 31.4.2 流程图

```
┌──────────┐
│ 发查询命令 │
└────┬─────┘
     ↓
┌──────────┐     否
│ 设备就绪？ │←────┐
└────┬─────┘      │
     │是           │
     ↓             │
┌──────────┐      │
│ 数据传输   │      │
└────┬─────┘      │
     ↓             │
┌──────────┐      │
│ 传输完成？ │──否──┘
└────┬─────┘
     │是
     ↓
┌──────────┐
│ 处理下一个 │
└──────────┘
```

### 31.4.3 优缺点

| 优点 | 缺点 |
|------|------|
| 实现简单 | **CPU利用率低** |
| 硬件少 | **实时性差** |
| 编程简单 | 不能处理多设备 |
| | CPU忙等待 |

### 31.4.4 适用场景

- 简单嵌入式系统
- 低速设备
- 对实时性要求不高的场合

## 31.5 本章典型例题

### 例题1：I/O编址方式

**题目**：某CPU采用统一编址，地址线16根，存储器占用0x0000~0x7FFF。I/O端口的地址范围是什么？

**解答**：

总地址空间 = $2^{16} = 64\text{KB}$

存储器占用 = 0x0000~0x7FFF = 32KB

I/O端口 = 0x8000~0xFFFF = 32KB

### 例题2：程序查询效率

**题目**：CPU时钟频率1GHz，设备数据传输率1MB/s，每次查询需要100个时钟周期。求CPU用于查询的时间占比。

**解答**：

每秒传输数据量 = 1MB = 1048576字节

每秒查询次数 = 1048576次

每秒查询时间 = 1048576 × 100 / 1GHz = 104.86ms

CPU用于查询的时间占比 = 104.86ms / 1000ms = **10.5%**

## 31.11 I/O接口的详细硬件结构

### 31.11.1 数据缓冲寄存器（DBR）

数据缓冲寄存器是CPU与I/O设备之间数据传输的暂存单元：

```
CPU侧                        设备侧
  │                            │
  ↓                            ↓
┌──────────────────────────────────────┐
│              数据缓冲寄存器（DBR）      │
│                                      │
│  ┌──────────────────────────────┐    │
│  │  D7  D6  D5  D4  D3  D2  D1  D0 │    │
│  └──────────────────────────────┘    │
│                                      │
│  写入：CPU → DBR → 设备              │
│  读取：设备 → DBR → CPU              │
└──────────────────────────────────────┘
```

**DBR的硬件实现**：

```
        数据总线
          │
          ↓
    ┌─────┴─────┐
    │  三态门    │←── OE（输出使能）
    └─────┬─────┘
          │
    ┌─────┴─────┐
    │  8D触发器   │←── CLK（时钟）
    │  (74LS374)  │←── LE（锁存使能）
    └─────┬─────┘
          │
    ┌─────┴─────┐
    │  三态门    │←── DIR（方向控制）
    └─────┬─────┘
          │
       I/O设备
```

**双向DBR的工作控制**：

| 操作 | OE | DIR | LE | 数据流向 |
|------|----|-----|-----|---------|
| CPU写DBR | 0 | 0 | ↑ | CPU → DBR |
| DBR输出到设备 | 1 | 1 | — | DBR → 设备 |
| 设备写DBR | 0 | 1 | ↑ | 设备 → DBR |
| DBR输出到CPU | 1 | 0 | — | DBR → CPU |

### 31.11.2 状态寄存器（SR）

状态寄存器记录I/O设备的当前状态，供CPU查询：

```
状态寄存器各位定义（典型）：

  D7    D6    D5    D4    D3    D2    D1    D0
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│ ERR │ RDY │ BUSY│ OVR │ PE  │ FE  │ OE  │ IRQ │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

ERR  - 错误标志（Error）
RDY  - 设备就绪（Ready）
BUSY - 设备忙（Busy）
OVR  - 溢出（Overrun）
PE   - 奇偶错误（Parity Error）
FE   - 帧错误（Frame Error）
OE   - 溢出错误（Overrun Error）
IRQ  - 中断请求（Interrupt Request）
```

**状态寄存器的硬件实现**：

```
               ┌─────────────────────────────────┐
               │          状态寄存器（SR）         │
               │                                 │
  设备状态 ────→│  ┌───┐ ┌───┐ ┌───┐ ┌───┐      │
  (ERR,RDY等)  │  │ Q7│ │ Q6│ │ Q5│ │ Q4│      │
               │  └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘      │
               │    │     │     │     │          │
  CPU读取 ─────│────┴─────┴─────┴─────┴──────→ 数据总线
               │                                 │
  设备状态 ────→│  ┌───┐ ┌───┐ ┌───┐ ┌───┐      │
               │  │ Q3│ │ Q2│ │ Q1│ │ Q0│      │
               │  └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘      │
               │    │     │     │     │          │
               │    └─────┴─────┴─────┴──────→ 数据总线
               └─────────────────────────────────┘
```

### 31.11.3 控制寄存器（CR）

控制寄存器接收CPU发送的控制命令，控制I/O设备的工作方式：

```
控制寄存器各位定义（典型）：

  D7    D6    D5    D4    D3    D2    D1    D0
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│ EN  │ IE  │ RST │MODE1│MODE0│ —   │ —   │ GO  │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

EN    - 设备使能（Enable）
IE    - 中断使能（Interrupt Enable）
RST   - 设备复位（Reset）
MODE1 - 工作模式选择位1
MODE0 - 工作模式选择位0
GO    - 启动操作
```

**控制寄存器的工作逻辑**：

```
CPU写入CR
     │
     ↓
┌────┴────┐
│ CR各位   │
├─────────┤
│ D7=EN   │──→ 设备使能门控
│ D6=IE   │──→ 中断控制门控
│ D5=RST  │──→ 复位信号产生
│ D4:3=MODE│──→ 模式选择多路器
│ D0=GO   │──→ 启动控制逻辑
└─────────┘
```

### 31.11.4 地址译码逻辑

地址译码逻辑负责选择I/O接口中的特定寄存器：

```
地址总线
  │
  ↓
┌─────────────────────────────────────────┐
│              地址译码逻辑                 │
│                                         │
│  A7 A6 A5 A4 A3 A2 A1 A0              │
│  │  │  │  │  │  │  │  │              │
│  ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓              │
│  ┌──────────────────────────┐          │
│  │  高位译码（设备选择）       │          │
│  │  1010 00xx → 本设备       │          │
│  └────────────┬─────────────┘          │
│               │ CS（片选）              │
│  ┌────────────┴─────────────┐          │
│  │  低位译码（寄存器选择）     │          │
│  │  A1 A0 → 选择寄存器       │          │
│  └──────────────────────────┘          │
│                                         │
│  A1=0, A0=0 → DBR（数据缓冲寄存器）     │
│  A1=0, A0=1 → SR（状态寄存器）          │
│  A1=1, A0=0 → CR（控制寄存器）          │
│  A1=1, A0=1 → 保留                     │
└─────────────────────────────────────────┘
```

**地址译码器的门级实现**：

```
CS = A7 · A6' · A5 · A4' · A3' · A2' · (IOR# + IOW#)

DBR选择 = CS · A1' · A0'
SR选择  = CS · A1' · A0
CR选择  = CS · A1  · A0'

读信号：
  DBR_OE = DBR选择 · IOR#
  SR_OE  = SR选择  · IOR#

写信号：
  DBR_WR = DBR选择 · IOW#
  CR_WR  = CR选择  · IOW#
```

### 31.11.5 完整I/O接口电路

```

│     │    │ DBR  │ ←── DBR_WR  │                    │
│     │    └──┬───┘             │                    │
│     │       │                 │                    │
│     ├──→ ┌──┴───┐ ←── SR_OE  │  地址译码            │
│     │    │  SR  │             │  ┌─────────┐        │
│     │    └──┬───┘             ├──│ A[7:0]  │        │
│     │       │                 │  │         │        │
│     ├──→ ┌──┴───┐ ←── CR_WR  │  │ CS=1010 │        │
│     │    │  CR  │             │  │ A1:A0   │        │
│     │    └──┬───┘             │  └─────────┘        │
│     │       │                 │        ↑            │
│     │       ↓                 │    IOR#/IOW#        │
│     │    ┌─────────────────┐  │                     │
│     │    │  控制逻辑        │  │                     │
│     │    │                 │  │                     │
│     │    └────────┬────────┘  │                     │
│     │             │           │                     │
```

### 31.13.3 程序查询方式的汇编实现

**x86汇编示例（独立编址）**：

```asm
; 查询方式读取串口数据
QUERY_LOOP:
    IN  AL, STATUS_PORT    ; 读取状态端口
    TEST AL, 01H           ; 测试RDY位（D0）
    JZ  QUERY_LOOP         ; 未就绪，继续查询
    
    IN  AL, DATA_PORT      ; 就绪，读取数据端口
    MOV [BUFFER], AL       ; 存储数据
    INC BUFFER             ; 缓冲区指针+1
    
    DEC CX                 ; 计数器-1
    JNZ QUERY_LOOP         ; 未完成，继续
```

**ARM汇编示例（统一编址）**：

```asm
; 查询方式读取GPIO数据
QUERY_LOOP:
    LDR R0, =GPIO_STATUS   ; 加载状态寄存器地址
    LDR R1, [R0]           ; 读取状态
    TST R1, #0x01          ; 测试就绪位
    BEQ QUERY_LOOP         ; 未就绪，继续查询
    
    LDR R0, =GPIO_DATA     ; 加载数据寄存器地址
    LDR R2, [R0]           ; 读取数据
    STR R2, [R3], #4       ; 存储到缓冲区，R3+4
    
    SUBS R4, R4, #1        ; 计数器-1
    BNE QUERY_LOOP         ; 未完成，继续
```

### 31.13.4 程序查询方式的C语言实现

```c
// 查询方式从串口读取N个字节
void poll_read(uint8_t *buffer, int n) {
    for (int i = 0; i < n; i++) {
        // 忙等待，直到设备就绪
        while ((inb(STATUS_PORT) & STATUS_RDY) == 0) {
            // 空循环
        }
        // 读取数据
        buffer[i] = inb(DATA_PORT);
    }
}

// 多设备轮询
void poll_multiple_devices(void) {
    while (1) {
        // 检查设备0（键盘）
        if (inb(KBD_STATUS) & KBD_RDY) {
            uint8_t key = inb(KBD_DATA);
            handle_keyboard(key);
        }
        // 检查设备1（串口）
        if (inb(UART_STATUS) & UART_RDY) {
            uint8_t data = inb(UART_DATA);
            handle_uart(data);
        }
        // 检查设备2（ADC）
        if (inb(ADC_STATUS) & ADC_RDY) {
            uint16_t value = inw(ADC_DATA);
            handle_adc(value);
        }
    }
}
```

> [!NOTE]
> 程序查询方式的核心是"忙等待"（Busy Waiting），CPU在循环中不断检查设备状态，直到设备就绪。这种方式实现简单，但CPU利用率低。

## 31.14 I/O编址方式的硬件实现

### 31.14.1 统一编址的硬件实现

**地址空间划分**：

```
地址总线（32位）：
0x00000000 ─┐
            │  存储器区域
0x7FFFFFFF ─┤
            │  I/O区域（存储器映射）
0xFFFFFFFF ─┘

存储器映射I/O的地址分配示例：
0x00000000 ~ 0x3FFFFFFF：DDR内存
0x40000000 ~ 0x4000FFFF：GPIO控制器
0x40010000 ~ 0x4001FFFF：UART控制器
0x40020000 ~ 0x4002FFFF：SPI控制器
```

**统一编址的地址译码电路**：

```
地址总线 A[31:0]
    │
    ├──→ ┌─────────────┐
    │    │  高位译码器   │
    │    │  A[31:28]    │
    │    │  0100 → I/O  │
    │    └──────┬───────┘
    │           │ CS_I/O
    │           │
    ├──→ ┌──────┴───────┐
    │    │  中位译码器   │
    │    │  A[27:16]    │
    │    │  0001 → UART │
    │    └──────┬───────┘
    │           │ CS_UART
    │           │
    ├──→ ┌──────┴───────┐
    │    │  低位译码器   │
    │    │  A[3:2]      │
    │    │  00 → RBR    │
    │    │  01 → THR    │
    │    │  10 → SR     │
    │    │  11 → CR     │
    │    └──────────────┘
    │
    └──→ 存储器控制器（当A[31:28]!=0100时）
```

**统一编址的读写控制**：

```
读操作：
  MEMR# = 存储器地址范围 AND READ信号
  IOR#  = I/O地址范围 AND READ信号

写操作：
  MEMW# = 存储器地址范围 AND WRITE信号
  IOW#  = I/O地址范围 AND WRITE信号
```

### 31.14.2 独立编址的硬件实现

**独立编址的特点**：

| 特性 | 统一编址 | 独立编址 |
|------|---------|---------|
| 地址线 | 共用存储器地址线 | 独立I/O地址线或复用 |
| 控制信号 | MEMR#/MEMW# | IOR#/IOW#（独立） |
| 指令 | MOV等访存指令 | IN/OUT专用指令 |
| 译码 | 需区分存储器和I/O | 专用I/O译码 |

**x86独立编址的硬件实现**：

```
CPU
 │
 ├── AD[15:0]（地址/数据复用）
 │
 ├── IOR#（I/O读）
 ├── IOW#（I/O写）
 ├── MEMR#（存储器读）
 ├── MEMW#（存储器写）
 │
 └── M/IO#（存储器/I/O选择）
      │
      ├── M/IO#=1 → 存储器操作
      │             IOR#/IOW#无效
      │
      └── M/IO#=0 → I/O操作
                    MEMR#/MEMW#无效
                    IOR#/IOW#有效
```

**独立编址的地址译码电路**：

```
AD[15:0] ──→ ┌────────────────┐
             │  I/O地址译码器   │
IOR#    ──→ │                │
IOW#    ──→ │  A[15:8]：高位  │
M/IO#   ──→ │  A[7:0]：端口地址│
             └────────┬───────┘
                      │
          ┌───────────┼───────────┐
          ↓           ↓           ↓
       CS_PORT0    CS_PORT1    CS_PORT2
       (0x3F8)     (0x3F9)     (0x3FA)
```

### 31.14.3 两种编址方式的电路对比

```
统一编址：
┌──────────────────────────────────────────┐
│ 地址总线 ──→ 地址译码 ──→ 存储器选择信号  │
│                        ──→ I/O选择信号    │
│ 控制总线 ──→ MEMR#/MEMW#（通用）         │
└──────────────────────────────────────────┘

独立编址：
┌──────────────────────────────────────────┐
│ 地址总线 ──→ I/O地址译码 ──→ I/O选择信号 │
│ 控制总线 ──→ IOR#/IOW#（专用）           │
│            ──→ MEMR#/MEMW#（存储器专用）  │

```

### 31.16.1 I/O接口在总线上的挂接

```
   │       │       │       │
```

### 31.16.2 接口与总线的信号连接

**数据线连接**：

```
数据总线 D[7:0]
    │
    ├── 三态缓冲器 ──→ 接口DBR
    │     ↑
    │   OE#（由CS和IOR#控制）
    │
    └── 数据总线上的三态门由接口的CS控制
        未选中时输出高阻态
```

**地址线连接**：

```
地址总线 A[7:0]
    │
    └──→ 接口地址译码逻辑
          │
          ├── CS（片选）
          └── 内部端口选择
```

**控制线连接**：

```
控制总线
    │
    ├── IOR# ──→ 接口（配合CS产生读信号）
    ├── IOW# ──→ 接口（配合CS产生写信号）
    ├── CLK  ──→ 接口（同步操作）
    ├── RST  ──→ 接口（复位）
    └── INT  ←── 接口（中断请求）
```

### 31.16.3 总线负载问题

**总线负载计算**：

```
总线驱动能力：能驱动N个标准负载（TTL门）

每个接口输入负载：1个标准负载

总线上的接口数量 ≤ N

如果接口数量超过N，需要增加总线驱动器：

总线 ──→ 总线驱动器(74LS244) ──→ 接口0
                                ──→ 接口1
                                ──→ ...
                                ──→ 接口N
```

**三态缓冲器的使用**：

```
总线
 │
 ├── 三态缓冲器 ←── 接口0.CS
 │     输出 = 数据0 或 高阻
 │
 ├── 三态缓冲器 ←── 接口1.CS
 │     输出 = 数据1 或 高阻
 │
 └── 三态缓冲器 ←── 接口2.CS
       输出 = 数据2 或 高阻

任何时候只有一个CS有效，其他输出高阻
```

### 31.16.4 接口的总线时序配合

**读操作时序配合**：


## 31.17 程序查询方式的性能分析

<div data-component="IOSystemPerformanceCalc"></div>

### 31.17.1 CPU利用率计算

**程序查询方式的CPU利用率公式**：

设：
- $f_{CPU}$：CPU时钟频率
- $T_{query}$：每次查询所需的CPU周期数
- $R_{device}$：设备数据传输速率（字节/秒）
- $B_{transfer}$：每次传输的数据量（字节）

**单设备查询**：

$$
\text{查询次数/秒} = R_{device} / B_{transfer}
$$

$$
\text{查询时间/秒} = \frac{R_{device}}{B_{transfer}} \times \frac{T_{query}}{f_{CPU}}
$$

$$
\eta_{CPU} = 1 - \frac{R_{device} \times T_{query}}{B_{transfer} \times f_{CPU}}
$$

**【例 31-1】** CPU频率1GHz，设备速率1MB/s，每次查询100周期，每次传输1字节。

$$
\text{查询次数/秒} = 1\text{MB/s} / 1\text{B} = 1048576
$$

$$
\text{查询时间/秒} = 1048576 \times 100 / 10^9 = 0.105\text{s}
$$

$$
\eta_{CPU} = 1 - 0.105 = 89.5\%
$$

### 31.17.2 多设备查询的CPU利用率

**N个设备的轮询查询**：

$$
\eta_{CPU} = \sum_{i=1}^{N} \frac{R_i \times T_{query,i}}{B_i \times f_{CPU}}
$$

**【例 31-2】** 系统有3个设备：

| 设备 | 速率 | 查询周期 | 传输字节数 |
|------|------|---------|-----------|
| 键盘 | 10 B/s | 50 | 1 |
| 鼠标 | 100 B/s | 30 | 3 |
| 串口 | 10 KB/s | 80 | 1 |

CPU频率100MHz。

**计算**：

$$
\eta_{键盘} = \frac{10 \times 50}{1 \times 10^8} = 5 \times 10^{-6}
$$

$$
\eta_{鼠标} = \frac{100 \times 30}{3 \times 10^8} = 10^{-5}
$$

$$
\eta_{串口} = \frac{10240 \times 80}{1 \times 10^8} = 8.192 \times 10^{-3}
$$

$$
\eta_{总} = 5 \times 10^{-6} + 10^{-5} + 8.192 \times 10^{-3} \approx 0.82\%
$$

### 31.17.3 查询延迟分析

**最坏情况延迟**：

在轮询多个设备时，最坏情况下设备等待被查询的时间：

$$
T_{max\_delay} = \sum_{i=1}^{N} T_{query,i}
$$

**平均延迟**：

$$
T_{avg\_delay} = \frac{T_{max\_delay}}{2}
$$

**【例 31-3】** 轮询3个设备，每个查询需要100μs。

$$
T_{max\_delay} = 3 \times 100\mu\text{s} = 300\mu\text{s}
$$

$$
T_{avg\_delay} = 150\mu\text{s}
$$

> [!NOTE]
> 程序查询方式的实时性差，因为设备必须等待CPU来查询。对于实时性要求高的系统，应使用中断方式。

### 31.17.4 查询方式的优化策略

**优化1：优先级轮询**

```
// 高优先级设备查询更频繁
loop:
    查询高优先级设备   // 每次循环都查
    if (计数器 % 2 == 0):
        查询中优先级设备  // 每2次循环查一次
    if (计数器 % 4 == 0):
        查询低优先级设备  // 每4次循环查一次
    计数器++
```

**优化2：批量传输**

```
// 一次查询传输多个数据
loop:
    读取状态寄存器
    if (RDY):
        读取FIFO中的所有数据（而不是单个字节）
```

**优化3：延迟查询**

```
// 不忙等待，而是执行其他任务后再查询
loop:
    执行计算任务
    查询设备
    if (RDY):
        处理数据
    继续计算任务
```

## 31.19 扩展例题

### 例题3：接口寄存器分析

**题目**：某I/O接口有4个端口，地址分别为0x300~0x303。端口0是数据寄存器（R/W），端口1是状态寄存器（只读），端口2是控制寄存器（只写），端口3是保留。画出地址译码电路。

**解答**：

```
地址线 A[7:0]

高位译码（基地址0x300）：
  A7=1, A6=1, A5=0, A4=0, A3=0, A2=0

CS = A7 · A6 · A5' · A4' · A3' · A2' · (IOR# + IOW#)

端口选择：
  PORT0 = CS · A1' · A0'  (0x300) - R/W
  PORT1 = CS · A1' · A0   (0x301) - R only
  PORT2 = CS · A1  · A0'  (0x302) - W only
  PORT3 = CS · A1  · A0   (0x303) - 保留

读信号：
  PORT0_RD = PORT0 · IOR#
  PORT1_RD = PORT1 · IOR#

写信号：
  PORT0_WR = PORT0 · IOW#
  PORT2_WR = PORT2 · IOW#
```

### 例题4：程序查询CPU利用率

**题目**：CPU频率500MHz，串口传输速率9600bps，每次查询需要200周期。计算CPU用于查询的时间占比。

**解答**：

$$
\text{每秒传输字节数} = 9600 / 8 = 1200 \text{字节/秒}
$$

$$
\text{每秒查询次数} = 1200 \text{（每字节一次）}
$$

$$
\text{查询时间/秒} = 1200 \times 200 / 500 \times 10^6 = 4.8 \times 10^{-3}\text{s}
$$

$$
\eta = 4.8 \times 10^{-3} / 1 = 0.48\%
$$

CPU利用率很低（查询开销仅0.48%），这是因为串口速率远低于CPU速度。

### 例题5：多设备查询优化

**题目**：某系统轮询4个设备，查询周期分别为50、100、80、120个时钟周期。如果需要保证任何设备的最大响应延迟不超过1000个时钟周期，这种方案是否可行？

**解答**：

$$
T_{max\_delay} = 50 + 100 + 80 + 120 = 350 \text{个时钟周期}
$$

$$
350 < 1000
$$

**结论**：可行，最大响应延迟为350个时钟周期，满足1000周期的要求。

### 例题6：I/O编址硬件设计

**题目**：某系统使用统一编址，存储器地址范围0x0000~0x7FFF，I/O地址范围0x8000~0xFFFF。设计一个电路，当地址在I/O范围且为读操作时产生IOR信号。

**解答**：

```
IOR = A15 · (MEMR信号)

电路实现：
         A15
          │
          ↓
    ┌─────┴─────┐
    │           │
    └─────┬─────┘
          │
          ├──── AND ──── MEMR ────→ IOR
          │
```

当A15=1（地址在0x8000以上）且MEMR有效时，产生IOR信号。
