# 第十八章 微程序控制器

微程序控制器将"存储程序"的思想应用于控制器设计，用微程序代替硬布线逻辑产生控制信号。**本章是考研重点难点**，需要掌握微指令的编码设计和微地址的形成方式。

---

## 18.1 微程序控制的基本思想

### 18.1.1 核心概念

| 概念 | 定义 | 类比 |
|------|------|------|
| 微命令 | 控制信号 | 机器指令中的操作码 |
| 微操作 | 微命令控制下的操作 | 机器指令的执行 |
| 微指令 | 若干微命令的集合 | 机器指令 |
| 微程序 | 若干微指令的有序集合 | 程序 |
| 机器指令 | 一段微程序 | — |

### 18.1.2 层次关系

```
机器指令1 → 微程序1（微指令1, 微指令2, ..., 微指令n）
机器指令2 → 微程序2（微指令1, 微指令2, ..., 微指令m）
...
```

### 18.1.3 微程序控制的基本原理

1. 每条机器指令对应一段**微程序**
2. 微程序存放在**控制存储器**（CM）中
3. 执行机器指令时，从CM中取出对应的微程序
4. 逐条执行微指令，产生控制信号

### 18.1.4 与硬布线控制器的对比

| 特性 | 硬布线 | 微程序 |
|------|--------|--------|
| 控制信号来源 | 组合逻辑 | 控制存储器 |
| 速度 | **快** | 较慢 |
| 灵活性 | 差 | **好** |
| 设计复杂度 | 高（指令多时） | 中等 |
| 修改/扩展 | 困难 | **容易** |
| 适用场景 | **RISC** | **CISC** |

## 18.2 控制存储器（CM）

### 18.2.1 概念

存放所有微程序的ROM，位于CPU内部。

$$
\text{CM容量} = \text{微指令条数} \times \text{微指令字长}
$$

### 18.2.2 与主存的区别

| 特性 | 控制存储器（CM） | 主存 |
|------|-----------------|------|
| 位置 | CPU内部 | CPU外部 |
| 存储内容 | 微程序 | 程序和数据 |
| 速度 | 极快 | 较慢 |
| 用户可见 | **否** | 是 |
| 读写 | 只读 | 可读写 |

### 18.2.3 CM的容量计算

**【例 18-1】** 某指令系统有64条指令，每条指令的微程序平均有8条微指令，微指令字长32位。求控制存储器容量。

CM容量 = $64 \times 8 \times 32 = 16384$ 位 = **2KB**

## 18.3 微指令格式

### 18.3.1 水平型微指令

一条微指令可以**同时发出多个微命令**。

```
┌─────────────────────────────────────────────────┐
│ 微命令字段1 │ 微命令字段2 │ ... │ 下地址字段  │
└─────────────────────────────────────────────────┘
```

**编码方式**：

**一、直接编码**

每个微命令占1位，可同时发出。

```
位0: PC_out    位1: MAR_in    位2: Read    ...
```

- 优点：译码简单，速度快
- 缺点：微指令字长很长

**二、字段直接编码**

将**互斥的微命令**分组，每组用编码表示。

```
字段1（3位）: 000=无操作, 001=PC_out, 010=ACC_out, 011=MDR_out
字段2（2位）: 00=无操作, 01=MAR_in, 10=IR_in, 11=ACC_in
```

- 优点：减少微指令字长
- 缺点：需要译码，速度稍慢

**三、字段间接编码**

一个字段的含义由另一个字段决定。

### 18.3.2 垂直型微指令

一条微指令只发出**一个微命令**。

```
┌──────────┬──────────┐
│ 微操作码  │ 微地址   │
└──────────┴──────────┘
```

### 18.3.3 水平型 vs 垂直型

| 特性 | 水平型 | 垂直型 |
|------|--------|--------|
| 并行性 | **高** | 低 |
| 微指令字长 | **长** | 短 |
| 微程序长度 | **短** | 长 |
| 编译难度 | 难 | **易** |
| 速度 | **快** | 慢 |

## 18.4 微地址的形成方式

### 18.4.1 初始微地址

取指微程序的入口地址，由机器指令的操作码**映射**得到。

**映射方法**：操作码 × 微程序长度 + 基地址

**【例 18-2】** 操作码为5，每条指令的微程序有8条微指令，基地址为0。取指微程序的入口地址是多少？

入口地址 = 5 × 8 + 0 = 40

### 18.4.2 后继微地址

**一、计数器方式（µPC自增）**

```
µPC + 1 → µPC
```

类似程序计数器PC，简单但不灵活。

**二、下地址字段方式**

微指令中有下一条微指令的地址字段。

```
┌────────────────┬──────────────┐
│   微命令字段    │  下地址字段   │
└────────────────┴──────────────┘
```

灵活但增加微指令字长。

**三、测试方式**

根据条件选择后继地址。

```
若条件成立 → 地址A
否则 → 地址B
```

### 18.4.3 微地址形成方式对比

| 方式 | 优点 | 缺点 |
|------|------|------|
| 计数器方式 | 简单 | 不灵活 |
| 下地址字段 | 灵活 | 增加字长 |
| 测试方式 | 支持条件分支 | 复杂 |

## 18.5 微程序的执行过程

```
1. 取机器指令 → 操作码映射到微程序入口地址
2. 从CM取出微指令
3. 执行微操作
4. 形成后继微地址
5. 重复2~4直到该微程序执行完毕
6. 返回取指微程序
```

### 18.5.1 执行流程图

```
┌──────────┐
│ 取机器指令 │
└────┬─────┘
     ↓
┌──────────┐
│ 操作码映射 │
│ →入口地址  │
└────┬─────┘
     ↓
┌──────────┐
│ 取微指令   │←─────┐
└────┬─────┘      │
     ↓             │
┌──────────┐      │
│ 执行微操作 │      │
└────┬─────┘      │
     ↓             │
┌──────────┐      │
│ 形成后继   │──────┘
│ 微地址     │
└────┬─────┘
     ↓
┌──────────┐
│ 微程序结束？│──是→ 返回取指
└────┬─────┘
     │否
     └──→ 继续取微指令
```

## 18.6 微程序的优化

### 18.6.1 微指令的编码优化

- 将常用的微命令组合编码

### 18.6.2 微程序的共享

公共微程序段（如取指微程序）可以共享，减少CM容量。

```
取指微程序（共享）
├── 指令1的微程序
├── 指令2的微程序
└── 指令3的微程序
```

### 18.6.3 动态微程序设计

微程序可以在运行时修改，实现指令集的动态扩展。

## 18.8 本章典型例题

### 例题1：微指令编码设计

**题目**：某CPU有15个微命令，其中5个互斥（同一时刻只能发出1个），其余10个可同时发出。设计微指令格式。

**解答**：

**互斥的5个微命令**：用3位编码（$2^3 = 8 > 5$）

**可同时发出的10个微命令**：用10位直接编码

**下地址字段**：假设CM有64个单元，需要6位

微指令字长 = 3 + 10 + 6 = **19位**

### 例题2：微程序长度

**题目**：某指令系统有64条指令，每条指令的微程序平均有8条微指令，微指令字长32位。求控制存储器容量。

**解答**：

CM容量 = $64 \times 8 \times 32 = 16384$ 位 = **2KB**

### 例题3：微地址计算

**题目**：操作码为3，每条指令的微程序有4条微指令，基地址为16。取指微程序的入口地址是多少？

**解答**：

入口地址 = 3 × 4 + 16 = 28

### 例题4：水平型微指令设计

**题目**：设计一个水平型微指令，支持以下微命令：PC_out, ACC_out, MDR_out（互斥），MAR_in, IR_in, ACC_in（互斥），Read, Write（互斥），PC_inc（独立）。

**解答**：

- 字段1（2位）：00=无, 01=PC_out, 10=ACC_out, 11=MDR_out
- 字段2（2位）：00=无, 01=MAR_in, 10=IR_in, 11=ACC_in
- 字段3（1位）：0=无, 1=Read
- 字段4（1位）：0=无, 1=Write
- 字段5（1位）：0=无, 1=PC_inc
- 下地址字段（6位）

微指令字长 = 2 + 2 + 1 + 1 + 1 + 6 = **13位**

## 18.9 本章常见误区与易错点

1. **微程序控制器的控制信号来自控制存储器**：不是来自组合逻辑。

2. **一条机器指令对应一段微程序**：不是一条微指令。

3. **水平型微指令并行性好但字长长**：垂直型相反。

4. **微地址的形成方式有多种**：计数器方式、下地址字段方式、测试方式。

5. **控制存储器在CPU内部**：不是主存。

6. **微程序的共享可以减少CM容量**：取指微程序是典型的共享段。

7. **字段直接编码中互斥的微命令在同一字段**：可同时发出的微命令在不同字段。

## 18.12 深入理解微程序控制器

### 18.12.1 微程序控制器的工作原理

**核心思想**：用"存储程序"的思想来产生控制信号。

**工作流程**：
1. 取机器指令
2. 根据操作码找到对应的微程序入口地址
3. 从控制存储器取出微指令
4. 执行微操作
5. 形成后继微地址
6. 重复3~5直到微程序结束
7. 返回取指微程序

### 18.12.2 控制存储器详解

**控制存储器（CM）** 是存放所有微程序的ROM。

**特点**：
- 位于CPU内部
- 只读
- 速度极快
- 用户不可见

**容量计算**：

$$
\text{CM容量} = \text{微指令条数} \times \text{微指令字长}
$$

CM容量 = $64 \times 8 \times 32 = 16384$ 位 = **2KB**

## 18.13 微程序控制器的设计

### 18.13.1 设计步骤

1. **确定指令系统**：确定支持哪些指令
2. **列出微操作序列**：为每条指令列出微操作
3. **设计微指令格式**：确定编码方式
4. **分配微地址**：为每条微指令分配地址
5. **编写微程序**：将微指令写入控制存储器
6. **验证**：验证微程序的正确性

### 18.13.2 设计示例

**【例 18-4】** 设计一个简单的微程序控制器，支持以下指令：
- ADD（加法）
- LDA（取数）
- STA（存数）
- JMP（跳转）

**步骤1：列出微操作序列**

| 指令 | 取指周期 | 执行周期 |
|------|---------|---------|
| ADD | T1:PC→MAR T2:1→R T3:M→MDR T4:MDR→IR,PC+1 | T1:Ad(IR)→MAR T2:1→R T3:M→MDR T4:ACC+MDR→ACC |
| LDA | 同上 | T1:Ad(IR)→MAR T2:1→R T3:M→MDR T4:MDR→ACC |
| STA | 同上 | T1:Ad(IR)→MAR T2:ACC→MDR T3:1→W T4:MDR→M |
| JMP | 同上 | T1:Ad(IR)→PC |

**步骤2：设计微指令格式**

使用字段直接编码：
- 字段1（2位）：00=无, 01=PC_out, 10=ACC_out, 11=MDR_out
- 字段2（2位）：00=无, 01=MAR_in, 10=IR_in, 11=ACC_in
- 字段3（1位）：0=无, 1=Read
- 字段4（1位）：0=无, 1=Write
- 字段5（1位）：0=无, 1=PC_inc
- 下地址字段（6位）

微指令字长 = 2+2+1+1+1+6 = **13位**

**步骤3：分配微地址**

- 取指微程序：地址 0~3
- ADD微程序：地址 4~7
- LDA微程序：地址 8~11
- STA微程序：地址 12~15
- JMP微程序：地址 16

## 18.14 微程序的优化

### 18.14.1 微指令的编码优化

**字段直接编码**：
- 将互斥的微命令分组
- 每组用编码表示
- 减少微指令字长

**字段间接编码**：
- 一个字段的含义由另一个字段决定
- 进一步减少字长
- 增加译码复杂度

### 18.14.2 微程序的共享

**公共微程序段**：取指微程序可以共享。

```
取指微程序（共享）
├── 指令1的微程序
├── 指令2的微程序
└── 指令3的微程序
```

### 18.14.3 动态微程序设计

**动态微程序设计**：微程序可以在运行时修改。

**优点**：
- 可以动态改变指令集
- 支持仿真不同的指令集

**缺点**：
- 硬件复杂
- 需要可写的控制存储器

│   ├── 计数器方式（µPC自增）

│   └── 测试方式（条件分支）
└── 设计与优化

<div data-component="MicroOpMapping"></div>

    ├── 设计步骤
    ├── 微程序共享
    └── 动态微程序设计

## 18.18 控制存储器（CM）的硬件结构

<div data-component="MicroprogramExecution"></div>

### 18.18.1 CM的物理结构

控制存储器（Control Store）是微程序控制器的核心部件，通常采用ROM或PLA实现。

```
┌─────────────────────────────────────────────────────────────┐
│                    控制存储器（CM）                            │
│                                                             │
│  ┌──────────┐                                               │
│  │ CMAR     │  ← 微地址寄存器                                │
│  │(μPC/CMAR)│                                               │
│  └────┬─────┘                                               │
│       │ 微地址                                              │
│       ▼                                                     │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                    ROM阵列                             │  │
│  │                                                      │  │
│  │  地址    │ 微指令内容                                   │  │
│  │  ────────┼────────────────────────────────────────── │  │
│  │  0x00    │ 1010 0110 0001 0011 0000 0001             │  │
│  │  0x01    │ 0101 1001 0010 0100 0000 0010             │  │
│  │  0x02    │ 0011 0101 0100 1000 0000 0011             │  │
│  │  ...     │ ...                                       │  │
│  └──────────────────────────────────────────────────────┘  │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────┐                                               │
│  │ μIR      │  ← 微指令寄存器                               │
│  │(微指令   │                                               │
│  │ 寄存器)  │                                               │
│  └────┬─────┘                                               │
│       │                                                     │
│       ▼                                                     │
│  控制信号输出                                                │
└─────────────────────────────────────────────────────────────┘
```

### 18.18.2 CM的存储单元

**ROM存储单元**：

```
         字线（地址译码输出）
         │
         ▼
    ┌─────────┐
    │  位线    │
    │  ┌───┐  │
    │  │   │  │  ← 存储单元（有晶体管=1，无晶体管=0）
    │  └─┬─┘  │
    │    │    │
    └────┼────┘
         │
         ▼
      数据输出
```

**ROM的组织方式**：

| 参数 | 说明 | 典型值 |
|------|------|--------|
| 字数 | 微指令条数 | 256~4096 |
| 字长 | 微指令位数 | 32~128位 |
| 读出时间 | 访问延迟 | 5~20ns |
| 功耗 | 静态功耗 | 低 |

### 18.18.3 CM的容量计算

$$
\text{CM容量} = \text{微指令条数} \times \text{微指令字长}
$$

**【例 18-5】** 某指令系统有128条指令，每条指令的微程序平均有12条微指令，微指令字长48位。求CM容量。

$$
\text{CM容量} = 128 \times 12 \times 48 = 73728 \text{位} = 9\text{KB}
$$

**【例 18-6】** CM有256个单元，微指令字长64位。求CM的地址线和数据线数量。

$$
\text{地址线} = \log_2 256 = 8 \text{根}
$$

$$
\text{数据线} = 64 \text{根}
$$

### 18.18.4 CM与主存的对比

| 特性 | 控制存储器（CM） | 主存 |
|------|-----------------|------|
| 位置 | CPU内部 | CPU外部（主板） |
| 存储介质 | ROM/SRAM | DRAM |
| 存储内容 | 微程序 | 程序和数据 |
| 访问速度 | 5~20ns | 50~100ns |
| 容量 | 几KB~几十KB | 几GB~几百GB |
| 用户可见 | 否 | 是 |
| 读写属性 | 只读（静态微程序） | 可读写 |
| 寻址方式 | 微地址 | 物理/虚拟地址 |

> [!NOTE]
> 控制存储器的速度直接影响微程序控制器的性能。为了提高速度，CM通常采用高速SRAM工艺，而非普通的DRAM。

## 18.19 微指令寄存器（CMAR/μIR）的工作过程

### 18.19.1 CMAR（控制存储器地址寄存器）

CMAR也称为μPC（微程序计数器），用于存放当前要执行的微指令地址。

**CMAR的结构**：

```
┌──────────────────────┐
│        CMAR           │
│                      │
│  ┌────────────────┐  │
│  │  微地址（8位）   │  │
│  └────────────────┘  │
│                      │
│  输入来源：           │
│  1. 操作码映射逻辑    │
│  2. μPC自增          │
│  3. 下地址字段       │
│  4. 测试逻辑输出     │
└──────────────────────┘
```

**CMAR的输入选择**：

```
         操作码映射    μPC+1      下地址字段    测试逻辑
            │           │           │           │
            ▼           ▼           ▼           ▼
        ┌───────────────────────────────────────────┐
        │              MUX（多路选择器）              │
        └────────────────────┬──────────────────────┘
                             │
                             ▼
                        ┌─────────┐
                        │  CMAR   │
                        └─────────┘
```

### 18.19.2 μIR（微指令寄存器）

μIR用于存放从CM读出的当前微指令。

**μIR的结构**：

```
┌────────────────────────────────────────────────────────────┐
│                         μIR                                 │
│                                                            │
│  ┌──────────┬──────────┬──────────┬──────────┬──────────┐ │
│  │ 微命令字段│ 微命令字段│ 微命令字段│  测试字段 │ 下地址字段│ │
│  │   字段1   │   字段2   │   字段3   │          │          │ │
│  └────┬─────┴────┬─────┴────┬─────┴────┬─────┴────┬─────┘ │
│       │          │          │          │          │        │
│       ▼          ▼          ▼          ▼          ▼        │
│   译码器1    译码器2    译码器3    测试逻辑   地址选择     │
│       │          │          │          │          │        │
│       ▼          ▼          ▼          ▼          ▼        │
│   控制信号   控制信号   控制信号   条件选择   后继地址     │
└────────────────────────────────────────────────────────────┘
```

### 18.19.3 微指令的读取过程

**时序图**：

```
时钟    ┌──┐  ┌──┐  ┌──┐  ┌──┐
       ┘  └──┘  └──┘  └──┘  └──┘

CMAR   ══════════════════════════
       │← CM读取时间 →│
CM输出 ──────────════════════════
                 │
μIR    ──────────════════════════
                 │
控制信号 ────────────═════════════
```

**读取过程**：

1. CMAR中的微地址送到CM的地址线
2. CM根据地址读出微指令
3. 微指令存入μIR
4. μIR中的微命令字段经译码产生控制信号
5. μIR中的下地址字段用于形成后继微地址

### 18.19.4 微指令的执行时序

**一条微指令的完整执行过程**：

| 阶段 | 操作 | 时钟周期 |
|------|------|---------|
| 1 | CMAR → CM地址线 | T1 |
| 2 | CM读取微指令 | T1~T2 |
| 3 | 微指令 → μIR | T2 |
| 4 | μIR译码产生控制信号 | T2~T3 |
| 5 | 执行微操作 | T3 |
| 6 | 形成后继微地址 → CMAR | T3~T4 |

> [!TIP]
> 为了提高性能，可以采用流水线方式：在执行当前微指令的同时，预取下一条微指令。

## 18.20 水平型vs垂直型微指令的硬件影响

### 18.20.1 水平型微指令的硬件结构

**水平型微指令的特点**：
- 微命令字段直接控制硬件
- 一条微指令可同时发出多个微命令
- 不需要译码或仅需简单译码

**水平型微指令的字长计算**：

$$
\text{字长} = \sum_{i=1}^{n} \lceil \log_2(k_i + 1) \rceil + \text{下地址位数}
$$

其中 $k_i$ 为第 $i$ 个互斥组中的微命令数。

**【例 18-7】** 设计水平型微指令：

| 互斥组 | 微命令 | 编码位数 |
|--------|--------|---------|
| 组1 | PC_out, ACC_out, MDR_out, SP_out | 3位（$2^3=8>4$） |
| 组2 | MAR_in, IR_in, ACC_in, PC_in | 3位 |
| 组3 | Read, Write | 2位（$2^2=4>2$） |
| 独立位 | PC_inc, ALU_add, ALU_sub | 3位 |
| 下地址 | CM有256个单元 | 8位 |

$$
\text{字长} = 3 + 3 + 2 + 3 + 8 = 19\text{位}
$$

**水平型微指令的硬件实现**：

```
μIR（19位）
┌─────┬─────┬────┬─────┬──────────┐
│组1  │组2  │组3 │独立  │ 下地址    │
│3位  │3位  │2位 │3位   │ 8位      │
└──┬──┴──┬──┴──┬─┴──┬──┴────┬─────┘
   │     │     │    │       │
   ▼     ▼     ▼    ▼       ▼
┌─────┐┌─────┐┌──┐┌─────┐┌──────┐
│3-8  ││3-8  ││2-││直接 ││CMAR  │
│译码器││译码器││4 ││输出 ││      │
└──┬──┘└──┬──┘│译│└──┬──┘└──────┘
   │     │   │码│   │
   ▼     ▼   ▼  ▼   │
PC_out  MAR_in Read PC_inc
ACC_out IR_in  Write ALU_add
MDR_out ACC_in      ALU_sub
SP_out  PC_in
```

### 18.20.2 垂直型微指令的硬件结构

**垂直型微指令的特点**：
- 一条微指令只发出一个微命令
- 微操作码需要译码才能产生控制信号
- 字长短，但微程序长

**垂直型微指令的格式**：

```
┌──────────────┬──────────────┐
│  微操作码     │  微地址/操作数 │
│  (8位)       │  (8位)        │
└──────────────┴──────────────┘
```

**垂直型微指令的硬件实现**：

```
μIR（16位）
┌──────────────┬──────────────┐
│  微操作码     │  微地址       │
│  (8位)       │  (8位)        │
└──────┬───────┴──────┬───────┘
       │              │
       ▼              ▼
┌──────────────┐  ┌──────────────┐
│  8-to-256    │  │  地址选择     │
│  译码器       │  │  逻辑        │
└──────┬───────┘  └──────┬───────┘
       │                 │
       ▼                 ▼
    控制信号          后继地址
```

### 18.20.3 硬件影响对比

| 方面 | 水平型 | 垂直型 |
|------|--------|--------|
| μIR字长 | 长（32~128位） | 短（16~32位） |
| CM容量 | 大 | 小 |
| 译码电路 | 少（直接译码） | 多（需要译码器） |
| 控制信号延迟 | 短（直接输出） | 长（需要译码） |
| 微程序长度 | 短 | 长 |
| 并行度 | 高 | 低 |

### 18.20.4 混合型微指令

实际处理器常采用混合型微指令，结合水平型和垂直型的优点：

```
┌──────────┬──────────┬──────────┬──────────┐
│ 直接控制  │ 编码字段  │ 编码字段  │ 下地址    │
│ (水平型)  │ (垂直型)  │ (垂直型)  │          │
└──────────┴──────────┴──────────┴──────────┘
```

## 18.21 微程序的执行过程详细时序

### 18.21.1 微指令周期

一条微指令的执行时间称为**微指令周期**，通常等于一个时钟周期。

**微指令周期的组成**：

```

│                                                             │

│  │ 取微指令  │  │ 译码     │  │ 执行     │                  │
│  │ (FETCH)  │→│ (DECODE) │→│ (EXECUTE)│                  │
│  └──────────┘  └──────────┘  └──────────┘                  │
│                                                             │
│  T1           T2           T3           T4                  │

```

### 18.21.2 微程序执行的时序图

以执行一条ADD指令的微程序为例：

```

       ┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘
       T1     T2     T3     T4     T5     T6     T7     T8

```

### 18.21.3 微程序执行的详细步骤

**取指微程序**（地址0x00~0x03）：

| 微地址 | 微指令内容 | 微操作 | 控制信号 |
|--------|-----------|--------|---------|
| 0x00 | PC_out, MAR_in | PC → MAR | PC_out=1, MAR_in=1 |
| 0x01 | Read, MDR_in | M(MAR) → MDR | Read=1, MDR_in=1 |
| 0x02 | MDR_out, IR_in | MDR → IR | MDR_out=1, IR_in=1 |
| 0x03 | PC_inc, 跳转到执行微程序 | PC+1, 转执行 | PC_inc=1, 下地址=映射 |

**ADD指令的执行微程序**（地址0x04~0x07）：

| 微地址 | 微指令内容 | 微操作 | 控制信号 |
|--------|-----------|--------|---------|
| 0x04 | IR_addr_out, MAR_in | Ad(IR) → MAR | IR_addr_out=1, MAR_in=1 |
| 0x05 | Read, MDR_in | M(MAR) → MDR | Read=1, MDR_in=1 |
| 0x06 | ACC_out, MDR_out, ALU_add, ACC_in | ACC+MDR → ACC | 多个控制信号 |
| 0x07 | 跳转到取指微程序 | 返回取指 | 下地址=0x00 |

## 18.22 微程序控制器的性能分析

### 18.22.1 性能指标

**微指令周期**：

$$
T_{\mu} = T_{CM\_read} + T_{decode} + T_{execute}
$$

其中：
- $T_{CM\_read}$：CM读取时间
- $T_{decode}$：译码时间
- $T_{execute}$：执行时间

**典型值**：

| 参数 | 典型值 |
|------|--------|
| $T_{CM\_read}$ | 10~20ns |
| $T_{decode}$ | 5~10ns |
| $T_{execute}$ | 5~10ns |
| $T_{\mu}$ | 20~40ns |

### 18.22.2 指令执行时间

一条机器指令的执行时间：

$$
T_{inst} = N_{\mu} \times T_{\mu}
$$

其中 $N_{\mu}$ 为该指令的微程序长度（微指令条数）。

**【例 18-8】** 某CPU的微指令周期为30ns，ADD指令的微程序有8条微指令。求ADD指令的执行时间。

$$
T_{ADD} = 8 \times 30\text{ns} = 240\text{ns}
$$

### 18.22.3 与硬布线控制器的性能对比

| 指标 | 硬布线 | 微程序 |
|------|--------|--------|
| 控制信号产生时间 | 1~3ns | 20~40ns |
| 单条指令执行时间 | 短 | 长 |
| CPI | 1~2 | 5~20 |
| 时钟频率 | 高 | 低 |

### 18.22.4 性能优化方法

**1. 减少微程序长度**

通过优化微指令编码，减少每条机器指令的微指令条数。

**2. 提高CM访问速度**

采用高速SRAM作为CM，减少读取时间。

**3. 微指令预取**

在执行当前微指令的同时，预取下一条微指令：

```
时钟    ┌──┐  ┌──┐  ┌──┐  ┌──┐
       ┘  └──┘  └──┘  └──┘  └──┘

执行    ═══════════════════════
预取       ═══════════════════════
```

**4. 流水线执行**

将微指令的取指、译码、执行过程流水线化：

```
微指令  | 时钟周期
        | 1  2  3  4  5  6  7
μI1     | F  D  E
μI2     |    F  D  E
μI3     |       F  D  E
```

### 18.22.5 性能计算示例

**【例 18-9】** 某微程序控制器：
- CM读取时间：15ns
- 译码时间：5ns
- 执行时间：10ns
- 取指微程序：4条微指令
- ADD指令执行微程序：4条微指令
- LDA指令执行微程序：5条微指令

**微指令周期**：

$$
T_{\mu} = 15 + 5 + 10 = 30\text{ns}
$$

**ADD指令执行时间**：

$$
T_{ADD} = (4 + 4) \times 30 = 240\text{ns}
$$

**LDA指令执行时间**：

$$
T_{LDA} = (4 + 5) \times 30 = 270\text{ns}
$$

**平均CPI**（假设ADD和LDA各占50%）：

$$
\text{平均微指令数} = \frac{8 + 9}{2} = 8.5
$$

$$
\text{CPI} = 8.5
$$

## 18.23 动态微程序与可写控制存储

### 18.23.1 静态微程序 vs 动态微程序

| 特性 | 静态微程序 | 动态微程序 |
|------|-----------|-----------|
| CM类型 | ROM | RAM（可写） |
| 微程序修改 | 不可修改 | 可在运行时修改 |
| 灵活性 | 低 | 高 |
| 硬件成本 | 低 | 高 |
| 典型应用 | 固定指令集 | 可配置处理器 |

### 18.23.2 可写控制存储器（WCS）

**WCS的结构**：

```
┌─────────────────────────────────────────────────────────────┐
│                可写控制存储器（WCS）                           │
│                                                             │
│  ┌──────────┐                    ┌──────────┐              │
│  │ CMAR     │───────────────────→│          │              │
│  └──────────┘                    │  SRAM    │              │
│                                  │  阵列    │              │
│  ┌──────────┐                    │          │              │
│  │ 写地址   │───────────────────→│          │              │
│  └──────────┘                    └────┬─────┘              │
│                                       │                    │
│  ┌──────────┐                         │                    │
│  │ 写数据   │────────────────────────→│                    │
│  └──────────┘                         │                    │
│                                       ▼                    │
│                                  ┌──────────┐              │
│                                  │ μIR      │              │
│                                  └──────────┘              │
└─────────────────────────────────────────────────────────────┘
```

### 18.23.3 动态微程序的工作原理

**加载微程序的过程**：

1. CPU从外部存储器读取微程序
2. 通过写操作将微程序写入WCS
3. 写入完成后，WCS作为CM使用

**微程序加载时序**：

```
阶段1：加载微程序
┌──────────────────────────────────────────┐
│ 外部存储器 ──→ 数据总线 ──→ WCS         │
│ 地址总线 ──→ WCS地址                      │
│ 写命令 ──→ WCS写使能                      │
└──────────────────────────────────────────┘

阶段2：执行微程序
┌──────────────────────────────────────────┐
│ CMAR ──→ WCS地址                          │
│ WCS ──→ μIR ──→ 控制信号                  │
└──────────────────────────────────────────┘
```

### 18.23.4 动态微程序的应用

**1. 指令集仿真**

通过加载不同的微程序，可以在同一硬件上仿真不同的指令集：

```
┌─────────────────────────────────────────┐
│              硬件平台                     │
│                                         │
│  微程序A ──→ 仿真x86指令集               │
│  微程序B ──→ 仿真ARM指令集               │
│  微程序C ──→ 仿真MIPS指令集              │
└─────────────────────────────────────────┘
```

**2. 指令集扩展**

在原有指令集基础上，通过加载新的微程序来扩展指令：

```
原有微程序：ADD, SUB, LDA, STA, JMP
新增微程序：MUL, DIV, SIMD指令
```

**3. 性能优化**

针对特定应用，加载优化的微程序：

```
通用微程序：适用于一般应用
优化微程序：针对特定算法（如FFT、矩阵运算）
```

### 18.23.5 动态微程序的挑战

| 挑战 | 说明 |
|------|------|
| 硬件复杂 | 需要可写的CM（SRAM），成本高 |
| 验证困难 | 动态加载的微程序需要在线验证 |
| 安全性 | 恶意微程序可能破坏系统 |
| 兼容性 | 不同的微程序可能不兼容 |

## 18.24 微程序控制器的完整设计实例

### 18.24.1 设计要求

设计一个微程序控制器，支持以下指令：

| 指令 | 操作码 | 功能 |
|------|--------|------|
| ADD | 000 | ACC ← ACC + M[addr] |
| SUB | 001 | ACC ← ACC - M[addr] |
| LDA | 010 | ACC ← M[addr] |
| STA | 011 | M[addr] ← ACC |
| JMP | 100 | PC ← addr |
| BAN | 101 | if ACC<0 then PC←addr |

### 18.24.2 微指令格式设计

采用字段直接编码：

| 字段 | 位数 | 编码 |
|------|------|------|
| 字段1（总线源） | 3位 | 000=无, 001=PC_out, 010=ACC_out, 011=MDR_out, 100=IR_addr_out, 101=SP_out |
| 字段2（总线目的） | 3位 | 000=无, 001=MAR_in, 010=IR_in, 011=ACC_in, 100=PC_in, 101=MDR_in |
| 字段3（存储器） | 2位 | 00=无, 01=Read, 10=Write |
| 字段4（ALU） | 2位 | 00=无, 01=ADD, 10=SUB |
| 字段5（特殊） | 1位 | 0=无, 1=PC_inc |
| 测试条件 | 2位 | 00=无条件, 01=SF=1, 10=ZF=1 |
| 下地址 | 6位 | 后继微地址 |

**微指令字长** = 3 + 3 + 2 + 2 + 1 + 2 + 6 = **19位**

## 18.25 本章补充例题

### 例题5：CM容量与性能

**题目**：某指令系统有64条指令，每条指令的微程序平均有10条微指令，微指令字长40位，微指令周期为25ns。求CM容量和最长指令的执行时间。

**解答**：

CM容量：

$$
\text{CM容量} = 64 \times 10 \times 40 = 25600\text{位} = 3.2\text{KB}
$$

最长指令的执行时间（假设最长微程序为15条微指令）：

$$
T_{max} = 15 \times 25\text{ns} = 375\text{ns}
$$

### 例题6：微指令编码设计

**题目**：某CPU有20个微命令，其中：
- 5个互斥的总线源信号
- 4个互斥的总线目的信号
- 3个互斥的存储器控制信号
- 8个可同时发出的独立信号

CM有128个单元。设计微指令格式。

**解答**：

- 字段1（总线源）：3位（$2^3=8>5$）
- 字段2（总线目的）：3位（$2^3=8>4$）
- 字段3（存储器）：2位（$2^2=4>3$）
- 字段4（独立信号）：8位
- 下地址字段：7位（$2^7=128$）

$$
\text{字长} = 3 + 3 + 2 + 8 + 7 = 23\text{位}
$$

### 例题7：微地址计算

**题目****：某CPU的CM有256个单元，采用操作码映射方式。64条指令，每条指令的微程序有8条微指令，取指微程序在地址0~3。操作码为5的指令的执行微程序入口地址是多少？

**解答****：

取指微程序占用地址0~3，执行微程序从地址4开始。

$$
\text{入口地址} = 4 + 5 \times 8 = 44
$$

### 例题8：微程序与硬布线对比

**题目**：比较微程序控制器和硬布线控制器在以下方面的差异：速度、灵活性、设计复杂度、适用场景。

**解答**：

| 方面 | 硬布线 | 微程序 |
|------|--------|--------|
| 速度 | 快（1~3ns） | 慢（20~40ns） |
| 灵活性 | 差（修改需重新设计） | 好（修改微程序即可） |
| 设计复杂度 | 高（指令多时） | 中等 |
| 适用场景 | RISC、高性能 | CISC、可配置 |

### 例题9：动态微程序设计

**题目**：说明动态微程序设计的优点和缺点。

**解答**：

**优点**：
- 可以在运行时修改指令集
- 支持指令集仿真
- 可以针对特定应用优化

**缺点**：
- 硬件成本高（需要可写CM）
- 验证困难
- 安全性风险
- 启动时需要加载微程序
