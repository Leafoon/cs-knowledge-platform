## 14.1 排序问题定义

### 14.1.1 什么是排序？——一个精确的数学定义

"排序"是我们日常生活中再熟悉不过的动作：图书馆按书号归架、超市货架按保质期摆放、搜索引擎把最相关的结果排在最前面……但要让计算机执行排序，我们需要一个**精确**的定义。

**排序问题的形式化定义**：

> **输入**：$n$ 个元素的序列 $\langle a_1, a_2, \ldots, a_n \rangle$，这些元素来自某个**全序集**（total order），即任意两个元素之间可以比较大小。  
> **输出**：输入序列的一个**排列**（permutation）$\langle a'_1, a'_2, \ldots, a'_n \rangle$，使得 $a'_1 \leq a'_2 \leq \cdots \leq a'_n$。

几个关键术语澄清：

- **全序集**（Total Order）：集合中任意两个元素 $x, y$ 满足：要么 $x \leq y$，要么 $y \leq x$（或两者都成立，即相等）。整数、浮点数、字符串（字典序）都是全序集。
- **排列**（Permutation）：不改变元素本身，只改变元素的**位置**。排序不增加也不删除元素。
- **稳定排序**（Stable Sort）：若 $a_i = a_j$ 且 $i < j$，则排序后 $a_i$ 仍在 $a_j$ 前面——即**相等元素**保持原来的**相对顺序**不变。

> 💡 **为什么"全序"很重要？** 如果元素之间的比较不满足全序（如浮点 NaN：`NaN != NaN`），排序的行为就是未定义的，这是很多 bug 的来源。Python 的 `sorted()` 会在碰到 NaN 时产生不一致结果。

### 14.1.2 稳定排序 vs 不稳定排序——一个被低估的属性

想象你在管理一张**学生成绩表**，初始数据已经按**姓名字母序**排好：

```
[Alice, 85], [Bob, 72], [Carol, 85], [Dave, 90]
```

现在你想按**分数升序**再排一次。

| 排序结果（稳定） | 排序结果（不稳定的某种实现） |
|---|---|
| `[Bob,72], [Alice,85], [Carol,85], [Dave,90]` | `[Bob,72], [Carol,85], [Alice,85], [Dave,90]` |

稳定排序保证了同分数（85）的 Alice 和 Carol 保持**原有的字母序**（Alice 在 Carol 前），而不稳定排序可能打乱这个关系。

**稳定性的实际重要性**：

1. **多键排序**（Multi-key Sort）：先按次要键排序，再按主要键稳定排序，最终得到多键有序结果。这在数据库 ORDER BY 多字段时极为常见。
2. **增量排序**：对已经按某个字段排好的数据追加少量元素后重新排序时，稳定性可避免不必要的位置交换。
3. **外部排序**：磁盘归并时，稳定性保证了分块内部秩序的继承。

**各算法稳定性速查**：
- ✅ **稳定**：插入排序、归并排序、计数排序、基数排序、冒泡排序（标准实现）
- ❌ **不稳定**：选择排序、快速排序（标准实现）、堆排序

### 14.1.3 内排序 vs 外排序

根据数据是否完全放在内存中，排序分为两大类：

| 维度 | 内排序（In-Memory Sort） | 外排序（External Sort） |
|---|---|---|
| 数据规模 | 数据全部放入 RAM | 数据超过 RAM 容量，存于磁盘 |
| 典型算法 | 插入、快排、归并、堆排 | 多路归并、Replacement Selection |
| 瓶颈 | CPU 操作次数（比较/交换） | I/O 次数（磁盘读写代价极大）|
| 复杂度单位 | 比较次数 / 交换次数 | 磁盘块读写次数 |

本章讨论的所有算法均为**内排序**。外排序将在 Chapter 15（归并排序）中通过 K-路归并介绍。

### 14.1.4 衡量排序算法的四个维度

工程上选择排序算法时，需要综合考虑以下四个维度：

| 维度 | 说明 |
|---|---|
| **时间复杂度** | 最坏 / 平均 / 最好三种情况下的比较次数、交换次数 |
| **空间复杂度** | 额外辅助空间（不含输入数组），$O(1)$ 称"原地"（in-place） |
| **稳定性** | 相等元素排序后相对顺序是否不变 |
| **适用场景** | 小数组、几乎有序、大量重复元素、链表、磁盘……|

### 14.1.5 各基础排序算法总览对比表

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 | 适用场景 |
|------|------|------|------|------|------|---------|

| 选择排序 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ❌ | 交换代价极大时 |

## 14.2 插入排序（Insertion Sort）

### 14.2.1 算法直觉：打扑克牌时的自然动作

想象你在打牌，手里已经整理好了一把从小到大的牌（左边小、右边大）。现在发牌员给你递来一张新牌。

你会怎么做？

1. 从右往左扫描你手里的牌，找到第一张比新牌**小**的牌
2. 把新牌插到那张牌的**右边**

这就是插入排序的全部直觉。

**图解过程**（以 `[5, 2, 4, 6, 1, 3]` 为例）：

```
初始：[ 5 | 2  4  6  1  3 ]   ← 竖线左边是"手里的牌"，右边是待发的牌
                                  手里只有一张牌时，自然有序

第1轮：取出 2，与 5 比较，2 < 5，5 右移
       [ 2  5 | 4  6  1  3 ]

第2轮：取出 4，与 5 比较，4 < 5，5 右移；与 2 比较，4 > 2，停止，4 插入
       [ 2  4  5 | 6  1  3 ]

第3轮：取出 6，与 5 比较，6 > 5，停止，6 插入
       [ 2  4  5  6 | 1  3 ]

第4轮：取出 1，连续右移 5、6、4、2，1 插到最左边
       [ 1  2  4  5  6 | 3 ]

第5轮：取出 3，连续右移 4、5、6，3 插入
       [ 1  2  3  4  5  6 ]   ✓ 排序完成
```

**关键动作**：
- **外层循环**：逐一取出待插入元素（从位置 1 到位置 n-1）
- **内层循环**：将当前元素与左侧已排序部分逐一比较，遇到比它大的就向右"腾出位置"，直到找到合适的插入位置

### 14.2.2 Python 与 C++ 实现

```python
def insertion_sort(arr: list[int]) -> None:
    """
    原地插入排序（升序）。
    时间：O(n^2) 最坏，O(n) 最好（已排序）
    空间：O(1)（只用一个 key 变量）
    稳定性：稳定（内层循环条件是严格大于，相等时停止，不移动）

    参数：
        arr: 待排序列表，排序在原地进行，无返回值
    """
    n = len(arr)
    for i in range(1, n):           # 外层：从第 2 个元素开始逐一处理
        key = arr[i]                 # 取出当前待插入元素
        j = i - 1                   # j 从已排序部分的最右端开始

        # 内层：向左找插入位置
        # 条件 1：j >= 0       防止越过数组左边界（边界条件！）
        # 条件 2：arr[j] > key 找到比 key 小的停止（注意：等号时不移动，保证稳定性）
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]     # 元素右移，为 key 腾出位置
            j -= 1

        arr[j + 1] = key            # 最终插入位置：j+1（循环结束时 j 是最后一个 <= key 的位置）

# === 使用示例 ===
arr = [5, 2, 4, 6, 1, 3]
insertion_sort(arr)
print(arr)  # [1, 2, 3, 4, 5, 6]

# === 验证稳定性 ===
data = [(85, 'Alice'), (72, 'Bob'), (85, 'Carol')]
# 按第一个元素（分数）排序，验证相同分数的相对顺序不变
insertion_sort_stable = lambda lst: sorted(lst, key=lambda x: x[0])
print(insertion_sort_stable(data))
# [(72, 'Bob'), (85, 'Alice'), (85, 'Carol')] ← Alice 在 Carol 前：稳定！
```
```cpp
#include <vector>
using namespace std;

/**
 * 原地插入排序（升序）
 *
 * 时间：O(n^2) 最坏，O(n) 最好
 * 空间：O(1)
 * 稳定性：稳定（内层用严格大于，等号时不移动）
 *
 * @param arr  待排序向量，in-place 修改
 */
void insertion_sort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 1; i < n; i++) {
        int key = arr[i];   // 取出当前元素
        int j = i - 1;      // 已排序部分的最右端

        // 边界检查：j >= 0 防止下标越界（C++ 中越界是未定义行为！）
        // 条件：arr[j] > key 而非 >= key，保证相等时不移动（稳定性关键）
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];   // 右移腾位
            j--;
        }
        arr[j + 1] = key;          // 插入到正确位置
    }
}

// 泛型版本（支持任意可比较类型）
template<typename T, typename Compare = less<T>>
void insertion_sort(vector<T>& arr, Compare cmp = Compare{}) {
    int n = arr.size();
    for (int i = 1; i < n; i++) {
        T key = arr[i];
        int j = i - 1;
        while (j >= 0 && cmp(key, arr[j])) {   // cmp(key, arr[j]) 即 key < arr[j]
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

### 14.2.3 循环不变式完整三步证明

循环不变式（Loop Invariant）是证明迭代算法正确性的标准工具，由 CLRS 定义为需要满足以下三个性质的断言：

> **不变式**：在外层 for 循环每次迭代开始时，子数组 `arr[0..i-1]` 包含了原来 `arr[0..i-1]` 的相同 $i$ 个元素，且已经**按顺序排好**。

**① 初始化（Initialization）**：循环开始前（$i = 1$）不变式成立吗？

此时 `arr[0..0]` 只有一个元素 `arr[0]`，单个元素的数组**天然有序**。✓

**② 保持（Maintenance）**：若第 $i$ 次迭代开始前不变式成立，第 $i+1$ 次迭代开始前不变式仍然成立吗？

设第 $i$ 次迭代开始前，`arr[0..i-1]` 已排好序。内层循环将 `arr[i]`（记为 `key`）与 `arr[0..i-1]` 的元素从右往左比较，把所有大于 `key` 的元素右移一格，最终将 `key` 插入正确位置。操作结束后：
- `arr[0..i]` 包含原来 `arr[0..i]` 的全部 $i+1$ 个元素
- 这 $i+1$ 个元素按升序排列

因此第 $i+1$ 次迭代开始前不变式成立。✓

**③ 终止（Termination）**：循环在什么条件下终止？终止时不变式能推出什么？

外层循环在 $i = n$ 时终止（Python 的 `range(1, n)` 不含 $n$）。将 $i = n$ 代入不变式：`arr[0..n-1]`（即完整数组）已排好序，包含所有 $n$ 个原始元素。这正是我们期望的输出。✓

> 🎯 **循环不变式的实用价值**：它不仅是理论工具，更是**调试利器**。写代码前先确定不变式，在关键位置加断言 `assert` 验证，能快速定位 off-by-one 等边界错误。

### 14.2.4 时间复杂度深度分析

设数组大小为 $n$，分析外层循环第 $i$ 次迭代时，内层循环执行次数（即元素移动次数）$t_i$：

$$\text{总比较次数} = \sum_{i=1}^{n-1} t_i$$

**最好情况**（数组已升序排列）：

每次内层循环 `arr[j] > key` 立即为假，$t_i = 1$（只做一次比较，不移动）：

$$T_{\text{best}} = \sum_{i=1}^{n-1} 1 = n - 1 = \Theta(n)$$

插入排序在**几乎有序**数据上非常高效，这是实践中非常重要的特性。

**最坏情况**（数组逆序排列）：

每次内层循环要移动 $i$ 次（`key` 最终插到最左边），$t_i = i$：

$$T_{\text{worst}} = \sum_{i=1}^{n-1} i = \frac{n(n-1)}{2} = \Theta(n^2)$$

**平均情况**（随机排列）：

每次内层循环平均执行 $i/2$ 次（随机位置的元素比已排序部分一半大，一半小）：

$$T_{\text{avg}} = \sum_{i=1}^{n-1} \frac{i}{2} = \frac{1}{2} \cdot \frac{n(n-1)}{2} = \Theta(n^2)$$

**空间复杂度**：只用了一个额外变量 `key`，$S = O(1)$，属于原地排序。

### 14.2.5 适用场景

插入排序在以下几个场景中是**最优或近优**的选择：

1. **小规模数组（$n \leq 20$）**：常数因子极小（代码紧凑，无递归开销），通常比快排快。这也是为什么 STL `std::sort` 在子数组长度低于阈值时切换回插入排序（称为 Introsort 策略）。

2. **几乎有序的数组**：若数组中逆序对数量为 $k$，插入排序总比较次数为 $O(n + k)$。当 $k = O(n)$，时间为 $O(n)$。

3. **在线排序（Online Sorting）**：数据一个一个到来，每来一个就需要维护已知数据有序。插入排序天然支持这一模式。

4. **数据量少的末尾阶段**：排序库（TimSort、Introsort）在子问题缩小到较小规模时，统一切换为插入排序。

### 14.2.6 二分插入排序（Binary Insertion Sort）

标准插入排序的内层循环做了两件事：**查找插入位置**（顺序 $O(i)$ 比较）+ **移动元素**（$O(i)$ 赋值）。

能否优化"查找"这一步？

答案是：可以。由于 `arr[0..i-1]` 已是有序数组，用**二分查找**可以将比较次数降为 $O(\log i)$。

```python
import bisect

def binary_insertion_sort(arr: list[int]) -> None:
    """
    二分插入排序：用 bisect 二分查找插入位置，减少比较次数。
    比较次数：O(n log n)
    移动次数：依然 O(n^2)（插入位置左边所有元素仍需右移）
    总时间：依然 O(n^2)，但常数因子更小（比较操作减少）

    ⚠️ 注意：比较次数的减少并不影响整体渐进复杂度，
           因为移动次数仍是 O(n^2) 的瓶颈。
           但在比较代价远大于移动代价时（如比较字符串），
           二分插入有明显优势。
    """
    for i in range(1, len(arr)):
        key = arr[i]
        # bisect_left：找到第一个 >= key 的位置（保证稳定性）
        # bisect_right：找到第一个 > key 的位置
        pos = bisect.bisect_left(arr, key, 0, i)   # 在 arr[0..i-1] 中二分查找

        # 将 arr[pos..i-1] 整体右移一格（Python 切片方式）
        arr[pos + 1 : i + 1] = arr[pos : i]        # O(i) 次移动，无法避免
        arr[pos] = key
```
```cpp
#include <vector>
#include <algorithm>
using namespace std;

void binary_insertion_sort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 1; i < n; i++) {
        int key = arr[i];

        // lower_bound 在 arr[0..i-1] 中找第一个 >= key 的位置
        // 等价于 bisect_left，保证稳定性
        auto it = lower_bound(arr.begin(), arr.begin() + i, key);
        int pos = it - arr.begin();

        // 将 arr[pos..i-1] 整体右移一格
        // memmove/std::move 比逐个赋值更高效（连续内存）
        for (int j = i; j > pos; j--) {
            arr[j] = arr[j - 1];
        }
        arr[pos] = key;
    }
}
```

**思考**：为什么"减少比较不减少移动"仍然有意义？

在对象类型的排序中（如比较两个大字符串、两个复杂对象），**一次比较**可能花费 $O(L)$ 时间（$L$ 是字符串长度），而**一次移动**只是指针的交换，$O(1)$。此时 $O(n \log n)$ vs $O(n^2)$ 的比较次数差异非常显著。

## 14.3 选择排序（Selection Sort）

### 14.3.1 算法直觉：每次从剩余中找"最小的那个"

插入排序每轮把一个新元素"插入"到已排序区，选择排序的思路恰好相反：每轮从**未排序区**找最小元素，把它"选出来"放到已排序区的末尾。

**图解过程**（以 `[64, 25, 12, 22, 11]` 为例）：

```
第 0 轮：未排序区 [64 25 12 22 11]，最小值是 11（位置 4），与 arr[0]=64 交换
         → [11 | 25 12 22 64]   已排序：[11]，未排序：[25 12 22 64]

第 1 轮：未排序区 [25 12 22 64]，最小值是 12（位置 2），与 arr[1]=25 交换
         → [11 12 | 22 25 64]   已排序：[11 12]，未排序：[22 25 64]

第 2 轮：未排序区 [22 25 64]，最小值是 22（位置 2），已在正确位置，"自交换"
         → [11 12 22 | 25 64]

第 3 轮：未排序区 [25 64]，最小值是 25，已在正确位置
         → [11 12 22 25 | 64]

第 4 轮：只剩一个元素，自然有序 ✓
         → [11 12 22 25 64]
```

**外层循环**：从位置 0 到 n-2，共 n-1 轮  
**内层循环**：每轮扫描所有未排序元素，找最小值下标  
**交换**：将最小值与未排序区首元素交换

### 14.3.2 Python 与 C++ 实现

```python
def selection_sort(arr: list[int]) -> None:
    """
    原地选择排序（升序）。
    时间：Θ(n^2)（无论输入如何，永远执行 n(n-1)/2 次比较）
    空间：O(1)
    稳定性：不稳定（见 14.3.3）
    交换次数：最多 n-1 次（每轮一次交换）—— 在交换代价极高时有优势

    注意：比较次数固定，但交换次数仅 O(n)，这是选择排序相对于其他 O(n^2)
          算法的唯一优势：适合"移动代价很高（如磁带、大块内存移动）"的场景。
    """
    n = len(arr)
    for i in range(n - 1):          # 外层：已排序区域右边界，从 0 到 n-2
        min_idx = i                 # 假设当前位置就是最小值
        for j in range(i + 1, n):  # 内层：在未排序区扫描，找真正的最小值位置
            if arr[j] < arr[min_idx]:
                min_idx = j
        # 将找到的最小值与未排序区首元素交换（即使 min_idx == i 也没关系）
        arr[i], arr[min_idx] = arr[min_idx], arr[i]

# === 使用示例 ===
arr = [64, 25, 12, 22, 11]
selection_sort(arr)
print(arr)  # [11, 12, 22, 25, 64]
```

```cpp
        if (min_idx != i) {

        }
    }
}
```

### 14.3.3 时间复杂度：永远的 Θ(n²)

选择排序的比较次数与输入无关：

- 第 0 轮内层比较：$n-1$ 次
- 第 1 轮内层比较：$n-2$ 次
- ...
- 第 $n-2$ 轮内层比较：$1$ 次

总比较次数：

$$T = (n-1) + (n-2) + \cdots + 1 = \frac{n(n-1)}{2} = \Theta(n^2)$$

**无论输入是否有序，这个数字不变。** 这是选择排序的最大弱点：它不能从"几乎有序"的输入中获益。

相比之下，交换次数只有 $O(n)$（至多 $n-1$ 次），这在某些特殊场景中反而有优势：若"移动数据"的代价远大于"比较数据"的代价（如将大型数据块写入磁带），选择排序可能更合适。

### 14.3.4 选择排序的不稳定性——一个反例

选择排序**不是稳定排序**，下面给出反例：

```
初始：[5a, 5b, 1]   （下标 a < b，即 5a 在 5b 之前，两者值相等）

第 0 轮：未排序区 [5a, 5b, 1]，最小值是 1（位置 2）
         与 arr[0]=5a 交换：[1, 5b, 5a]

现在 5b 在 5a 之前！原来的相对顺序被破坏了。
排序结果：[1, 5b, 5a]
期望（稳定）结果：[1, 5a, 5b]
```

**根本原因**：选择排序的"交换"操作可能把某个元素"跳过"另一个相等元素，将它们的相对顺序打乱。这不是实现 bug，而是算法本身的属性。

> 🎯 **面试考点**：面试官极喜欢问"为什么选择排序不稳定？请举例"。记住这个 `[5a, 5b, 1]` 的例子。

## 14.4 冒泡排序（Bubble Sort）

### 14.4.1 算法直觉：重球沉底，气泡上浮

想象一排装满水的玻璃柱，每个柱子里有一个小球，球越**重**（值越大）越沉。每一"趟"扫描时，我们从左到右两两比较相邻的球，如果左边比右边重，就把它们**对调**。

经过一趟扫描后：**最重的球必然沉到了最右端**。再经过 $n-1$ 趟，所有球都到了正确位置。

**图解过程**（以 `[5, 1, 4, 2, 8]` 为例，目标升序）：

```
第 1 趟（找最大值）：
5,1 → 交换 → 1,5   [1, 5, 4, 2, 8]
5,4 → 交换 → 4,5   [1, 4, 5, 2, 8]
5,2 → 交换 → 2,5   [1, 4, 2, 5, 8]
5,8 → 不交换        [1, 4, 2, 5, 8]  ← 最大值 8 沉底

第 2 趟（在前 n-1 个中找最大值）：
1,4 → 不交换
4,2 → 交换 → 2,4   [1, 2, 4, 5, 8]
4,5 → 不交换        [1, 2, 4, 5, 8]  ← 5 找到正确位置

第 3 趟：
1,2 → 不交换
2,4 → 不交换        [1, 2, 4, 5, 8]  ← 无交换，可提前终止！
```

### 14.4.2 标准实现与"提前终止"优化

```python
def bubble_sort(arr: list[int]) -> None:
    """
    冒泡排序，含提前终止优化。
    时间：最坏 O(n^2)，最好 O(n)（已排序时，第一趟无交换即终止）
    空间：O(1)
    稳定性：稳定（相等元素不交换：条件是严格大于）

    核心优化：若某一趟没有发生任何交换，说明数组已经有序，立即停止。
    这使得冒泡排序在"几乎有序"数据上的性能接近 O(n)。
    """
    n = len(arr)
    for i in range(n - 1):          # 共需 n-1 趟
        swapped = False              # 本趟是否发生了交换（优化标志位）
        for j in range(n - 1 - i): # 每趟比较范围缩短（最右端已是有序的）
            if arr[j] > arr[j + 1]:          # 严格大于：相等时不交换（保稳定性）
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:              # 本趟无交换 → 已全局有序 → 提前结束
            break

# === 使用示例 ===
arr = [5, 1, 4, 2, 8]
bubble_sort(arr)
print(arr)   # [1, 2, 4, 5, 8]

# 已排序输入：只需 1 趟
arr2 = [1, 2, 3, 4, 5]
bubble_sort(arr2)   # O(n) 时间就退出
print(arr2)  # [1, 2, 3, 4, 5]
```
```cpp
#include <vector>

    int n = arr.size();
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;           // 本趟交换标志

        // 每趟只需扫描未排序部分：[0, n-1-i)
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) { // 严格大于，等号不换（保稳定性）
                swap(arr[j], arr[j + 1]);
                swapped = true;
            }
        }

        if (!swapped) break;           // 提前终止：本趟无交换说明已排序
    }
}
```

### 14.4.3 逆序数（Inversions）——量化"无序程度"

这是冒泡排序最精彩的理论洞见，也是面试和竞赛的高频考点。

**定义**：给定序列 $\langle a_1, a_2, \ldots, a_n \rangle$，若 $i < j$ 且 $a_i > a_j$，则称 $(i, j)$ 为一个**逆序对**（Inversion Pair）。序列的**逆序数**（Inversion Count）是所有逆序对的数量。

**例子**：
- 序列 `[2, 3, 8, 6, 1]` 的逆序对：(8,6), (8,1), (6,1), (2,1), (3,1) → 逆序数 = 5
- 完全升序序列 `[1, 2, 3, 4, 5]` → 逆序数 = 0
- 完全降序序列 `[5, 4, 3, 2, 1]` → 逆序数 = $\binom{5}{2} = 10$（最大值）

**逆序数的核心定理**：

> **定理**：冒泡排序（以及插入排序）每次**相邻交换**（swap of adjacent elements）恰好消除一个逆序对，且不会产生新的逆序对。
>
> **推论**：任何仅通过**相邻交换**实现的排序算法，其总交换次数**等于**输入的逆序数。

**证明**：
设 $a_i > a_{i+1}$（发生一次相邻交换），交换后 $(i, i+1)$ 这对逆序消除。对所有其他位置对 $(k, l)$ 的逆序关系不受影响（因为只有位置 $i$ 和 $i+1$ 的元素移动了）。故每次相邻交换恰消除 1 个逆序对。□

**这个定理的惊人推论**：

$$\text{排序代价（相邻交换次数）} = \text{输入的逆序数}$$

这让"逆序数"成为衡量数组"无序程度"的**最自然的指标**：
- 逆序数 = 0：已完全有序，$O(n)$ 时间
- 逆序数 = $\Theta(n)$：几乎有序，$O(n)$ 时间
- 逆序数 = $\Theta(n^2)$：完全随机，$O(n^2)$ 时间

### 14.4.4 随机排列的期望逆序数

**定理**：均匀随机排列的期望逆序数为 $\dfrac{n(n-1)}{4}$。

**证明**：考虑任意一对下标 $i < j$，序列中元素 $a_i$ 与 $a_j$ 构成逆序对的概率是 $\dfrac{1}{2}$（因为随机排列中大数在前和小数在前等概率）。共有 $\binom{n}{2} = \dfrac{n(n-1)}{2}$ 对，由期望的线性性：

$$E[\text{逆序数}] = \binom{n}{2} \cdot \frac{1}{2} = \frac{n(n-1)}{4}$$

因此，对随机输入，冒泡排序平均需要 $\dfrac{n(n-1)}{4} = \Theta(n^2)$ 次交换，平均时间复杂度 $\Theta(n^2)$。

### 14.4.5 使用归并排序计算逆序数（进阶内容）

虽然可以暴力 $O(n^2)$ 枚举所有对来数逆序对，但在 Chapter 15 学完归并排序后，我们会看到**分治法**可以在 $O(n \log n)$ 时间内计算逆序数（CLRS 习题 2.4-2）。  

这是 LeetCode #315（计算右侧小于当前元素的个数）的核心思路，也是理解**归并排序与逆序对关系**的精华。

## 14.5 希尔排序（Shell Sort）

### 14.5.1 插入排序的瓶颈与希尔的突破思想

回看插入排序的痛点：若待插入元素 `key` 非常小，它必须通过**逐步移动**越过所有比它大的元素，每次只移动 1 格。最坏情况（逆序数组）下，第 $i$ 个元素要移动 $i$ 格，总代价 $\Theta(n^2)$。

1959 年，Donald Shell 提出了一个天才想法：

> **不要一步一步移动，先进行"大步跳跃"的粗排，再细化。**

**核心思想**：选定一个**间隔序列**（Gap Sequence）$h_1 > h_2 > \cdots > h_t = 1$，依次对每个间隔 $h_k$ 做一次"$h_k$-sort"（即对所有间隔为 $h_k$ 的子序列各自做插入排序），最后当间隔缩减到 1 时，做一次完整的插入排序。

**关键洞察**：
- **大间隔阶段**：一次移动可以从位置 $j$ 直接跳到位置 $j - h$，大幅减少远距离移动的步数
- **小间隔阶段**：由于前几轮大间隔已经将数组"粗略排好"，元素大致在正确位置附近，此时插入排序的代价很低（逆序数很小）
- **间隔 = 1**：就是标准插入排序，但此时数组几乎有序，只需 $O(n)$

### 14.5.2 间隔序列的选择——不同方案的复杂度

不同的间隔序列导致截然不同的复杂度，这也是希尔排序理论研究至今未完结的原因：

| 间隔序列 | 来源 | 最坏时间复杂度 |
|---|---|---|
| $\lfloor n/2 \rfloor, \lfloor n/4 \rfloor, \ldots, 1$ | Shell（1959） | $O(n^2)$ |
| $2^k - 1$（即 1, 3, 7, 15, 31…） | Hibbard（1963） | $O(n^{3/2})$ |
| $1, 4, 13, 40, 121, \ldots$（$3h+1$） | Knuth（1973） | $O(n^{3/2})$ |
| $\{1,5,19,41,109,\ldots\}$（$9 \cdot 4^k - 9 \cdot 2^k + 1$） | Sedgewick（1986） | $O(n^{4/3})$ |
| 已知最优 | Ciura（2001）实验序列 | 实践最优，理论未知 |

> 💡 **为什么理论上难以分析？**  
> 不同轮次的 $h$-sort 之间存在复杂的相互作用（前一轮的"成果"会被后一轮利用），目前数学上还无法对一般间隔序列给出紧界。这是计算机科学中少数"实践跑在理论前面"的案例之一。

**Knuth 序列**是实践中最常用的：从 1 开始，反复乘以 3 再加 1，直到超过 $n/3$：

$$h_1 = 1, \quad h_{k+1} = 3h_k + 1 \quad \Rightarrow \quad 1, 4, 13, 40, 121, 364, \ldots$$

使用时从序列最大值倒退回 1。

### 14.5.3 Python 与 C++ 实现（Knuth 序列）

```python
def shell_sort(arr: list[int]) -> None:
    """
    希尔排序，使用 Knuth 间隔序列（1, 4, 13, 40, 121...）
    时间复杂度：O(n^(3/2))（Knuth 序列），远优于朴素 O(n^2)
    空间：O(1)（原地排序）
    稳定性：不稳定（大步跳跃的交换可能破坏相对顺序）

    算法设计：
    1. 计算初始间隔 h（不超过 n//3 的最大 Knuth 数）
    2. 对每个间隔 h 做 h-sort（即间隔为 h 的子序列分别插入排序）
    3. 间隔按 Knuth 公式缩小：h = (h - 1) / 3
    4. 直到 h == 1 完成最后一趟（标准插入排序，但此时已几乎有序）
    """
    n = len(arr)

    # 步骤 1：求初始最大间隔
    h = 1
    while h < n // 3:
        h = 3 * h + 1   # Knuth 序列：1, 4, 13, 40, 121...

    # 步骤 2：从大间隔到小间隔逐步缩减
    while h >= 1:
        # h-sort：对所有起始位置 i 在 [h, n) 内执行插入排序
        # 关键：这里的"相邻"是间隔为 h 的元素，而非真正相邻
        for i in range(h, n):
            key = arr[i]
            j = i - h            # 从 i 往左跳，步长为 h

            # 间隔为 h 的子序列中执行插入
            # 边界：j >= 0 防止越界（同标准插入排序）
            while j >= 0 and arr[j] > key:
                arr[j + h] = arr[j]  # 注意：移动步长是 h，不是 1
                j -= h

            arr[j + h] = key         # 插入

        # 步骤 3：缩减间隔（Knuth 公式逆运算）
        h = (h - 1) // 3

# === 使用示例 ===
arr = [23, 29, 15, 19, 31, 7, 9, 5, 2]
shell_sort(arr)
print(arr)  # [2, 5, 7, 9, 15, 19, 23, 29, 31]

# 验证：随机大数组
import random
arr2 = random.sample(range(1000), 100)
shell_sort(arr2)
print(arr2 == sorted(arr2))  # True
```
```cpp
#include <vector>
using namespace std;

void shell_sort(vector<int>& arr) {
    int n = arr.size();

    // 步骤 1：求初始最大 Knuth 间隔
    int h = 1;
    while (h < n / 3) {
        h = 3 * h + 1;   // 1, 4, 13, 40, 121, 364...
    }

    // 步骤 2：从大间隔到 1 逐步 h-sort
    while (h >= 1) {
        // h-sort：对每个起始位置 i 做"间隔为 h"的插入排序
        for (int i = h; i < n; i++) {
            int key = arr[i];
            int j = i - h;

            // 在间隔为 h 的子序列上做插入
            // 注意移动步长是 h（不是 1）
            while (j >= 0 && arr[j] > key) {
                arr[j + h] = arr[j];
                j -= h;
            }
            arr[j + h] = key;
        }

        // 步骤 3：缩减间隔
        h = (h - 1) / 3;   // Knuth 逆运算：h = (h-1)/3
    }
}
```

### 14.5.4 可视化理解：为什么大步跳跃有效？

以 `[9, 8, 7, 6, 5, 4, 3, 2, 1]`（完全逆序）为例，$n = 9$，Knuth 初始间隔 $h = 4$：

```
原数组：[9, 8, 7, 6, 5, 4, 3, 2, 1]

h = 4 时，数组被分成 4 个间隔为 4 的子序列：
  子序列 0: arr[0], arr[4], arr[8] = [9, 5, 1] → 4-sort → [1, 5, 9]
  子序列 1: arr[1], arr[5]         = [8, 4]    → 4-sort → [4, 8]
  子序列 2: arr[2], arr[6]         = [7, 3]    → 4-sort → [3, 7]
  子序列 3: arr[3], arr[7]         = [6, 2]    → 4-sort → [2, 6]

h=4 结束后：[1, 4, 3, 2, 5, 8, 7, 6, 9]
（9 已经从位置 0 跳到位置 8，越过了 4 格！插入排序只能移动 1 格）

h = 1 时（标准插入排序）：
此时逆序数远少于原始逆序数 36，代价大幅降低。
```

**直觉**：$h=4$ 时一次移动跨越 4 格，相当于"消除了距离为 4 的逆序对"，而插入排序每次只消除距离为 1 的逆序对。希尔排序先用大步快速消除大量逆序，再用小步精细调整。

### 14.5.5 复杂度的难分析性与实践表现

**理论困境**：希尔排序的精确复杂度高度依赖间隔序列，当前最优已知结果：
- Shell 原始序列：$O(n^2)$（与选择排序一样差！大步无效）
- Hibbard / Knuth：$O(n^{3/2})$（实验证实，严格证明很难）
- Sedgewick 1986：$O(n^{4/3})$（目前已知最优）
- 理论下界：未知（甚至不知道是否存在接近 $O(n \log n)$ 的间隔序列）

**实践表现**：
- 对 $10^4$ 级别的数据，Knuth 序列的希尔排序实践上非常快，常数因子小
- 原地排序（$O(1)$ 空间），无递归调用栈开销，代码简洁
- 在嵌入式系统（RAM 极有限）中有时是首选
- 现代 C 标准库（如 glibc 的早期版本）就使用了希尔排序

```
需要原地排序且空间极度受限（嵌入式/中等规模）？
 └─ YES → 希尔排序（O(n^4/3) 或更优，无递归栈）

教学/竞赛中需要统计逆序数？
 └─ YES → 先了解冒泡排序的逆序对定理，实际计算用归并（Ch15）

通用场景（n 较大，无特殊约束）？
 └─ YES → 快速排序 / TimSort（见 Chapter 15）
```

### 14.7.2 链表上的插入排序

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def insertion_sort_list(head: ListNode) -> ListNode:
    """
    链表版插入排序（LeetCode #147）
    链表上无法随机访问，但指针实现"插入"代价很低（不需要移动其他元素）
    时间：O(n^2)，空间：O(1)（只修改指针，不用辅助数组）

    技巧：用哨兵节点（dummy node）简化链表头部的边界处理
    """
    # 哨兵节点：值为最小值，作为排好序链表的虚拟头
    dummy = ListNode(float('-inf'))
    dummy.next = None

    curr = head   # 当前待插入节点
    while curr:
        next_node = curr.next   # 保存下一个（因为 curr.next 会被修改）

        # 在已排序链表中找插入位置
        # 从 dummy 开始往后扫，找到第一个 prev.next.val > curr.val 的位置
        prev = dummy
        while prev.next and prev.next.val <= curr.val:
            prev = prev.next

        # 把 curr 插入到 prev 和 prev.next 之间
        curr.next = prev.next
        prev.next = curr

        curr = next_node   # 处理下一个节点

    return dummy.next   # 返回排好序的链表头

# 辅助函数：创建链表
def create_list(arr):
    dummy = ListNode(0)
    cur = dummy
    for v in arr:
        cur.next = ListNode(v)
        cur = cur.next
    return dummy.next

# 辅助函数：打印链表
def print_list(head):
    res = []
    while head:
        res.append(head.val)
        head = head.next
    print(res)

# === 测试 ===
head = create_list([4, 2, 1, 3])
sorted_head = insertion_sort_list(head)
print_list(sorted_head)   # [1, 2, 3, 4]
```
```cpp
struct ListNode {
    int val;
    ListNode* next;
    ListNode(int x) : val(x), next(nullptr) {}
};

class Solution {
public:
    ListNode* insertionSortList(ListNode* head) {
        // 哨兵节点，值尽量小
        ListNode* dummy = new ListNode(INT_MIN);
        dummy->next = nullptr;

        ListNode* curr = head;
        while (curr) {
            ListNode* nextNode = curr->next;   // 保存下一个

            // 找插入位置
            ListNode* prev = dummy;
            while (prev->next && prev->next->val <= curr->val) {
                prev = prev->next;
            }

            // 插入 curr 到 prev 之后
            curr->next = prev->next;
            prev->next = curr;

            curr = nextNode;
        }

        ListNode* result = dummy->next;
        delete dummy;   // 内存管理
        return result;
    }
};
```

### 14.7.3 高频面试题 Q&A

**Q1：插入排序和选择排序最本质的区别是什么？**

A：
- 插入排序每轮**确定一个元素的最终位置**的方式是：找到它在**已排序区**中的正确位置；已排序区从左端扩展，未排序区从左端缩减。它是**自适应**的（已排序时只需 $O(n)$）。
- 选择排序每轮**从未排序区找出最小元素**放到已排序区末尾；比较次数固定不可减少，**永远 $\Theta(n^2)$**，但交换次数仅 $O(n)$。

**Q2：如何证明基于比较的排序下界是 $\Omega(n \log n)$？**

A（决策树论证，完整在 Chapter 15 证明）：  
任何基于比较的排序算法可以抽象为一棵二叉决策树，叶子节点对应输入的某个排列。$n$ 个元素共有 $n!$ 种排列，决策树至少有 $n!$ 个叶子，二叉树高度 $\geq \lceil \log_2 n! \rceil$。由 Stirling 近似 $n! \approx \left(\frac{n}{e}\right)^n \sqrt{2\pi n}$，得 $\log_2 n! = \Omega(n \log n)$。故任何基于比较的排序最坏情况至少需要 $\Omega(n \log n)$ 次比较。

**Q3：什么情况下冒泡排序优于插入排序？**

A：理论上，**几乎没有场景**冒泡排序优于插入排序。两者最好/最坏复杂度相同，但插入排序的常数因子更小（内层循环每步只做一次赋值，而冒泡每步做一次交换即三次赋值）。冒泡排序的主要价值在于：①教学（概念最直观）②逆序数定理（理论分析）。

**Q4：希尔排序是稳定的吗？**

A：不稳定。大步跳跃的交换可以将相等元素的相对顺序打乱。例如间隔 $h=2$ 时，位置 0 和位置 2 的相等元素可能因为中间元素的存在而发生相对顺序改变。

## 14.8 常见错误与调试技巧

### ⚠️ 错误 1：插入排序内层循环忘记边界检查

```python
# ❌ 错误：缺少 j >= 0，当 key 是最小值时会越界
while arr[j] > key:    # 当 j = -1 时，arr[-1] 在 Python 中是最后一个元素！
    arr[j + 1] = arr[j]
    j -= 1

# ✅ 正确：两个条件缺一不可，且 j >= 0 必须在前（短路求值防止 arr[-1] 被访问）
while j >= 0 and arr[j] > key:
    arr[j + 1] = arr[j]
    j -= 1
```

> 注意：Python 的 `arr[-1]` 不会抛出异常，而是返回最后一个元素！这会导致**静默错误**，非常难以调试。C++ 中 `arr[-1]` 是未定义行为，可能随机崩溃。

### ⚠️ 错误 2：冒泡排序"提前终止"写法错误

```python
# ❌ 错误：swapped 放在外层初始化，导致只要历史上有过交换就不终止
swapped = True
for i in range(n - 1):
    for j in range(n - 1 - i):
        if arr[j] > arr[j + 1]:
            arr[j], arr[j + 1] = arr[j + 1], arr[j]
            swapped = True   # ❌ swapped 从 True 变 True，永远不 break

# ✅ 正确：swapped 必须在每趟开始时重置为 False
for i in range(n - 1):
    swapped = False       # ← 每趟开始重置！
    for j in range(n - 1 - i):
        if arr[j] > arr[j + 1]:
            arr[j], arr[j + 1] = arr[j + 1], arr[j]
            swapped = True
    if not swapped:
        break
```

### ⚠️ 错误 3：选择排序写成稳定的"假象"

```python
# 有些实现尝试让选择排序"看起来稳定"，但这是徒劳的
# 标准交换版选择排序天然不稳定，无法通过简单修改变稳定
# 若需要稳定选择排序，应改用插入来替代"交换"：
def stable_selection_sort(arr):
    for i in range(len(arr)):
        min_idx = min(range(i, len(arr)), key=lambda k: arr[k])
        # ⚠️ 仍然不稳定！即使用 min() 也不能保证稳定性
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
# 正确做法：需要真正稳定选择排序时，其实复杂度会变为 O(n^2) 且实现更复杂，
# 不如直接改用插入排序。
```

### ⚠️ 错误 4：希尔排序间隔序列计算错误

```python
# ❌ 错误：Shell 原始序列（n//2 减半）
h = n // 2
while h > 0:
    # ... h-sort ...
    h //= 2   # 复杂度 O(n^2)，选错了序列

# ✅ 正确：Knuth 序列（3h+1），复杂度 O(n^(3/2))
h = 1
while h < n // 3:
    h = 3 * h + 1   # 1, 4, 13, 40...
while h >= 1:
    # ... h-sort ...
    h = (h - 1) // 3   # ← 注意是 (h-1)//3 而非 h//3（逆运算）
```

## 本章小结

| 知识点 | 核心结论 |
|---|---|
| 稳定性 | 插入、冒泡稳定；选择、希尔不稳定；"相等不换"是稳定的充分条件 |
| 插入排序 | 循环不变式证明；$O(n)$ 最好（已排序）；适合小数组和几乎有序 |
| 选择排序 | 永远 $\Theta(n^2)$，交换次数仅 $O(n)$；不稳定（例：$[5a, 5b, 1]$） |
| 冒泡排序 | 相邻交换每次恰消除 1 个逆序对；逆序数 = 总交换次数 |
| 逆序数 | 随机排列期望 $n(n-1)/4$；是"无序程度"的自然度量；$O(n \log n)$ 计数见 Ch15 |
| 希尔排序 | 大间隔跳跃消远距逆序；Knuth 序列 $O(n^{3/2})$；理论下界待研究 |
| 决策树下界 | 基于比较的排序 $\Omega(n \log n)$；线性时间排序需利用数据额外信息（Ch15）|

**下一章**（Chapter 15）将介绍时间复杂度突破到 $\Theta(n \log n)$ 的高效排序算法：归并排序（分治精华）、快速排序（工程首选）、以及突破比较下界的三种线性时间排序。
